<dl> <dt>

<span id="_security_national_institute_of_standards_and_technology_gly"></span><span id="_SECURITY_NATIONAL_INSTITUTE_OF_STANDARDS_AND_TECHNOLOGY_GLY"></span>**National Institute of Standards and Technology**
</dt> <dd>

NIST División del Departamento de comercio de Estados Unidos que publica estándares oficiales para sistemas de equipos gubernamentales y de sectores privados. Estos estándares se publican como publicaciones de estándares federales de procesamiento de información (FIPS). En 1987, NIST se dirigió para definir estándares con el fin de garantizar la seguridad de la información confidencial pero sin clasificar en los sistemas de equipos gubernamentales.

</dd> <dt>

<span id="_security_negotiate_gly"></span><span id="_SECURITY_NEGOTIATE_GLY"></span>**Busca**
</dt> <dd>

[*Proveedor de compatibilidad para seguridad*](s-gly.md) (SSP) que actúa como nivel de aplicación entre la [*interfaz del proveedor de compatibilidad para seguridad*](s-gly.md) (SSPI) y el otro SSP. Cuando una aplicación llama en SSPI para iniciar sesión en una red, puede especificar un SSP para que procese la solicitud. Si la aplicación especifica Negotiate, Negotiate analiza la solicitud y elige el mejor SSP para controlar la solicitud en función de la Directiva de seguridad configurada por el cliente.

</dd> <dt>

<span id="_security_nist_gly"></span><span id="_SECURITY_NIST_GLY"></span>**NIST**
</dt> <dd>

Consulte *National Institute of Standards and Technology*.

</dd> <dt>

<span id="_security_nonce_gly"></span><span id="_SECURITY_NONCE_GLY"></span>**emisor**
</dt> <dd>

Un valor generado de forma aleatoria y que se utiliza para acabar con los ataques basados en “repeticiones”.

</dd> <dt>

<span id="_security_nonrepudiation_gly"></span><span id="_SECURITY_NONREPUDIATION_GLY"></span>**no rechazo**
</dt> <dd>

La capacidad para identificar a los usuarios que realizaron ciertas acciones, contrarrestando de manera irrefutable cualquier intento de negación de responsabilidades por parte de un usuario. Por ejemplo, un sistema puede registrar el identificador de un usuario cada vez que se elimine un archivo.

</dd> </dl>
