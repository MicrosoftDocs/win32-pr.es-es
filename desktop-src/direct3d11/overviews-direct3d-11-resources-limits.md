 Una aplicación puede intentar asignar más memoria para un recurso que el tamaño máximo de recurso especificado. Es decir, el tiempo de ejecución de Direct3D 11 permite estos intentos de asignación de memoria en caso de que el hardware pueda admitirlos. Sin embargo, el entorno de ejecución de Direct3D 11 solo garantiza que las asignaciones dentro del tamaño máximo de recursos sean compatibles con todo el hardware de nivel [de](overviews-direct3d-11-devices-downlevel-intro.md) característica 11. Si una aplicación intenta asignar memoria para un recurso dentro del tamaño máximo del recurso, el tiempo de ejecución produce un error en el intento solo si el sistema operativo se queda sin recursos. Si una aplicación intenta asignar memoria para un recurso por encima del tamaño máximo del recurso, el tiempo de ejecución puede producir un error en el intento porque el sistema operativo está sobreextenido o el hardware no admite asignaciones por encima del tamaño máximo del recurso. La capa de depuración solo comprueba el TAMAÑO DE RECURSO DE REQ de D3D11 \_ \_ EN \_ \_ \_ MEGABYTES EXPRESSION A TERM \_ \_ \_ (128) MB.

 Las ranuras de salida del sombreador de píxeles se comparten entre los registros de salida de píxeles (cuatro componentes de 32 bits) y los UAV.

 El número total de componentes de todos los sombreadores de casco a puntos de control de sombreador de dominio se limita a 3968, que es 128 menos que el número máximo de puntos de control que el punto de control máximo registra por cuatro componentes.

4Para los perfiles de sombreador de proceso CS \_ \_ 4 0 y CS \_ 4 \_ 1, solo hay 1 UAV disponible. Para obtener más información sobre los perfiles de sombreador, vea [Shader Model 5](/windows/desktop/direct3dhlsl/d3d11-graphics-reference-sm5).
