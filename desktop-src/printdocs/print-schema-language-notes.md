# <a name="print-schema-language-notes"></a>Imprimir notas del lenguaje de esquema

Este tema no es actual. Para obtener la información más reciente, vea [La especificación de esquema de impresión](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

En esta especificación, las palabras que se usan para definir la importancia de cada requisito se escriben en mayúsculas. Estas palabras se usan de acuerdo con sus definiciones en RFC 2119 y sus significados respectivos se reproducen a continuación:



| Término                                                                | Descripción                                                                                                                                                                                                                                                                                         |
|---------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="MUST"></span><span id="must"></span>Debe<br/>       | Esta palabra, o el adjetivo REQUIRED, significa que el elemento es un requisito absoluto de la especificación.<br/>                                                                                                                                                                               |
| <span id="SHOULD"></span><span id="should"></span>Debe<br/> | Esta palabra, o el adjetivo RECOMENDADO, significa que puede haber razones válidas en determinadas circunstancias para omitir este elemento, pero se deben comprender todas las implicaciones y sopesar cuidadosamente el caso antes de elegir un curso diferente.<br/>                                     |
| <span id="MAY"></span><span id="may"></span>Puede<br/>          | Esta palabra, o el adjetivo OPTIONAL, significa que este elemento es realmente opcional. Por ejemplo, una implementación puede optar por incluir el elemento porque un marketplace o escenario determinado lo requiere o porque mejora el producto. Otra implementación puede omitir el mismo elemento.<br/> |
