Esta sección contiene una lista de libros y recursos recomendados relacionados con el diseño de la interfaz de usuario. (Estos libros y recursos pueden no estar disponibles en algunos idiomas y países).

## <a name="reading-list"></a>Lista de lectura

<dl> <dt>

<span id="_Windows_Interface_Guidelines_for_Software_Design"></span><span id="_windows_interface_guidelines_for_software_design"></span><span id="_WINDOWS_INTERFACE_GUIDELINES_FOR_SOFTWARE_DESIGN"></span>*Directrices de la interfaz de Windows para el diseño de software*
</dt> <dd>

(Microsoft Press, 1995; ISBN 1-55615-679-0)

En este libro se explican todas las reglas clave de la interfaz de usuario que deben seguirse, con numerosos ejemplos de cómo diseñar cuadros de diálogo, asistentes, barras de herramientas y menús contextuales.

</dd> <dt>

<span id="Usability_Engineering_by_Jakob_Nielsen_"></span><span id="usability_engineering_by_jakob_nielsen_"></span><span id="USABILITY_ENGINEERING_BY_JAKOB_NIELSEN_"></span>*Ingeniería de facilidad de uso* de Jakob Nielsen 
</dt> <dd>

(AP Professional, 1994; ISBN 0125184069)

Este libro se centra en las técnicas para medir eficazmente la facilidad de uso de una interfaz. Se explican muchas técnicas de prueba diferentes para comparar distintas ideas de diseño, incluidas sugerencias para mantener los costos de las pruebas de facilidad de uso. Este libro cubre técnicas de análisis que se pueden aplicar a sitios web, páginas web o cualquier tipo de software. Es excelente para las fases iniciales de aprendizaje de los usuarios y las fases medias de las pruebas de uso e iteración.

</dd> <dt>

<span id="Design_of_Everyday_Things_by_Don_Norman"></span><span id="design_of_everyday_things_by_don_norman"></span><span id="DESIGN_OF_EVERYDAY_THINGS_BY_DON_NORMAN"></span>*Diseño de cosas diarias de* Don Norman
</dt> <dd>

(Currency/Doubleday, 1988; ISBN: 0385267746)

En este libro se describen los principios psicológico en el trabajo que hacen que las interfaces sean difíciles de usar. Recomendado para cualquier persona interesada en las interfaces de usuario y que desea una amplia comprensión de los problemas implicados. Entre los ejemplos se incluyen aparatos de cocina, puertas de oficina y máquinas expendedoras.

</dd> <dt>

<span id="About_Face_by_Alan_Cooper_"></span><span id="about_face_by_alan_cooper_"></span><span id="ABOUT_FACE_BY_ALAN_COOPER_"></span>*Acerca de la esfera* de Alan Cooper 
</dt> <dd>

(Libros de IDG en todo el mundo, 1995; ISBN 1568843224)

Al pensar en las soluciones de la interfaz de usuario, un breve paso a través de este libro le ayudará a obtener ideas.

</dd> <dt>

<span id="The_Elements_of_Friendly_Software_Design_by_Paul_Heckel_"></span><span id="the_elements_of_friendly_software_design_by_paul_heckel_"></span><span id="THE_ELEMENTS_OF_FRIENDLY_SOFTWARE_DESIGN_BY_PAUL_HECKEL_"></span>*Los elementos de diseño de software sencillo de* Paul Heckel 
</dt> <dd>

(SYBEX, 1991; ISBN 0-89588-768-1)

</dd> <dt>

<span id="Microsoft_Windows_User_Experience"></span><span id="microsoft_windows_user_experience"></span><span id="MICROSOFT_WINDOWS_USER_EXPERIENCE"></span>*Experiencia del usuario de Microsoft Windows*
</dt> <dd>

(Microsoft Press, 1999; ISBN 0735605661)

Las directrices oficiales de Microsoft para crear interfaces de usuario bien diseñadas, visualmente y coherentes funcionalmente para las aplicaciones.

</dd> <dt>
