# <a name="d1150-unsupported-mip-level-selection"></a>D1150: selección de nivel de MIP no admitida

El IDXGISurface2 proporcionado tiene varios niveles de MIP, pero no se admite la selección de nivel de MIP en este nivel de dispositivo (D3D9).






 

## <a name="possible-causes"></a>Causas posibles

Interopped las texturas de Direct3D pueden estar asignadas a MIP; Direct2D solo funcionará en un nivel de MIP a la vez. Los demás niveles de MIP de la textura primaria no se verán afectados ni representarán en Direct2D. Sin embargo, los dispositivos de nivel de característica 9 no admiten la selección de nivel de MIP y no pueden usar esta característica de interoperabilidad.

## <a name="possible-fixes"></a>Posibles correcciones

En el caso de las aplicaciones que se espera que se ejecuten correctamente en los dispositivos de nivel de característica 9, considere la posibilidad de modificar la lógica de representación para evitar la necesidad de interoperar con las texturas de mipmapped. Si es posible, interopere un intermediario en su lugar. Para minimizar los efectos de solucionar esta limitación, la aplicación puede optar por implementar esto como una ruta de acceso de reserva específica de nivel de característica 9.
