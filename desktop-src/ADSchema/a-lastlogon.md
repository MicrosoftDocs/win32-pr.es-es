# <a name="last-logon-attribute"></a>Last-Logon atributo)

La última vez que el usuario inició sesión. Este valor se almacena como un entero grande que representa el número de intervalos de 100 nanosegundos desde el 1 de enero de 1601 (UTC). Un valor de cero significa que la hora del último inicio de sesión es desconocida.



| Entrada | Value |
|-------------------|--------------------------------------|
| CN                | Last-Logon                           |
| Nombre para mostrar de LDAP | lastLogon                            |
| Tamaño              | 8 bytes                              |
| Actualizar privilegio  | El sistema establece este valor.     |
| Frecuencia de actualización  | Cada vez que el usuario inicia sesión.          |
| Attribute-Id      | 1.2.840.113556.1.4.52                |
| System-ID-GUID    | bf967997-0de6-11d0-a285-00aa003049e2 |
| Sintaxis            | [**Interval**](s-interval.md)       |

## <a name="remarks"></a>Observaciones

La parte alta de este entero grande corresponde al miembro **dwHighDateTime** de la estructura [**FILETIME**](/windows/desktop/api/minwinbase/ns-minwinbase-filetime) y la parte baja corresponde al miembro **dwLowDateTime** de la estructura **FILETIME** .

Este atributo no se replica y se mantiene por separado en cada controlador de dominio del dominio. Para obtener un valor preciso para el último inicio de sesión del usuario en el dominio, el atributo **Last-Logon** del usuario se debe recuperar de cada controlador de dominio del dominio. El valor más grande que se recupera es la hora real del último inicio de sesión de ese usuario.
