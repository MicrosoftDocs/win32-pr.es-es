<span data-ttu-id="7d746-104">Una ventana secundaria es una ventana con el \_ estilo WS secundario o WS \_ CHILDWINDOW.</span><span class="sxs-lookup"><span data-stu-id="7d746-104">A child window is a window with the WS\_CHILD or WS\_CHILDWINDOW style.</span></span> <span data-ttu-id="7d746-105">Al igual que otros estilos de ventana, las ventanas secundarias reciben mensajes de [**WM \_ Paint**](wm-paint.md) para solicitar la actualización.</span><span class="sxs-lookup"><span data-stu-id="7d746-105">Like other window styles, child windows receive [**WM\_PAINT**](wm-paint.md) messages to prompt updating.</span></span> <span data-ttu-id="7d746-106">Cada ventana secundaria tiene una región de actualización, que puede configurar el sistema o la aplicación para generar mensajes de la **\_ pintura de WM** eventuales.</span><span class="sxs-lookup"><span data-stu-id="7d746-106">Each child window has an update region, which either the system or the application can set to generate eventual **WM\_PAINT** messages.</span></span>

<span data-ttu-id="7d746-107">La ventana primaria del elemento secundario afecta a las regiones de actualización y visible de una ventana secundaria. Esto no es cierto para ventanas de otros estilos.</span><span class="sxs-lookup"><span data-stu-id="7d746-107">A child window's update and visible regions are affected by the child's parent window; this is not true for windows of other styles.</span></span> <span data-ttu-id="7d746-108">El sistema suele establecer la región de actualización de la ventana secundaria cuando establece la región de actualización de la ventana primaria, lo que hace que la ventana secundaria reciba mensajes de [**WM \_ Paint**](wm-paint.md) cuando la ventana primaria las recibe.</span><span class="sxs-lookup"><span data-stu-id="7d746-108">The system often sets the child window's update region when it sets the parent window's update region, causing the child window to receive [**WM\_PAINT**](wm-paint.md) messages when the parent window receives them.</span></span> <span data-ttu-id="7d746-109">El sistema limita la ubicación del área visible de la ventana secundaria a dentro del área cliente de la ventana primaria y recorta cualquier parte de la ventana secundaria que se ha despasado fuera de la ventana primaria.</span><span class="sxs-lookup"><span data-stu-id="7d746-109">The system limits the location of the child window's visible region to within the client area of the parent window and clips any portion of the child window moved outside the parent window.</span></span>

<span data-ttu-id="7d746-110">El sistema establece la región de actualización de una ventana secundaria siempre que parte de la región de actualización de la ventana primaria incluya una parte de la ventana secundaria.</span><span class="sxs-lookup"><span data-stu-id="7d746-110">The system sets the update region for a child window whenever part of the parent window's update region includes a portion of the child window.</span></span> <span data-ttu-id="7d746-111">En tales casos, el sistema envía primero un mensaje de [**\_ dibujo de WM**](wm-paint.md) a la ventana primaria y, a continuación, envía un mensaje a la ventana secundaria, lo que permite que el elemento secundario restaure las partes de la ventana que puede haber dibujado el elemento primario.</span><span class="sxs-lookup"><span data-stu-id="7d746-111">In such cases, the system first sends a [**WM\_PAINT**](wm-paint.md) message to the parent window and then sends a message to the child window, allowing the child to restore any portions of the window that the parent may have drawn over.</span></span>

<span data-ttu-id="7d746-112">El sistema no establece la región de actualización del elemento primario cuando se establece el elemento secundario.</span><span class="sxs-lookup"><span data-stu-id="7d746-112">The system does not set the parent's update region when the child's is set.</span></span> <span data-ttu-id="7d746-113">Una aplicación no puede generar un mensaje de [**\_ dibujo de WM**](wm-paint.md) para la ventana primaria mediante la invalidación de la ventana secundaria.</span><span class="sxs-lookup"><span data-stu-id="7d746-113">An application cannot generate a [**WM\_PAINT**](wm-paint.md) message for the parent window by invalidating the child window.</span></span> <span data-ttu-id="7d746-114">Del mismo modo, una aplicación no puede generar un mensaje de **\_ dibujo de WM** para el elemento secundario mediante la invalidación de una parte del área de cliente del elemento primario que se encuentra por completo en la ventana secundaria.</span><span class="sxs-lookup"><span data-stu-id="7d746-114">Similarly, an application cannot generate a **WM\_PAINT** message for the child by invalidating a portion of the parent's client area that lies entirely under the child window.</span></span> <span data-ttu-id="7d746-115">En tales casos, ninguna ventana recibe un mensaje de **\_ dibujo de WM** .</span><span class="sxs-lookup"><span data-stu-id="7d746-115">In such cases, neither window receives a **WM\_PAINT** message.</span></span>

<span data-ttu-id="7d746-116">Una aplicación puede impedir que la región de actualización de una ventana secundaria se establezca cuando el valor de la ventana primaria se establece especificando el \_ estilo WS CLIPCHILDREN al crear la ventana primaria.</span><span class="sxs-lookup"><span data-stu-id="7d746-116">An application can prevent a child window's update region from being set when the parent window's is set by specifying the WS\_CLIPCHILDREN style when creating the parent window.</span></span> <span data-ttu-id="7d746-117">Cuando se establece este estilo, el sistema excluye las ventanas secundarias del área visible del elemento primario y, por tanto, omite cualquier parte de la región de actualización que pueda contener las ventanas secundarias.</span><span class="sxs-lookup"><span data-stu-id="7d746-117">When this style is set, the system excludes the child windows from the parent's visible region and therefore ignores any portion of the update region that may contain the child windows.</span></span> <span data-ttu-id="7d746-118">Cuando la aplicación pinta en la ventana primaria, se recorta cualquier dibujo que abarque la ventana secundaria, lo que convierte en innecesariamente un mensaje de [**\_ dibujo de WM**](wm-paint.md) posterior a la ventana secundaria.</span><span class="sxs-lookup"><span data-stu-id="7d746-118">When the application paints in the parent window, any drawing that would cover the child window is clipped, making a subsequent [**WM\_PAINT**](wm-paint.md) message to the child window unnecessary.</span></span>
