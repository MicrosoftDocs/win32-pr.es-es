# <a name="non-default-backup-and-restore-locations"></a>Ubicaciones de copia de seguridad y restauración no predeterminadas

Normalmente, las operaciones de copia de seguridad y restauración copian archivos de una ubicación predeterminada determinada del disco a un medio de copia de seguridad y, a continuación, restauran desde ese medio a la misma ubicación predeterminada del disco.

Hay muchas razones, especialmente cuando se trabaja con procesos en ejecución, para no seguir este modelo simple. VSS admite el uso de orígenes no estándar para la copia de seguridad y destinos alternativos para la restauración, e incluye mecanismos para trabajar con subconjuntos de archivos y reasignar archivos.

-   [Trabajar con rutas de acceso alternativas durante la copia de seguridad](working-with-alternate-paths-during-backup.md)
-   [Trabajar con ubicaciones alternativas durante la restauración](working-with-alternate-locations-during-restore.md)
-   [Trabajar con nuevos destinos durante la restauración](working-with-new-targets-during-restore.md)
-   [Trabajar con archivos parciales](working-with-partial-files.md)
-   [Trabajar con destinos dirigidos](working-with-directed-targets.md)
