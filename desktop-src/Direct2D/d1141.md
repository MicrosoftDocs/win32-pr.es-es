# <a name="d1141-print-requires-srgb-color-space"></a>D1141: Print requiere el espacio de color sRGB

Impresión en Direct2D: no se admite el espacio de colores personalizado. Usar sRGB en su lugar.






 

## <a name="possible-causes"></a>Causas posibles

Al crear un control de impresión de Direct2D, una aplicación usa \_ las propiedades del control de impresión D2D1 \_ \_ con un atributo de espacio de colores. Direct2D admite [**el \_ espacio de colores D2D1 \_ \_ personalizado**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) para algunas operaciones, como la creación de un contexto de color mediante datos ICC. Sin embargo, al crear un control de impresión, Direct2D solo admite espacios de colores integrados como sRGB o scRGB.

## <a name="possible-fixes"></a>Posibles correcciones

Para direccionar el mensaje, especifique un espacio de colores no personalizado como sRGB o scRGB al crear un control de impresión. El mensaje no indica un error, pero indica que Direct2D omitirá el especificador personalizado y creará el control de impresión mediante un espacio de colores sRGB.
