<span data-ttu-id="9704c-104">DirectShow usa una arquitectura modular, donde cada fase de procesamiento se realiza mediante un objeto COM denominado filtro.</span><span class="sxs-lookup"><span data-stu-id="9704c-104">DirectShow uses a modular architecture, where each stage of processing is done by a COM object called a filter.</span></span> <span data-ttu-id="9704c-105">DirectShow proporciona un conjunto de filtros estándar para que los usen las aplicaciones, y los desarrolladores pueden escribir sus propios filtros personalizados que amplían la funcionalidad de DirectShow.</span><span class="sxs-lookup"><span data-stu-id="9704c-105">DirectShow provides a set of standard filters for applications to use, and developers can write their own custom filters that extend the functionality of DirectShow.</span></span> <span data-ttu-id="9704c-106">Para ilustrar, estos son los pasos necesarios para reproducir un archivo de vídeo AVI, junto con los filtros que realizan cada paso:</span><span class="sxs-lookup"><span data-stu-id="9704c-106">To illustrate, here are the steps needed to play an AVI video file, along with the filters that perform each step:</span></span>

-   <span data-ttu-id="9704c-107">Lea los datos sin procesar del archivo como una secuencia de bytes (filtro de origen de archivo).</span><span class="sxs-lookup"><span data-stu-id="9704c-107">Read the raw data from the file as a byte stream (File Source filter).</span></span>

-   <span data-ttu-id="9704c-109">Descodifique los fotogramas de vídeo (varios filtros del descodificador, dependiendo del formato de compresión).</span><span class="sxs-lookup"><span data-stu-id="9704c-109">Decode the video frames (various decoder filters, depending on the compression format).</span></span>
-   <span data-ttu-id="9704c-110">Dibuje los fotogramas de vídeo (filtro de representador de vídeo).</span><span class="sxs-lookup"><span data-stu-id="9704c-110">Draw the video frames (Video Renderer filter).</span></span>

<span data-ttu-id="9704c-112">Estos filtros se muestran en el diagrama siguiente.</span><span class="sxs-lookup"><span data-stu-id="9704c-112">These filters are shown in the following diagram.</span></span>

<span data-ttu-id="9704c-114">Como se muestra en el diagrama, cada filtro está conectado a uno o varios filtros.</span><span class="sxs-lookup"><span data-stu-id="9704c-114">As the diagram shows, each filter is connected to one or more other filters.</span></span> <span data-ttu-id="9704c-115">Los puntos de conexión también son objetos COM, denominados *PIN*.</span><span class="sxs-lookup"><span data-stu-id="9704c-115">The connection points are also COM objects, called *pins*.</span></span> <span data-ttu-id="9704c-116">Los filtros utilizan PIN para trasladar los datos de un filtro a la siguiente.</span><span class="sxs-lookup"><span data-stu-id="9704c-116">Filters use pins to move data from one filter the next.</span></span> <span data-ttu-id="9704c-117">Las flechas del diagrama muestran la dirección en la que los datos viajan.</span><span class="sxs-lookup"><span data-stu-id="9704c-117">The arrows in the diagram show the direction in which the data travels.</span></span> <span data-ttu-id="9704c-118">En DirectShow, un conjunto de filtros se denomina *gráfico de filtro*.</span><span class="sxs-lookup"><span data-stu-id="9704c-118">In DirectShow, a set of filters is called a *filter graph*.</span></span>

<span data-ttu-id="9704c-119">Los filtros tienen tres Estados posibles: en ejecución, detenido y en pausa.</span><span class="sxs-lookup"><span data-stu-id="9704c-119">Filters have three possible states: running, stopped, and paused.</span></span> <span data-ttu-id="9704c-120">Cuando se ejecuta un filtro, procesa los datos multimedia.</span><span class="sxs-lookup"><span data-stu-id="9704c-120">When a filter is running, it processes media data.</span></span> <span data-ttu-id="9704c-121">Cuando se detiene, detiene el procesamiento de los datos.</span><span class="sxs-lookup"><span data-stu-id="9704c-121">When it is stopped, it stops processing data.</span></span> <span data-ttu-id="9704c-122">El estado pausado se usa para señalar los datos antes de ejecutarse; en el [flujo de datos de](data-flow-in-the-filter-graph.md) la sección del gráfico de filtros se describe con más detalle este concepto.</span><span class="sxs-lookup"><span data-stu-id="9704c-122">The paused state is used to cue data before running; the section [Data Flow in the Filter Graph](data-flow-in-the-filter-graph.md) describes this concept in more detail.</span></span> <span data-ttu-id="9704c-123">Con excepciones muy poco frecuentes, los cambios de estado se coordinan en todo el gráfico de filtros. todos los filtros del gráfico cambian de estado en el unísono.</span><span class="sxs-lookup"><span data-stu-id="9704c-123">With very rare exceptions, state changes are coordinated throughout the entire filter graph; all the filters in the graph switch states in unison.</span></span> <span data-ttu-id="9704c-124">Por lo tanto, también se dice que todo el gráfico de filtros está en ejecución, detenido o en pausa.</span><span class="sxs-lookup"><span data-stu-id="9704c-124">Thus, the entire filter graph is also said to be running, stopped, or paused.</span></span>

<span data-ttu-id="9704c-125">Los filtros se pueden agrupar en varias categorías amplias:</span><span class="sxs-lookup"><span data-stu-id="9704c-125">Filters can be grouped into several broad categories:</span></span>

-   <span data-ttu-id="9704c-126">Un filtro de *origen* introduce los datos en el gráfico.</span><span class="sxs-lookup"><span data-stu-id="9704c-126">A *source* filter introduces data into the graph.</span></span> <span data-ttu-id="9704c-127">Los datos pueden provienen de un archivo, una red, una cámara o cualquier otro lugar.</span><span class="sxs-lookup"><span data-stu-id="9704c-127">The data might come from a file, a network, a camera, or anywhere else.</span></span> <span data-ttu-id="9704c-128">Cada filtro de origen controla un tipo diferente de origen de datos.</span><span class="sxs-lookup"><span data-stu-id="9704c-128">Each source filter handles a different type of data source.</span></span>
-   <span data-ttu-id="9704c-129">Un filtro de *transformación* toma un flujo de entrada, procesa los datos y crea un flujo de salida.</span><span class="sxs-lookup"><span data-stu-id="9704c-129">A *transform* filter takes an input stream, processes the data, and creates an output stream.</span></span> <span data-ttu-id="9704c-130">Los codificadores y descodificadores son ejemplos de filtros de transformación.</span><span class="sxs-lookup"><span data-stu-id="9704c-130">Encoders and decoders are examples of transform filters.</span></span>
-   <span data-ttu-id="9704c-131">Los filtros de *representador* se encuentran al final de la cadena.</span><span class="sxs-lookup"><span data-stu-id="9704c-131">*Renderer* filters sit at the end of the chain.</span></span> <span data-ttu-id="9704c-132">Reciben datos y los presentan al usuario.</span><span class="sxs-lookup"><span data-stu-id="9704c-132">They receive data and present it to the user.</span></span> <span data-ttu-id="9704c-133">Por ejemplo, un representador de vídeo dibuja fotogramas de vídeo en la pantalla. un representador de audio envía datos de audio a la tarjeta de sonido. y un filtro de escritura de archivos escribe datos en un archivo.</span><span class="sxs-lookup"><span data-stu-id="9704c-133">For example, a video renderer draws video frames on the display; an audio renderer sends audio data to the sound card; and a file-writer filter writes data to a file.</span></span>
-   <span data-ttu-id="9704c-134">Un filtro *divisor* divide un flujo de entrada en dos o más salidas, normalmente analizando el flujo de entrada a lo largo del proceso.</span><span class="sxs-lookup"><span data-stu-id="9704c-134">A *splitter* filter splits an input stream into two or more outputs, typically parsing the input stream along the way.</span></span> <span data-ttu-id="9704c-135">Por ejemplo, el divisor AVI analiza una secuencia de bytes en secuencias de audio y vídeo independientes.</span><span class="sxs-lookup"><span data-stu-id="9704c-135">For example, the AVI Splitter parses a byte stream into separate video and audio streams.</span></span>
-   <span data-ttu-id="9704c-136">Un filtro *MUX* toma varias entradas y las combina en una única secuencia.</span><span class="sxs-lookup"><span data-stu-id="9704c-136">A *mux* filter takes multiple inputs and combines them into a single stream.</span></span> <span data-ttu-id="9704c-137">Por ejemplo, AVI MUX realiza la operación inversa del divisor de AVI.</span><span class="sxs-lookup"><span data-stu-id="9704c-137">For example, the AVI Mux performs the inverse operation of the AVI Splitter.</span></span> <span data-ttu-id="9704c-138">Toma secuencias de audio y vídeo y genera una secuencia de bytes con formato AVI.</span><span class="sxs-lookup"><span data-stu-id="9704c-138">It takes audio and video streams and produces an AVI-formatted byte stream.</span></span>

<span data-ttu-id="9704c-139">Las distinciones entre estas categorías no son absolutas.</span><span class="sxs-lookup"><span data-stu-id="9704c-139">The distinctions between these categories are not absolute.</span></span> <span data-ttu-id="9704c-140">Por ejemplo, el filtro lector ASF actúa como filtro de origen y como filtro divisor.</span><span class="sxs-lookup"><span data-stu-id="9704c-140">For example, the ASF Reader filter acts as both a source filter and a splitter filter.</span></span>

<span data-ttu-id="9704c-141">Todos los filtros de DirectShow exponen la interfaz [**IBaseFilter**](/windows/desktop/api/Strmif/nn-strmif-ibasefilter) y todos los pin exponen la interfaz [**IPin**](/windows/desktop/api/Strmif/nn-strmif-ipin) .</span><span class="sxs-lookup"><span data-stu-id="9704c-141">All DirectShow filters expose the [**IBaseFilter**](/windows/desktop/api/Strmif/nn-strmif-ibasefilter) interface, and all pins expose the [**IPin**](/windows/desktop/api/Strmif/nn-strmif-ipin) interface.</span></span> <span data-ttu-id="9704c-142">DirectShow también define muchas otras interfaces que admiten una funcionalidad más específica.</span><span class="sxs-lookup"><span data-stu-id="9704c-142">DirectShow also defines many other interfaces that support more specific functionality.</span></span>
