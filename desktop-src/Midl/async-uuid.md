## <a name="remarks"></a><span data-ttu-id="4a230-121">Observaciones</span><span class="sxs-lookup"><span data-stu-id="4a230-121">Remarks</span></span>

<span data-ttu-id="4a230-123">Al aplicar el atributo de **\[ \_ UUID \] asincrónico** a una interfaz com (es decir, una interfaz que tiene el atributo de [**\[ objeto \]**](object.md) ), el compilador MIDL genera una definición asincrónica de la interfaz, además de la versión sincrónica tradicional.</span><span class="sxs-lookup"><span data-stu-id="4a230-123">When you apply the **\[async\_uuid\]** attribute to a COM interface (that is, an interface that has the [**\[object\]**](object.md) attribute), the MIDL compiler generates an asynchronous definition of the interface, in addition to the traditional, synchronous version.</span></span> <span data-ttu-id="4a230-124">La interfaz asincrónica tendrá los mismos nombres que la interfaz sincrónica, pero con un prefijo "Async".</span><span class="sxs-lookup"><span data-stu-id="4a230-124">The asynchronous interface will have the same names as the synchronous interface, but with an "Async" prefix.</span></span> <span data-ttu-id="4a230-125">El identificador de interfaz (IID) será el UUID especificado como un parámetro para el atributo **\[ Async \_ UUID \]** .</span><span class="sxs-lookup"><span data-stu-id="4a230-125">The interface identifier (IID) will be the UUID specified as a parameter to the **\[async\_uuid\]** attribute.</span></span>

<span data-ttu-id="4a230-126">En el caso de la interfaz asincrónica, MIDL divide cada método en métodos independientes de *Inicio* y *finalización* .</span><span class="sxs-lookup"><span data-stu-id="4a230-126">For the asynchronous interface, MIDL splits each method into separate *begin* and *finish* methods.</span></span> <span data-ttu-id="4a230-127">El método *Begin* tiene el nombre de método sincrónico con un \_ prefijo "begin" e incluye todos los parámetros [**\[ in \]**](in.md) del método sincrónico.</span><span class="sxs-lookup"><span data-stu-id="4a230-127">The *begin* method has the synchronous method name with a "Begin\_" prefix and includes all of the [**\[in\]**](in.md) parameters from the synchronous method.</span></span> <span data-ttu-id="4a230-128">El método *Finish* tiene el nombre del método sincrónico con un \_ prefijo "Finish" e incluye todos los parámetros [**\[ out \]**](out-idl.md) del método sincrónico.</span><span class="sxs-lookup"><span data-stu-id="4a230-128">The *finish* method has the synchronous method's name with a "Finish\_" prefix and includes all of the [**\[out\]**](out-idl.md) parameters from the synchronous method.</span></span> <span data-ttu-id="4a230-129">Si el método sincrónico tiene un parámetro **\[ in, out \] ,** se incluirán en los métodos asincrónicos de *Inicio* y *finalización* .</span><span class="sxs-lookup"><span data-stu-id="4a230-129">If the synchronous method has any **\[in, out\]** parameters they will be included in both the *begin* and *finish* asynchronous methods.</span></span>

<span data-ttu-id="4a230-130">Si un método de interfaz asincrónica tiene la [**\[ llamada \_ como \]**](call-as.md) atributo, MIDL generará declaraciones para los métodos *Begin* y *Finish* .</span><span class="sxs-lookup"><span data-stu-id="4a230-130">If an asynchronous interface method has the [**\[call\_as\]**](call-as.md) attribute, MIDL will generate declarations for the *begin* and the *finish* methods.</span></span> <span data-ttu-id="4a230-131">Debe implementar ambos métodos.</span><span class="sxs-lookup"><span data-stu-id="4a230-131">You must implement both methods.</span></span>

<span data-ttu-id="4a230-132">Cada interfaz asincrónica es un modificador en una interfaz sincrónica y, como tal, no tiene un gráfico de herencia independiente.</span><span class="sxs-lookup"><span data-stu-id="4a230-132">Each asynchronous interface is a modifier on a synchronous interface and, as such, does not have a separate inheritance graph.</span></span> <span data-ttu-id="4a230-133">Esto significa que no se puede definir una interfaz sincrónica a partir de una interfaz asincrónica (distinta de **IUnknown**).</span><span class="sxs-lookup"><span data-stu-id="4a230-133">This means that you cannot define a synchronous interface from an asynchronous interface (other than **IUnknown**).</span></span> <span data-ttu-id="4a230-134">Ni las interfaces sincrónicas se pueden heredar de interfaces asincrónicas.</span><span class="sxs-lookup"><span data-stu-id="4a230-134">Nor can synchronous interfaces inherit from asynchronous interfaces.</span></span> <span data-ttu-id="4a230-135">El compilador MIDL emitirá un mensaje de error si se intenta cualquiera de ellos.</span><span class="sxs-lookup"><span data-stu-id="4a230-135">The MIDL compiler will issue an error message if you attempt either one.</span></span>
