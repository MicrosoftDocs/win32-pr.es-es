# <a name="rmshutdownandrestart-controlevent"></a>RmShutdownAndRestart ControlEvent

Este evento notifica al instalador de Windows [](../rstmgr/restart-manager-portal.md) que use el Administrador de reinicio para apagar todas las aplicaciones que tienen archivos en uso y reiniciarlas al final de la instalación.

Este control ControlEvent no tiene ningún efecto si se cumple alguna de las condiciones siguientes.

-   El sistema operativo que ejecuta la instalación no Windows Vista ni Windows Server 2008.
-   La propiedad [](../rstmgr/restart-manager-portal.md) [**MSIRESTARTMANAGERCONTROL**](msirestartmanagercontrol.md) o la directiva [DisableAutomaticApplicationShutdown](disableautomaticapplicationshutdown.md) han deshabilitado las interacciones con el Administrador de reinicio.
-   Actualmente no hay ninguna sesión [de Restart Manager](../rstmgr/restart-manager-portal.md) abierta.
-   Las llamadas del instalador Windows al [Administrador de reinicio](../rstmgr/restart-manager-portal.md) devuelven un error.

## <a name="typical-use"></a>Uso típico

El evento de control RMShutdownAndRestart solo lo publica un control [PushButton](pushbutton-control.md) en el cuadro de diálogo [MsiRMFilesInUse.](msirmfilesinuse-dialog.md)
