# <a name="d1150-unsupported-mip-level-selection"></a>D1150: Selección de nivel de mip no admitida

El IDXGISurface2 proporcionado tiene varios niveles de mip, pero la selección de nivel de mip no se admite en este nivel de dispositivo (D3D9).






 

## <a name="possible-causes"></a>Causas posibles

Las texturas de Direct3D interoperadas pueden estar asignadas a mip; Direct2D solo funcionará en un nivel mip a la vez. Otros niveles mip de la textura primaria no se verán afectados ni representados por Direct2D. Sin embargo, los dispositivos de nivel de característica 9 no admiten la selección de nivel de mip y no pueden usar esta característica de interoperabilidad.

## <a name="possible-fixes"></a>Posibles correcciones

En el caso de las aplicaciones que se espera que se ejecuten correctamente en dispositivos de nivel de característica 9, considere la posibilidad de modificar la lógica de representación para evitar la necesidad de interoperar texturas mipmapped. Si es posible, interoperar un elemento intermedio en su lugar. Para minimizar los efectos de evitar esta limitación, la aplicación puede optar por implementarla como una ruta de acceso de reserva específica de nivel 9 de características.
