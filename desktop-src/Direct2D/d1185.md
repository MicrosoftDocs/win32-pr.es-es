# <a name="d1185-fillgeometry-opacity-mask-brush-restriction"></a>D1185: FillGeometry restricción de pincel de máscara de opacidad

Si FillGeometry se utiliza con una máscara de opacidad, el pincel principal debe ser un pincel de mapa de bits con compresión.






 

## <a name="possible-causes"></a>Causas posibles

Al usar ID2D1RenderTarget:: FillGeometry, el pincel principal era un tipo que no es ID2D1BitmapBrush. O bien, el pincel primario tenía un modo de extensión x o y distinto de la \_ abrazadera del modo de extensión D2D1 \_ \_ .

## <a name="possible-fixes"></a>Posibles correcciones

Si la aplicación usa FillGeometry con una máscara de opacidad, asegúrese de que el pincel primario (no debe confundirse con el pincel de opacidad) es un ID2D1BitmapBrush con modos de extensión con compresión. Si el comportamiento de enmascaramiento de opacidad no es importante, deje el argumento de pincel de opacidad como NULL para quitar estas restricciones en el pincel principal.
