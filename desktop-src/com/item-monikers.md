<span data-ttu-id="72b27-104">Otra clase moniker implementada por OLE es el *moniker del elemento*, que se puede usar para identificar un objeto contenido en otro objeto.</span><span class="sxs-lookup"><span data-stu-id="72b27-104">Another OLE-implemented moniker class is the *item moniker*, which can be used to identify an object contained in another object.</span></span> <span data-ttu-id="72b27-105">Un tipo de objeto contenido es un objeto OLE incrustado en un documento compuesto.</span><span class="sxs-lookup"><span data-stu-id="72b27-105">One type of contained object is an OLE object embedded in a compound document.</span></span> <span data-ttu-id="72b27-106">Un documento compuesto puede identificar los objetos incrustados que contiene asignando cada uno un nombre arbitrario, como "embedobj1", "embedobj2", etc.</span><span class="sxs-lookup"><span data-stu-id="72b27-106">A compound document could identify the embedded objects it contains by assigning each one an arbitrary name, such as "embedobj1," "embedobj2," and so forth.</span></span> <span data-ttu-id="72b27-107">Otro tipo de objeto contenido es una selección de usuario en un documento, como un rango de celdas de una hoja de cálculo o un intervalo de caracteres en un documento de texto.</span><span class="sxs-lookup"><span data-stu-id="72b27-107">Another type of contained object is a user selection in a document, such as a range of cells in a spreadsheet or a range of characters in a text document.</span></span> <span data-ttu-id="72b27-108">Un objeto que consta de una selección se denomina *pseudo objeto* porque no se trata como un objeto distinto hasta que un usuario marca la selección.</span><span class="sxs-lookup"><span data-stu-id="72b27-108">An object that consists of a class="sxs-lookup"><span data-stu-id="72b27-109">A spreadsheet might identify a cell range using a name such as "1A:7F," while a word processing document might identify a range of characters using the <span data-ttu-id="72b27-110">Un moniker de elemento es útil principalmente cuando se concatena, o se *crea*, con otro moniker, uno que identifica el contenedor.</span><span class="sxs-lookup"><span data-stu-id="72b27-110">An item moniker is useful primarily when concatenated, or *composed*, with another moniker, one that identifies the container.</span></span> <span data-ttu-id="72b27-111">Normalmente, un moniker de elemento se crea y se compone en (por ejemplo) un moniker de archivo para crear el equivalente de una ruta de acceso completa al objeto.</span><span class="sxs-lookup"><span data-stu-id="72b27-111">An item moniker is usually created and then composed onto (for example) a file moniker to create the equivalent of a complete path to the object.</span></span> <span data-ttu-id="72b27-112">Por ejemplo, puede crear el moniker de archivo "c: \\ work \\report.doc" (que identifica el objeto contenedor) con el moniker de elemento "embedobj1" (que identifica un objeto dentro del contenedor) para formar el moniker "c: \\ Work \\report.doc\\ embedobj1", que identifica de forma única un objeto determinado dentro de un archivo determinado.</span><span class="sxs-lookup"><span data-stu-id="72b27-112">For example, you can compose the file moniker "c:\\work\\report.doc" (which identifies the container object) with the item moniker "embedobj1" (which identifies an object within the container) to form the moniker "c:\\work\\report.doc\\embedobj1," which uniquely identifies a particular object within a particular file.</span></span> <span data-ttu-id="72b27-113">También puede concatenar monikers de elementos adicionales para identificar objetos profundamente anidados.</span><span class="sxs-lookup"><span data-stu-id="72b27-113">You can also concatenate additional item monikers to identify deeply nested objects.</span></span> <span data-ttu-id="72b27-114">Por ejemplo, si "embedobj1" es el nombre de un objeto de hoja de cálculo, para identificar un determinado intervalo de celdas en ese objeto de hoja de cálculo podría anexar otro moniker de elemento para crear un moniker que sería equivalente a "c: \\ work \\report.doc\\ Embedobj1 \\ 1A: 7F".</span><span class="sxs-lookup"><span data-stu-id="72b27-114">For example, if "embedobj1" is the name of a spreadsheet object, to identify a certain range of cells in that spreadsheet object you could append another item moniker to create a moniker that would be the equivalent of "c:\\work\\report.doc\\embedobj1\\1A:7F."</span></span>

<span data-ttu-id="72b27-115">Cuando se combina con un moniker de archivo, un moniker de elemento forma una ruta de acceso completa.</span><span class="sxs-lookup"><span data-stu-id="72b27-115">When combined with a file moniker, an item moniker forms a complete path.</span></span> <span data-ttu-id="72b27-116">Así, los monikers de elemento amplían la noción de nombres de ruta de acceso más allá del sistema de archivos, lo que define los nombres de ruta de acceso para identificar objetos individuales, no solo archivos.</span><span class="sxs-lookup"><span data-stu-id="72b27-116">Item monikers thus extend the notion of path names beyond the file system, defining path names to identify individual objects, not just files.</span></span>

<span data-ttu-id="72b27-117">Existe una diferencia significativa entre un moniker de elemento y un moniker de archivo.</span><span class="sxs-lookup"><span data-stu-id="72b27-117">There is a significant difference between an item moniker and a file moniker.</span></span> <span data-ttu-id="72b27-118">La ruta de acceso contenida en un moniker de archivo es significativa para cualquier persona que entienda el sistema de archivos, mientras que la ruta de acceso parcial contenida en un moniker de elemento solo es significativa para un contenedor determinado.</span><span class="sxs-lookup"><span data-stu-id="72b27-118">The path contained in a file moniker is meaningful to anyone who understands the file system, while the partial path contained in an item moniker is meaningful only to a particular container.</span></span> <span data-ttu-id="72b27-119">Todos saben qué "c: \\ work \\report.doc" hace referencia a, pero solo un objeto contenedor concreto sabe lo que hace referencia "1A: 7F".</span><span class="sxs-lookup"><span data-stu-id="72b27-119">Everyone knows what "c:\\work\\report.doc" refers to, but only one particular container object knows what "1A:7F" refers to.</span></span> <span data-ttu-id="72b27-120">Un contenedor no puede interpretar un moniker de elemento creado por otra aplicación. el único contenedor que sabe a qué objeto se hace referencia mediante un moniker de elemento es el contenedor que asignó el moniker de elemento al objeto en primer lugar.</span><span class="sxs-lookup"><span data-stu-id="72b27-120">One container cannot interpret an item moniker created by another application; the only container that knows which object is referred to by an item moniker is the container that assigned the item moniker to the object in the first place.</span></span> <span data-ttu-id="72b27-121">Por esta razón, el origen del objeto denominado por la combinación de un moniker de archivo y elemento no debe implementar solo [**IPersistFile**](/windows/desktop/api/ObjIdl/nn-objidl-ipersistfile), para facilitar el enlace del moniker de archivo, sino también [**IOleItemContainer**](/windows/desktop/api/OleIdl/nn-oleidl-ioleitemcontainer) para facilitar la resolución del nombre del moniker del elemento en el objeto adecuado, en el contexto de un archivo.</span><span class="sxs-lookup"><span data-stu-id="72b27-121">For this reason, the source of the object named by the combination of a file and item moniker must not only implement [**IPersistFile**](/windows/desktop/api/ObjIdl/nn-objidl-ipersistfile), to facilitate binding the file moniker, but also [**IOleItemContainer**](/windows/desktop/api/OleIdl/nn-oleidl-ioleitemcontainer) to facilitate resolving the name of the item moniker into the appropriate object, in the context of a file.</span></span>

<span data-ttu-id="72b27-122">La ventaja de los monikers es que alguien que use un moniker para buscar un objeto no necesita comprender el nombre contenido en el moniker del elemento, siempre que el moniker del elemento forme parte de un compuesto.</span><span class="sxs-lookup"><span data-stu-id="72b27-122">The advantage of monikers is that someone using a moniker to locate an object doesn't need to understand the name contained within the item moniker, as long as the item moniker is part of a composite.</span></span> <span data-ttu-id="72b27-123">Por lo general, no tendría sentido que el moniker de un elemento existiera por sí solo.</span><span class="sxs-lookup"><span data-stu-id="72b27-123">Generally, it would not make sense for an item moniker to exist on its own.</span></span> <span data-ttu-id="72b27-124">En su lugar, crearía un moniker de elemento en un moniker de archivo.</span><span class="sxs-lookup"><span data-stu-id="72b27-124">Instead, you would compose an item moniker onto a file moniker.</span></span> <span data-ttu-id="72b27-125">A continuación, llamaría a [**IMoniker:: BindToObject**](/windows/desktop/api/ObjIdl/nf-objidl-imoniker-bindtoobject) en el compuesto, que enlaza los monikers individuales dentro de él, interpretando los nombres.</span><span class="sxs-lookup"><span data-stu-id="72b27-125">You would then call [**IMoniker::BindToObject**](/windows/desktop/api/ObjIdl/nf-objidl-imoniker-bindtoobject) on the composite, which binds the individual monikers within it, interpreting the names.</span></span>

<span data-ttu-id="72b27-126">Para crear un objeto de moniker de elemento y devolver su puntero al proveedor de moniker, OLE proporciona la función auxiliar [**CreateItemMoniker**](/windows/desktop/api/Objbase/nf-objbase-createitemmoniker).</span><span class="sxs-lookup"><span data-stu-id="72b27-126">To create an item moniker object and return its pointer to the moniker provider, OLE provides the helper function [**CreateItemMoniker**](/windows/desktop/api/Objbase/nf-objbase-createitemmoniker).</span></span> <span data-ttu-id="72b27-127">Esta función crea un objeto de moniker de elemento y devuelve su puntero al proveedor.</span><span class="sxs-lookup"><span data-stu-id="72b27-127">This function creates an item moniker object and returns its pointer to the provider.</span></span>
