# <a name="object-namespaces"></a><span data-ttu-id="e64c2-103">Espacios de nombres de objetos</span><span class="sxs-lookup"><span data-stu-id="e64c2-103">Object Namespaces</span></span>

<span data-ttu-id="e64c2-104">Un *espacio de nombres de objeto* protege los objetos con nombre del acceso no autorizado.</span><span class="sxs-lookup"><span data-stu-id="e64c2-104">An *object namespace* protects named objects from unauthorized access.</span></span> <span data-ttu-id="e64c2-105">La creación de un espacio de nombres privado permite a las aplicaciones y los servicios crear un entorno más seguro.</span><span class="sxs-lookup"><span data-stu-id="e64c2-105">Creating a private namespace enables applications and services to build a more secure environment.</span></span>

<span data-ttu-id="e64c2-106">Un proceso puede crear un espacio de nombres privado mediante la función [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) .</span><span class="sxs-lookup"><span data-stu-id="e64c2-106">A process can create a private namespace using the [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) function.</span></span> <span data-ttu-id="e64c2-107">Esta función requiere que se especifique un *límite* que defina cómo se van a aislar los objetos del espacio de nombres.</span><span class="sxs-lookup"><span data-stu-id="e64c2-107">This function requires that you specify a *boundary* that defines how the objects in the namespace are to be isolated.</span></span> <span data-ttu-id="e64c2-108">El autor de la llamada debe estar dentro del límite especificado para que la operación de creación se realice correctamente.</span><span class="sxs-lookup"><span data-stu-id="e64c2-108">The caller must be within the specified boundary for the create operation to succeed.</span></span> <span data-ttu-id="e64c2-109">Para especificar un límite, use las funciones [**CreateBoundaryDescriptor**](/windows/desktop/api/WinBase/nf-winbase-createboundarydescriptora) y [**AddSIDToBoundaryDescriptor**](/windows/win32/api/namespaceapi/nf-namespaceapi-addsidtoboundarydescriptor) .</span><span class="sxs-lookup"><span data-stu-id="e64c2-109">To specify a boundary, use the [**CreateBoundaryDescriptor**](/windows/desktop/api/WinBase/nf-winbase-createboundarydescriptora) and [**AddSIDToBoundaryDescriptor**](/windows/win32/api/namespaceapi/nf-namespaceapi-addsidtoboundarydescriptor) functions.</span></span>

<span data-ttu-id="e64c2-110">El parámetro *lpAliasPrefix* de [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) sirve como nombre del espacio de nombres.</span><span class="sxs-lookup"><span data-stu-id="e64c2-110">The *lpAliasPrefix* parameter of [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) serves as the name of the namespace.</span></span> <span data-ttu-id="e64c2-111">Cada espacio de nombres se identifica de forma única mediante su nombre y sus límites.</span><span class="sxs-lookup"><span data-stu-id="e64c2-111">Each namespace is uniquely identified by its name and boundaries.</span></span> <span data-ttu-id="e64c2-112">El sistema admite varios espacios de nombres privados con el mismo nombre, siempre y cuando especifiquen límites diferentes.</span><span class="sxs-lookup"><span data-stu-id="e64c2-112">The system supports multiple private namespaces with the same name, as long as they specify different boundaries.</span></span>

<span data-ttu-id="e64c2-113">Supongamos que un proceso solicita la creación de un espacio de nombres, NS1, que define un límite que contiene dos elementos: el SID de administrador y el número de sesión actual.</span><span class="sxs-lookup"><span data-stu-id="e64c2-113">Suppose that a process requests the creation of a namespace, NS1, that defines a boundary containing two elements: the administrator SID and the current session number.</span></span> <span data-ttu-id="e64c2-114">El espacio de nombres se crea si el proceso se ejecuta con la cuenta de administrador en la sesión especificada.</span><span class="sxs-lookup"><span data-stu-id="e64c2-114">The namespace is created if the process is running under the Administrator account in the specified session.</span></span> <span data-ttu-id="e64c2-115">Otro proceso puede tener acceso a este espacio de nombres mediante la función [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) .</span><span class="sxs-lookup"><span data-stu-id="e64c2-115">Another process can access this namespace using the [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) function.</span></span> <span data-ttu-id="e64c2-116">Tanto el nombre como el límite especificados deben coincidir si este proceso es para abrir el espacio de nombres creado por el primer proceso.</span><span class="sxs-lookup"><span data-stu-id="e64c2-116">Both the specified name and boundary must match if this process is to open the namespace created by the first process.</span></span> <span data-ttu-id="e64c2-117">Tenga en cuenta que un proceso puede abrir un espacio de nombres existente aunque no esté dentro del límite a menos que el creador restrinja el acceso al espacio de nombres mediante el parámetro *lpPrivateNamespaceAttributes* .</span><span class="sxs-lookup"><span data-stu-id="e64c2-117">Note that a process can open an existing namespace even if it is not within the boundary unless the creator restricted access to the namespace using the *lpPrivateNamespaceAttributes* parameter.</span></span>

<span data-ttu-id="e64c2-118">Los objetos que se crean en este espacio de nombres tienen nombres con el formato *prefijo* \\ *objectname*.</span><span class="sxs-lookup"><span data-stu-id="e64c2-118">The objects that are created in this namespace have names that are of the form *prefix*\\*objectname*.</span></span> <span data-ttu-id="e64c2-119">El prefijo es el nombre del espacio de nombres especificado por el parámetro *lpAliasPrefix* de [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea).</span><span class="sxs-lookup"><span data-stu-id="e64c2-119">The prefix is the namespace name specified by the *lpAliasPrefix* parameter of [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea).</span></span> <span data-ttu-id="e64c2-120">Por ejemplo, para crear un objeto de evento denominado onEvent en el espacio de nombres de NS1, llame a la función [**CreateEvent**](/windows/win32/api/synchapi/nf-synchapi-createeventa) con el parámetro *LPNAME* establecido en ns1 \\ onEvent.</span><span class="sxs-lookup"><span data-stu-id="e64c2-120">For example, to create an event object named MyEvent in the NS1 namespace, call the [**CreateEvent**](/windows/win32/api/synchapi/nf-synchapi-createeventa) function with the *lpName* parameter set to NS1\\MyEvent.</span></span>

<span data-ttu-id="e64c2-121">El proceso que creó el espacio de nombres puede usar la función [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) para cerrar el identificador del espacio de nombres.</span><span class="sxs-lookup"><span data-stu-id="e64c2-121">The process that created the namespace can use the [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) function to close the handle to the namespace.</span></span> <span data-ttu-id="e64c2-122">El identificador también se cierra cuando finaliza el proceso que creó el espacio de nombres.</span><span class="sxs-lookup"><span data-stu-id="e64c2-122">The handle is also closed when the process that created the namespace terminates.</span></span> <span data-ttu-id="e64c2-123">Una vez cerrado el identificador del espacio de nombres, las llamadas subsiguientes a [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) generan un error, pero todas las operaciones en los objetos del espacio de nombres se realizan correctamente.</span><span class="sxs-lookup"><span data-stu-id="e64c2-123">After the namespace handle is closed, subsequent calls to [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) fail, but all operations on objects in the namespace succeed.</span></span>
