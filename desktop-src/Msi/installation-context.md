# <a name="installation-context"></a>Contexto de instalación

Windows Installer puede instalar un paquete en un equipo en dos contextos de instalación: por equipo y por usuario. Se requiere una instalación por equipo del paquete para permitir que todos los usuarios del equipo tengan acceso a la aplicación y la utilicen. Dado que una instalación por equipo realiza cambios en el sistema que afectan a todos los usuarios, se puede impedir que los usuarios estándar con privilegios limitados instalen un paquete en el contexto por equipo sin obtener primero el permiso.

Puede especificar el contexto de instalación mediante la creación del paquete para la instalación por usuario o por equipo y mediante las propiedades [**AllUsers**](allusers.md) y [**MSIINSTALLPERUSER**](msiinstallperuser.md) . En función de estas propiedades, Windows Installer redirige automáticamente los valores de las propiedades de la carpeta y los registros a las ubicaciones del contexto por usuario o por equipo.

> [!Note]  
> La propiedad [**MSIINSTALLPERUSER**](msiinstallperuser.md) , disponible a partir de Windows Installer 5,0 y windows Server 2008 R2 y Windows 7, puede facilitar el desarrollo de un único paquete capaz de instalarse en el contexto por equipo o por usuario. Para obtener información sobre el desarrollo de un paquete de doble uso que proporciona al usuario la capacidad de elegir el contexto de instalación en el momento de la instalación, vea [creación de un solo paquete](single-package-authoring.md). Windows Installer omite la propiedad **MSIINSTALLPERUSER** si el valor de [**AllUsers**](allusers.md) no es 2. Windows Installer restablece siempre el valor de **AllUsers** en 1 cuando se instala en el contexto por equipo y restablece el valor de **AllUsers** en una cadena vacía ("") cuando se instala en el contexto por usuario.

| Los métodos abreviados se instalan en el perfil todos los usuarios.<br/>                                              | Los accesos directos solo se instalan en el perfil de ese usuario.<br/>                                                                 |
| Los iconos y las transformaciones se almacenan en% WINDOWS% \\ Installer \\ {ProductCode}.<br/>                        | Los iconos y las transformaciones se almacenan en% USERPROFILE% \\ Application Data \\ Microsoft \\ Installer \\ {ProductCode GUID}<br/>         |

## <a name="folder-redirection"></a>Redirección de carpetas

> [!Note]  
> Las carpetas se identifican mediante sus constantes [**KNOWNFOLDERID**](../shell/knownfolderid.md) y [**CSIDL**](/windows/win32/api/shlobj_core/nf-shlobj_core-shgetfolderpatha) . A partir de Windows Vista, las aplicaciones deben usar la función [**SHGetKnownFolderPath**](/windows/win32/api/shlobj_core/nf-shlobj_core-shgetknownfolderpath) y **KNOWNFOLDERID** para determinar la ruta de acceso completa a las carpetas especiales. Las aplicaciones existentes que usan la función **SHGetFolderPath** y los identificadores de elementos especiales ([**CSIDL**](/windows/win32/api/shlobj_core/nf-shlobj_core-shgetfolderpatha)) seguirán funcionando.
