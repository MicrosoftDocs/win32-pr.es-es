# <a name="d1251-d3d-state-issue"></a>D1251: problema de estado de D3D

El estado del dispositivo Direct3D cambió cuando Direct2D todavía lo estaba usando. Puede ser indicativo de una infracción de subprocesamiento, ya que solo un subproceso puede usar el dispositivo a la vez.






 

## <a name="possible-causes"></a>Causas posibles

Es posible que la aplicación use un único generador de D2D de subprocesos en dos subprocesos diferentes al mismo tiempo.

## <a name="possible-fixes"></a>Posibles correcciones

Asegúrese de que la aplicación respeta correctamente las directivas de subprocesos de Direct2d.
