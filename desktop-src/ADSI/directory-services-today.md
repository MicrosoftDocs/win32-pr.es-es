# <a name="directory-services-today"></a>Servicios de directorio hoy

Es habitual encontrar varios directorios administrativos implementados en una sola organización. Estos directorios incluyen directorios de recursos de red, como los directorios basados en LDAP, como Microsoft Active Directory Directory Service, el servicio de directorio del sistema operativo Windows, así como directorios específicos de la aplicación, como Microsoft Exchange.

![implementación de varios directorios](images/ds2chal.png)

## <a name="the-directory-challenge"></a>El desafío del directorio

Varios directorios de la organización plantean desafíos complejos a los usuarios, administradores y desarrolladores. Estos problemas tienen una implementación limitada del directorio amplio. Los usuarios se ven con varios inicios de sesión y una variedad de interfaces con información en varios directorios. Los administradores se comparan en la complejidad de la administración de varios directorios. Los usuarios finales y los administradores quieren que los desarrolladores de aplicaciones utilicen un directorio administrativo existente, pero los desarrolladores se ven con un dilema sobre cuál usar. Cada directorio ofrece interfaces de aplicación únicas. Un desarrollador debe elegir una implementación de directorio específica o admitir varias versiones de su aplicación. Como resultado, los desarrolladores rara vez usan servicios de directorio existentes.
