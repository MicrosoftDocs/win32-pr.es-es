# <a name="m-isolated-applications-and-side-by-side-assemblies"></a>M (aplicaciones aisladas y ensamblados en paralelo)

[A](a-sbscs-gly.md) B C [D](d-sbscs-gly.md) E F [G](g-sbscs-gly.md) H [I](i-sbscs-gly.md) J K L M N o [p](p-sbscs-gly.md) Q [R](r-sbscs-gly.md) [s](s-sbscs-gly.md) T U V W X Y Z

<dl> <dt>

<span id="_win32_manifest_file_gly"></span><span id="_WIN32_MANIFEST_FILE_GLY"></span>**archivo de manifiesto**
</dt> <dd>

Archivo XML que describe un ensamblado en paralelo o una aplicación aislada. Un archivo de manifiesto puede contener metadatos de enlace y activación almacenados tradicionalmente en el registro, como clases, interfaces y bibliotecas de tipos COM. Cada aplicación aislada o ensamblado en paralelo está acompañado de un archivo de manifiesto. Un manifiesto de aplicación describe una aplicación totalmente o parcialmente aislada y un manifiesto del ensamblado describe un ensamblado en paralelo.

</dd> <dt>

<span id="_win32_metadata_gly"></span><span id="_WIN32_METADATA_GLY"></span>**repositorio**
</dt> <dd>

Información almacenada en un archivo de manifiesto sobre el enlace o activación de una aplicación aislada o un ensamblado en paralelo. Por ejemplo, la información sobre las clases COM, las interfaces y las bibliotecas de tipos, tradicionalmente almacenadas en el registro, se puede almacenar como metadatos en un manifiesto. Los metadatos describen los elementos que componen un ensamblado y sus dependencias en otros ensamblados en paralelo. Los metadatos especificados en el manifiesto de aplicación y los ensamblados dependientes se pueden usar para rellenar el contexto de activación.

</dd> </dl>
