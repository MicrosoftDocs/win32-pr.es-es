# <a name="prov_dss"></a>DSS de PROV \_

El tipo de proveedor de DSS de PROV \_ , como Prov \_ RSA \_ SIG, solo admite [*hashes*](../secgloss/h-gly.md) y [*firmas digitales*](../secgloss/d-gly.md). El algoritmo de firma especificado por el \_ tipo de proveedor de DSS DSS es el [*algoritmo de firma digital*](../secgloss/d-gly.md) (DSA).

## <a name="algorithms-supported"></a>Algoritmos admitidos

Para obtener descripciones de cada uno de estos algoritmos, vea el glosario.



| Propósito      | Algoritmos admitidos                                                                                                              |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------|
| Intercambio de claves | None                                                                                                                              |
| Firma    | [*DSS*](../secgloss/d-gly.md)                                                                       |
| Cifrado   | Ninguno                                                                                                                              |
| Aplicación de algoritmo hash      | [*MD5*](../secgloss/m-gly.md)[*Sha*](../secgloss/s-gly.md)<br/> |



 

## <a name="related-documentation"></a>Documentación relacionada

El DSA fue propuesto por el [*Instituto Nacional de estándares y Tecnología*](../secgloss/n-gly.md) (NIST). Una descripción del algoritmo se puede encontrar en la siguiente referencia gubernamental: "propuesto Estándar federal de procesamiento de información para el estándar de firma digital (DSS)," Federal Register, v. 57, no. 21, 31 de enero de 1992, PP. 3747-3749.
