# <a name="call-center-control"></a>Control de centro de llamadas

TapI 2.2 básico se puede usar para administrar centros de llamadas y otros elementos de la infraestructura de red de telefonía (como servidores de correo de voz y IVR) a través del control de llamadas de terceros. Para ayudar a crear servidores proxy de ACD que se ejecutarán en servidores, se han agregado funciones a TAPI 2.2 para proporcionar asistencia adicional a los desarrolladores de aplicaciones.

En los temas siguientes se describen las características de TAPI 2.2 que puede usar para implementar controles de centro de llamadas:

-   [Modelado de un centro de llamadas](modeling-of-a-call-center.md)
-   [Estaciones](stations.md)
-   [Marcado predictivo](predictive-dialing.md)
-   [Colas de llamadas y puntos de ruta](call-queues-and-route-points.md)
-   [Supervisión y control del agente de ACD](acd-agent-monitoring-and-control.md)
-   [Control de estado de la estación](station-status-control.md)
-   [Control de estado de la estación](station-status-control.md)
-   [Temporizador de estado de llamada](call-state-timer.md)
-   [Temporizadores de eventos multimedia](media-event-timers.md)

TAPI 3.x es la API preferida para escribir aplicaciones del agente ACD. Consulte la sección [Controles del centro de](./about-call-center-controls.md) llamadas para obtener información sobre el uso de estas interfaces y métodos. Sin embargo, TAPI 2.2 sigue siendo utilizable para conjuntos de aplicaciones ACD completos si lo desea.
