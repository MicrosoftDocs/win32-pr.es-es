# <a name="windows-8-and-windows-server-2012-client-and-server-compatibility"></a>Windows 8 y Windows Server 2012 cliente y servidor

En esta sección se describen los cambios en el sistema operativo a los que debe prestar especial atención debido a los posibles impactos en las aplicaciones existentes y cómo se deben diseñar nuevas aplicaciones en clientes, en servidores o en clientes y servidores. A continuación se muestra la lista de temas que se tratan en esta sección, agrupada por área de características:

**AppCompat**

-   Actualización de las reglas de detección de aplicaciones de seguridad
-   Determinar el estado de corrección de compatibilidad (shim)
-   Las aplicaciones de servidor deben poder ejecutarse sin el Shell gráfico de servidor
-   Los componentes del servidor RDS se quitan de Windows 8
-   Modelo de asociaciones de protocolo y tipo de archivo
-   Moderador de la actividad del escritorio
-   .NET Framework 4.5 es el valor predeterminado y .NET Framework 3.5 es opcional
-   Es posible que las aplicaciones de escritorio no sean visibles después de iniciar el explorador web predeterminado
-   Modo de contraste alto
-   Manifiesto de aplicación (ejecutable)
-   El modelo presente en cola está en desuso
-   Escenarios del Asistente de compatibilidad de programas para Windows 8
-   Dispositivos de escritorio eliminados

**Storage y sistema de archivos**

-   Actualización de compatibilidad de disco de formato avanzado (4K)
-   Aprovisionamiento fino de unidades lógicas
-   El almacenamiento mejorado ahora es opcional para Windows entorno de preinstalación (WinPE) y SKU de servidor
-   Virtual Disk Service está transfiriendo al Windows de instrumental de administración de recursos (WMI) basado en Windows Storage API de Administración
-   Interfaz de usuario de versiones anteriores quitada para volúmenes locales
-   StorAHCI reemplaza a MSAHCI
-   Windows 7 copias de seguridad y restauración en desuso
-   Transferencias de datos descargados

**Otros**

-   Administrador de ventanas de escritorio está siempre on
-   Direct2D no admite la representación en metarchivos "enriquecidos" en Internet Explorer 9
-   Cambios en la compatibilidad con hardware heredado dx9
-   MSAA no está disponible para aplicaciones Windows Store
-   El puerto 3 está en desuso para los controladores NDIS 6.30
