<dl> <dt>

<span id="_rras_cyclic_redundancy_check_crc__gly"></span><span id="_RRAS_CYCLIC_REDUNDANCY_CHECK_CRC__GLY"></span>**Comprobación de redundancia cíclica (CRC)**
</dt> <dd>

Un CRC es un valor numérico relacionado con un bloque de datos. Este valor proporciona información que permite que una aplicación determine si se han modificado los datos. Una CRC es similar a una suma de comprobación, pero se considera un mecanismo de comprobación de errores más fuerte.

</dd> <dt>

<span id="_rras_compression_control_protocol_ccp__gly"></span><span id="_RRAS_COMPRESSION_CONTROL_PROTOCOL_CCP__GLY"></span>**Protocolo de control de compresión (CCP)**
</dt> <dd>

El protocolo de control de compresión (CCP) permite que dos equipos que se comunican a través de PPP negocien algoritmos compatibles para enviar y recibir tramas PPP comprimidas. Es posible que los dos equipos no utilicen CCP hasta la fase Network-Control-Protocol de la conexión PPP.

Para obtener más información, consulte [RFC 1962](remote-access-request-for-comments.md). Consulte también [**\_ \_ información de CCP de PPP**](/windows/desktop/api/Mprapi/ns-mprapi-ppp_ccp_info) y [**RASPPPCCP**](/previous-versions/windows/desktop/legacy/aa377620(v=vs.85)).

</dd> </dl>
