# <a name="documenting-schema-extensions"></a>Documentar extensiones de esquema

Además de proporcionar un procedimiento de instalación independiente para las extensiones de esquema, la naturaleza de las extensiones de esquema debe estar claramente documentada. La documentación debe contener:

-   Una lista de clases y atributos nuevos con una explicación de cada uno
-   Una lista de las clases estándar que se extienden con nuevos atributos y una lista de los atributos.
-   Una lista de las nuevas páginas de propiedades con una explicación de cómo utilizar cada una
-   Explicación de cómo el servicio utiliza las extensiones de esquema
