# <a name="properties-and-property-sets"></a><span data-ttu-id="b14e1-104">Propiedades y conjuntos de propiedades</span><span class="sxs-lookup"><span data-stu-id="b14e1-104">Properties and Property Sets</span></span>

<span data-ttu-id="b14e1-105">Aunque los tipos de propiedades de tiempo de ejecución que ofrecen Automation y los controles ActiveX de Microsoft son importantes, no solucionan directamente la necesidad de almacenar información con objetos almacenados de forma persistente en el sistema de archivos.</span><span class="sxs-lookup"><span data-stu-id="b14e1-105">While the kinds of run-time properties that Automation and Microsoft ActiveX Controls offer are important, they do not directly address the need to store information with objects persistently stored in the file system.</span></span> <span data-ttu-id="b14e1-106">Estas entidades podrían incluir archivos (estructurados, compuestos, etc.), directorios y catálogos de resumen.</span><span class="sxs-lookup"><span data-stu-id="b14e1-106">These entities could include files (structured, compound, and so forth), directories, and summary catalogs.</span></span> <span data-ttu-id="b14e1-107">COM proporciona tanto un formato serializado estándar para estas propiedades persistentes como un conjunto de interfaces y funciones que permiten crear y manipular los conjuntos de propiedades y sus propiedades.</span><span class="sxs-lookup"><span data-stu-id="b14e1-107">COM provides both a standard serialized format for these persistent properties, and a set of interfaces and functions that allow you to create and manipulate the property sets and their properties.</span></span>

<span data-ttu-id="b14e1-108">Las propiedades persistentes se almacenan como conjuntos y uno o más conjuntos pueden estar asociados a una entidad del sistema de archivos.</span><span class="sxs-lookup"><span data-stu-id="b14e1-108">Persistent properties are stored as sets, and one or more sets may be associated with a file system entity.</span></span> <span data-ttu-id="b14e1-109">Estos conjuntos de propiedades persistentes están diseñados para almacenar datos que se pueden representar como una colección de valores específicos.</span><span class="sxs-lookup"><span data-stu-id="b14e1-109">These persistent property sets are intended to be used to store data that is suited to being represented as a collection of fine-grained values.</span></span> <span data-ttu-id="b14e1-110">No están diseñados para usarse como base de datos de gran tamaño.</span><span class="sxs-lookup"><span data-stu-id="b14e1-110">They are not intended to be used as a large data base.</span></span> <span data-ttu-id="b14e1-111">Se pueden usar para almacenar información de resumen sobre un objeto en el sistema, al que puede tener acceso cualquier otro objeto que entienda cómo interpretar ese conjunto de propiedades.</span><span class="sxs-lookup"><span data-stu-id="b14e1-111">They can be used to store summary information about an object on the system, which can then be accessed by any other object that understands how to interpret that property set.</span></span>

<span data-ttu-id="b14e1-112">En las versiones anteriores de COM se especificaba muy poco con respecto a las propiedades y su uso, pero se definió un formato serializado que permitía a los desarrolladores almacenar propiedades y conjuntos de propiedades en una instancia de [**IStorage**](/windows/desktop/api/Objidl/nn-objidl-istorage) .</span><span class="sxs-lookup"><span data-stu-id="b14e1-112">Previous versions of COM specified very little with respect to properties and their usage, but did define a serialized format that allowed developers to store properties and property sets in an [**IStorage**](/windows/desktop/api/Objidl/nn-objidl-istorage) instance.</span></span> <span data-ttu-id="b14e1-113">También se definieron los identificadores de propiedad y la semántica de un único conjunto de propiedades, que se usa para obtener información de resumen sobre un documento.</span><span class="sxs-lookup"><span data-stu-id="b14e1-113">The property identifiers and semantics of a single property set, used for summary information about a document, were also defined.</span></span> <span data-ttu-id="b14e1-114">En ese momento, era necesario crear y manipular directamente esa estructura como un flujo de datos.</span><span class="sxs-lookup"><span data-stu-id="b14e1-114">At that time, it was necessary to create and manipulate that structure directly as a data stream.</span></span> <span data-ttu-id="b14e1-115">Consulte [formato del conjunto de propiedades serializado de almacenamiento estructurado](structured-storage-serialized-property-set-format.md).</span><span class="sxs-lookup"><span data-stu-id="b14e1-115">See [Structured Storage Serialized Property Set Format](structured-storage-serialized-property-set-format.md).</span></span>

<span data-ttu-id="b14e1-116">Sin embargo, ahora COM define dos interfaces principales para administrar los conjuntos de propiedades:</span><span class="sxs-lookup"><span data-stu-id="b14e1-116">Now, however, COM defines two primary interfaces to manage property sets:</span></span>

-   [<span data-ttu-id="b14e1-117">**IPropertyStorage**</span><span class="sxs-lookup"><span data-stu-id="b14e1-117">**IPropertyStorage**</span></span>](/windows/desktop/api/Propidl/nn-propidl-ipropertystorage)
-   [<span data-ttu-id="b14e1-118">**IPropertySetStorage**</span><span class="sxs-lookup"><span data-stu-id="b14e1-118">**IPropertySetStorage**</span></span>](/windows/desktop/api/Propidl/nn-propidl-ipropertysetstorage)

<span data-ttu-id="b14e1-119">Ya no es necesario tratar el formato serializado directamente cuando estas interfaces se implementan en un objeto que admite la interfaz [**IStorage**](/windows/desktop/api/Objidl/nn-objidl-istorage) (como archivos compuestos).</span><span class="sxs-lookup"><span data-stu-id="b14e1-119">It is no longer necessary to deal with the files).</span></span> <span data-ttu-id="b14e1-120">La escritura de propiedades a través de [**IPropertySetStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertysetstorage) y [**IPropertyStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertystorage) crea datos que se ajustan exactamente al formato del conjunto de propiedades com, tal como se ve a través de métodos **IStorage** .</span><span class="sxs-lookup"><span data-stu-id="b14e1-120">Writing properties through [**IPropertySetStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertysetstorage) and [**IPropertyStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertystorage) creates data that exactly conforms to the COM property set format, as viewed through **IStorage** de **IPropertySetStorage** y **IPropertyStorage** (aunque no se puede esperar escribir en [**IStream**](/windows/desktop/api/Objidl/nn-objidl-istream) y tener las propiedades a través de **IPropertyStorage** disponibles de inmediato, o viceversa).</span><span class="sxs-lookup"><span data-stu-id="b14e1-121">The converse is also true — properties written to the COM property set format the properties through **IPropertyStorage** immediately available, or vice versa).</span></span>

<span data-ttu-id="b14e1-122">La interfaz [**IPropertySetStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertysetstorage) define métodos que crean y administran conjuntos de propiedades.</span><span class="sxs-lookup"><span data-stu-id="b14e1-122">The [**IPropertySetStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertysetstorage) interface defines methods that these interfaces, an application developer can manage whatever property sets are appropriate for a given file system entity.</span></span> <span data-ttu-id="b14e1-125">El uso de estas interfaces proporciona una implementación optimizada de lectura y escritura para las propiedades, en lugar de tener una implementación en cada aplicación, donde podría haber cuellos de botella de rendimiento como la búsqueda de incessant.</span><span class="sxs-lookup"><span data-stu-id="b14e1-125">Use of these interfaces provides one tuned reading and writing implementation for properties, rather than mejorar el rendimiento, por lo que las propiedades se pueden leer y escribir más rápidamente mediante, por ejemplo, un almacenamiento en caché más eficaz.</span><span class="sxs-lookup"><span data-stu-id="b14e1-126">You can implement the interfaces to enhance performance, so properties can be read and written more quickly by, for example, more efficient caching.</span></span> <span data-ttu-id="b14e1-127">Además, **IPropertyStorage** y **IPropertySetStorage** permiten manipular propiedades en entidades que no admiten [**IStorage**](/windows/desktop/api/Objidl/nn-objidl-istorage), aunque en general, la mayoría de las aplicaciones no lo harán.</span><span class="sxs-lookup"><span data-stu-id="b14e1-127">Furthermore, **IPropertyStorage** and **IPropertySetStorage** make it possible to manipulate properties on entities that do not support [**IStorage**](/windows/desktop/api/Objidl/nn-objidl-istorage), although in
