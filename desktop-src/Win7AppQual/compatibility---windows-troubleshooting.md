# <a name="windows-troubleshooting"></a>Solución de problemas de Windows

## <a name="affected-platforms"></a>Plataformas afectadas

**Clientes** : Windows 7  
**Servidores** : Windows Server 2008 R2  









## <a name="description"></a>Descripción

Una nueva característica del centro de actividades de Windows 7 (conocido anteriormente como centro de soluciones), solución de problemas de Windows, ofrece una experiencia de solución de problemas sistemática para el usuario. El centro de actividades es uno de los cinco iconos anclados en la Systray. La solución de problemas de Windows le permite examinar o buscar paquetes de solución de problemas integrados y para solucionar problemas de los paquetes que se almacenan en un servidor de Microsoft en Internet, una experiencia mejor cuando está conectado (BWC). Puede seleccionar y ejecutar un paquete de solución de problemas para intentar resolver el problema. Si no puede identificar una solución para el problema, tiene la opción de buscar ayuda, contenido de la comunidad y artículos de soporte técnico, u otros paquetes de solución de problemas para el contenido relacionado pertinente. Si no proporciona una respuesta al problema, puede restaurar el equipo a un momento anterior a la hora en que se produjo el problema u obtener ayuda a través de asistencia remota. El objetivo es permitirle encontrar una solución al problema con facilidad y rapidez.

## <a name="usage"></a>Uso

El centro de actividades es claramente visible y está disponible desde varias ubicaciones. Puede iniciarlo desde el menú contextual del centro de actividades de la Systray, desde el panel de control como un vínculo de acceso directo en sistema y mantenimiento, desde la Página principal del centro de actividades y desde el contenido de la ayuda.

Los paquetes de solución de problemas se basan en scripts de PowerShell. El proceso de creación y publicación de un paquete de solución de problemas se hará público para que los OEM, IHV, ISV y profesionales de ti puedan desarrollar y enviar su propio contenido de solución de problemas.

Para obtener una experiencia de usuario coherente, asegúrese de seguir los procedimientos recomendados y las directrices que se describen en el kit de herramientas de solución de problemas de Windows al diseñar sus propios paquetes de solución de problemas.
