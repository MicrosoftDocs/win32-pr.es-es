# <a name="d1181-enddraw-incorrect-state"></a>D1181: EndDraw estado incorrecto

Se llamó a EndDraw, pero el estado de rendertarget no era válido. Esto puede deberse a la llamada a EndDraw sin un BeginDraw coincidente.






 

## <a name="possible-causes"></a>Causas posibles

La aplicación realizó varias llamadas a EndDraw sin llamar a BeginDraw en el mismo destino. Al usar Direct2D, las aplicaciones deben coincidir con cada llamada a EndDraw con una llamada anterior a BeginDraw. Como alternativa, este mensaje puede producirse si el destino está en un estado erróneo y no se pudo representar.

## <a name="possible-fixes"></a>Posibles correcciones

Asegúrese de que cada llamada a BeginDraw coincida con una EndDraw subsiguiente. Al enviar comandos a un destino mediante flush o EndDraw, compruebe que el resultado devuelto es S \_ OK antes de intentar volver a usar el destino.
