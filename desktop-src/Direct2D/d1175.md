# <a name="d1175-dxgi-target-must-not-use-software-flag"></a>D1175: el destino de DXGI no debe usar la marca de software

Este tipo de rendertarget debe marcarse como predeterminado o como HARDWARE.






 

## <a name="possible-causes"></a>Causas posibles

Una aplicación denominada ID2D1Factory:: CreateDxgiSurfaceRenderTarget con un \_ objeto de propiedades de destino de representación D2D1 \_ \_ cuyo campo de tipo se estableció en D2D1 \_ representación de software de tipo de \_ destino \_ \_ . Direct2D considera que esta combinación de especificadores es incorrecta, porque la marca de SOFTWARE está reservada para destinos creados con CreateWicBitmapRenderTarget y CreateHwndRenderTarget.

## <a name="possible-fixes"></a>Posibles correcciones

Al usar de CreateDxgiSurfaceRenderTarget, las aplicaciones deben especificar el tipo de destino de representación D2D1 de \_ \_ \_ \_ hardware.
