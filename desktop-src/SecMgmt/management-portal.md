# <a name="security-management"></a>Administración de seguridad

## <a name="purpose"></a>Propósito

Las tecnologías de administración de seguridad se pueden usar para administrar la Directiva de la Directiva de [*seguridad local*](/windows/desktop/SecGloss/l-gly) (LSA) y la Directiva de [*filtro de contraseña*](/windows/desktop/SecGloss/p-gly) , consultar la capacidad de los programas de orígenes externos y datos adjuntos de servicio que amplían la funcionalidad de la herramienta de configuración de seguridad.

Las tecnologías de administración de seguridad de Microsoft incluyen la API de directiva de LSA, la API de filtro de contraseñas, la API más segura y la API de datos adjuntos de seguridad del servicio. Estas tecnologías permiten a los desarrolladores de software crear aplicaciones que administran sistemas y aplicaciones.

## <a name="developer-audience"></a>Audiencia de desarrolladores

La API de directiva de LSA, la API de filtro de contraseñas, la API más segura y la API de datos adjuntos de seguridad del servicio están pensadas para que las usen los desarrolladores de aplicaciones que permiten a los administradores administrar y proteger sus sistemas. Los desarrolladores deben estar familiarizados con los lenguajes de programación de C y C++ y con la programación basada en Windows.

Para obtener información sobre los requisitos de tiempo de ejecución para un elemento de programación determinado, vea la sección de requisitos de la página de referencia de ese elemento.
