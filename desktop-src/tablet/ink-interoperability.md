# <a name="ink-interoperability"></a>Interoperabilidad de entradas manuscritas

La persistencia y la interoperabilidad son dos características de vital importancia, especialmente para las aplicaciones que usan entradas manuscritas. Para habilitar la persistencia y la interoperabilidad de la entrada manuscrita, una aplicación debe permitir a los usuarios:

-   Guarde y abra sus documentos con plena fidelidad de los datos de tinta.
-   Mueva los datos hacia y desde otras aplicaciones basadas en Microsoft Windows mediante el portapapeles.

En las secciones siguientes se detalla la interoperabilidad de entradas manuscritas.

-   [Formatos de datos de tinta](ink-data-formats.md)
-   [Escenarios de interoperabilidad importantes](important-interoperability-scenarios.md)
-   [Formatos de persistencia](persistence-formats.md)
-   [Elección de los formatos admitidos](choosing-which-formats-to-support.md)
-   [Bloques de creación](building-blocks.md)
-   [Almacenar entradas manuscritas en HTML](storing-ink-in-html.md)

En este documento se da por supuesto un conocimiento detallado del portapapeles. Para obtener más información sobre el portapapeles, vea la [documentación del portapapeles](/windows/desktop/dataxchg/clipboard).
