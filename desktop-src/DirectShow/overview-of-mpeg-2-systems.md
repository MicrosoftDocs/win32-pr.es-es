# <a name="overview-of-mpeg-2-systems"></a><span data-ttu-id="15dc6-103">Información general de los sistemas MPEG-2</span><span class="sxs-lookup"><span data-stu-id="15dc6-103">Overview of MPEG-2 Systems</span></span>

<span data-ttu-id="15dc6-104">En esta sección se ofrece una descripción general no técnica de la capa de sistemas MPEG-2.</span><span class="sxs-lookup"><span data-stu-id="15dc6-104">This section gives a general, non-technical overview of the MPEG-2 Systems layer.</span></span> <span data-ttu-id="15dc6-105">MPEG-2 es el estándar que define cómo se multiplexan las secuencias de audio y vídeo en MPEG-2.</span><span class="sxs-lookup"><span data-stu-id="15dc6-105">MPEG-2 Systems is the standard that defines how audio and video streams are multiplexed in MPEG-2.</span></span>

<span data-ttu-id="15dc6-106">**Flujos elementales**</span><span class="sxs-lookup"><span data-stu-id="15dc6-106">**Elementary Streams**</span></span>

<span data-ttu-id="15dc6-107">La multiplexación MPEG-2 comienza con uno o varios flujos de bytes, denominados secuencias elementales, que contienen vídeo, audio u otros datos.</span><span class="sxs-lookup"><span data-stu-id="15dc6-107">MPEG-2 multiplexing starts with one or more byte streams, called elementary streams (ES), that contain video, audio, or other data.</span></span> <span data-ttu-id="15dc6-108">Por ejemplo, un vídeo ES contiene fotogramas de vídeo comprimidos, más encabezados de secuencia, encabezados de grupo de imágenes (GOP) y cualquier otro elemento que necesite el descodificador para descodificar la secuencia.</span><span class="sxs-lookup"><span data-stu-id="15dc6-108">For example, a video ES contains compressed video frames, plus sequence headers, group-of-picture (GOP) headers, and anything else needed by the decoder to decode the stream.</span></span> <span data-ttu-id="15dc6-109">La capa de sistemas no define el contenido de la secuencia de bytes ES.</span><span class="sxs-lookup"><span data-stu-id="15dc6-109">The Systems layer does not define the contents of the ES byte stream.</span></span>

<span data-ttu-id="15dc6-110">Una secuencia elemental se divide en paquetes, formando una *secuencia elemental* (PES) con paquetes.</span><span class="sxs-lookup"><span data-stu-id="15dc6-110">An elementary stream is broken up into packets, forming a *packetized elementary stream* (PES).</span></span> <span data-ttu-id="15dc6-111">Los paquetes PES tienen una longitud variable.</span><span class="sxs-lookup"><span data-stu-id="15dc6-111">PES packets have variable length.</span></span> <span data-ttu-id="15dc6-112">El contenido del paquete se denomina *carga*.</span><span class="sxs-lookup"><span data-stu-id="15dc6-112">The contents of the packet are called the *payload*.</span></span> <span data-ttu-id="15dc6-113">Cada paquete de PES también incluye un encabezado.</span><span class="sxs-lookup"><span data-stu-id="15dc6-113">Each PES packet also includes a header.</span></span> <span data-ttu-id="15dc6-114">El multiplexor asigna un identificador de secuencia de 1 byte a cada PES; los paquetes PES individuales se identifican mediante el identificador de secuencia en el encabezado del paquete.</span><span class="sxs-lookup"><span data-stu-id="15dc6-114">The multiplexer assigns a 1-byte stream ID to every PES; individual PES packets are identified by the stream ID in the packet header.</span></span> <span data-ttu-id="15dc6-115">En el caso de las secuencias de audio, el ID. de secuencia tiene el formato 110 *xxxxx*.</span><span class="sxs-lookup"><span data-stu-id="15dc6-115">For audio streams, the stream ID has the form 110 *xxxxx*.</span></span> <span data-ttu-id="15dc6-116">En el caso de vídeo, el ID. de secuencia tiene el formato 1110 *AAAA*.</span><span class="sxs-lookup"><span data-stu-id="15dc6-116">For video, the stream ID has the form 1110 *yyyy*.</span></span>

<span data-ttu-id="15dc6-117">El estándar MPEG-2 define dos maneras de proporcionar flujos elementales con paquetes: *secuencias de programa* y secuencias de *transporte*.</span><span class="sxs-lookup"><span data-stu-id="15dc6-117">The MPEG-2 standard defines two ways to deliver packetized elementary streams: *program streams* and *transport streams*.</span></span>

<span data-ttu-id="15dc6-118">**Flujos de programa**</span><span class="sxs-lookup"><span data-stu-id="15dc6-118">**Program Streams**</span></span>

<span data-ttu-id="15dc6-119">Los flujos de programa están diseñados para entornos que no tienen errores relativamente, como el almacenamiento de archivos local.</span><span class="sxs-lookup"><span data-stu-id="15dc6-119">Program streams are designed for environments that are relatively error-free, such as local file storage.</span></span> <span data-ttu-id="15dc6-120">En una secuencia de programa, los paquetes de PES se multiplexan y se organizan en unidades denominadas *paquetes*.</span><span class="sxs-lookup"><span data-stu-id="15dc6-120">In a program stream, the PES packets are multiplexed and organized into units called *packs*.</span></span> <span data-ttu-id="15dc6-121">Todas las secuencias de PES en una secuencia de programa se sincronizan con el mismo reloj.</span><span class="sxs-lookup"><span data-stu-id="15dc6-121">All of the PES streams in a program stream are synchronized to the same clock.</span></span>

<span data-ttu-id="15dc6-122">**Flujos de transporte**</span><span class="sxs-lookup"><span data-stu-id="15dc6-122">**Transport Streams**</span></span>

<span data-ttu-id="15dc6-123">Los flujos de transporte (TS) están diseñados para entornos no confiables o propensos a errores, como difusiones de red.</span><span class="sxs-lookup"><span data-stu-id="15dc6-123">Transport streams (TS) are designed for unreliable or error-prone environments, such as network broadcasts.</span></span> <span data-ttu-id="15dc6-124">Además, pueden contener varios programas que se sincronizan con distintos relojes.</span><span class="sxs-lookup"><span data-stu-id="15dc6-124">Also, they can contain multiple programs that are synchronized to different clocks.</span></span> <span data-ttu-id="15dc6-125">Un flujo de transporte agrega una segunda capa de empaquetado: los flujos de PES se empaquetan dentro de los paquetes de flujo de transporte, que tienen un tamaño fijo de 188 bytes por paquete.</span><span class="sxs-lookup"><span data-stu-id="15dc6-125">A transport stream adds a second layer of packetizing — the PES streams are packaged inside transport stream packets, which have a fixed size of 188 bytes per packet.</span></span> <span data-ttu-id="15dc6-126">Los paquetes de TS también pueden contener secuencias de información del programa, que se describen en la sección siguiente.</span><span class="sxs-lookup"><span data-stu-id="15dc6-126">TS packets can also contain program information streams, which are described in the following section.</span></span>

<span data-ttu-id="15dc6-127">Cada paquete de TS tiene un encabezado de 4 bytes, más un campo de adaptación opcional que contiene información de encabezado adicional.</span><span class="sxs-lookup"><span data-stu-id="15dc6-127">Each TS packet has a 4-byte header, plus an optional adaptation field that contains additional header information.</span></span> <span data-ttu-id="15dc6-128">El multiplexor asigna un identificador de programa (PID) a cada secuencia de PES o de información de programa.</span><span class="sxs-lookup"><span data-stu-id="15dc6-128">The multiplexer assigns a program ID (PID) to each PES stream or program information stream.</span></span> <span data-ttu-id="15dc6-129">Los PID se usan para identificar los paquetes de TS, de manera similar a la forma en que los identificadores de flujo identifican los paquetes de PES.</span><span class="sxs-lookup"><span data-stu-id="15dc6-129">The PIDs are used to identify the TS packets, similar to the way stream IDs identify PES packets.</span></span> <span data-ttu-id="15dc6-130">(Si una secuencia de transporte contiene varios programas, es posible que los identificadores de *flujo* no sean únicos, pero las asignaciones de PID son únicas en el flujo de transporte).</span><span class="sxs-lookup"><span data-stu-id="15dc6-130">(If a transport stream contains multiple programs, the *stream* IDs might not be unique, but the PID assignments are unique within the transport stream.)</span></span>

<span data-ttu-id="15dc6-132">Dado que una secuencia de transporte puede llevar varios programas, debe haber una manera de asociar los distintos paquetes de PES con los programas a los que pertenecen.</span><span class="sxs-lookup"><span data-stu-id="15dc6-132">Because a transport stream can carry multiple programs, there must be a way to associate the various PES packets with the programs they belong to.</span></span> <span data-ttu-id="15dc6-133">Esto se logra mediante el uso de tablas que identifican los flujos de programa.</span><span class="sxs-lookup"><span data-stu-id="15dc6-133">This is accomplished using tables that identify the program streams.</span></span> <span data-ttu-id="15dc6-134">Colectivamente, estos datos se denominan información específica del programa (PSI).</span><span class="sxs-lookup"><span data-stu-id="15dc6-134">Collectively, this data is called Program Specific Information (PSI).</span></span> <span data-ttu-id="15dc6-135">Los datos de la PSI se transfieren en paquetes de TS, al igual que los datos de PES.</span><span class="sxs-lookup"><span data-stu-id="15dc6-135">The PSI data is carried in TS packets, just like the PES data.</span></span> <span data-ttu-id="15dc6-136">Hay varios tipos de datos de PSI, entre los que se incluyen:</span><span class="sxs-lookup"><span data-stu-id="15dc6-136">There are various types of PSI data, including:</span></span>
