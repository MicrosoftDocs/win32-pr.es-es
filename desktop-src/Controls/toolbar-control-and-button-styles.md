# <a name="toolbar-control-and-button-styles"></a>Controles de barra de herramientas y estilos de botón

Los siguientes estilos de ventana son específicos de las barras de herramientas. Se combinan con otros estilos de ventana cuando se crea la barra de herramientas.

**Nota:** En el caso de los controles comunes [versión 6,00](common-control-versions.md), si se usa un [estilo visual](themes-overview.md) con la barra de herramientas, los botones siempre son transparentes independientemente de la configuración de estilo. De lo contrario, el comportamiento de transparencia es normal, tal y como se indica mediante el \_ estilo transparente TBSTYLE o TBSTYLE \_ .

> [!Note]  
> Comctl32.dll versión 6 no es redistribuible, pero se incluye en Windows. Para usar Comctl32.dll versión 6, especifíquelo en un manifiesto. Para obtener más información sobre los manifiestos, vea [habilitar estilos visuales](cookbook-overview.md).

**Nota:** Un botón de la barra de herramientas puede tener una combinación de los siguientes estilos. Para especificar un estilo de botón, establezca las marcas adecuadas en el miembro **fsStyle** de la estructura [**TBBUTTON**](/windows/desktop/api/Commctrl/ns-commctrl-tbbutton) del botón. No todos los estilos se pueden combinar.

En el caso de la versión de Shell 4,72 y versiones anteriores, las marcas de estilo de ventana y de botón tenían el formato TBSTYLE \_ xxx. Si está compilando una aplicación con la [versión 4,72](common-control-versions.md) o anterior de commctrl. h, debe usar las \_ marcas de estilo de botón de TBSTYLE xxx. En la versión 5,80 y versiones posteriores, todos los estilos de botón tienen el formato BTNS \_ xxx. Todos los valores de TBSTYLE \_ XXX tienen valores equivalentes de BTNS \_ xxx, con el mismo significado y el mismo valor numérico. Para mayor comodidad, ambos formularios se proporcionan en la tabla siguiente.
