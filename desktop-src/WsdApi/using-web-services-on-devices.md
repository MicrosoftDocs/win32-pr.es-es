# <a name="using-web-services-on-devices"></a>Uso de servicios web en dispositivos

Microsoft Web Services on Devices API (WSDAPI) admite la implementación de dispositivos y servicios controlados por el cliente, y hosts de dispositivos que se ajustan al perfil de dispositivos para servicios [web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS). WSDAPI se puede usar para el desarrollo de implementaciones de cliente y servidor (dispositivo).

Siempre que sea posible, use WSDAPI para crear una aplicación host de dispositivo o cliente WSD basada en Windows. El uso de WSDAPI reduce la complejidad del desarrollo y ahorra tiempo. Para obtener más información, vea WSD Application Development on Windows (Desarrollo de aplicaciones [WSD en Windows)](wsd-application-development-on-windows.md) y WSDAPI Development Tools (Herramientas de [desarrollo de WSDAPI).](wsdapi-development-tools.md)

También puede crear dispositivos WSD que no ejecuten Windows y que sean interoperables con clientes WSD basados en Windows y hosts de dispositivo. Estos dispositivos deben cumplir requisitos específicos para mantener la interoperabilidad. Para obtener más información, vea [Desarrollo de dispositivos WSD.](wsd-device-development.md)
