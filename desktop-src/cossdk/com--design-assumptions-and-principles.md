## <a name="assumptions"></a><span data-ttu-id="0dae8-109">Supuestos</span><span class="sxs-lookup"><span data-stu-id="0dae8-109">Assumptions</span></span>

<span data-ttu-id="0dae8-110">Las suposiciones son las siguientes:</span><span class="sxs-lookup"><span data-stu-id="0dae8-110">The assumptions are as follows:</span></span>

-   <span data-ttu-id="0dae8-111">**La aplicación COM+ será compatible con varios usuarios en varios servidores.**</span><span class="sxs-lookup"><span data-stu-id="0dae8-111">**The COM+ application will support multiple users across multiple servers.**</span></span> <span data-ttu-id="0dae8-112">En otras palabras, está creando una aplicación distribuida y estos varios usuarios van a estar en distintos equipos host desde donde se ejecuta el código.</span><span class="sxs-lookup"><span data-stu-id="0dae8-112">In other words, you are building a distributed application, and these multiple users are going to be on different host computers from where the code runs.</span></span> <span data-ttu-id="0dae8-113">Los usuarios van a tener acceso al código a través de Internet o a través de una red privada.</span><span class="sxs-lookup"><span data-stu-id="0dae8-113">Users are going to access the code via the Internet or via a private network.</span></span> <span data-ttu-id="0dae8-114">La interfaz de usuario se va a presentar a través del explorador o quizás una aplicación personalizada, como una aplicación basada en formularios escrita con Microsoft Visual Basic o MFC.</span><span class="sxs-lookup"><span data-stu-id="0dae8-114">The user interface is going to be presented via browser or perhaps a custom application, such as a form-based application written with Microsoft Visual Basic or MFC.</span></span> <span data-ttu-id="0dae8-115">Esa interfaz de usuario se va a ubicar en el equipo cliente.</span><span class="sxs-lookup"><span data-stu-id="0dae8-115">That user interface is going to be located on the client computer.</span></span>
-   <span data-ttu-id="0dae8-116">**La aplicación COM+ se puede convertir en escalable y puede proporcionar mayor disponibilidad y confiabilidad mediante la implementación de la aplicación en varios equipos servidor.**</span><span class="sxs-lookup"><span data-stu-id="0dae8-116">**The COM+ application can be made scalable and can provide greater availability and reliability by deploying the application across multiple server computers.**</span></span> <span data-ttu-id="0dae8-117">Al hacerlo, puede equilibrar la carga de trabajo de la aplicación y proporcionar tolerancia a errores mediante el uso de la organización por clústeres de Windows.</span><span class="sxs-lookup"><span data-stu-id="0dae8-117">By doing this, you can balance the workload of the application and provide fault tolerance by using Windows Clustering.</span></span>

## <a name="principles"></a><span data-ttu-id="0dae8-120">Principios</span><span class="sxs-lookup"><span data-stu-id="0dae8-120">Principles</span></span>

<span data-ttu-id="0dae8-121">Además de estas tres suposiciones, los siguientes principios pervade el modelo de programación de COM+:</span><span class="sxs-lookup"><span data-stu-id="0dae8-121">In addition to these three assumptions, the following principles pervade the COM+ programming model:</span></span>

-   <span data-ttu-id="0dae8-122">**La lógica de la aplicación residirá en los equipos servidor, no en los equipos cliente.**</span><span class="sxs-lookup"><span data-stu-id="0dae8-122">**The application logic will reside on server computers, not on client computers.**</span></span> <span data-ttu-id="0dae8-123">Hay tres razones principales para hacerlo:</span><span class="sxs-lookup"><span data-stu-id="0dae8-123">There are three major reasons to do this:</span></span>
    -   <span data-ttu-id="0dae8-124">Es posible que el equipo cliente no tenga la potencia de procesamiento o las características necesarias para ejecutar la lógica de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="0dae8-124">The client computer may not have the processing power or features needed to run the application logic.</span></span> <span data-ttu-id="0dae8-125">Además, mantener la lógica de la aplicación en el servidor simplifica la implementación.</span><span class="sxs-lookup"><span data-stu-id="0dae8-125">In addition, keeping the application logic on the server simplifies deployment.</span></span>
    -   <span data-ttu-id="0dae8-126">Los equipos servidor suelen estar más cerca de los datos y estos datos suelen estar en una base de datos.</span><span class="sxs-lookup"><span data-stu-id="0dae8-126">The server computers are often closer to the data, and this data is most often in a database.</span></span> <span data-ttu-id="0dae8-127">Dado que la aplicación está accediendo a las bases de datos, querrá ser muy sensible al costo de las conexiones de base de datos.</span><span class="sxs-lookup"><span data-stu-id="0dae8-127">Because your application is accessing databases, you want to be very sensitive to the cost of database connections.</span></span> <span data-ttu-id="0dae8-128">Al colocar la mayor parte de la lógica en los equipos servidor, puede compartir las conexiones de base de datos y obtener una mejora significativa en el rendimiento.</span><span class="sxs-lookup"><span data-stu-id="0dae8-128">By putting most of the logic on the server computers, you can share database connections and get a significant performance improvement.</span></span> <span data-ttu-id="0dae8-129">Hay otros recursos en los equipos del servidor que también se pueden compartir, de nuevo con una ventaja de rendimiento.</span><span class="sxs-lookup"><span data-stu-id="0dae8-129">There are other resources on the server computers that can be shared as well, again with a performance benefit.</span></span>
    -   <span data-ttu-id="0dae8-130">Tener la lógica de aplicación en los equipos servidor mantiene el control del contexto de seguridad con la aplicación.</span><span class="sxs-lookup"><span data-stu-id="0dae8-130">Having application logic reside on server computers keeps control of the security context with the application.</span></span> <span data-ttu-id="0dae8-131">Tiene más control sobre la seguridad si mantiene esa seguridad en componentes de aplicaciones que se ejecutan en equipos servidor en lugar de en equipos cliente.</span><span class="sxs-lookup"><span data-stu-id="0dae8-131">You have more control over security if you maintain that security on application components running on server computers rather than on client computers.</span></span>
-   <span data-ttu-id="0dae8-132">**Las transacciones son el núcleo.**</span><span class="sxs-lookup"><span data-stu-id="0dae8-132">**Transactions are the core.**</span></span> <span data-ttu-id="0dae8-133">Por diseño, las transacciones de forma que contengan el modelo de programación COM+ que sea muy importante para comprenderlos.</span><span class="sxs-lookup"><span data-stu-id="0dae8-133">By design, transactions so permeate the COM+ programming model that it is very important for you to understand them.</span></span> <span data-ttu-id="0dae8-134">Aunque puede usar muchos de los servicios de COM+ sin usar transacciones, si decide no utilizarlos, no puede aprovechar al máximo los servicios COM+ que tiene a su disposición.</span><span class="sxs-lookup"><span data-stu-id="0dae8-134">While you can make use of many of the services of COM+ without using transactions, if you choose not to use them, you cannot take full advantage of the COM+ services available to you.</span></span> <span data-ttu-id="0dae8-135">Entre las ventajas importantes del uso de transacciones se incluyen las siguientes:</span><span class="sxs-lookup"><span data-stu-id="0dae8-135">Some important benefits of using transactions include the following:</span></span>
    -   <span data-ttu-id="0dae8-136">Las transacciones son una solución razonable para el problema de la administración de simultaneidad.</span><span class="sxs-lookup"><span data-stu-id="0dae8-136">Transactions are a reasonable solution for the problem of concurrency management.</span></span> <span data-ttu-id="0dae8-137">Además, las transacciones ayudan a protegerse contra los bloqueos y tienen un buen modelo de recuperación de errores.</span><span class="sxs-lookup"><span data-stu-id="0dae8-137">Also, transactions help protect against crashes and have a good error recovery model.</span></span> <span data-ttu-id="0dae8-138">Además, las transacciones se convierten en una excelente manera de administrar tareas en varios sistemas.</span><span class="sxs-lookup"><span data-stu-id="0dae8-138">In addition, transactions turn out to be a great way to manage tasks across multiple systems.</span></span>
    -   <span data-ttu-id="0dae8-139">Puede diseñar una aplicación COM+ basada en transacciones para trabajar con un administrador de recursos y una base de datos, lo que ayuda a proteger la mayoría de la información de estado.</span><span class="sxs-lookup"><span data-stu-id="0dae8-139">You can design a transaction-based COM+ application to work with a resource manager and a database, which helps protect most state information.</span></span> <span data-ttu-id="0dae8-140">Al mantener el estado dentro de una base de datos u otro almacenamiento administrado por un administrador de recursos, no es necesario mantener muchos Estados dentro de los objetos reales que crea la aplicación.</span><span class="sxs-lookup"><span data-stu-id="0dae8-140">By keeping state inside a database or some other storage managed by a resource manager, you don't have to keep much state inside the actual objects your application creates.</span></span> <span data-ttu-id="0dae8-141">Aunque esto es una salida de un modelo orientado a objetos puro, funciona bien para crear aplicaciones distribuidas con recuperación de errores.</span><span class="sxs-lookup"><span data-stu-id="0dae8-141">While this is a departure from a pure object-oriented model, it works well for creating distributed applications with error recovery.</span></span>
-   <span data-ttu-id="0dae8-142">**La funcionalidad de la aplicación se genera como objetos COM, ajustando los protocolos que se usan para comunicarse con otros sistemas o tecnologías.**</span><span class="sxs-lookup"><span data-stu-id="0dae8-142">**Application functionality is built as COM objects, wrapping the protocols used to communicate with other systems or technologies.**</span></span> <span data-ttu-id="0dae8-143">Dado que es probable que cree componentes que van a unir varias tecnologías o sistemas heredados, planee el uso de diversos protocolos de comunicación.</span><span class="sxs-lookup"><span data-stu-id="0dae8-143">Because you are probably building components that are going to stitch together several technologies or legacy systems, plan on using a variety of communication protocols.</span></span> <span data-ttu-id="0dae8-144">Use HTTP para la comunicación de cliente/servidor o de aplicación a aplicación a través de Internet.</span><span class="sxs-lookup"><span data-stu-id="0dae8-144">Use HTTP for client/server communication or application-to-application communication over the Internet.</span></span> <span data-ttu-id="0dae8-145">Use DCOM para la comunicación de aplicación a aplicación o de componente a componente en el servidor.</span><span class="sxs-lookup"><span data-stu-id="0dae8-145">Use DCOM for application-to-application or component-to-component communication on the server.</span></span>
