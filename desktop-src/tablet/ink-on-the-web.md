# <a name="ink-on-the-web"></a>Entrada manuscrita en la web

El kit de desarrollo de software de Windows Vista y el kit de desarrollo de Windows XP Tablet PC Edition permiten crear aplicaciones web habilitadas para tinta para los usuarios de Tablet PC que deben tener acceso a aplicaciones remotas. Hay dos técnicas básicas para lograr esto: una implementación no táctil, que permite implementar aplicaciones .NET desde un servidor web o de archivos, y la otra es crear páginas web habilitadas para tinta con controles de Windows Forms. En ambos casos, la tinta se controla en el cliente, en lugar de en el servidor. Tenga en cuenta que la API COM no es compatible con la Web.

Para usar el procesamiento del lado cliente a través de la web, debe comprender el modelo de seguridad de .NET y cómo el funcionamiento de la confianza parcial afecta a la aplicación. Por esta razón, también se describe la seguridad y la confianza de las aplicaciones de Tablet PC.

Los temas siguientes contienen notas sobre las distintas formas de crear aplicaciones web habilitadas para tinta.

-   [Sin implementación táctil](no-touch-deployment.md)
-   [Controles Web](web-controls.md)
-   [Aplicaciones web habilitadas para tinta](ink-enabled-web-applications.md)
-   [Seguridad y confianza](security-and-trust.md)
