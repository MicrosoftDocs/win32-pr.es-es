# <a name="using-capicom"></a>Usar CAPICOM

\[CAPICOM es un componente de solo bits de 32 que está disponible para su uso en los siguientes sistemas operativos: Windows Server 2008, Windows Vista y Windows XP. En su lugar, use la .NET Framework para implementar características de seguridad. Para obtener más información, vea [alternativas al uso de CAPICOM](alternatives-to-using-capicom.md).\]

En esta sección se incluyen escenarios en los que se usan los procedimientos de CAPICOM.

> [!Note]  
> La creación de [*firmas digitales*](../secgloss/d-gly.md) y la desprotección de mensajes con CAPICOM se realiza mediante la criptografía de infraestructura de clave pública (PKI) y solo se puede hacer si el firmante o el usuario que descifra un mensaje con doble cifrado tiene acceso a un certificado con una clave privada asociada disponible. Para descifrar un mensaje con doble cifrado, debe haber un certificado con acceso a la clave privada en el almacén MY.
