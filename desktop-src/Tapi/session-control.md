# <a name="session-control"></a>Control de sesión

Una sesión o llamada representa una conexión entre dos o más direcciones. Esta conexión es dinámica y los objetos de programación asociados se deben crear, manipular y liberar según sea necesario. En el caso más simple, esto significa que se realiza y desconecta una llamada telefónica. En el caso de las aplicaciones avanzadas, el control de sesión puede implicar la administración de conferencias multimedia a través de una red IP hasta el nivel de participantes individuales.

El control de sesión implica dos áreas básicas:

-   [Las operaciones de sesión](session-operations-ovr.md) son controles que inician, mantienen y finalizan una sesión de comunicaciones.
-   La información de la [sesión](session-information-ovr.md) describe los datos que se pueden obtener en relación con una sesión de comunicaciones.

En esta sección no se describen dos tipos especializados de control de sesión. Para obtener información acerca de los centros de distribución de llamadas automáticas, consulte [control de centro de llamadas](./call-center-control.md) (TAPI 2. x) o [acerca de los controles del centro de llamadas](about-call-center-controls.md) (TAPI 3. x). Para obtener información sobre las conferencias IP, consulte Acerca de las [conferencias de telefonía IP de encuentro](about-rendezvous-ip-telephony-conferencing.md).
