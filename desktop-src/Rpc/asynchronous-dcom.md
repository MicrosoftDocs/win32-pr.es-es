# <a name="asynchronous-dcom"></a>DCOM asincrónico

Para aprovechar las ventajas de la RPC asincrónica de forma transparente, los programadores de COM pueden utilizar las interfaces asincrónicas \* generadas por el compilador MIDL tal y como se describe en el tema del atributo MIDL, [**\_ UUID Async**](../midl/async-uuid.md). Para obtener más información, vea [clientes y servidores com](../com/com-clients-and-servers.md),[realizar y procesar llamadas asincrónicas](../com/making-and-processing-asynchronous-calls.md), y [llamadas a métodos asincrónicos en clientes y servidores com](/archive/msdn-magazine/2000/april/windows-2000-asynchronous-method-calls-eliminate-the-wait-for-com-clients-and-servers).
