# <a name="system-administration"></a>Administración del sistema

> [!Note]  
> Estos temas son para desarrolladores de software. Para problemas de soporte al cliente, consulte [soporte técnico: ayuda de Windows](https://windows.microsoft.com/windows/support#1TC=windows-10).

 

## <a name="in-this-section"></a>En esta sección

<dl> <dt>

[datos de la configuración de arranque (BCD) proveedor WMI](/previous-versions/windows/desktop/bcd/boot-configuration-data-portal)
</dt> <dd>

El proveedor de datos de la configuración de arranque (BCD) (BCD) Instrumental de administración de Windows (WMI) proporciona acceso mediante programación a los almacenes de BCD, que describen las aplicaciones de arranque y la configuración de la aplicación de arranque.

</dd> <dt>

[Directiva de grupo](group-policy.md)
</dt> <dd>

Directiva de grupo habilita la administración basada en directivas mediante los servicios de directorio de Microsoft Active Directory. Directiva de grupo usa servicios de directorio y pertenencia a grupos de seguridad para proporcionar flexibilidad y admitir información de configuración extensa.

</dd> <dt>

[Microsoft Management Console (MMC)](microsoft-management-console--mmc-.md)
</dt> <dd>

Microsoft Management Console (MMC) proporciona un modelo de administración y una interfaz de usuario de administración sencilla, coherente e integrada.

</dd> <dt>

[Proveedor de la aplicación de administración de dispositivos móviles](/previous-versions/windows/desktop/mdmappprov/mobile-device-management-application-provider-portal)
</dt> <dd>

El proveedor de aplicaciones de administración de dispositivos móviles (MDM) administra las aplicaciones en los dispositivos que se suscriben a un servicio MDM.

</dd> <dt>

[Registro de administración de dispositivos móviles](/windows/desktop/MDMReg/mobile-device-management-registration-portal)
</dt> <dd>

El registro de MDM inscribe los dispositivos en un servicio MDM.

</dd> <dt>

[Proveedor de configuración de administración de dispositivos móviles](/previous-versions/windows/desktop/mdmsettingsprov/mobile-device-management-settings-provider-portal)
</dt> <dd>

El proveedor de configuración de MDM habilita la administración de la configuración en los dispositivos que se suscriben a un servicio MDM.

</dd> <dt>

[NetShell](/previous-versions/windows/desktop/netshell/netshell-start-page)
</dt> <dd>

NetShell es una herramienta basada en línea de comandos que permite a los administradores administrar y configurar de forma remota los servicios de red críticos.

</dd> <dt>

[Infraestructura de administración de configuración](/previous-versions/windows/desktop/smi/settings-management-infrastructure--smi-)
</dt> <dd>

La infraestructura de administración de configuración (SMI) proporciona una infraestructura estandarizada para obtener acceso y manipular la configuración que pueden modificar los usuarios y las aplicaciones.

</dd> <dt>

[Registro de inventario de software](/previous-versions/windows/desktop/sil/software-inventory-logging-portal)
</dt> <dd>

El registro de inventario de software recopila datos de licencias sobre el software instalado en un servidor de Windows y proporciona acceso remoto a los datos para que un centro de datos pueda agregarlos fácilmente.

</dd> <dt>

[API de licencias de software](/previous-versions/windows/desktop/secslapi/software-licensing-api-portal)
</dt> <dd>

La API de licencias de software (SLAPI) se puede usar para determinar una instalación original de Microsoft Windows, instalar y registrar una licencia de administración de activos y recuperar información acerca de la Directiva de licencias de un componente de software.

</dd> <dt>

[Restaurar sistema](/windows/desktop/sr/system-restore-portal)
</dt> <dd>

Restaurar sistema supervisa y registra automáticamente los cambios del sistema de claves en el equipo de un usuario. Está diseñado para reducir los costos de soporte técnico y aumentar la satisfacción del cliente al permitir que un usuario deshaga un cambio que puede haber causado un problema en el sistema o revertir a un día en el que el sistema funcionaba de forma óptima.

</dd> <dt>

[Cierre del sistema](/windows/desktop/Shutdown/system-shutdown)
</dt> <dd>

Las funciones y los mensajes de cierre del sistema permiten a las aplicaciones cerrar la sesión del usuario actual, apagar el sistema o bloquear la estación de trabajo.

</dd> <dt>

[Programador de tareas](/windows/desktop/TaskSchd/task-scheduler-start-page)
</dt> <dd>

El Programador de tareas permite realizar automáticamente las tareas rutinarias en un equipo seleccionado. Para ello, el Programador de tareas supervisa los criterios que elija para iniciar las tareas (denominados desencadenadores) y, a continuación, ejecutar las tareas cuando se cumplan los criterios.

</dd> <dt>

[Registro de acceso de usuarios](/previous-versions/windows/desktop/ual/user-access-logging)
</dt> <dd>

El registro de acceso de usuarios (UAL) es un marco común para que los roles de Windows Server informen de sus métricas de consumo respectivos. Este marco de trabajo de UAL es un componente fundamental y crítico de la solución de administración de licencias más grande.

</dd> <dt>

[Servicios de implementación de Windows](/windows/desktop/Wds/windows-deployment-services-portal)
</dt> <dd>

Servicios de implementación de Windows (WDS) es la versión revisada de los servicios de instalación remota (RIS). WDS permite la implementación de sistemas operativos Windows. Puede usar WDS para configurar nuevos clientes con una instalación basada en red sin necesidad de que los administradores visiten cada equipo o instalen directamente desde un CD o DVD.

</dd> <dt>

[API de Windows Genuine Advantage](/previous-versions/windows/desktop/wingen/windows-genuine-advantage-api-portal)
</dt> <dd>

La API de Windows original Advantage se usa para determinar si el sistema operativo Windows que se está ejecutando en el sistema actual es una copia original.

</dd> <dt>

[Instrumental de administración de Windows](/windows/desktop/WmiSdk/wmi-start-page)
</dt> <dd>

Instrumental de administración de Windows (WMI) es la infraestructura de datos y operaciones de administración en sistemas operativos basados en Windows. Puede escribir scripts o aplicaciones de WMI para automatizar las tareas administrativas en equipos remotos.

</dd> <dt>

[API de Windows Defender WMIv2](/previous-versions/windows/desktop/defender/windows-defender-wmiv2-apis-portal)
</dt> <dd>

Las API de WMI de Windows Defender se pueden usar para administrar la protección contra malware a través de scripts o aplicaciones.

</dd> <dt>

[Windows PowerShell](https://msdn.microsoft.com/library/Dd835506(v=VS.85).aspx)
</dt> <dd>

Windows PowerShell es un shell de línea de comandos basado en tareas y un lenguaje de scripting diseñado especialmente para la administración del sistema. Basado en .NET Framework, Windows PowerShell ayuda a los profesionales de TI y a los usuarios avanzados a controlar y automatizar la administración tanto del sistema operativo Windows como de las aplicaciones que se ejecutan en Windows.

</dd> <dt>

[Administración remota de Windows](/windows/desktop/WinRM/portal)
</dt> <dd>

El Administración remota de Windows (WinRM) es la implementación de Microsoft del Protocolo de WS-Management, un protocolo estándar compatible con firewall y basado en el Protocolo simple de acceso a objetos (SOAP) que permite que interoperen el hardware y los sistemas operativos de diferentes proveedores. La especificación del protocolo WS-Management proporciona una manera común de que los sistemas tengan acceso a la información de administración de una infraestructura de ti y la intercambie.

</dd> <dt>

[Protección de recursos de Windows](/windows/desktop/Wfp/windows-resource-protection-portal)
</dt> <dd>

Protección de recursos de Windows (WRP) impide el reemplazo de archivos esenciales del sistema, carpetas y claves del registro que se instalan como parte del sistema operativo.

</dd> <dt>

[Windows Server Update Services](windows-server-update-services.md)
</dt> <dd>

Los administradores del sistema pueden usar la API de Windows Server Update Services (WSUS) para determinar qué actualizaciones se aplican a un equipo o grupo de equipos, descargar dichas actualizaciones e instalarlas con poca o ninguna intervención del usuario.

Los fabricantes de software independientes y los desarrolladores de usuarios finales pueden integrar características de WSUS en el software de administración de equipos o de administración de actualizaciones para proporcionar un entorno operativo sin problemas.

</dd> <dt>

[Herramienta de evaluación del sistema de Windows](/windows/desktop/WinSAT/windows-system-assessment-tool-portal)
</dt> <dd>

Cómo usar WinSAT para evaluar las características y capacidades de rendimiento de un equipo.

</dd> <dt>

[Carpetas de trabajo](/previous-versions/windows/desktop/syncshareservermgmt/sync-share-server-management-portal)
</dt> <dd>

Carpetas de trabajo permite a los usuarios almacenar y obtener acceso a archivos de trabajo en equipos y dispositivos personales.

</dd> </dl>
