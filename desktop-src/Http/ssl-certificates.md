# <a name="ssl-certificates"></a><span data-ttu-id="8bb9e-104">Certificados SSL</span><span class="sxs-lookup"><span data-stu-id="8bb9e-104">SSL Certificates</span></span>

<span data-ttu-id="8bb9e-105">Capa de sockets seguros (SSL), también conocido como seguridad de la capa de transporte (TLS), se ha convertido en un estándar para proteger las conexiones a Internet y se usa para evitar la interceptación en la red.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-105">Secure Sockets Layer (SSL), also known as Transport Layer Security (TLS), has become a standard for securing Internet connections and is used to prevent eavesdropping on the network.</span></span> <span data-ttu-id="8bb9e-106">El protocolo SSL/TLS permite que un cliente y un servidor se autentiquen entre sí y negocien los algoritmos de cifrado.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-106">The SSL/TLS protocol allows a client and server to authenticate each other and negotiate encryption algorithms.</span></span>

<span data-ttu-id="8bb9e-107">SSL utiliza una clave de cifrado y un algoritmo de cifrado para proteger la conexión HTTP.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-107">SSL uses an encryption key and an encryption algorithm to secure the HTTP connection.</span></span> <span data-ttu-id="8bb9e-108">Las claves de cifrado están contenidas en certificados SSL utilizados por el cliente y el servidor.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-108">The encryption keys are contained in SSL certificates used by both the client and the server.</span></span> <span data-ttu-id="8bb9e-109">Normalmente, el certificado es un documento X. 509 ([RFC 2459](https://www.ietf.org/rfc/rfc2459.txt)).</span><span class="sxs-lookup"><span data-stu-id="8bb9e-109">The certificate is typically an X.509 ([RFC 2459](https://www.ietf.org/rfc/rfc2459.txt)) document.</span></span> <span data-ttu-id="8bb9e-110">El servidor proporciona el certificado SSL para la sesión y lo envía al cliente en la fase de protocolo de enlace.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-110">The server provides the SSL certificate for the session and sends the certificate to the client in the handshake phase.</span></span> <span data-ttu-id="8bb9e-111">El cliente envía su certificado al servidor sólo si el servidor envía una solicitud al cliente para un certificado.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-111">The client sends its certificate to the server only if the server sends a request to the client for a certificate.</span></span> <span data-ttu-id="8bb9e-112">Por lo tanto, el cliente siempre autentica el servidor, pero el servidor tiene la opción de autenticar el cliente.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-112">Thus the client always authenticates the server but the server has the option whether or not to authenticate the client.</span></span>

<span data-ttu-id="8bb9e-113">Los certificados de servidor deben almacenarse en el almacenamiento persistente local de la API del servidor HTTP, para su uso cada vez que se cree una conexión segura.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-113">Server certificates must be stored in the HTTP Server API's local persistent storage, for use each time a secure connection is created.</span></span> <span data-ttu-id="8bb9e-114">Cada entrada del almacén de certificados también contiene la dirección IP y el puerto del servidor, el hash del certificado (usado para firmar los mensajes) y el identificador de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-114">Each certificate store entry also contains the IP Address and port of the server, the certificate hash (used for signing the messages) and the application ID.</span></span> <span data-ttu-id="8bb9e-115">El identificador de la aplicación se usa para identificar la aplicación que posee el certificado.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-115">The application ID is used to identify the application that owns the certificate.</span></span>

<span data-ttu-id="8bb9e-116">Los administradores del sistema pueden almacenar la información del certificado de servidor SSL con las API de configuración.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-116">System administrators can store SSL server certificate information with the configuration APIs.</span></span> <span data-ttu-id="8bb9e-117">Una herramienta administrativa llama a la función [**HttpSetServiceConfiguration**](/windows/desktop/api/Http/nf-http-httpsetserviceconfiguration) y especifica el valor HttpServiceConfigSSLCertInfo para el parámetro de configuración del servicio para establecer la información de un certificado SSL.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-117">An administrative tool calls the [**HttpSetServiceConfiguration**](/windows/desktop/api/Http/nf-http-httpsetserviceconfiguration) function and specifies HttpServiceConfigSSLCertInfo value for the service configuration parameter to set information for an SSL certificate.</span></span> <span data-ttu-id="8bb9e-118">Solo se puede configurar un certificado de servidor para cada par de puerto y dirección IP del equipo.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-118">Only one server certificate can be configured for each IP Address and port pair on the machine.</span></span> <span data-ttu-id="8bb9e-119">La API del servidor HTTP también proporciona funciones de [**consulta**](/windows/desktop/api/Http/nf-http-httpqueryserviceconfiguration) y [**eliminación**](/windows/desktop/api/Http/nf-http-httpdeleteserviceconfiguration) para obtener acceso a los certificados existentes o eliminarlos.</span><span class="sxs-lookup"><span data-stu-id="8bb9e-119">The HTTP Server API also provides [**query**](/windows/desktop/api/Http/nf-http-httpqueryserviceconfiguration) and [**delete**](/windows/desktop/api/Http/nf-http-httpdeleteserviceconfiguration) functions to access or delete existing certificates.</span></span>
