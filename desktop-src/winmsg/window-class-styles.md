# <a name="window-class-styles"></a>Estilos de clase de ventana

Los estilos de clase definen elementos adicionales de la clase de ventana. Se pueden combinar dos o más estilos mediante el operador OR bit a bit ( \| ). Para asignar un estilo a una clase de ventana, asigne el estilo al miembro **de** estilo de la [**estructura WNDCLASSEX.**](/windows/win32/api/winuser/ns-winuser-wndclassexa)

## <a name="example"></a>Ejemplo

```cpp
    WNDCLASS wc = {};
    wc.lpfnWndProc = s_DropDownWndProc;
    wc.cbWndExtra = sizeof(CTipACDialog *);
    wc.hInstance = g_hInstance;
    wc.hCursor = LoadCursor(NULL, IDC_ARROW);
    wc.hbrBackground = (HBRUSH)(COLOR_WINDOW + 1);
    wc.style = CS_SAVEBITS | CS_DROPSHADOW;
    wc.lpszClassName = s_wzClassName;
    RegisterClass(&wc);
```

Ejemplo de [Windows ejemplos clásicos](https://github.com/microsoft/Windows-classic-samples/blob/1d363ff4bd17d8e20415b92e2ee989d615cc0d91/Samples/Win7Samples/tabletpc/TipAutoComplete/TIPAutoCompleteSDKSample.cpp) en GitHub.

| <span id="CS_BYTEALIGNWINDOW"></span><span id="cs_bytealignwindow"></span><dl> <dt>**CS \_ ByteALIGNWINDOW**</dt> <dt>0x2000</dt> </dl> | Alinea la ventana en un límite de bytes (en la dirección x). Este estilo afecta al ancho de la ventana y a su posición horizontal en la pantalla.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="CS_CLASSDC"></span><span id="cs_classdc"></span><dl> <dt>**CS \_ ClassDC**</dt> <dt>0x0040</dt> </dl>                         | Asigna un contexto de dispositivo que todas las ventanas de la clase compartirán. Dado que las clases de ventana son específicas del proceso, es posible que varios subprocesos de una aplicación creen una ventana de la misma clase. También es posible que los subprocesos intenten usar el contexto del dispositivo simultáneamente. Cuando esto sucede, el sistema permite que solo un subproceso finalice correctamente su operación de dibujo. <br/>                                                                                                                                                                                                                                                                                                                                                                     |
| <span id="CS_DBLCLKS"></span><span id="cs_dblclks"></span><dl> <dt>**CS \_ DBLCLKS**</dt> <dt>0x0008</dt> </dl>                         | Envía un mensaje de doble clic al procedimiento de ventana cuando el usuario hace doble clic en el mouse mientras el cursor está dentro de una ventana que pertenece a la clase . <br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| <span id="CS_DROPSHADOW"></span><span id="cs_dropshadow"></span><dl> <dt>**CS \_ DROPSHADOW**</dt> <dt>0x00020000</dt> </dl>            | Habilita el efecto de sombra paralela en una ventana. El efecto se ha activado y desactivado a través **de SPI \_ SETDROPSHADOW**. Normalmente, esto se habilita para ventanas pequeñas y de corta duración, como los menús, para resaltar su relación de orden Z con otras ventanas. Windows creado a partir de una clase con este estilo debe ser ventanas de nivel superior; es posible que no sean ventanas secundarias.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                             |

| <span id="CS_PARENTDC"></span><span id="cs_parentdc"></span><dl> <dt>**CS \_ ParentDC**</dt> <dt>0x0080</dt> </dl>                      | Establece el rectángulo de recorte de la ventana secundaria en el de la ventana primaria para que el elemento secundario pueda dibujar en el elemento primario. Una ventana con el bit **de estilo \_ PARENTDC de CS** recibe un contexto de dispositivo normal de la memoria caché del sistema de contextos de dispositivo. No proporciona al elemento secundario el contexto del dispositivo primario ni la configuración del contexto del dispositivo. La **especificación de \_ PARENTDC de CS** mejora el rendimiento de una aplicación. <br/>                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="CS_SAVEBITS"></span><span id="cs_savebits"></span><dl> <dt>**CS \_ SAVEBITS**</dt> <dt>0x0800</dt> </dl>                      | Guarda, como un mapa de bits, la parte de la imagen de pantalla ocultada por una ventana de esta clase. Cuando se quita la ventana, el sistema usa el mapa de bits guardado para restaurar la imagen de pantalla, incluidas otras ventanas ocultas. Por lo tanto, el sistema no envía mensajes [**WM \_ PAINT**](../gdi/wm-paint.md) a ventanas ocultas si no se ha descartado la memoria usada por el mapa de bits y si otras acciones de pantalla no han invalidado la imagen almacenada. <br/> Este estilo es útil para ventanas pequeñas (por ejemplo, menús o cuadros de diálogo) que se muestran brevemente y luego se quitan antes de que se lleve a cabo otra actividad de pantalla. Este estilo aumenta el tiempo necesario para mostrar la ventana, ya que el sistema primero debe asignar memoria para almacenar el mapa de bits.<br/> |
| <span id="CS_VREDRAW"></span><span id="cs_vredraw"></span><dl> <dt>**CS \_ DRAWDRAW**</dt> <dt>0x0001</dt> </dl>                         | Vuelve a dibujar toda la ventana si un ajuste de tamaño o movimiento cambia el alto del área cliente.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
