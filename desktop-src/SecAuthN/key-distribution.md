# <a name="key-distribution"></a><span data-ttu-id="26c7e-103">Distribución de claves</span><span class="sxs-lookup"><span data-stu-id="26c7e-103">Key Distribution</span></span>

<span data-ttu-id="26c7e-104">La técnica de autenticación de clave secreta no explica cómo el cliente y el servidor obtienen la [*clave de sesión*](../secgloss/s-gly.md) secreta que se utiliza en las sesiones entre sí.</span><span class="sxs-lookup"><span data-stu-id="26c7e-104">The secret key authentication technique does not explain how the client and server get the secret [*session key*](../secgloss/s-gly.md) to use in sessions with each other.</span></span> <span data-ttu-id="26c7e-105">Si son personas, pueden reunirse en secreto y aceptar la clave.</span><span class="sxs-lookup"><span data-stu-id="26c7e-105">If they are people, they could meet in secret and agree on the key.</span></span> <span data-ttu-id="26c7e-106">Pero si el cliente es un programa que se ejecuta en una estación de trabajo y el servidor es un servicio que se ejecuta en un servidor de red, ese método no funcionará.</span><span class="sxs-lookup"><span data-stu-id="26c7e-106">But if the client is a program running on a workstation and the server is a service running on a network server, that method will not work.</span></span>

<span data-ttu-id="26c7e-107">Un cliente querrá comunicarse con muchos servidores y necesitará claves diferentes para cada uno de ellos.</span><span class="sxs-lookup"><span data-stu-id="26c7e-107">A client will want to communicate with many servers and will need different keys for each of them.</span></span> <span data-ttu-id="26c7e-108">Un servidor se comunica con muchos clientes y necesita claves diferentes para cada uno de ellos.</span><span class="sxs-lookup"><span data-stu-id="26c7e-108">A server communicates with many clients and needs different keys for each of them, as well.</span></span> <span data-ttu-id="26c7e-109">Si cada cliente necesita una clave diferente para cada servidor y cada servidor necesita una clave diferente para cada cliente, la distribución de claves se convierte en un problema.</span><span class="sxs-lookup"><span data-stu-id="26c7e-109">If each client needs a different key for every server, and each server needs a different key for every client, key distribution becomes a problem.</span></span> <span data-ttu-id="26c7e-110">Además, la necesidad de almacenar y proteger muchas claves en muchos equipos crea un riesgo de seguridad enorme.</span><span class="sxs-lookup"><span data-stu-id="26c7e-110">In addition, the need to store and protect many keys on many computers creates an enormous security risk.</span></span>

<span data-ttu-id="26c7e-111">El nombre del [*protocolo Kerberos*](../secgloss/k-gly.md) sugiere su solución al problema de distribución de claves.</span><span class="sxs-lookup"><span data-stu-id="26c7e-111">The name of the [*Kerberos protocol*](../secgloss/k-gly.md) suggests its solution to the problem of key distribution.</span></span> <span data-ttu-id="26c7e-112">Kerberos (o Cerberus) fue una figura en la mitología griega clásica, un perro de tres puntas y un paso prolongado que mantuvo a los intrusos en el mundo.</span><span class="sxs-lookup"><span data-stu-id="26c7e-112">Kerberos (or Cerberus) was a figure in classical Greek mythology—a fierce, three-headed dog who kept living intruders from entering the Underworld.</span></span> <span data-ttu-id="26c7e-113">Como la protección de mitos, el protocolo Kerberos tiene tres cabezas: un cliente, un servidor y un tercero de confianza para mediar entre ellos.</span><span class="sxs-lookup"><span data-stu-id="26c7e-113">Like the mythical guard, the Kerberos protocol has three heads: a client, a server, and a trusted third party to mediate between them.</span></span> <span data-ttu-id="26c7e-114">El intermediario de confianza de este protocolo es el Centro de distribución de claves (KDC).</span><span class="sxs-lookup"><span data-stu-id="26c7e-114">The trusted intermediary in this protocol is the Key Distribution Center (KDC).</span></span>

<span data-ttu-id="26c7e-115">El KDC es un servicio que se ejecuta en un servidor protegido físicamente.</span><span class="sxs-lookup"><span data-stu-id="26c7e-115">The KDC is a service running on a physically secure server.</span></span> <span data-ttu-id="26c7e-116">Mantiene una base de datos con información de cuenta para todas las [*entidades de seguridad*](../secgloss/s-gly.md) de su dominio Kerberos.</span><span class="sxs-lookup"><span data-stu-id="26c7e-116">It maintains a database with account information for all [*security principals*](../secgloss/s-gly.md) in its realm.</span></span> <span data-ttu-id="26c7e-117">Un dominio Kerberos es el equivalente de Kerberos de un dominio en Windows.</span><span class="sxs-lookup"><span data-stu-id="26c7e-117">A realm is the Kerberos equivalent of a domain in Windows.</span></span>

<span data-ttu-id="26c7e-118">Junto con otra información acerca de cada entidad de seguridad, el KDC almacena una clave criptográfica conocida únicamente para el servidor principal y el KDC.</span><span class="sxs-lookup"><span data-stu-id="26c7e-118">Along with other information about each security principal, the KDC stores a cryptographic key known only to the principal and the KDC.</span></span> <span data-ttu-id="26c7e-119">Esta es la [*clave maestra*](../secgloss/m-gly.md) que se usa en los intercambios entre cada entidad de seguridad y el KDC.</span><span class="sxs-lookup"><span data-stu-id="26c7e-119">This is the [*master key*](../secgloss/m-gly.md) used in exchanges between each security principal and the KDC.</span></span> <span data-ttu-id="26c7e-120">En la mayoría de las implementaciones del protocolo Kerberos, esta clave maestra se deriva usando una función [*hash*](../secgloss/h-gly.md) de la contraseña de una entidad de seguridad.</span><span class="sxs-lookup"><span data-stu-id="26c7e-120">In most implementations of the Kerberos protocol, this master key is derived using a [*hash*](../secgloss/h-gly.md) function from a security principal's password.</span></span>

<span data-ttu-id="26c7e-121">Cuando un cliente desea crear una conexión segura con un servidor, el cliente comienza enviando una solicitud al KDC, no al servidor al que desea tener acceso.</span><span class="sxs-lookup"><span data-stu-id="26c7e-121">When a client wants to create a secure connection with a server, the client begins by sending a request to the KDC, not to the server that it wants to reach.</span></span> <span data-ttu-id="26c7e-122">El KDC crea y envía al cliente una clave de [*sesión*](../secgloss/s-gly.md) única para el cliente y un servidor que se utiliza para autenticarse entre sí.</span><span class="sxs-lookup"><span data-stu-id="26c7e-122">The KDC creates and sends to the client a unique [*session key*](../secgloss/s-gly.md) for the client and a server to use to authenticate each other.</span></span> <span data-ttu-id="26c7e-123">El KDC tiene acceso tanto a la clave maestra del cliente como a la clave maestra del servidor.</span><span class="sxs-lookup"><span data-stu-id="26c7e-123">The KDC has access to both the client's master key and the server's master key.</span></span> <span data-ttu-id="26c7e-124">El KDC cifra la copia del servidor de la clave de sesión mediante la clave maestra del servidor y la copia del cliente mediante la clave maestra del cliente.</span><span class="sxs-lookup"><span data-stu-id="26c7e-124">The KDC encrypts the server's copy of the session key using the server's master key, and the client's copy using the client's master key.</span></span>

<span data-ttu-id="26c7e-125">El KDC podría cumplir su rol como intermediario de confianza enviando la clave de sesión directamente a cada una de las entidades de seguridad implicadas, pero en la práctica este procedimiento no funcionará, por una serie de motivos.</span><span class="sxs-lookup"><span data-stu-id="26c7e-125">The KDC could fulfill its role as trusted intermediary by sending the session key directly to each of the security principals involved, but in practice this procedure will not work, for a number of reasons.</span></span> <span data-ttu-id="26c7e-126">En su lugar, el KDC envía al cliente ambas claves de sesión cifrada.</span><span class="sxs-lookup"><span data-stu-id="26c7e-126">Instead, the KDC sends both encrypted session keys to the client.</span></span> <span data-ttu-id="26c7e-127">La clave de sesión del servidor se incluye en un [vale de sesión](session-tickets.md).</span><span class="sxs-lookup"><span data-stu-id="26c7e-127">The session key for the server is included in a [session ticket](session-tickets.md).</span></span>
