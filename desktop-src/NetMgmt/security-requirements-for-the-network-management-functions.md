# <a name="network-management-function-security-requirements"></a>Requisitos de seguridad de la función de administración de redes

Llamar a algunas de las funciones de administración de red no requiere la pertenencia a grupos especiales. Otras funciones requieren que los usuarios tengan un nivel de privilegios específico para ejecutarse correctamente. Si procede, la sección Comentarios de la página de referencia de una función indica el nivel de privilegio que un usuario debe tener para ejecutar la función determinada.

Los requisitos de seguridad que se aplican Active Directory controladores de dominio pueden diferir de los que se aplican a servidores y estaciones de trabajo.

Para obtener más información y una lista de las funciones afectadas, vea los temas siguientes:

-   [Requisitos para las funciones de administración de redes Active Directory controladores de dominio](requirements-for-network-management-functions-on-active-directory-domain-controllers.md)
-   [Requisitos para las funciones de administración de redes en servidores y estaciones de trabajo](requirements-for-network-management-functions-on-servers-and-workstations.md)

Para obtener más información sobre cómo controlar el acceso a objetos protegibles, [vea Access Control](/windows/desktop/SecAuthZ/access-control), [Privileges](/windows/desktop/SecAuthZ/privileges)y [Securable Objects](/windows/desktop/SecAuthZ/securable-objects).

Para obtener más información sobre descriptores de seguridad específicos que se usan durante las comprobaciones de acceso, vea la página de referencia de función individual. Para obtener más información sobre cómo llamar a funciones que requieren privilegios de administrador, vea [Running with Special Privileges](/windows/desktop/SecBP/running-with-special-privileges).
