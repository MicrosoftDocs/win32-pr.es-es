# <a name="large-page-support"></a><span data-ttu-id="2c259-103">Compatibilidad con Large-Page</span><span class="sxs-lookup"><span data-stu-id="2c259-103">Large-Page Support</span></span>

<span data-ttu-id="2c259-104">La compatibilidad con páginas de gran tamaño permite a las aplicaciones de servidor establecer regiones de memoria de gran tamaño, lo que es especialmente útil en Windows de 64 bits.</span><span class="sxs-lookup"><span data-stu-id="2c259-104">Large-page support enables server applications to establish large-page memory regions, which is particularly useful on 64-bit Windows.</span></span> <span data-ttu-id="2c259-105">Cada traducción de página grande utiliza un único búfer de traducción dentro de la CPU.</span><span class="sxs-lookup"><span data-stu-id="2c259-105">Each large-page translation uses a single translation buffer inside the CPU.</span></span> <span data-ttu-id="2c259-106">El tamaño de este búfer suele ser tres pedidos de magnitud mayor que el tamaño de página nativo; Esto aumenta la eficacia del búfer de traducción, lo que puede aumentar el rendimiento de la memoria a la que se accede con frecuencia.</span><span class="sxs-lookup"><span data-stu-id="2c259-106">The size of this buffer is typically three orders of magnitude larger than the native page size; this increases the efficiency of the translation buffer, which can increase performance for frequently accessed memory.</span></span>

<span data-ttu-id="2c259-107">En el procedimiento siguiente se describe cómo usar la compatibilidad con páginas de gran tamaño.</span><span class="sxs-lookup"><span data-stu-id="2c259-107">The following procedure describes how to use large-page support.</span></span>

<span data-ttu-id="2c259-108">**Para usar la compatibilidad con páginas grandes**</span><span class="sxs-lookup"><span data-stu-id="2c259-108">**To use large-page support**</span></span>

1.  <span data-ttu-id="2c259-109">Obtenga el privilegio **SeLockMemoryPrivilege** mediante una llamada a la función [**AdjustTokenPrivileges**](/windows/win32/api/securitybaseapi/nf-securitybaseapi-adjusttokenprivileges) .</span><span class="sxs-lookup"><span data-stu-id="2c259-109">Obtain the **SeLockMemoryPrivilege** privilege by calling the [**AdjustTokenPrivileges**](/windows/win32/api/securitybaseapi/nf-securitybaseapi-adjusttokenprivileges) function.</span></span> <span data-ttu-id="2c259-110">Para obtener más información, consulte [asignación de privilegios a una cuenta](../secbp/assigning-privileges-to-an-account.md) y [cambio de privilegios en un token](../secbp/changing-privileges-in-a-token.md).</span><span class="sxs-lookup"><span data-stu-id="2c259-110">For more information, see [Assigning Privileges to an Account](../secbp/assigning-privileges-to-an-account.md) and [Changing Privileges in a Token](../secbp/changing-privileges-in-a-token.md).</span></span>
2.  <span data-ttu-id="2c259-111">Recupere el tamaño mínimo de página grande mediante una llamada a la función [**GetLargePageMinimum**](/windows/win32/api/memoryapi/nf-memoryapi-getlargepageminimum) .</span><span class="sxs-lookup"><span data-stu-id="2c259-111">Retrieve the minimum large-page size by calling the [**GetLargePageMinimum**](/windows/win32/api/memoryapi/nf-memoryapi-getlargepageminimum) function.</span></span>
3.  <span data-ttu-id="2c259-112">Incluya el valor de **MEM \_ Large \_ pages** al llamar a la función [**VirtualAlloc**](/windows/win32/api/memoryapi/nf-memoryapi-virtualalloc) .</span><span class="sxs-lookup"><span data-stu-id="2c259-112">Include the **MEM\_LARGE\_PAGES** value when calling the [**VirtualAlloc**](/windows/win32/api/memoryapi/nf-memoryapi-virtualalloc) function.</span></span> <span data-ttu-id="2c259-113">El tamaño y la alineación deben ser un múltiplo del mínimo de la página grande.</span><span class="sxs-lookup"><span data-stu-id="2c259-113">The size and alignment must be a multiple of the large-page minimum.</span></span>

<span data-ttu-id="2c259-114">Al escribir aplicaciones que utilizan memoria de página grande, tenga en cuenta las consideraciones siguientes:</span><span class="sxs-lookup"><span data-stu-id="2c259-114">When writing applications that use large-page memory, keep the following considerations in mind:</span></span>

-   <span data-ttu-id="2c259-115">Las regiones de memoria de gran tamaño pueden resultar difíciles de obtener después de que el sistema se ejecute durante mucho tiempo, ya que el espacio físico para cada página grande debe ser contiguo, pero es posible que se haya fragmentado la memoria.</span><span class="sxs-lookup"><span data-stu-id="2c259-115">Large-page memory regions may be difficult to obtain after the system has been running for a long time because the physical space for each large page must be contiguous, but the memory may have become fragmented.</span></span> <span data-ttu-id="2c259-116">Asignar páginas grandes en estas condiciones puede afectar significativamente al rendimiento del sistema.</span><span class="sxs-lookup"><span data-stu-id="2c259-116">Allocating large pages under these conditions can significantly affect system performance.</span></span> <span data-ttu-id="2c259-117">Por lo tanto, las aplicaciones deben evitar que se repitan las asignaciones de páginas grandes y, en su lugar, asignar todas las páginas grandes una vez, en el inicio.</span><span class="sxs-lookup"><span data-stu-id="2c259-117">Therefore, applications should avoid making repeated large-page allocations and instead allocate all large pages one time, at startup.</span></span>
-   <span data-ttu-id="2c259-118">La memoria es siempre de lectura/escritura y no paginable (siempre residente en la memoria física).</span><span class="sxs-lookup"><span data-stu-id="2c259-118">The memory is always read/write and nonpageable (always resident in physical memory).</span></span>
-   <span data-ttu-id="2c259-119">La memoria forma parte de los bytes privados del proceso pero no de parte del espacio de trabajo, porque el espacio de trabajo establecido por definición contiene solo memoria paginable.</span><span class="sxs-lookup"><span data-stu-id="2c259-119">The memory is part of the process private bytes but not part of the working set, because the working set by definition contains only pageable memory.</span></span>
-   <span data-ttu-id="2c259-120">Las asignaciones de páginas grandes no están sujetas a límites de trabajos.</span><span class="sxs-lookup"><span data-stu-id="2c259-120">Large-page allocations are not subject to job limits.</span></span>
-   <span data-ttu-id="2c259-121">La memoria de páginas grandes debe estar reservada y confirmada como una sola operación.</span><span class="sxs-lookup"><span data-stu-id="2c259-121">Large-page memory must be reserved and committed as a single operation.</span></span> <span data-ttu-id="2c259-122">En otras palabras, las páginas grandes no se pueden usar para confirmar un intervalo de memoria reservado previamente.</span><span class="sxs-lookup"><span data-stu-id="2c259-122">In other words, large pages cannot be used to commit a previously reserved range of memory.</span></span>
-   <span data-ttu-id="2c259-123">WOW64 en sistemas basados en Intel Itanium no admite aplicaciones de 32 bits que usan esta característica.</span><span class="sxs-lookup"><span data-stu-id="2c259-123">WOW64 on Intel Itanium-based systems does not support 32-bit applications that use this feature.</span></span> <span data-ttu-id="2c259-124">Las aplicaciones deben volver a compilarse como aplicaciones nativas de 64 bits.</span><span class="sxs-lookup"><span data-stu-id="2c259-124">The applications should be recompiled as native 64-bit applications.</span></span>
