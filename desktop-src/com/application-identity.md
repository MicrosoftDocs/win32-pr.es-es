# <a name="application-identity"></a>Identidad de aplicación

La identidad de una aplicación es la cuenta que se usa para ejecutar la aplicación. La identidad puede ser la del usuario que ha iniciado sesión actualmente (el usuario interactivo), el usuario que inició el servidor, un usuario especificado o un servicio. Para obtener información sobre cómo establecer la identidad de la aplicación Dcomcnfg.exe, consulte Establecer la seguridad de todo el proceso [mediante DCOMCNFG.](setting-processwide-security-using-dcomcnfg.md)

En los temas siguientes se describe cada uno de los cuatro tipos de identidad de aplicación y se explican las limitaciones de cada tipo:

-   [Usuario interactivo](interactive-user.md)
-   [Iniciar usuario](launching-user.md)
-   [Usuario especificado](specified-user.md)
-   [Identidad de servicio](service-identity.md)
