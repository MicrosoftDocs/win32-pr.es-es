# <a name="prov_dss"></a>PROV \_ DSS

El tipo de proveedor DSS de PROV, como \_ PROV RSA SIG, solo admite \_ \_ [*hashes*](../secgloss/h-gly.md) y [*firmas digitales.*](../secgloss/d-gly.md) El algoritmo de firma especificado por el tipo de proveedor de DSS prov \_ es el algoritmo de firma [*digital*](../secgloss/d-gly.md) (DSA).

## <a name="algorithms-supported"></a>Algoritmos admitidos

Para obtener descripciones de cada uno de estos algoritmos, consulte el glosario.



| Propósito      | Algoritmos admitidos                                                                                                              |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------|
| Intercambio de claves | Ninguno                                                                                                                              |
| Firma    | [*Dss*](../secgloss/d-gly.md)                                                                       |
| Cifrado   | Ninguno                                                                                                                              |
| Aplicación de algoritmo hash      | [*MD5*](../secgloss/m-gly.md)[*SHA*](../secgloss/s-gly.md)<br/> |



 

## <a name="related-documentation"></a>Documentación relacionada

El DSA fue propuesto por el [*Instituto Nacional de Estándares y Tecnología*](../secgloss/n-gly.md) (NIST). Puede encontrar una descripción del algoritmo en la siguiente referencia gubernamental: "Proposed Estándar federal de procesamiento de información for Digital Signature Standard (DSS)" Federal Register, v. 57, no. 21, 31 ene 1992, pp. 3747-3749.
