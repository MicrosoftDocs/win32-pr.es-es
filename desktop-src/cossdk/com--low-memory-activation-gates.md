# <a name="com-low-memory-activation-gates"></a>Puertas de activación de Low-Memory COM+

El servicio de puertas de activación de memoria baja de COM+ comprueba automáticamente la memoria antes de crear un servidor o un objeto COM+. Si el porcentaje de memoria virtual disponible para la aplicación está por debajo de un umbral fijo, se produce un error en la activación antes de crear el objeto. De esta manera, el servicio de puertas de activación de memoria baja mejora en gran medida la confiabilidad del sistema.

En los temas siguientes se proporciona información general y de tareas sobre el servicio de puertas de activación de memoria baja de COM+.
