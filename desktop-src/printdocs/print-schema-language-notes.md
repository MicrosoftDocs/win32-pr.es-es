# <a name="print-schema-language-notes"></a>Notas del lenguaje de esquema de impresión

Este tema no está actualizado. Para obtener la información más reciente, consulte la [especificación del esquema de impresión](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

En esta especificación, las palabras que se usan para definir la importancia de cada requisito se escriben en mayúsculas. Estas palabras se usan de acuerdo con sus definiciones en RFC 2119, y se reproducen los significados correspondientes:



| Término                                                                | Descripción                                                                                                                                                                                                                                                                                         |
|---------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="MUST"></span><span id="must"></span>DEBE<br/>       | Esta palabra, o el adjetivo requerido, significa que el elemento es un requisito absoluto de la especificación.<br/>                                                                                                                                                                               |
| <span id="SHOULD"></span><span id="should"></span>PODRÁ<br/> | Esta palabra, o el adjetivo recomendado, significa que pueden existir motivos válidos en determinadas circunstancias para omitir este elemento, pero se deben entender las implicaciones completas y el caso se sopesa cuidadosamente antes de elegir otro curso.<br/>                                     |
| <span id="MAY"></span><span id="may"></span>PUEDE<br/>          | Esta palabra, o el adjetivo opcional, significa que este elemento es realmente opcional. Por ejemplo, una implementación puede decidir incluir el elemento porque un determinado Marketplace o escenario lo requiere o porque mejora el producto. Otra implementación puede omitir el mismo elemento.<br/> |
