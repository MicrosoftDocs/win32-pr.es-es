# <a name="ink-interoperability"></a>Interoperabilidad de entrada manuscrita

La persistencia y la interoperabilidad son dos características de vital importancia, especialmente para las aplicaciones que usan la entrada de lápiz. Para habilitar la persistencia y la interoperabilidad de la entrada de lápiz, una aplicación debe permitir a los usuarios:

-   Guarde y abra sus documentos con plena fidelidad de los datos de entrada de lápiz.
-   Mueva datos hacia y desde otras aplicaciones basadas Windows microsoft mediante el Portapapeles.

En las secciones siguientes se detalla la interoperabilidad de la entrada de lápiz.

-   [Formatos de datos de entrada manuscrita](ink-data-formats.md)
-   [Escenarios de interoperabilidad importantes](important-interoperability-scenarios.md)
-   [Formatos de persistencia](persistence-formats.md)
-   [Elección de los formatos que se admitirán](choosing-which-formats-to-support.md)
-   [Bloques de creación](building-blocks.md)
-   [Almacenamiento de entrada manuscrita en HTML](storing-ink-in-html.md)

En este documento se da por supuesto un conocimiento detallado del Portapapeles. Para obtener más información sobre el Portapapeles, vea la documentación [del Portapapeles](/windows/desktop/dataxchg/clipboard).
