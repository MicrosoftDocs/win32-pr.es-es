## <a name="session-establishment-in-the-control-plane"></a><span data-ttu-id="d54c9-105">Establecimiento de la sesión en el plano de control</span><span class="sxs-lookup"><span data-stu-id="d54c9-105">Session Establishment in the Control Plane</span></span>

<span data-ttu-id="d54c9-106">En el plano de control hay dos tipos distintos de establecimiento de sesión:</span><span class="sxs-lookup"><span data-stu-id="d54c9-106">In the control plane there are two distinct types of session establishment:</span></span>

-   <span data-ttu-id="d54c9-108">no raíz</span><span class="sxs-lookup"><span data-stu-id="d54c9-108">nonrooted</span></span>

<span data-ttu-id="d54c9-109">En el caso del control con raíz, existe un participante especial, la \_ raíz de c, que es diferente del resto de los miembros de esta sesión Multipoint, cada uno de los cuales se denomina \_ hoja c.</span><span class="sxs-lookup"><span data-stu-id="d54c9-109">In the case of rooted control, a special participant, c\_root, exists that is different from the rest of the members of this multipoint session, each of which are called a c\_leaf.</span></span> <span data-ttu-id="d54c9-110">La \_ raíz de c debe permanecer presente durante todo el tiempo de la sesión de Multipoint, ya que la sesión se divide en ausencia de la raíz de c \_ .</span><span class="sxs-lookup"><span data-stu-id="d54c9-110">The c\_root must remain present for the whole duration of the multipoint session, as the session is broken up in the absence of the c\_root.</span></span> <span data-ttu-id="d54c9-111">La raíz de c \_ normalmente inicia la sesión de Multipoint mediante la configuración de la conexión a una \_ hoja de c o un número de hojas de c \_ .</span><span class="sxs-lookup"><span data-stu-id="d54c9-111">The c\_root usually initiates the multipoint session by setting up the connection to a c\_leaf, or a number of c\_leafs.</span></span> <span data-ttu-id="d54c9-112">La raíz de c \_ puede agregar más \_ hojas de c o, en algunos casos, una \_ hoja c puede unir la raíz de c \_ en un momento posterior.</span><span class="sxs-lookup"><span data-stu-id="d54c9-112">The c\_root may add more c\_leafs, or (in some cases) a c\_leaf can join the c\_root at a later time.</span></span> <span data-ttu-id="d54c9-113">Los ejemplos del plano de control con raíz se pueden encontrar en ATM y ST-II.</span><span class="sxs-lookup"><span data-stu-id="d54c9-113">Examples of the rooted control plane can be found in ATM and ST-II.</span></span>

<span data-ttu-id="d54c9-116">Otra forma de ver esto es que una \_ raíz de c todavía existe, pero se puede considerar que está en la nube de red en lugar de uno de los participantes.</span><span class="sxs-lookup"><span data-stu-id="d54c9-116">Another way to look at this is that a c\_root still exists, but can be considered to be in the network cloud as opposed to one of the participants.</span></span> <span data-ttu-id="d54c9-117">Dado que una raíz de control sigue existiendo, también se podría considerar que un plano de control no raíz se ha modificado de forma implícita.</span><span class="sxs-lookup"><span data-stu-id="d54c9-117">Because a control root still exists, a nonrooted control plane could also be considered to be implicitly rooted.</span></span> <span data-ttu-id="d54c9-118">Ejemplos de este tipo de esquemas de multipoint con raíz implícita:</span><span class="sxs-lookup"><span data-stu-id="d54c9-118">Examples for this kind of implicitly rooted multipoint schemes are:</span></span>

## <a name="data-transfer-in-the-data-plane"></a><span data-ttu-id="d54c9-122">Transferencia de datos en el plano de datos</span><span class="sxs-lookup"><span data-stu-id="d54c9-122">Data Transfer in the Data Plane</span></span>

<span data-ttu-id="d54c9-123">En el plano de datos, hay dos tipos de estilos de transferencia de datos:</span><span class="sxs-lookup"><span data-stu-id="d54c9-123">In the data plane, there are two types of data transfer styles:</span></span>

<span data-ttu-id="d54c9-126">En un plano de datos raíz, existe un participante especial denominado \_ raíz d.</span><span class="sxs-lookup"><span data-stu-id="d54c9-126">In a rooted data plane, a special participant called d\_root exists.</span></span> <span data-ttu-id="d54c9-127">La transferencia de datos solo se produce entre la \_ raíz d y el resto de los miembros de esta sesión Multipoint, cada una de las cuales se conoce como \_ hoja d.</span><span class="sxs-lookup"><span data-stu-id="d54c9-127">Data transfer only occurs between the d\_root and the rest of the members of this multipoint session, each of which are referred to as a d\_leaf.</span></span> <span data-ttu-id="d54c9-128">El tráfico puede ser unidireccional o bidireccional.</span><span class="sxs-lookup"><span data-stu-id="d54c9-128">The traffic could be unidirectional or bidirectional.</span></span> <span data-ttu-id="d54c9-129">Los datos que se envían desde la \_ raíz d están duplicados (si es necesario) y se entregan a cada \_ hoja d, mientras que los datos de las \_ hojas d solo van a la \_ raíz d.</span><span class="sxs-lookup"><span data-stu-id="d54c9-129">The data sent out from the d\_root is duplicated (if required) and delivered to every d\_leaf, while the data from d\_leafs only goes to the d\_root.</span></span> <span data-ttu-id="d54c9-130">En el caso de un plano de datos con raíz, no se permite ningún tráfico entre las \_ hojas d.</span><span class="sxs-lookup"><span data-stu-id="d54c9-130">In the case of a rooted data plane, no traffic is allowed among d\_leafs.</span></span> <span data-ttu-id="d54c9-131">Un ejemplo de un protocolo cuya raíz se encuentra en el plano de datos es ST-II.</span><span class="sxs-lookup"><span data-stu-id="d54c9-131">An example of a protocol that is rooted in the data plane is ST-II.</span></span>
