</dt> <dd>

NetShell es una herramienta basada en la línea de comandos que permite a los administradores administrar y configurar servicios de red críticos de forma remota.

</dd> <dt>

[Configuración Infraestructura de administración](/previous-versions/windows/desktop/smi/settings-management-infrastructure--smi-)
</dt> <dd>

Configuración La infraestructura de administración (SMI) proporciona una infraestructura estandarizada para acceder y manipular configuraciones que los usuarios y las aplicaciones pueden modificables.

</dd> <dt>

[Registro de inventario de software](/previous-versions/windows/desktop/sil/software-inventory-logging-portal)
</dt> <dd>

El Registro de inventario de software recopila datos de licencias sobre el software instalado en un servidor de Windows y proporciona acceso remoto a los datos para que un centro de datos pueda agregarlo fácilmente.

</dd> <dt>

[API de licencias de software](/previous-versions/windows/desktop/secslapi/software-licensing-api-portal)
</dt> <dd>

La API de licencias de software (ESTAT) se puede usar para determinar una instalación original de Microsoft Windows, instalar y registrar una licencia de administración de recursos y recuperar información sobre la directiva de licencias de un componente de software.

</dd> <dt>

[Restaurar sistema](/windows/desktop/sr/system-restore-portal)
</dt> <dd>

Restaurar sistema supervisa y registra automáticamente los cambios clave del sistema en el equipo de un usuario. Está diseñado para reducir los costos de soporte técnico y aumentar la satisfacción del cliente, ya que permite a un usuario deshacer un cambio que puede haber provocado un problema con el sistema o revertir a un día en el que el sistema funcionaba de forma óptima.

</dd> <dt>

[Cierre del sistema](/windows/desktop/Shutdown/system-shutdown)
</dt> <dd>

Las funciones y mensajes de apagado del sistema permiten a las aplicaciones cerrar la sesión del usuario actual, apagar el sistema o bloquear la estación de trabajo.

</dd> <dt>

[Programador de tareas](/windows/desktop/TaskSchd/task-scheduler-start-page)
</dt> <dd>

El Programador de tareas permite realizar automáticamente tareas rutinarias en un equipo elegido. El Programador de tareas hace esto supervisando los criterios que elija para iniciar las tareas (denominadas desencadenadores) y, a continuación, ejecutando las tareas cuando se cumplen los criterios.

</dd> <dt>

[Registro de acceso de usuarios](/previous-versions/windows/desktop/ual/user-access-logging)
</dt> <dd>

El registro de acceso de usuario (UAL) es un marco común para que los roles Windows Server informen de sus respectivas métricas de consumo. Este marco de UAL es un componente fundamental y crítico de la solución de administración de licencias más grande.

</dd> <dt>

[Windows Servicios de implementación](/windows/desktop/Wds/windows-deployment-services-portal)
</dt> <dd>

Windows Deployment Services (WDS) es la versión revisada de Servicios de instalación remota (RIS). WDS permite la implementación de Windows sistemas operativos. Puede usar WDS para configurar nuevos clientes con una instalación basada en red sin necesidad de que los administradores visiten cada equipo o se instalen directamente desde medios de CD o DVD.

</dd> <dt>

[Windows API de Ventaja original](/previous-versions/windows/desktop/wingen/windows-genuine-advantage-api-portal)
</dt> <dd>

La Windows Genuine Advantage API se usa para determinar si el sistema operativo Windows que se ejecuta en el sistema actual es una copia original.

</dd> <dt>

[Instrumental de administración de Windows](/windows/desktop/WmiSdk/wmi-start-page)
</dt> <dd>

Windows Instrumental de administración (WMI) es la infraestructura para la administración de datos y operaciones Windows sistemas operativos basados en la administración. Puede escribir scripts WMI o aplicaciones para automatizar tareas administrativas en equipos remotos.

</dd> <dt>

[API de Windows Defender WMIv2](/previous-versions/windows/desktop/defender/windows-defender-wmiv2-apis-portal)
</dt> <dd>

Windows Defender Las API WMI se pueden usar para administrar la protección contra malware a través de scripts o aplicaciones.

</dd> <dt>

[Windows PowerShell](https://msdn.microsoft.com/library/Dd835506(v=VS.85).aspx)
</dt> <dd>

Windows PowerShell es un shell de línea de comandos basado en tareas y un lenguaje de scripting diseñado especialmente para la administración del sistema. Basado en .NET Framework, Windows PowerShell ayuda a los profesionales de TI y a los usuarios avanzados a controlar y automatizar la administración tanto del sistema operativo Windows como de las aplicaciones que se ejecutan en Windows.

</dd> <dt>

[Administración remota de Windows](/windows/desktop/WinRM/portal)
</dt> <dd>

La administración remota de Windows (WinRM) es la implementación de Microsoft del protocolo WS-Management, un protocolo estándar basado en el Protocolo simple de acceso a objetos (SOAP) y compatible con el firewall que permite que el hardware y los sistemas operativos, de distintos proveedores, interoperan. La especificación WS-Management protocolo proporciona una manera común para que los sistemas accedan a la información de administración de una infraestructura de TI e intercamba información de administración a través de ella.

</dd> <dt>

[Windows Protección de recursos](/windows/desktop/Wfp/windows-resource-protection-portal)
</dt> <dd>

Windows Resource Protection (WRP) impide el reemplazo de archivos esenciales del sistema, carpetas y claves del Registro que se instalan como parte del sistema operativo.

</dd> <dt>

[Windows Server Update Services](windows-server-update-services.md)
</dt> <dd>

Los administradores del sistema pueden usar la API de Windows Server Update Services (WSUS) para determinar qué actualizaciones se aplican a un equipo o grupo de equipos, descargar esas actualizaciones e instalarlas con poca o ninguna intervención del usuario.

Los proveedores de software independientes y los desarrolladores de usuarios finales pueden integrar características de WSUS en la administración de equipos o en el software de administración de actualizaciones para proporcionar un entorno operativo sin problemas.

</dd> <dt>

[Windows Herramienta de evaluación del sistema](/windows/desktop/WinSAT/windows-system-assessment-tool-portal)
</dt> <dd>

Cómo usar WinSAT para evaluar las características y funcionalidades de rendimiento de un equipo.

</dd> <dt>

[Carpetas de trabajo](/previous-versions/windows/desktop/syncshareservermgmt/sync-share-server-management-portal)
</dt> <dd>

Carpetas de trabajo permite a los usuarios almacenar y acceder a archivos de trabajo en equipos y dispositivos personales.

</dd> </dl>
