# <a name="wsd-device-development"></a>Desarrollo de dispositivos WSD

Los fabricantes de dispositivos y otros implementadores de dispositivos pueden crear dispositivos que interoperen con clientes y hosts de WSDAPI basados en Windows sin usar WSDAPI.

Además de desarrollar la funcionalidad del dispositivo, los implementadores de dispositivos deben implementar el [Perfil de dispositivos para servicios web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS).

Siempre que sea posible, use WSDAPI para crear un cliente WSD basado en Windows o una aplicación host de dispositivo. El uso de WSDAPI reduce la complejidad del desarrollo y ahorra tiempo. Para obtener más información, vea [desarrollo de aplicaciones WSD en Windows](wsd-application-development-on-windows.md).
