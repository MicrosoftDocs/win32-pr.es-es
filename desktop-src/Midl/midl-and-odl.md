# <a name="midl-and-odl"></a>MIDL y ODL

El Lenguaje de definición de interfaz de Microsoft (MIDL) incluye ahora la sintaxis completa del lenguaje de definición de objetos (ODL). Esto le permite usar el compilador MIDL de 32 bits para generar una biblioteca de tipos y archivos de encabezado opcionales para una aplicación COM.

> [!Note]  
> Cuando la documentación hace referencia a un archivo ODL, hace referencia a un archivo que Mktyplib.exe puede analizar. Cuando hace referencia a un archivo IDL, hace referencia a un archivo que el compilador MIDL puede analizar. Esta es estrictamente una Convención de nomenclatura. El compilador MIDL analizará un archivo de entrada independientemente de su extensión de nombre de archivo.

-   [Archivos adicionales necesarios para generar una biblioteca de tipos](additional-files-required-to-generate-a-type-library-2.md)
-   [Diferencias entre MIDL y MkTypLib](differences-between-midl-and-mktyplib.md)
-   [Características del lenguaje ODL en MIDL](odl-language-features-in-midl.md)
-   [Generar un archivo DLL de proxy y una biblioteca de tipos desde un solo archivo IDL](generating-a-proxy-dll-and-a-type-library-from-a-single-idl-file-2.md)
