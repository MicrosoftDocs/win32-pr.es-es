# <a name="role-based-access-control"></a>Control de acceso basado en rol

La API de administrador de autorización y el complemento MMC proporcionan aplicaciones con un marco de control de acceso basado en roles. La API de administrador de autorización, también conocida como AzMan, proporciona un modelo de desarrollo simplificado para administrar grupos flexibles y reglas de negocios y almacenar directivas de autorización. Para obtener más información, vea los temas siguientes:

-   [Ventajas de la autorización basada en roles](advantages-of-role-based-authorization.md)
-   [Modelo de administrador de autorización](authorization-manager-model.md)

**Windows XP:** La API de administrador de autorización y el complemento MMC están disponibles como descarga (paquete de herramientas de administración de Windows Server 2003) en [https://www.microsoft.com/downloads](https://www.microsoft.com/Downloads/) . Esta descarga admite la administración y el desarrollo de directivas de administrador de autorización. no admite la implementación de aplicaciones que usan administrador de autorización para controlar el acceso.
