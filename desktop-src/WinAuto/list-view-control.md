# <a name="list-view-control-msaa-ui-element-reference"></a>Control List-View (referencia de elementos de interfaz de usuario de MSAA)

> [!Note]  
> En este tema se describen los objetos de **control de vista de lista** para fines de referencia de elementos de interfaz de usuario de MSAA. Cómo crear objetos de **control de vista de lista** en varios marcos de interfaz de usuario no se describe aquí. Consulte la documentación de referencia de la API del marco de interfaz de usuario que está usando.

 

Un control de vista de lista muestra una colección de elementos, cada uno de los cuales consta de un icono y una etiqueta, y proporciona varias maneras de mostrar y organizar los elementos. Por ejemplo, se muestra información adicional sobre cada elemento en columnas a la derecha del icono y la etiqueta. El explorador de Windows utiliza un control de vista de lista para mostrar los archivos.

El nombre de clase de ventana de un control de vista de lista es WC \_ ListView, que se define como "SysListView" en commctrl. h.

## <a name="notes"></a>Notas

-   Microsoft Active Accessibility no expone texto de elementos de vista de lista en aplicaciones que usan una función de devolución de llamada para agregar el texto al elemento.
-   Los elementos de la vista de lista que están recortados no tienen establecido el [**sistema de estado \_ \_ invisible**](object-state-constants.md) . Para determinar si un elemento de lista está recortado, los clientes pueden comparar las coordenadas del elemento devueltas por [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) con la región de recorte de la ventana.
-   Microsoft Active Accessibility no admite el uso del método [**accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) para desplazarse hasta el control de encabezado dentro de un control de vista de lista que se encuentra en la vista detalles. Sin embargo, los clientes utilizan la función [**AccessibleObjectFromPoint**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint) para obtener acceso al control de encabezado.
-   En el caso de los objetos no rectangulares, como los elementos de vista de lista en la vista de detalles, las coordenadas del rectángulo delimitador del elemento recuperado por [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) producen un error si se prueban con [**accHitTest**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest) porque **accHitTest** determina los límites del objeto píxel a píxel.
