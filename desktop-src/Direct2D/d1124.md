# <a name="d1124-dxgi-format-unknown-with-null-target-bitmap"></a>D1124: formato de DXGI desconocido con mapa de bits de destino nulo

\_ \_ No se permite el formato de DXGI desconocido como parámetro de esta API cuando el destino está establecido en null o es una lista de comandos.






 

## <a name="possible-causes"></a>Causas posibles

En el caso de operaciones como ID2D1RenderTarget:: CreateCompatibleRenderTarget, el destino de representación de origen debe tener asignado un mapa de bits de destino. Sin un mapa de bits de destino, Direct2D no puede determinar el formato de superficie que tendrá el nuevo recurso de destino de representación.

## <a name="possible-fixes"></a>Posibles correcciones

Para solucionar este error, asegúrese de que el ID2D1DeviceContext correspondiente tiene un mapa de bits de destino asignado mediante ID2D1DeviceContext:: SetTarget. O bien, si la aplicación simplemente necesita la capacidad de representar en dos superficies de destino diferentes, considere la posibilidad de crear un ID2D1Bitmap y establecerlo como el destino del contexto del dispositivo en lugar de usar ID2D1RenderTarget:: CreateCompatibleRenderTarget.
