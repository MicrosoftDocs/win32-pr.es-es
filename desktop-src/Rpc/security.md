# <a name="security-rpc"></a>Seguridad (RPC)

Con el mayor uso de las aplicaciones distribuidas, es primordial la necesidad de comunicaciones seguras entre las partes de las aplicaciones cliente y servidor. La biblioteca en tiempo de ejecución de llamada a procedimiento remoto (RPC) proporciona una interfaz estandarizada a los servicios de autenticación para clientes y servidores. Los servicios de autenticación del sistema host de servidor proporcionan autenticación de RPC. Las aplicaciones utilizan llamadas a procedimientos remotos autenticados para asegurarse de que todas las llamadas provienen de clientes autorizados. También pueden ayudar a garantizar que todas las respuestas del servidor provienen de servidores autenticados.

En esta sección se presenta una introducción a la RPC autenticada en las siguientes secciones:

-   [Aspectos básicos de seguridad de RPC](rpc-security-essentials.md)
-   [Métodos de seguridad](security-methods.md)
