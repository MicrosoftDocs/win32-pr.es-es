# <a name="s-upnp-apis"></a>S (API de UPnP)

Contiene términos relacionados con UPnP que comienzan por la letra S.

[A](a-gly.md) B [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F G [H](h-gly.md) I J K L M [N](n-gly.md) o [p](p-gly.md) Q [R](r-gly.md) s T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.s_1_gly"></span><span id="UPNP.S_1_GLY"></span>**servicio**
</dt> <dd>

Un servicio es una unidad funcional lógica. Los servicios exponen acciones y modelan algunos o todos los Estados de un dispositivo físico mediante variables de estado.

</dd> <dt>

<span id="upnp.s_2_gly"></span><span id="UPNP.S_2_GLY"></span>**Descripción del servicio**
</dt> <dd>

Una descripción del servicio es una lista de las acciones que proporciona un servicio y las variables de estado asociadas a las acciones.

</dd> <dt>

<span id="upnp.s_3_gly"></span><span id="UPNP.S_3_GLY"></span>**objeto de servicio**
</dt> <dd>

Un objeto de servicio es la representación de la API de punto de control de un servicio. Un objeto de servicio implementa la interfaz [**IUPnPService**](/windows/desktop/api/Upnp/nn-upnp-iupnpservice) .

</dd> <dt>

<span id="upnp.s_4_gly"></span><span id="UPNP.S_4_GLY"></span>**tipo de servicio**
</dt> <dd>

Un tipo de servicio es un URN que identifica un servicio. Hay una relación uno a uno entre una plantilla de servicio UPnP y un tipo de servicio. Los comités de trabajo del Foro de UPnP definen varios tipos de servicio estándar. Los tipos de servicio tienen el formato urn: schemas-UPnP-org: Service: serviceType: version. Por ejemplo, un tipo de servicio de analizador podría ser urn: schemas-UPnP-org: Service: Scanner: 1.

Los proveedores UPnP pueden especificar servicios adicionales. Estos servicios se indican mediante urn: Domain-Name: Service: serviceType: version, donde Domain-Name es un nombre de dominio registrado en el proveedor.

</dd> <dt>

<span id="upnp.s_5_gly"></span><span id="UPNP.S_5_GLY"></span>**variable de estado**
</dt> <dd>

Una variable de estado es un fragmento de datos que describe parte del estado de un servicio.

</dd> </dl>
