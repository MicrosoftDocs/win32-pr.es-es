# <a name="d1158-pixels-unit-mode-unsupported-with-print"></a>D1158: modo de unidad de píxeles no compatible con Print

Impresión de Direct2D: el comando SetUnitMode (píxeles) no compatible estaba presente en la lista de comandos enviada para la impresión.



|             |             |
|-------------|-------------|
| Nivel de error | Información |



 

## <a name="possible-causes"></a>Causas posibles

La impresión en Direct2D no permite el modo de unidad de píxeles. El modo de unidad predeterminado es DIP.

## <a name="possible-fixes"></a>Posibles correcciones

Para tratar este mensaje, no establezca el modo de unidad del contexto del dispositivo en píxeles mientras crea contenido imprimible. Si es necesario aplicar una transformación de escala global que afecte a todas las operaciones de dibujo primitivo, considere si es suficiente una llamada equivalente a ID2D1RenderTarget:: SetDpi o ID2D1RenderTarget:: SetTransform.
