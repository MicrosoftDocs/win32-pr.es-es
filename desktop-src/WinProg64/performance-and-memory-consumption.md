<span data-ttu-id="066aa-108">El rendimiento y el consumo de memoria en WOW64 vienen determinados por los siguientes factores:</span><span class="sxs-lookup"><span data-stu-id="066aa-108">Performance and memory consumption under WOW64 are determined by the following factors:</span></span>

-   <span data-ttu-id="066aa-109">Hardware del procesador.</span><span class="sxs-lookup"><span data-stu-id="066aa-109">Processor hardware.</span></span> <span data-ttu-id="066aa-110">La emulación de instrucciones se realiza en el chip.</span><span class="sxs-lookup"><span data-stu-id="066aa-110">Instruction emulation is performed on the chip.</span></span> <span data-ttu-id="066aa-111">En el procesador x64, el procesador ejecuta de forma nativa las instrucciones x86.</span><span class="sxs-lookup"><span data-stu-id="066aa-111">On the x64 processor, x86 instructions are executed natively by the processor.</span></span> <span data-ttu-id="066aa-112">Por lo tanto, la velocidad de ejecución en WOW64 en x64 es similar a la velocidad en Windows de 32 bits.</span><span class="sxs-lookup"><span data-stu-id="066aa-112">Therefore, execution speed under WOW64 on x64 is similar to its speed under 32-bit Windows.</span></span> <span data-ttu-id="066aa-113">En el procesador Intel Itanium y en todos los procesadores ARM64, hay más software implicado en la emulación y el rendimiento se ve afectado.</span><span class="sxs-lookup"><span data-stu-id="066aa-113">On the Intel Itanium processor and any ARM64 processors, more software is involved in the emulation, and performance suffers as a result.</span></span>

-   <span data-ttu-id="066aa-117">Tamaño de la memoria virtual.</span><span class="sxs-lookup"><span data-stu-id="066aa-117">Virtual memory size.</span></span> <span data-ttu-id="066aa-118">En el procesador Intel Itanium, WOW64 agrega una sobrecarga significativa si dos o más instancias de la misma aplicación de 32 bits se ejecutan simultáneamente.</span><span class="sxs-lookup"><span data-stu-id="066aa-118">On the Intel Itanium processor, WOW64 adds significant overhead if two or more instances of the same 32-bit application are running concurrently.</span></span> <span data-ttu-id="066aa-119">Esto se debe a las páginas nativas de 8 KB en Intel Itanium, que complica la emulación de las páginas nativas de 4 KB en la arquitectura x86 (más páginas se marcan como modificables; todas las páginas que se pueden escribir son privadas en el proceso).</span><span class="sxs-lookup"><span data-stu-id="066aa-119">This is due to the native 8 KB pages on the Intel Itanium, which complicates the emulation of the native 4 KB pages on the x86 architecture (more pages are marked as writable; all writable pages are private to the process).</span></span> <span data-ttu-id="066aa-120">Esto puede afectar negativamente a la escalabilidad de Terminal Services en determinados procesadores.</span><span class="sxs-lookup"><span data-stu-id="066aa-120">This can adversely affect the scalability of Terminal Services on certain processors.</span></span> <span data-ttu-id="066aa-121">Este no es el caso del procesador x64.</span><span class="sxs-lookup"><span data-stu-id="066aa-121">This is not the case for the x64 processor.</span></span>

<span data-ttu-id="066aa-124">WOW64 permite que las aplicaciones de 32 bits aprovechen el kernel de 64 bits.</span><span class="sxs-lookup"><span data-stu-id="066aa-124">WOW64 enables 32-bit applications to take advantage of the 64-bit kernel.</span></span> <span data-ttu-id="066aa-125">Por lo tanto, las aplicaciones de 32 bits pueden usar un mayor número de identificadores de kernel y de ventana.</span><span class="sxs-lookup"><span data-stu-id="066aa-125">Therefore, 32-bit applications can use a larger number of kernel handles and window handles.</span></span> <span data-ttu-id="066aa-126">Sin embargo, es posible que las aplicaciones de 32 bits no puedan crear tantos subprocesos en WOW64 como se ejecuten de forma nativa en sistemas basados en x86, porque WOW64 asigna una pila de 64 bits adicional (normalmente 512 KB) para cada subproceso.</span><span class="sxs-lookup"><span data-stu-id="066aa-126">However, 32-bit applications may not be able to create as many threads under WOW64 as they can when running natively on x86-based systems because WOW64 allocates an additional 64-bit stack (usually 512 KB) for each thread.</span></span> <span data-ttu-id="066aa-127">Además, se reserva una cantidad de espacio de direcciones para el propio WOW64 y las estructuras de datos que usa.</span><span class="sxs-lookup"><span data-stu-id="066aa-127">In addition, some amount of address space is reserved for WOW64 itself and the data structures it uses.</span></span> <span data-ttu-id="066aa-128">La cantidad reservada depende del procesador; hay más reservada en Intel Itanium que en los procesadores x64 o ARM64.</span><span class="sxs-lookup"><span data-stu-id="066aa-128">The amount reserved depends on the processor; more is reserved on the Intel Itanium than on the x64 or ARM64 processors.</span></span>
