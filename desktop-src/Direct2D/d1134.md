# <a name="d1134-buffer-precision-disallowed-by-caps"></a>D1134: precisión de búfer no permitida por Cap

El dispositivo no tiene la capacidad de admitir la precisión de búfer especificada.






 

## <a name="possible-causes"></a>Causas posibles

Direct2D permite que una aplicación especifique una precisión de búfer deseada para algunas operaciones. Por ejemplo, la precisión usada para la canalización de imágenes se especifica mediante [**ID2D1DeviceContext:: SetRenderingControls**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1devicecontext-setrenderingcontrols(constd2d1_rendering_controls)). Sin embargo, es posible que los dispositivos de hardware Direct2D no admitan todas las precisiones de búfer, en función de su nivel de características. Por ejemplo, es posible que los dispositivos de nivel de característica 9 no admitan opciones de precisión de búfer distintas de 8 bits por canal.

## <a name="possible-fixes"></a>Posibles correcciones

En el caso de las aplicaciones que se ejecutan en hardware, asegúrese de que el nivel de características del hardware admite todos los especificadores de precisión del búfer. Considere la posibilidad de usar [**ID3D11Device:: CheckFormatSupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkformatsupport) y [**ID3D11Device:: CheckFeatureSupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkfeaturesupport) para obtener más información acerca de los formatos admitidos por el hardware y para determinar qué precisión de superficie se permitirán. Para una operación coherente con todos los dispositivos de nivel de característica 9, diseñe un mecanismo de reserva para evitar búferes de precisión alta.
