# <a name="asynchronous-monikers"></a><span data-ttu-id="a5759-103">Monikers asincrónicos</span><span class="sxs-lookup"><span data-stu-id="a5759-103">Asynchronous Monikers</span></span>

<span data-ttu-id="a5759-104">La arquitectura de moniker de OLE proporciona un modelo de programación coherente y extensible para trabajar con objetos de Internet, proporcionando métodos para analizar nombres, que representan localizadores de recursos universales (URL) como nombres imprimibles, y localizar y enlazar a los objetos representados por cadenas de dirección URL.</span><span class="sxs-lookup"><span data-stu-id="a5759-104">The OLE moniker architecture provides a consistent, extensible programming model for working with Internet objects, providing methods for parsing names, representing Universal Resource Locators (URLs) as printable names, and locating and binding to the objects represented by URL strings.</span></span> <span data-ttu-id="a5759-105">(Consulte también [monikers de dirección URL](url-monikers.md)). Sin embargo, los monikers estándar de OLE (especialmente, los monikers de elementos, archivos y punteros) no son adecuados para Internet porque son sincrónicos y devuelven un puntero a un objeto o su almacenamiento solo en ese momento, ya que todos los datos están disponibles.</span><span class="sxs-lookup"><span data-stu-id="a5759-105">(Also see [URL Monikers](url-monikers.md).) Standard OLE monikers (notably, item, file, and pointer monikers), however, are inappropriate for the Internet because they are synchronous, returning a pointer to an object or its storage only at such time as all data is available.</span></span> <span data-ttu-id="a5759-106">Dependiendo de la cantidad de datos que se van a descargar, el enlace de forma sincrónica puede asociar la interfaz de usuario del cliente durante períodos prolongados.</span><span class="sxs-lookup"><span data-stu-id="a5759-106">Depending on the amount of data to be downloaded, binding synchronously can tie up the client's user interface for prolonged periods.</span></span>

<span data-ttu-id="a5759-107">Internet requiere nuevos enfoques para el diseño de aplicaciones.</span><span class="sxs-lookup"><span data-stu-id="a5759-107">The Internet requires new approaches to application design.</span></span> <span data-ttu-id="a5759-108">Las aplicaciones deben poder realizar todas las operaciones de red costosas de forma asincrónica para evitar la detención de la interfaz de usuario.</span><span class="sxs-lookup"><span data-stu-id="a5759-108">Applications should be able to perform all expensive network operations asynchronously to avoid stalling the user interface.</span></span> <span data-ttu-id="a5759-109">Una aplicación debe ser capaz de desencadenar una operación y recibir notificaciones sobre la finalización completa o parcial.</span><span class="sxs-lookup"><span data-stu-id="a5759-109">An application should be able to trigger an operation and receive notification on full or partial completion.</span></span> <span data-ttu-id="a5759-110">En ese momento, la aplicación debe elegir entre continuar con el siguiente paso de la operación o proporcionar información adicional según sea necesario.</span><span class="sxs-lookup"><span data-stu-id="a5759-110">At that point, the application should have the choice either of proceeding with the next step of the operation or providing additional information as needed.</span></span> <span data-ttu-id="a5759-111">A medida que se lleva a cabo una descarga, una aplicación también debe ser capaz de proporcionar información de progreso y la oportunidad de cancelar la operación en cualquier momento.</span><span class="sxs-lookup"><span data-stu-id="a5759-111">As a download proceeds, an application should also be able to provide users with progress information and the opportunity to cancel the operation at any time.</span></span>

<span data-ttu-id="a5759-112">Los monikers asincrónicos proporcionan estas capacidades, así como varios niveles de comportamiento de enlace asincrónico, a la vez que proporcionan compatibilidad con versiones anteriores para las aplicaciones que no se reconocen o que no requieren un comportamiento asincrónico.</span><span class="sxs-lookup"><span data-stu-id="a5759-112">Asynchronous monikers provide these capabilities, as well as various levels of asynchronous binding behavior, while providing backward compatibility for applications that are either unaware of or do not require asynchronous behavior.</span></span> <span data-ttu-id="a5759-113">Otra tecnología OLE, almacenamiento asincrónico, funciona con monikers asincrónicos para proporcionar descarga asincrónica del estado persistente de un objeto de Internet.</span><span class="sxs-lookup"><span data-stu-id="a5759-113">Another OLE technology, asynchronous storage, works with asynchronous monikers to provide asynchronous downloading of an Internet object's persistent state.</span></span> <span data-ttu-id="a5759-114">El moniker asincrónico desencadena la operación de enlace y configura los componentes necesarios, incluidos los objetos de almacenamiento y de secuencia, los objetos de matriz de bytes y los receptores de notificación.</span><span class="sxs-lookup"><span data-stu-id="a5759-114">The asynchronous moniker triggers the bind operation and sets up the necessary components, including storage and stream objects, byte-array objects, and notification sinks.</span></span> <span data-ttu-id="a5759-115">Una vez que los componentes están conectados, el moniker sale del camino y el resto del enlace se ejecuta principalmente entre los componentes que implementan los componentes de almacenamiento asincrónico y el objeto.</span><span class="sxs-lookup"><span data-stu-id="a5759-115">Once the components are connected, the moniker gets out of the way and the rest of the bind is executed mainly between the components implementing the asynchronous storage components and the object.</span></span>
