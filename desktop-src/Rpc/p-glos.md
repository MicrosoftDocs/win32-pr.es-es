</dt> <dd>

[*Identificador de enlace*](b-glos.md) que no incluye información de [*extremo*](e-glos.md) . Vea también [*identificador completamente enlazado*](f-glos.md).

</dd> <dt>

<span id="_rpc_pickling_glos"></span><span id="_RPC_PICKLING_GLOS"></span>**selección**
</dt> <dd>

Vea [*serialización*](s-glos.md).

</dd> <dt>

<span id="_rpc_pipe_glos"></span><span id="_RPC_PIPE_GLOS"></span>**codo**
</dt> <dd>

Constructor de tipo [*IDL*](i-glos.md)que admite la transmisión de un flujo de datos de un extremo abierto entre las aplicaciones cliente y servidor.

</dd> <dt>

<span id="_rpc_primary_enterprise_controller_glos"></span><span id="_RPC_PRIMARY_ENTERPRISE_CONTROLLER_GLOS"></span>**controlador Enterprise principal**
</dt> <dd>

La base de datos del servicio de información de cola de mensajes (MQIS) maestro para una red. Vea también [*servicio de información de cola de mensajes (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_primary_site_controller_glos"></span><span id="_RPC_PRIMARY_SITE_CONTROLLER_GLOS"></span>**controlador de sitio principal**
</dt> <dd>

Base de datos del servicio de información de cola de mensajes (MQIS) para un sitio determinado dentro de una empresa. Vea también [*servicio de información de cola de mensajes (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_procedure_serialization_glos"></span><span id="_RPC_PROCEDURE_SERIALIZATION_GLOS"></span>**serialización de procedimientos**
</dt> <dd>

[*Serialización*](s-glos.md) de datos que usa un código auxiliar de serialización generado por MIDL para realizar la codificación y la descodificación de uno o más tipos con una única llamada a procedimiento. La serialización de procedimientos se logra aplicando los atributos de \[ [codificación](/windows/desktop/Midl/encode) \] y \[ [descodificación](/windows/desktop/Midl/decode) \] a un prototipo de función en el archivo ACF. Vea también la [*serialización de tipos*](t-glos.md).

</dd> <dt>

<span id="_rpc_protocol_sequence_glos"></span><span id="_RPC_PROTOCOL_SEQUENCE_GLOS"></span>**secuencia de protocolos**
</dt> <dd>

Cadena de caracteres que representa una combinación válida de un protocolo RPC, un protocolo de nivel de red y un protocolo de capa de transporte. Por ejemplo, la secuencia de protocolos NCACN \_ IP \_ TCP describe una conexión de [*NCA*](n-glos.md) a través de un protocolo de Internet (IP) con un protocolo de control de transmisión (TCP) como transporte.

</dd> <dt>

<span id="_rpc_proxy_stub_glos"></span><span id="_RPC_PROXY_STUB_GLOS"></span>**código auxiliar de proxy**
</dt> <dd>

Lenguaje C o C++ generado por MIDL: código fuente que contiene todas las funciones necesarias para una interfaz OLE personalizada.

</dd> </dl>
