# <a name="d1180-begindraw-incorrect-state"></a>D1180: estado incorrecto de BeginDraw

Se llamó a BeginDraw varias veces, sin una llamada correspondiente a EndDraw.






 

## <a name="possible-causes"></a>Causas posibles

La aplicación realizó varias llamadas a BeginDraw sin llamar a EndDraw en el mismo destino. Al usar Direct2D, las aplicaciones deben coincidir con cada llamada a BeginDraw con un EndDraw subsiguiente. Como alternativa, este mensaje puede producirse si el destino está en un estado erróneo y la llamada BeginDraw no permitió que el destino se representara.

## <a name="possible-fixes"></a>Posibles correcciones

Asegúrese de que cada llamada a BeginDraw coincida con una EndDraw subsiguiente. Al enviar comandos a un destino mediante flush o EndDraw, compruebe que el resultado devuelto es S \_ OK antes de intentar volver a usar el destino.
