# <a name="the-evolution-of-file-systems"></a><span data-ttu-id="51814-103">La evolución de los sistemas de archivos</span><span class="sxs-lookup"><span data-stu-id="51814-103">The Evolution of File Systems</span></span>

<span data-ttu-id="51814-104">Antes de que los equipos se desarrollaron para funcionar en los sistemas operativos de disco, cada equipo se compiló para ejecutar una única aplicación de propiedad, que tenía un control completo y exclusivo de todo el equipo.</span><span class="sxs-lookup"><span data-stu-id="51814-104">Before computers were developed to function on disk operating systems, each computer was built to run a single, proprietary application, which had complete and exclusive control of the entire machine.</span></span> <span data-ttu-id="51814-105">La aplicación escribiría sus datos persistentes directamente en un disco, o tambor, enviando comandos directamente al controlador de disco.</span><span class="sxs-lookup"><span data-stu-id="51814-105">The application would write its persistent data directly to a disk, or drum, by sending commands directly to the disk controller.</span></span> <span data-ttu-id="51814-106">La aplicación era responsable de administrar las ubicaciones absolutas de los datos en el disco, asegurándose de que no se sobrescribiran los datos ya existentes.</span><span class="sxs-lookup"><span data-stu-id="51814-106">The application was responsible for managing the absolute locations of data on the disk, making sure that it was not overwriting already-existing data.</span></span> <span data-ttu-id="51814-107">Puesto que solo se estaba ejecutando una aplicación en el equipo en cualquier momento, esta tarea no era demasiado difícil.</span><span class="sxs-lookup"><span data-stu-id="51814-107">Since only one application was running on the computer at any time, this task was not too difficult.</span></span>

<span data-ttu-id="51814-108">La llegada de los sistemas informáticos que podían ejecutar más de una aplicación requiere un mecanismo para asegurarse de que las aplicaciones no han escrito los datos de los demás.</span><span class="sxs-lookup"><span data-stu-id="51814-108">The advent of computer systems that could run more than one application required a mechanism to ensure that applications did not write over each other's data.</span></span> <span data-ttu-id="51814-109">Para solucionar este problema, los desarrolladores de aplicaciones adoptan un estándar único para distinguir los sectores de disco en uso de los que estaban libres marcándolos.</span><span class="sxs-lookup"><span data-stu-id="51814-109">Application developers addressed this problem by adopting a single standard for distinguishing disk sectors in use from those that were free by marking them accordingly.</span></span> <span data-ttu-id="51814-110">En el tiempo, estos estándares se fusionaron para convertirse en un sistema operativo de disco, que proporcionaba varios servicios a las aplicaciones, incluido un sistema de archivos para administrar el almacenamiento persistente.</span><span class="sxs-lookup"><span data-stu-id="51814-110">In time, these standards coalesced to become a disk operating system, which provided various services to the applications, including a file system for managing persistent storage.</span></span> <span data-ttu-id="51814-111">Con la llegada de un sistema de archivos, las aplicaciones ya no tenían que tratar directamente con el medio de almacenamiento físico.</span><span class="sxs-lookup"><span data-stu-id="51814-111">With the advent of a file system, applications no longer had to deal directly with the physical storage medium.</span></span> <span data-ttu-id="51814-112">En su lugar, simplemente indicaban al sistema de archivos que escribira bloques de datos en el disco y dejar que el sistema de archivos se preocupe de cómo hacerlo.</span><span class="sxs-lookup"><span data-stu-id="51814-112">Instead, they simply told the file system to write blocks of data to the disk and let the file system worry about how to do it.</span></span> <span data-ttu-id="51814-113">Además, el sistema de archivos permitía a las aplicaciones crear jerarquías de datos a través de una abstracción conocida como directorio.</span><span class="sxs-lookup"><span data-stu-id="51814-113">In addition, the file system allowed applications to create data hierarchies through an abstraction known as a directory.</span></span> <span data-ttu-id="51814-114">Un directorio no puede contener solo archivos y otros directorios que, a su vez, pueden contener sus propios archivos y directorios, etc.</span><span class="sxs-lookup"><span data-stu-id="51814-114">A directory could contain not only files but other directories, which in turn could contain their own files and directories, and so on.</span></span>

<span data-ttu-id="51814-115">El sistema de archivos proporcionó un único nivel de direccionamiento indirecto entre las aplicaciones y el disco, y el resultado fue que cada aplicación vio un archivo como una única secuencia contigua de bytes en el disco, aunque el sistema de archivos almacenara realmente el archivo en sectores no contiguos.</span><span class="sxs-lookup"><span data-stu-id="51814-115">The file system provided a single level of indirection between applications and the disk, and the result was that every application saw a file as a single contiguous stream of bytes on the disk even though the file system was actually storing the file in discontiguous sectors.</span></span> <span data-ttu-id="51814-116">El direccionamiento indirecto ha liberado a las aplicaciones de tener que realizar un seguimiento de la posición absoluta de los datos en un dispositivo de almacenamiento.</span><span class="sxs-lookup"><span data-stu-id="51814-116">The indirection freed the applications from having to track the absolute position of data on a storage device.</span></span>

<span data-ttu-id="51814-117">En la actualidad, prácticamente todas las API del sistema para la entrada y salida de archivos proporcionan aplicaciones para escribir información en un archivo plano.</span><span class="sxs-lookup"><span data-stu-id="51814-117">Today, virtually all system APIs for file input and output provide applications for writing information into a flat file.</span></span> <span data-ttu-id="51814-118">Las aplicaciones ven este archivo como un flujo de bytes único que puede crecer tan grande como sea necesario hasta que el disco esté lleno.</span><span class="sxs-lookup"><span data-stu-id="51814-118">Applications see this file as a single stream of bytes that can grow as large as necessary until the disk is full.</span></span> <span data-ttu-id="51814-119">Durante mucho tiempo, estas API han sido suficientes para que las aplicaciones almacenen su información persistente.</span><span class="sxs-lookup"><span data-stu-id="51814-119">For a long time these APIs have been sufficient for applications to store their persistent information.</span></span> <span data-ttu-id="51814-120">Las aplicaciones han realizado importantes innovaciones en el modo en que se ocupan de una única secuencia de información para proporcionar características como el guardado "rápido" incremental.</span><span class="sxs-lookup"><span data-stu-id="51814-120">Applications have made significant innovations in how they deal with a single stream of information to provide features like incremental "fast" saves.</span></span>

<span data-ttu-id="51814-121">Sin embargo, en un mundo de objetos de componentes, el almacenamiento de datos en un solo archivo plano ya no es eficaz.</span><span class="sxs-lookup"><span data-stu-id="51814-121">However, in a world of component objects, storing data in a single flat file is no longer efficient.</span></span> <span data-ttu-id="51814-122">Del mismo modo que los sistemas de archivos surgieron de la necesidad de que varias aplicaciones compartan el mismo medio de almacenamiento, por lo que, ahora, los objetos de componente requieren un sistema que les permita compartir almacenamiento en el marco conceptual de un solo archivo.</span><span class="sxs-lookup"><span data-stu-id="51814-122">Just as file systems arose out of the need for multiple applications to share the same storage medium, so, now, do component objects require a system that allows them to share storage within the conceptual framework of a single file.</span></span> <span data-ttu-id="51814-123">Aunque es posible almacenar los objetos independientes mediante el almacenamiento de archivo sin formato convencional, si uno de los objetos aumenta de tamaño, o si simplemente agrega otro objeto, es necesario cargar el archivo completo en la memoria, insertar el nuevo objeto y, a continuación, guardar el archivo completo.</span><span class="sxs-lookup"><span data-stu-id="51814-123">Even though it is possible to store the separate objects using conventional flat-file storage, if one of the objects increases in size, or if you simply add another object, it becomes necessary to load the entire file into memory, insert the new object, and then save the whole file.</span></span> <span data-ttu-id="51814-124">Este proceso puede llevar mucho tiempo.</span><span class="sxs-lookup"><span data-stu-id="51814-124">This process can be extremely time-consuming.</span></span>

<span data-ttu-id="51814-125">La solución que proporciona COM es implementar un segundo nivel de direccionamiento indirecto: un sistema de archivos dentro de un archivo.</span><span class="sxs-lookup"><span data-stu-id="51814-125">The solution provided by COM is to implement a second level of indirection: a file system within a file.</span></span> <span data-ttu-id="51814-126">El almacenamiento de archivo sin formato requiere que una secuencia de bytes contigua grande del disco se manipule a través de un único identificador de archivo con un único puntero de búsqueda.</span><span class="sxs-lookup"><span data-stu-id="51814-126">Flat-file storage requires that a large contiguous sequence of bytes on the disk be manipulated through a single file handle with a single seek pointer.</span></span> <span data-ttu-id="51814-127">Por el contrario, el almacenamiento estructurado COM define cómo tratar una única entidad del sistema de archivos como una colección estructurada de dos tipos de objetos (almacenamientos y secuencias) que actúan como archivos y directorios.</span><span class="sxs-lookup"><span data-stu-id="51814-127">By contrast, COM structured storage defines how to treat a single file system entity as a structured collection of two types of objects — storages and streams — that act like directories and files.</span></span>
