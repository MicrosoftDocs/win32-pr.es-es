# <a name="radiobuttongroup-control"></a>Control RadioButtonGroup

El control RadioButtonGroup es un grupo de botones de radio. Permite al usuario seleccionar una cadena o un valor entero para una propiedad de una lista predeterminada de valores. Puede asociar este control a una propiedad escribiendo el nombre de la propiedad en la columna de propiedades de la [tabla de control](control-table.md). Especifique los valores posibles para la selección en la columna valor de la [tabla RadioButton](radiobutton-table.md). Tenga en cuenta que la cadena mostrada no es necesariamente la misma que el valor elegido por el usuario.

Cada control RadioButtonGroup está asociado a una propiedad. El valor predeterminado de esta propiedad se debe inicializar en la [tabla de propiedades](property-table.md). Dentro de cada RadioButtonGroup especificado en la [tabla RadioButton](radiobutton-table.md), puede haber un botón de radio que tenga un valor en el campo valor que coincida con el valor predeterminado de esta propiedad. Este es el botón predeterminado del control RadioButtonGroup. El botón **predeterminado** se muestra inicialmente como seleccionado en el control.

Tenga en cuenta que el foco en un cuadro de diálogo no puede moverse a un control RadioButtonGroup hasta que se haya seleccionado uno de los botones del grupo. Para que el foco se mueva a este grupo de botones, especifique uno de los botones como botón predeterminado para el grupo.

Los controles RadioButtonGroup solo establecen valores de propiedad y no se pueden usar para enviar un [ControlEvent,](control-events.md).

La implementación trata todo el grupo como un control, por lo que no es posible ocultar o deshabilitar botones individuales dentro del grupo. De igual forma, todos los botones deben tener el mismo estilo, es decir, todos ellos tienen texto o todos ellos tienen mapas de bits (u otras características similares). La posición de los botones se crea en relación con el grupo. De este modo, todo el grupo puede moverse cambiando solo las coordenadas del grupo sin cambiar los botones individuales. En la creación, el control comprueba que los botones individuales no se extienden más allá de los límites del grupo.

## <a name="remarks"></a>Observaciones

Debido al modo en que Windows dibuja el marco, hay un hueco entre la parte superior de la ventana del control y el marco visible, incluso cuando no hay ningún título.

Este control se puede crear a partir de la clase BUTTON mediante la función [**CreateWindowEx**](/windows/win32/api/winuser/nf-winuser-createwindowexa) . Si se establece el bit [HasBorder](hasborder-control-attribute.md) , tiene el estilo **de \_ GROUPBOX** , de lo contrario, tiene el estilo de **\_ OWNERDRAW BS** .

El control RadioButtonGroup no debe superponerse a otros controles y otros controles no deben superponerse a un RadioButtonGroup. La superposición de este control y otro puede hacer que los controles funcionen o se muestren incorrectamente. Para proporcionar a los programas lector de pantalla un texto descriptivo adicional sobre un control RadioButtonGroup, siga el ejemplo [que se proporciona en agregar texto adicional a los botones de radio](adding-extra-text-to-radio-buttons.md).
