## <a name="filter-chain-guidelines"></a><span data-ttu-id="9e42b-127">Instrucciones de la cadena de filtros</span><span class="sxs-lookup"><span data-stu-id="9e42b-127">Filter Chain Guidelines</span></span>

<span data-ttu-id="9e42b-128">Cuando se usan métodos **IFilterChain** , es importante asegurarse de que los filtros del gráfico pueden admitir operaciones de encadenamiento de filtros.</span><span class="sxs-lookup"><span data-stu-id="9e42b-128">When you use **IFilterChain** methods, it is important to make sure that the filters in the graph can support filter chaining operations.</span></span> <span data-ttu-id="9e42b-129">De lo contrario, podría causar interbloqueos o errores de grafo.</span><span class="sxs-lookup"><span data-stu-id="9e42b-129">Otherwise, you might cause deadlocks or graph errors.</span></span> <span data-ttu-id="9e42b-130">Los filtros conectados a la cadena deben funcionar correctamente después de que cambie el estado de la cadena.</span><span class="sxs-lookup"><span data-stu-id="9e42b-130">Filters connected to the chain must function correctly after the chain changes state.</span></span>

<span data-ttu-id="9e42b-131">La mejor manera de usar **IFilterChain** es con un conjunto de filtros que se han diseñado específicamente para el encadenamiento.</span><span class="sxs-lookup"><span data-stu-id="9e42b-131">The best way to use **IFilterChain** is with a set of filters that you have designed specifically for chaining.</span></span> <span data-ttu-id="9e42b-132">Utilice las siguientes directrices para asegurarse de que los filtros son seguros para las operaciones de cadena de filtros.</span><span class="sxs-lookup"><span data-stu-id="9e42b-132">Use the following guidelines to ensure that your filters are safe for filter chain operations.</span></span> <span data-ttu-id="9e42b-133">Estos puntos hacen referencia al diagrama siguiente.</span><span class="sxs-lookup"><span data-stu-id="9e42b-133">These points refer to the following diagram.</span></span>

-   <span data-ttu-id="9e42b-135">Antes de que cambie el estado de la cadena de filtros, se deben completar todas las llamadas de procesamiento de datos en el límite de la cadena de filtros.</span><span class="sxs-lookup"><span data-stu-id="9e42b-135">Before the filter chain's state changes, all data processing calls at the boundary of the filter chain must be completed.</span></span> <span data-ttu-id="9e42b-136">Esta regla se aplica a los métodos [**IMemInputPin:: Receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive), [**IPin:: NewSegment**](/windows/desktop/api/Strmif/nf-strmif-ipin-newsegment)y [**IPin:: EndOfStream**](/windows/desktop/api/Strmif/nf-strmif-ipin-endofstream).</span><span class="sxs-lookup"><span data-stu-id="9e42b-136">This rule applies to the methods [**IMemInputPin::Receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive), [**IPin::NewSegment**](/windows/desktop/api/Strmif/nf-strmif-ipin-newsegment), and [**IPin::EndOfStream**](/windows/desktop/api/Strmif/nf-strmif-ipin-endofstream).</span></span> <span data-ttu-id="9e42b-137">Los filtros de la cadena deben devolver de las llamadas a estos métodos realizados por filtros fuera de la cadena. y los filtros fuera de la cadena deben devolver desde las llamadas realizadas por filtros dentro de la cadena.</span><span class="sxs-lookup"><span data-stu-id="9e42b-137">Filters in the chain must return from calls to these methods made by filters outside the chain; and filters outside the chain must return from calls made by filters within the chain.</span></span>

-   <span data-ttu-id="9e42b-140">Los filtros ascendentes deben esperar que el estado de la cadena cambie.</span><span class="sxs-lookup"><span data-stu-id="9e42b-140">Upstream filters must expect the chain's state to change.</span></span> <span data-ttu-id="9e42b-141">Por ejemplo, en el diagrama anterior, supongamos que la cadena está detenida pero el filtro A llama a **IMemInputPin:: Receive**.</span><span class="sxs-lookup"><span data-stu-id="9e42b-141">For example, in the previous diagram, suppose the chain is stopped but filter A calls **IMemInputPin::Receive**.</span></span> <span data-ttu-id="9e42b-142">Se produce un error en la llamada y la respuesta del filtro a es detener el streaming.</span><span class="sxs-lookup"><span data-stu-id="9e42b-142">The call fails and the response of filter A is to stop streaming.</span></span> <span data-ttu-id="9e42b-143">Cuando la aplicación reinicia la cadena, no tiene ningún efecto porque el filtro A ya no transmite datos.</span><span class="sxs-lookup"><span data-stu-id="9e42b-143">When the application restarts the chain, it has no effect because filter A is no longer streaming data.</span></span>

-   <span data-ttu-id="9e42b-149">Cada conexión de PIN de un filtro fuera de la cadena a un filtro dentro de la cadena debe tener su propio asignador, que no se comparte con otras conexiones.</span><span class="sxs-lookup"><span data-stu-id="9e42b-149">Each pin connection from a filter outside the chain to a filter within the chain should have its own allocator, which is not shared by other connections.</span></span> <span data-ttu-id="9e42b-150">Cuando la cadena cambia de estado o se quita del gráfico, es posible que el asignador esté desasignado.</span><span class="sxs-lookup"><span data-stu-id="9e42b-150">When the chain changes state or is removed from the graph, the allocator might be decommitted.</span></span> <span data-ttu-id="9e42b-151">Si otras conexiones usaban el mismo asignador, ya no podrán procesar ejemplos.</span><span class="sxs-lookup"><span data-stu-id="9e42b-151">If other connections were using the same allocator, they can no longer process samples.</span></span>
