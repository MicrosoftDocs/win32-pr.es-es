# <a name="windows-8-and-windows-server-2012-client-and-server-compatibility"></a>Compatibilidad de servidor y cliente de Windows 8 y Windows Server 2012

En esta sección se describen los cambios en el sistema operativo a los que debe prestar atención especial debido a los posibles impactos en las aplicaciones existentes y cómo deben diseñarse las nuevas aplicaciones en los clientes, en servidores o en clientes y servidores. A continuación se muestra la lista de temas que se tratan en esta sección, agrupados por área de características:

**AppCompat**

-   Actualización de reglas de detección de aplicaciones de seguridad
-   Determinar el estado de la corrección de compatibilidad
-   Las aplicaciones de servidor deben poder ejecutarse sin el shell gráfico de servidor
-   Los componentes de servidor RDS se quitan de Windows 8
-   Modelo de tipo de archivo y asociaciones de protocolo
-   Moderador de la actividad del escritorio
-   .NET Framework 4,5 es el valor predeterminado y .NET Framework 3,5 es opcional
-   Es posible que las aplicaciones de escritorio no estén visibles después de iniciar el explorador Web predeterminado
-   Modo de contraste alto
-   Manifiesto de aplicación (ejecutable)
-   El modelo de presentación en cola está en desuso
-   Escenarios del Asistente para la compatibilidad de programas para Windows 8
-   Gadgets de escritorio quitados

**Almacenamiento y sistema de archivos**

-   Actualización de compatibilidad de disco Advanced Format (4K)
-   Aprovisionamiento fino de unidades lógicas
-   El almacenamiento mejorado ahora es opcional para el Entorno de preinstalación de Windows (WinPE) y el SKU de servidor
-   El servicio de disco virtual está cambiando a la API de administración de almacenamiento de Windows basada en Instrumental de administración de Windows (WMI)
-   Interfaz de usuario de versiones anteriores quitada para volúmenes locales
-   StorAHCI reemplaza a MSAHCI
-   Copias de seguridad y restauración de Windows 7 en desuso
-   Transferencias de datos descargados

**Otros**

-   Administrador de ventanas de escritorio está siempre activado
-   Direct2D no admite la representación en metaarchivos "ricos" en Internet Explorer 9
-   Cambios en la compatibilidad con hardware heredado de DX9
-   MSAA no está disponible para las aplicaciones de la tienda Windows
-   El puerto 3 está en desuso para los controladores NDIS 6,30
