# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: intento no válido de dibujar un mapa de bits

No se pudo realizar una operación porque el mapa de bits tiene las \_ Opciones de mapa de bits D2D1 \_ \_ no se puede \_ dibujar la opción.



|             |             |
|-------------|-------------|
| Nivel de error | Información |



 

## <a name="possible-causes"></a>Causas posibles

Este mensaje se produce cuando una aplicación intenta dibujar un mapa de bits marcado con \_ las opciones de mapa de bits de D2D1 \_ \_ no puede \_ dibujar. Esto incluye llamadas directas a DrawBitmap, así como el dibujo de primitivas mediante un pincel de mapa de bits.

## <a name="possible-fixes"></a>Posibles correcciones

La marca no se puede \_ dibujar se usa para anotar correctamente los recursos que solo se pueden usar como destinos, como los búferes de la cadena de intercambio. Sin embargo, también se permite omitir la marca no se puede \_ dibujar en algunos mapas de bits de destino si también se pueden usar como origen. Si este es el caso, simplemente omita la marca no se puede \_ dibujar. Si el mapa de bits se debe anotar con no puede \_ dibujar debido a su uso exclusivo como destino o para permitir la protección de la CPU, considere la posibilidad de usar una textura de ensayo con el mismo tamaño que el origen deseado, pero sin la designación no se puede \_ dibujar. Copie en la textura de ensayo mediante ID3D11DeviceContext:: CopyResource y use la textura de ensayo como origen para la operación de dibujo. El uso de una textura de ensayo da lugar a implicaciones en el rendimiento. pueden ser aceptables si el mapa de bits de origen no cambia con frecuencia.
