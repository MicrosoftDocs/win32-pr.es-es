# <a name="outputtype-complex-type"></a>Tipo complejo de OutputType

Define un tipo de datos de salida que determina cómo se representan los datos.

``` syntax
<xs:complexType name="OutputType">
    <xs:simpleContent>
        <xs:extension
            base="string"
        >
            <xs:attribute name="default"
                type="boolean"
                use="optional"
             />
            <xs:attribute name="xmlType"
                type="QName"
                use="required"
             />
        </xs:extension>
    </xs:simpleContent>
</xs:complexType>
```

## <a name="attributes"></a>Atributos



| Nombre    | Tipo      | Descripción                                                                                                                                                                                       |
|---------|-----------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| default | boolean   | Determina si este tipo de salida se debe utilizar como el tipo de salida predeterminado para el tipo de entrada primario. Establézcalo en **true** para utilizar este tipo de salida como el tipo predeterminado; en caso contrario, **false**.<br/> |
| xmlType | **QName** | Nombre del tipo de salida.<br/>                                                                                                                                                           |



## <a name="remarks"></a>Observaciones

A continuación se enumeran los tipos de salida reconocidos que se pueden especificar en el manifiesto. El tipo de salida determina el modo en que el servicio representa los datos. Los tipos de salida se definen en el \\ archivo de inclusión de \\Winmeta.xml, que se incluye en el Windows SDK.

**Windows Server 2008 y Windows Vista:** El servicio no utiliza el tipo de salida para representar los datos; en su lugar, el servicio usa el tipo de entrada para determinar cómo se representan los datos.
