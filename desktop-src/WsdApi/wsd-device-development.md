# <a name="wsd-device-development"></a>Desarrollo de dispositivos WSD

Los fabricantes de dispositivos y otros implementadores de dispositivos pueden crear dispositivos que interoperan con Windows clientes y hosts WSDAPI basados en Windows sin usar WSDAPI.

Además de desarrollar la funcionalidad del dispositivo, los implementadores de dispositivos deben implementar el perfil [de dispositivos](https://specs.xmlsoap.org/ws/2006/02/devprof/) para servicios web (DPWS).

Siempre que sea posible, use WSDAPI para crear una aplicación de host de dispositivo o cliente de WSD basada en Windows basado en dispositivos. El uso de WSDAPI reduce la complejidad del desarrollo y ahorra tiempo. Para obtener más información, vea [WSD Application Development on Windows](wsd-application-development-on-windows.md).
