# <a name="structured-storage-api-functions"></a>Funciones de api Storage estructurada

Algunas de las funciones de API para el almacenamiento estructurado son funciones auxiliares que realizan una secuencia de llamadas a otras funciones de API y métodos de interfaz. Puede usar estas funciones auxiliares como cortas.

Otras funciones de API proporcionan acceso a la implementación de COM de almacenamiento estructurado, Compound Files.

Otro conjunto de funciones de API le permite convertir objetos OLE 1 en almacenamiento estructurado. Puede usar estas funciones para determinar si una clase de objeto es de OLE 1 y para convertir objetos entre OLE 1 y los formatos de almacenamiento COM actuales.

Por último, hay funciones de API para convertir y emular otros objetos. Estas funciones proporcionan servicios para que una aplicación de servidor funcione con datos de otra aplicación. Los datos se pueden convertir al formato nativo de la aplicación de servidor si se leen los datos de su formato original, pero se escriben en el formato nativo de la aplicación de objeto. O bien, los datos pueden permanecer en su formato original mientras la aplicación de servidor lee y escribe los datos en su formato original.

Para obtener más información, vea la siguiente sección [Structured Storage Access Modes](structured-storage-access-modes.md).
