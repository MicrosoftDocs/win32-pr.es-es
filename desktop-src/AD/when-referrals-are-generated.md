# <a name="when-referrals-are-generated"></a><span data-ttu-id="24168-104">Cuándo se generan las referencias</span><span class="sxs-lookup"><span data-stu-id="24168-104">When Referrals Are Generated</span></span>

<span data-ttu-id="24168-105">Una referencia es la forma en que un servidor de directorio comunica que no contiene los datos necesarios para completar una consulta, pero tiene una referencia a un servidor que puede contener los datos necesarios.</span><span class="sxs-lookup"><span data-stu-id="24168-105">A referral is the way that a directory server communicates that it does not contain the data required to complete a query, but has a reference to a server that may contain the required data.</span></span> <span data-ttu-id="24168-106">Tenga en cuenta que las referencias no se generan simplemente mediante solicitudes de consulta.</span><span class="sxs-lookup"><span data-stu-id="24168-106">Be aware that referrals are not just generated by query requests.</span></span>

<span data-ttu-id="24168-107">Las operaciones siguientes pueden dar lugar a una o varias referencias:</span><span class="sxs-lookup"><span data-stu-id="24168-107">The following operations can result in one or more referrals:</span></span>

-   <span data-ttu-id="24168-108">Enlazar a un servidor que no contiene el objeto especificado por el nombre distintivo solicitado, pero que tiene datos sobre un servidor o dominio que puede contener ese objeto.</span><span class="sxs-lookup"><span data-stu-id="24168-108">Binding to a server that does not contain the object specified by the requested distinguished name but has data about a server or domain that may contain that object.</span></span> <span data-ttu-id="24168-109">Para ADSI, esto puede ocurrir si la aplicación llama a [**ADsGetObject**](/windows/desktop/api/adshlp/nf-adshlp-adsgetobject) o [**ADsOpenObject**](/windows/desktop/api/adshlp/nf-adshlp-adsopenobject) para enlazar con un objeto que existe en otro dominio del bosque (referencia interna) o un contexto de nomenclatura totalmente independiente del bosque (referencia externa).</span><span class="sxs-lookup"><span data-stu-id="24168-109">For ADSI, this can occur if the application calls [**ADsGetObject**](/windows/desktop/api/adshlp/nf-adshlp-adsgetobject) or [**ADsOpenObject**](/windows/desktop/api/adshlp/nf-adshlp-adsopenobject) to bind to an object that exists in another domain in the forest (internal referral) or a naming context that is completely separate from the forest (external referral).</span></span> <span data-ttu-id="24168-110">Para la API LDAP, esto puede ocurrir al realizar operaciones de agregar, modificar, eliminar o buscar que especifican un objeto que existe en otro dominio del bosque (referencia interna) o un contexto de nomenclatura totalmente independiente del bosque (referencia externa).</span><span class="sxs-lookup"><span data-stu-id="24168-110">For the LDAP API, this can occur when performing add, modify, delete, or search operations that specify an object that exists in another domain in the forest (internal referral) or a naming context that is completely separate from the forest (external referral).</span></span>

    <span data-ttu-id="24168-111">Si la resolución de nombres no encuentra un objeto localmente y no hay objetos **crossRef** para esa parte del espacio de nombres, el controlador de dominio intentará construir una referencia externa basada en los componentes de dominio del nombre distintivo.</span><span class="sxs-lookup"><span data-stu-id="24168-111">If name resolution fails to find an object locally and there are no **crossRef** objects for that portion of the namespace, the domain controller will attempt to construct an external referral based on the domain components of the distinguished name.</span></span> <span data-ttu-id="24168-112">Por ejemplo, si una búsqueda se basa en "CN = a, CN = b, DC = c, DC = d, DC = e", el controlador de dominio construirá una referencia al servidor LDAP en la dirección DNS "c. d. e".</span><span class="sxs-lookup"><span data-stu-id="24168-112">For example, if a search was based at "CN=a,CN=b,DC=c,DC=d,DC=e", the domain controller will construct a referral to the LDAP server at DNS address "c.d.e".</span></span>

    <span data-ttu-id="24168-113">Todos los controladores de dominio de Windows 2000 (que solo admiten la nomenclatura DC = para los componentes superiores) se reconocen entre sí, y no se requiere ninguna referencia cruzada externa para que un cliente se enlace desde un bosque a otro.</span><span class="sxs-lookup"><span data-stu-id="24168-113">All Windows 2000 domain controllers (which support only DC= naming for the upper components) recognize each other, and no external cross references are required for a client to bind from one forest to another.</span></span> <span data-ttu-id="24168-114">Si otros servidores de directorio que no son de Windows 2000, como un servidor de Netscape, usan DC = naming y tienen un RR SRV adecuado registrado en DNS, también obtendrá la ventaja de las referencias automáticas.</span><span class="sxs-lookup"><span data-stu-id="24168-114">If other non-Windows 2000 directory servers, such as a Netscape server, is using DC= naming and has an appropriate SRV RR registered in DNS, it will get the advantage of the automatic referrals as well.</span></span> <span data-ttu-id="24168-115">Si no es así, se debe agregar manualmente un objeto **crossRef** externo.</span><span class="sxs-lookup"><span data-stu-id="24168-115">If not, an external **crossRef** object must be added manually.</span></span>

-   <span data-ttu-id="24168-116">Realización de una búsqueda de subárbol en un dominio que contiene dominios subordinados en el bosque o dominios externos subordinados, esquemas o contenedores de configuración.</span><span class="sxs-lookup"><span data-stu-id="24168-116">Performing a subtree search on a domain that contains subordinate domains in the forest or subordinate external domains, schema, or configuration containers.</span></span> <span data-ttu-id="24168-117">Se creará una referencia al dominio subordinado, el dominio externo, el esquema o el contenedor de configuración.</span><span class="sxs-lookup"><span data-stu-id="24168-117">A referral to the subordinate domain, external domain, schema or configuration container will be created.</span></span> <span data-ttu-id="24168-118">Si está habilitado el seguimiento de referencias, la referencia será transparente para el autor de la llamada.</span><span class="sxs-lookup"><span data-stu-id="24168-118">If referral chasing is enabled, the referral will be transparent to the caller.</span></span>
