### <a name="best-practices-for-verb-implementations"></a><span data-ttu-id="bea56-113">Procedimientos recomendados para implementaciones de verbos</span><span class="sxs-lookup"><span data-stu-id="bea56-113">Best Practices for Verb Implementations</span></span>

-   <span data-ttu-id="bea56-115">Elija siempre el método de verbo más sencillo que satisfaga sus necesidades.</span><span class="sxs-lookup"><span data-stu-id="bea56-115">Always chose the simplest verb method that meets your needs.</span></span>
-   <span data-ttu-id="bea56-116">Use un método de verbo estático, si es posible.</span><span class="sxs-lookup"><span data-stu-id="bea56-116">Use a static verb method, if possible.</span></span>
-   <span data-ttu-id="bea56-117">No realice operaciones de uso intensivo de recursos ni E/S en el subproceso de la interfaz de usuario.</span><span class="sxs-lookup"><span data-stu-id="bea56-117">Do not perform resource-intensive operations or I/O on the UI thread.</span></span> <span data-ttu-id="bea56-118">Tanto [**IShellExtInit::Initialize**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-ishellextinit-initialize) como [**IContextMenu::QueryContextMenu**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-querycontextmenu) deben ser muy conservadores en el trabajo que realizan.</span><span class="sxs-lookup"><span data-stu-id="bea56-118">Both [**IShellExtInit::Initialize**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-ishellextinit-initialize) and [**IContextMenu::QueryContextMenu**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-querycontextmenu) need to be very conservative in the work they perform.</span></span> <span data-ttu-id="bea56-119">[**IContextMenu::InvokeCommand**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-invokecommand) debe realizarse en otro proceso o debe crear un subproceso nuevo para evitar el bloqueo del autor de la llamada.</span><span class="sxs-lookup"><span data-stu-id="bea56-119">[**IContextMenu::InvokeCommand**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-invokecommand) must be performed in another process, or you must create a new thread to avoid blocking the caller.</span></span>
-   <span data-ttu-id="bea56-120">Antefiena los verbos con el nombre del proveedor de software independiente (ISV) como se muestra a continuación, `ISVName.verb` .</span><span class="sxs-lookup"><span data-stu-id="bea56-120">Prefix verbs with the independent software vendor (ISV) name as follows, `ISVName.verb`.</span></span> <span data-ttu-id="bea56-121">El uso de nombres no calificados puede provocar colisiones con varios ISV que eligieron el mismo nombre de verbo.</span><span class="sxs-lookup"><span data-stu-id="bea56-121">Using unqualified names can result in collisions with multiple ISVs that chose the same verb name.</span></span>
-   <span data-ttu-id="bea56-122">Use siempre un ProgID específico de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="bea56-122">Always use an application-specific ProgID.</span></span> <span data-ttu-id="bea56-123">Dado que algunos tipos de elementos no usan esta asignación, es necesario usar nombres únicos del proveedor.</span><span class="sxs-lookup"><span data-stu-id="bea56-123">Because some item types do not use this mapping, there is a need for vendor-unique names.</span></span>

-   <span data-ttu-id="bea56-125">No acepte el valor devuelto **S \_ OK para** verbos canónicos pasados al método [**IContextMenu::InvokeCommand.**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-invokecommand)</span><span class="sxs-lookup"><span data-stu-id="bea56-125">Do not accept the return value **S\_OK** for canonical verbs passed to the [**IContextMenu::InvokeCommand**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-invokecommand) method.</span></span> <span data-ttu-id="bea56-126">Si lo hace, se produce un error en la implementación del verbo real que se va a invocar y devuelve un código de error para verbos canónicos.</span><span class="sxs-lookup"><span data-stu-id="bea56-126">Doing so causes a failure for the real verb implementation to be invoked, and returns a failure code for canonical verbs.</span></span> <span data-ttu-id="bea56-127">Si no admite verbos canónicos, devuelve un error cuando se encuentra un valor HIWORD(lpVerb) distinto de cero.</span><span class="sxs-lookup"><span data-stu-id="bea56-127">If you do not support canonical verbs, then return failure when a nonzero HIWORD(lpVerb) value is encountered.</span></span>
-   <span data-ttu-id="bea56-128">Evite el uso de **rundll32.exe** como host del verbo.</span><span class="sxs-lookup"><span data-stu-id="bea56-128">Avoid the use of **rundll32.exe** as the host for your verb.</span></span>
-   <span data-ttu-id="bea56-129">Al implementar [**IContextMenu::QueryContextMenu,**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-querycontextmenu) asegúrese de devolver el número de verbos que se han agregado al menú a través del **valor HRESULT** mediante la macro ResultFromShort.</span><span class="sxs-lookup"><span data-stu-id="bea56-129">When implementing [**IContextMenu::QueryContextMenu**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-querycontextmenu) be sure to return the number of verbs that have been added to the menu through the **HRESULT** value using the ResultFromShort macro.</span></span>
-   <span data-ttu-id="bea56-130">Si se registra en una de las siguientes entradas de clave del Registro, tenga cuidado y asegúrese de registrar el controlador en el tipo más específico para reducir las consecuencias posiblemente no deseadas:</span><span class="sxs-lookup"><span data-stu-id="bea56-130">If you register on one of the following registry key entries, then exercise caution and be sure to register your handler on the most specific type to reduce the possibly unintended consequences:</span></span>
    -   <span data-ttu-id="bea56-131">**RAÍZ DE CLASES HKEY \_ \_\\\***</span><span class="sxs-lookup"><span data-stu-id="bea56-131">**HKEY\_CLASSES\_ROOT\\\***</span></span>
    -   <span data-ttu-id="bea56-132">**HKEY \_ CLASSES \_ ROOT \\ AllFileSystemObjects**</span><span class="sxs-lookup"><span data-stu-id="bea56-132">**HKEY\_CLASSES\_ROOT\\AllFileSystemObjects**</span></span>
    -   <span data-ttu-id="bea56-133">**Carpeta RAÍZ \_ HKEY CLASSES \_ \\**</span><span class="sxs-lookup"><span data-stu-id="bea56-133">**HKEY\_CLASSES\_ROOT\\Folder**</span></span>
    -   <span data-ttu-id="bea56-134">**HKEY \_ CLASSES \_ ROOT \\ Directory**</span><span class="sxs-lookup"><span data-stu-id="bea56-134">**HKEY\_CLASSES\_ROOT\\Directory**</span></span>
-   <span data-ttu-id="bea56-135">Establezca la **tecla MayChangeDefaultMenu** solo si prevé que es posible que tenga que cambiar el verbo predeterminado en el menú contextual.</span><span class="sxs-lookup"><span data-stu-id="bea56-135">Set the **MayChangeDefaultMenu** key only if you anticipate that you might need to change the default verb in the shortcut menu.</span></span> <span data-ttu-id="bea56-136">Si el controlador no cambia el verbo predeterminado, no debe establecer esta clave porque, al hacerlo, el sistema carga innecesariamente el archivo DLL.</span><span class="sxs-lookup"><span data-stu-id="bea56-136">If your handler does not change the default verb, then you should not set this key because doing so causes the system to load your DLL unnecessarily.</span></span>
-   <span data-ttu-id="bea56-137">Minimice el trabajo que realiza [**en IShellExtInit::Initialize**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-ishellextinit-initialize).</span><span class="sxs-lookup"><span data-stu-id="bea56-137">Minimize the work you perform in [**IShellExtInit::Initialize**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-ishellextinit-initialize).</span></span> <span data-ttu-id="bea56-138">Para los controladores de menús contextuales, capture el objeto de datos pasado a **IShellExtInit::Initialize** y procese en [**IContextMenu::QueryContextMenu**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-querycontextmenu)o [**IContextMenu::InvokeCommand**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-invokecommand).</span><span class="sxs-lookup"><span data-stu-id="bea56-138">For shortcut menu handlers, capture the data object passed to **IShellExtInit::Initialize**, and then process it in [**IContextMenu::QueryContextMenu**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-querycontextmenu), or [**IContextMenu::InvokeCommand**](/windows/desktop/api/shobjidl_core/nf-shobjidl_core-icontextmenu-invokecommand).</span></span>

## <a name="best-practices-for-multiple-selection-verbs"></a><span data-ttu-id="bea56-139">Procedimientos recomendados para verbos de selección múltiple</span><span class="sxs-lookup"><span data-stu-id="bea56-139">Best Practices for Multiple Selection Verbs</span></span>

<span data-ttu-id="bea56-140">Dado que el número de elementos de un escenario de verbo de selección múltiple puede ser grande, es importante tener en cuenta las implicaciones de rendimiento de las implementaciones de verbos.</span><span class="sxs-lookup"><span data-stu-id="bea56-140">Because the number of items in a multi-selection verb scenario can be large, it is important that you consider the performance implications of your verb implementations.</span></span> <span data-ttu-id="bea56-141">Por ejemplo, cuando un usuario busca " " en un ámbito que incluye un gran número de elementos y, a continuación, hace clic en Seleccionar todo y hace clic con el botón derecho, el verbo se presenta con una selección que puede tener miles de elementos \* en él. </span><span class="sxs-lookup"><span data-stu-id="bea56-141">For example, when a user searches for "\*" over a scope that includes large numbers of items, and then clicks **Select All** and right-clicks, the verb is presented with a selection that can have thousands of items in it.</span></span> <span data-ttu-id="bea56-142">Como resultado, los verbos solo deben tener en cuenta el primer elemento de la selección y el recuento total de elementos.</span><span class="sxs-lookup"><span data-stu-id="bea56-142">As a result, verbs should only consider the first item in the selection, and the overall item count.</span></span> <span data-ttu-id="bea56-143">El primer elemento se define como el elemento de la parte superior de la vista o el elemento en el que el usuario hizo clic con el botón derecho por primera vez.</span><span class="sxs-lookup"><span data-stu-id="bea56-143">The first item is defined as either the item at the top of the view, or the item that the user first right-clicked.</span></span>

<span data-ttu-id="bea56-144">En Windows 7 y versiones posteriores, el número de elementos pasados a un verbo se limita a 16 cuando se consulta un menú contextual.</span><span class="sxs-lookup"><span data-stu-id="bea56-144">In Windows 7 and later, the number of items passed to a verb is limited to 16 when a shortcut menu is queried.</span></span> <span data-ttu-id="bea56-145">A continuación, el verbo se vuelve a crear y se vuelve a inicializar con la selección completa cuando se invoca ese verbo.</span><span class="sxs-lookup"><span data-stu-id="bea56-145">The verb is then re-created and re-initialized with the full selection when that verb is invoked.</span></span>

<span data-ttu-id="bea56-146">En algunos casos, es adecuado tener en cuenta un pequeño número de elementos fijos.</span><span class="sxs-lookup"><span data-stu-id="bea56-146">It is appropriate in some cases to consider a small number of fixed items.</span></span> <span data-ttu-id="bea56-147">Por ejemplo, es adecuado que un verbo "diff" tenga en cuenta solo los dos primeros elementos.</span><span class="sxs-lookup"><span data-stu-id="bea56-147">For example, it is appropriate for a "diff" verb to consider only the first two items.</span></span> <span data-ttu-id="bea56-148">Por lo general, no es necesario probar todos los elementos de la selección para ver si es un tipo determinado o consultar sus propiedades en cada elemento de la selección.</span><span class="sxs-lookup"><span data-stu-id="bea56-148">Generally, you need not test every item in the selection to see if it is a certain type, or query every item in the selection for its properties.</span></span> <span data-ttu-id="bea56-149">Mire en su lugar el primer elemento y decida si es adecuado agregar el verbo.</span><span class="sxs-lookup"><span data-stu-id="bea56-149">Look rather at the first item and decide if it is appropriate to add your verb.</span></span>

### <a name="heterogenous-selections"></a><span data-ttu-id="bea56-150">Selecciones heterogéneos</span><span class="sxs-lookup"><span data-stu-id="bea56-150">Heterogenous Selections</span></span>

<span data-ttu-id="bea56-151">Los verbos optimistas se agregan automáticamente en el caso de selección múltiple, suponiendo que el verbo pueda controlar los elementos no especificados.</span><span class="sxs-lookup"><span data-stu-id="bea56-151">Optimistic verbs are automatically added in the multi-selection case, assuming that uninspected items can be handled by the verb.</span></span> <span data-ttu-id="bea56-152">Por el contrario, los verbos pesimistas no se agregan cuando la selección contiene elementos no especificados y solo se agregan en casos en los que el número de elementos es pequeño.</span><span class="sxs-lookup"><span data-stu-id="bea56-152">In contrast, pessimistic verbs are not added when the selection contains uninspected items, and are only added in cases where the number of items is small.</span></span>

<span data-ttu-id="bea56-153">Los verbos de estilo de reproductor deben ser optimistas y omitir silenciosamente los elementos que no se controlan.</span><span class="sxs-lookup"><span data-stu-id="bea56-153">Player style verbs should be optimistic, and silently skip the items that are not handled.</span></span> <span data-ttu-id="bea56-154">Si un error al actuar en los elementos puede causar confusión o pérdida de datos, el verbo debe advertir a los usuarios sobre los elementos que no se pueden procesar.</span><span class="sxs-lookup"><span data-stu-id="bea56-154">If a failure to act on items can cause data loss or confusion, then the verb should warn users about items that cannot be processed.</span></span> <span data-ttu-id="bea56-155">Por ejemplo, un verbo de "copia de seguridad" debe indicar que no se pudo realizar una copia de seguridad de algunos elementos.</span><span class="sxs-lookup"><span data-stu-id="bea56-155">For example a "backup" verb should indicate that some items could not be backed up.</span></span>
