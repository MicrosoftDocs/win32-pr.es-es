# <a name="the-listening-tip"></a>La sugerencia de escucha

\[Microsoft Agent está en desuso a partir de Windows 7 y puede que no esté disponible en versiones posteriores de Windows.\]

La sugerencia de escucha es otro servicio de entrada de voz proporcionado por el agente de Microsoft. Cuando se instala la entrada de voz, el agente incluye una ventana de información sobre herramientas especial que aparece cuando el usuario presiona la tecla de método de escucha o llama al método Listen. La sugerencia de escucha solo aparece cuando los servicios de voz están disponibles. Si ningún cliente ha creado un comando de voz o ha cargado correctamente un motor de voz, la sugerencia de escucha no aparece. Además, tanto la entrada de voz como la opción Mostrar sugerencias de escucha en las opciones de caracteres avanzadas deben estar habilitadas para que aparezca la sugerencia.

La sugerencia de escucha agota el tiempo de espera automáticamente tras su presentación. Si el tiempo de espera del texto "escuchado" se completa mientras el usuario todavía mantiene presionada la tecla de tecla, la sugerencia vuelve al texto "escuchando" a menos que el servidor reciba otro utterance coincidente. En este caso, la sugerencia muestra el nuevo texto "escuchado" y comienza el tiempo de espera para ese texto de sugerencia. Si el usuario suelta la tecla de tecla y el servidor muestra el texto "escuchado", el tiempo de espera continúa y la ventana de la sugerencia de escucha se oculta cuando transcurre el intervalo de tiempo de espera.

Si el servidor todavía no ha intentado cargar un motor de reconocimiento de voz, la sugerencia de escucha no se mostrará. Del mismo modo, si el usuario ha deshabilitado la visualización de la sugerencia de escucha o de la entrada de voz deshabilitada en opciones de caracteres avanzadas, no se mostrará la sugerencia de escucha.

La sugerencia de escucha no aparece cuando el puntero se encuentra sobre el icono de la barra de tareas del carácter. En su lugar, aparece la ventana sugerencia de notificación estándar y muestra el nombre del carácter.

Las aplicaciones cliente no pueden escribir directamente en la sugerencia de escucha, pero puede especificar texto alternativo que el servidor muestre en el reconocimiento de un comando de voz coincidente. Para ello, establezca la propiedad [**Confidence**](confidence-property.md) y la nueva propiedad [**ConfidenceText**](confidencetext-property.md)para el comando. Si la entrada hablada coincide con el comando, pero la mejor coincidencia no supera la configuración de confianza, el servidor usa el texto establecido en la propiedad **ConfidenceText** de la ventana de sugerencia. Si el cliente no proporciona este valor, el servidor muestra el texto (gramática) con el que coincide.

El texto de la sugerencia de escucha aparece en el idioma según la configuración de ID. de idioma del cliente de entrada-activo, independientemente de si hay disponible un motor de reconocimiento de voz compatible con el idioma.
