# <a name="session-monikers"></a>Monikers de sesión

COM distribuido (DCOM) permite la activación de objetos por sesión mediante un moniker de sesión proporcionado por el sistema. Para obtener más información sobre cómo crear y usar monikers de sesión, incluido un ejemplo de código que muestra cómo activar un proceso de servidor local, vea los temas siguientes.

## <a name="in-this-section"></a>En esta sección

<dl> <dt>

[Activación de sesión a sesión con un Moniker de sesión](session-to-session-activation-with-a-session-moniker.md)
</dt> <dd>

La activación de sesión a sesión (también denominada activación entre sesiones) permite que un proceso de cliente inicie (active) un proceso de servidor local en una sesión especificada.

</dd> <dt>

[Uso de un Moniker de sesión](using-a-session-moniker.md)
</dt> <dd>

La activación de sesión a sesión permite que un proceso de cliente active un proceso de servidor local en una sesión especificada.

</dd> </dl>
