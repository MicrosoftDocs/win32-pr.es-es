# <a name="object-state-constants"></a>Constantes de estado de objeto

En este tema se describen los valores constantes que se usan para describir el estado de los objetos en una interfaz de usuario de la aplicación. Las constantes de estado se definen en oleacc. h.

Un objeto está asociado a uno o varios de estos valores de estado en cualquier momento. No se usan las constantes de estado de objeto siguientes: alerta de sistema de estado \_ \_ \_ alta, \_ \_ media de alerta de sistema \_ de estado, alerta de sistema de estado \_ \_ \_ baja y sistema de estado \_ \_ flotante.

Los clientes recuperan el estado de un objeto llamando a [**IAccessible:: get \_ accState**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-get_accstate), que devuelve un entero que es una combinación de las siguientes marcas de bits. Los clientes llaman a [**GetStateText**](/windows/desktop/api/Oleacc/nf-oleacc-getstatetexta) con el valor de estado para recuperar una cadena localizada que describe el estado del objeto.

Cuando cambia el estado de un objeto, los servidores deben llamar a [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) con la constante de evento [**STATECHANGE del \_ objeto \_ de evento**](event-constants.md) . Sin embargo, los objetos con las constantes de estado de objeto del sistema de estado \_ \_ invisible, el sistema de estado y el equipo \_ \_ \_ \_ seleccionado tienen sus propias constantes de evento. Para estos objetos, no use el **\_ objeto \_ STATECHANGE de evento**. En su lugar, use la constante de evento individual.
