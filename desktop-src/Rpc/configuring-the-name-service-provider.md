# <a name="configuring-the-name-service-provider"></a>Configurar el proveedor de servicios de nombres

Si la aplicación distribuida registra su interfaz con un servicio de nombres, tanto el cliente como el servidor deben usar el mismo servicio de nombres. Microsoft RPC interopera con el localizador de Microsoft y cualquier proveedor de servicios de nombres (NSP) que se adhiere a la interfaz de servicio de nombres RPC (NSI) de Microsoft, por ejemplo, el DCE Servicio de directorio de celdas tener acceso a través del demonio de servicio de nombres de Digital Equipment Corporation (NSID). El localizador de Microsoft, que está diseñado para su uso en entornos Windows, es el proveedor de servicios de nombres predeterminado.

En esta sección se presenta una descripción de la instalación y configuración de proveedores de servicios de nombres:

-   [Configuración del servicio de nombres](configuring-the-name-service-for-windows-xp-windows-2000-or-windows-nt.md)
-   [Configurar el servicio de nombres para Windows 95](configuring-the-name-service-for-windows-95.md)
-   [Configurar el servicio de nombres para Windows 3. x o MS-DOS](configuring-the-name-service-for-windows-3-x-or-ms-dos.md)
-   [Iniciar y detener el localizador de Microsoft](starting-and-stopping-microsoft-locator.md)
