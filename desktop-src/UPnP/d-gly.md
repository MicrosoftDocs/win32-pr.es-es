# <a name="d-upnp-apis"></a>D (API de UPnP)

Contiene términos relacionados con UPnP que comienzan por la letra D.

[A](a-gly.md) B [C](c-gly.md) D [E](e-gly.md) F G [H I](h-gly.md) J K L M [N O](n-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.d_1_gly"></span><span id="UPNP.D_1_GLY"></span>**Dispositivo**
</dt> <dd>

Un dispositivo es un contenedor de otros servicios o dispositivos. Un dispositivo puede contener otros dispositivos lógicos.

</dd> <dt>

<span id="upnp.d_2_gly"></span><span id="UPNP.D_2_GLY"></span>**objeto de control de dispositivo**
</dt> <dd>

Un objeto de control de dispositivo es el punto central de administración de los objetos de servicio de un dispositivo. Cuando se registra un dispositivo, su objeto de control de dispositivo se pasa al host del dispositivo.

</dd> <dt>

<span id="upnp.d_3_gly"></span><span id="UPNP.D_3_GLY"></span>**descripción del dispositivo**
</dt> <dd>

Una descripción del dispositivo es una lista de las propiedades genéricas de todos los dispositivos de un dispositivo físico, incluidos los servicios, la jerarquía del dispositivo y sus propiedades.

</dd> <dt>

<span id="upnp.d_4_gly"></span><span id="UPNP.D_4_GLY"></span>**host de dispositivo**
</dt> <dd>

El host de dispositivo controla las partes de detección, descripción, control y eventos de la funcionalidad de un dispositivo. Los dispositivos contienen el código para controlar la funcionalidad específica del dispositivo, como rebobinar un VCR.

</dd> <dt>

<span id="upnp.d_5_gly"></span><span id="UPNP.D_5_GLY"></span>**proveedor de dispositivos**
</dt> <dd>

Un proveedor de dispositivos es una entidad registrada con el host del dispositivo. Este proveedor de dispositivos no está publicado en la red. Sin embargo, el proveedor de dispositivos registra otros dispositivos y los publica en la red.

</dd> <dt>

<span id="upnp.d_6_gly"></span><span id="UPNP.D_6_GLY"></span>**tipo de dispositivo**
</dt> <dd>

Un tipo de dispositivo estándar se indica mediante urn:schemas-upnp-org:device: seguido de un nombre único asignado por el foro de UPnP.

Hay una relación uno a uno entre las plantillas de dispositivo basadas en UPnP y los tipos de dispositivo. Los proveedores de UPnP pueden especificar tipos de dispositivos adicionales; estos se indican mediante urn:domain-name:device: seguido de un nombre único asignado por el proveedor, donde domain-name es un nombre de dominio registrado en el proveedor.

</dd> </dl>
