## <a name="remarks"></a>Observaciones

Es posible que solo le interese si el método se realizó correctamente o no. La mejor manera de comprobar si un valor **HRESULT** indica que la operación se ha realizado correctamente o no es pasar el valor a una de las siguientes macros, definidas en WinError. h:

-   La macro [**Succeeded**](/windows/win32/api/winerror/nf-winerror-succeeded) devuelve **true** para un código de operación correcta y **false** para un código de error.
-   La macro [**failed**](/windows/win32/api/winerror/nf-winerror-failed) devuelve **true** para un código de error y **false** para un código de operación correcta.

El valor **HRESULT** para cada valor de **\_ error de DXGI** se determina a partir de esta macro que se define en DXGItype. h:


```
#define _FACDXGI    0x87a
#define MAKE_DXGI_HRESULT(code) MAKE_HRESULT(1, _FACDXGI, code)
```



Por ejemplo, **la \_ \_ llamada no válida \_ a error de DXGI** se define como **0x887A0001**:


```
#define DXGI_ERROR_INVALID_CALL                 MAKE_DXGI_HRESULT(1)
```
