# <a name="web-services-on-devices"></a>Servicios web en dispositivos

## <a name="purpose"></a>Propósito

La API de servicios Web de Microsoft en dispositivos (WSDAPI) admite la implementación de dispositivos y servicios controlados por el cliente, y los hosts de dispositivo que se ajustan al [Perfil de dispositivos para servicios web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS). WSDAPI usa [WS-Discovery para la](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) detección de dispositivos.

WSDAPI se puede usar para el desarrollo de implementaciones de cliente y de servicio.

## <a name="where-applicable"></a>Donde sea aplicable

Servicios web en dispositivos permite a un cliente detectar y tener acceso a un dispositivo remoto y sus servicios asociados a través de una red. Admite la detección de dispositivos, la descripción, el control y los eventos. Los desarrolladores pueden crear proxies de cliente de WSDAPI y códigos auxiliares correspondientes para los hosts de dispositivos.

## <a name="developer-audience"></a>Audiencia de desarrolladores

La documentación de servicios web en dispositivos está pensada para programadores de C/C++ y proveedores de dispositivos que crean productos conformes a DPWS.

## <a name="run-time-requirements"></a>Requisitos de tiempo de ejecución

Las aplicaciones cliente que usan WSDAPI se admiten a partir de Windows Vista y Windows Server 2008.
