# <a name="d1148-invalid-operation-on-closed-print-control"></a>D1148: operación no válida en el control de impresión cerrado

Impresión en Direct2D: se llama a [**addPage**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-addpage) o [**Close**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-close) una vez cerrado [**ID2D1PrintControl**](/windows/win32/api/d2d1_1/nn-d2d1_1-id2d1printcontrol)






 

## <a name="possible-causes"></a>Causas posibles

Este mensaje puede producirse si una aplicación llama a ID2D1PrintControl:: AddPage en un control de impresión que se ha cerrado. Direct2D considera que los objetos de control de impresión no se pueden cambiar después de cerrarse, de una manera similar a las de las geometrías de ruta de acceso y las listas de comandos.

## <a name="possible-fixes"></a>Posibles correcciones

Asegúrese de que la aplicación no intenta agregar páginas a un control de impresión cerrado. Modifique la lógica de la aplicación para que todas las páginas necesarias se envíen a un control de impresión antes de cerrarse.
