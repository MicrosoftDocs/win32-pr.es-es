# <a name="dispatching-soap-messages"></a><span data-ttu-id="18d0a-104">Enviar mensajes SOAP</span><span class="sxs-lookup"><span data-stu-id="18d0a-104">Dispatching SOAP Messages</span></span>

<span data-ttu-id="18d0a-105">Existen muchas formas de controlar la distribución de mensajes SOAP recibidos al servicio adecuado.</span><span class="sxs-lookup"><span data-stu-id="18d0a-105">There are many ways to handle dispatching received SOAP messages to the appropriate service.</span></span> <span data-ttu-id="18d0a-106">Los dos mecanismos más sencillos son envío de nivel de transporte y envío de dirección y acción.</span><span class="sxs-lookup"><span data-stu-id="18d0a-106">The two simplest mechanisms are transport level dispatch, and address and action dispatch.</span></span>

## <a name="transport-level-dispatch"></a><span data-ttu-id="18d0a-107">Envío de nivel de transporte</span><span class="sxs-lookup"><span data-stu-id="18d0a-107">Transport level dispatch</span></span>

<span data-ttu-id="18d0a-108">Con el envío de nivel de transporte, el servidor HTTP subyacente (como la [API http](/windows/desktop/Http/http-api-start-page)) se usa para administrar el enrutamiento de las solicitudes al dispositivo y sus servicios.</span><span class="sxs-lookup"><span data-stu-id="18d0a-108">With transport level dispatch, the underlying HTTP server (such as the [HTTP API](/windows/desktop/Http/http-api-start-page)) is used to manage routing of requests to the device and its services.</span></span> <span data-ttu-id="18d0a-109">El servidor proporciona una dirección URL diferente para cada servicio y para el dispositivo, y se registran receptores diferentes para cada dirección URL.</span><span class="sxs-lookup"><span data-stu-id="18d0a-109">The server provides a different URL for each service and for the device and different sinks are registered for each URL.</span></span> <span data-ttu-id="18d0a-110">Esto permite diseñar el código de forma que cada servicio esté aislado del otro, ya sea en ejecución como componentes independientes en el mismo proceso o en ejecución como procesos independientes.</span><span class="sxs-lookup"><span data-stu-id="18d0a-110">This allows code to be designed such that each service is isolated from the other, either running as separate components within the same process or running as separate processes.</span></span>

<span data-ttu-id="18d0a-111">El envío de nivel de transporte tiene algunas ventajas.</span><span class="sxs-lookup"><span data-stu-id="18d0a-111">Transport level dispatch has a few advantages.</span></span> <span data-ttu-id="18d0a-112">Los mensajes se pueden enviar al componente adecuado sin analizar primero la envoltura SOAP o el cuerpo del mensaje.</span><span class="sxs-lookup"><span data-stu-id="18d0a-112">Messages can be dispatched to the appropriate component without first parsing the SOAP envelope or message body.</span></span> <span data-ttu-id="18d0a-113">Además, se puede reutilizar el mecanismo existente para enrutar los mensajes proporcionados por la mayoría de las implementaciones de servidor HTTP, lo que significa que el código de distribución personalizado no es necesario.</span><span class="sxs-lookup"><span data-stu-id="18d0a-113">Also, the existing mechanism for routing messages provided by most HTTP server implementations can be reused, which means custom dispatching code is unnecessary.</span></span> <span data-ttu-id="18d0a-114">También aísla el código de procesamiento SOAP entre los servicios, lo que proporciona un nivel de seguridad, ya que los servicios seguros evitan que los mensajes viajen a través del código común.</span><span class="sxs-lookup"><span data-stu-id="18d0a-114">It also isolates the SOAP processing code between services, which provides a level of security, as secure services avoid having messages travel through common code.</span></span>

## <a name="address-and-action-dispatch"></a><span data-ttu-id="18d0a-115">Envío de dirección y acción</span><span class="sxs-lookup"><span data-stu-id="18d0a-115">Address and action dispatch</span></span>

<span data-ttu-id="18d0a-116">La distribución de la dirección y la acción se basa en los encabezados SOAP para determinar el servicio adecuado al que se envía el mensaje.</span><span class="sxs-lookup"><span data-stu-id="18d0a-116">Address and action dispatch relies on the SOAP headers to determine the appropriate service to which the message is dispatched.</span></span> <span data-ttu-id="18d0a-117">Este modelo también puede usar información adicional, como parámetros de referencia, para facilitar la distribución.</span><span class="sxs-lookup"><span data-stu-id="18d0a-117">This model may also use additional information, such as reference parameters, to further help dispatching.</span></span>

<span data-ttu-id="18d0a-118">Este modelo fomenta la reutilización de código en una pila de mensajería por capas, ya que todos los servicios comparten todo el código hasta el procesador SOAP.</span><span class="sxs-lookup"><span data-stu-id="18d0a-118">This model encourages code reuse throughout a layered messaging stack, as all code up to the SOAP processor is shared by all services.</span></span> <span data-ttu-id="18d0a-119">Además, no se requieren direcciones de transporte diferentes para los servicios, lo que significa que las direcciones UUID se pueden usar para los puntos de conexión de servicio.</span><span class="sxs-lookup"><span data-stu-id="18d0a-119">Also, distinct transport addresses for services are not required, which means that UUID addresses can be used for service endpoints.</span></span> <span data-ttu-id="18d0a-120">La distribución de la dirección y la acción también traduce más directamente a un modelo de programación.</span><span class="sxs-lookup"><span data-stu-id="18d0a-120">Address and action dispatch also translates more directly to a programming model.</span></span> <span data-ttu-id="18d0a-121">Los desarrolladores pueden conectar servicios y dispositivos en un único componente que administra el enrutamiento, en lugar de tener que asociarse a una capa HTTP o crear componentes independientes para cada servicio.</span><span class="sxs-lookup"><span data-stu-id="18d0a-121">Developers can plug services and devices into a single component which manages routing, rather than having to tie into a HTTP layer or creating separate components for each service.</span></span>
