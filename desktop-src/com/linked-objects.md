# <a name="linked-objects"></a>Objetos vinculados

Cuando se inserta un vínculo a un objeto en un documento compuesto, los datos de origen o el *origen del vínculo* continúan residiendo dondequiera que se crearon inicialmente, normalmente en otro documento. El documento compuesto solo contiene una referencia, o un *vínculo*, a los datos reales almacenados en el origen del vínculo, junto con información sobre cómo presentar esos datos al usuario.

La activación de un vínculo ejecuta la aplicación de servidor del origen del vínculo, que el usuario necesita para editar o manipular de otro modo los datos del vínculo. La vinculación mantiene el tamaño de un documento compuesto pequeño. También resulta útil cuando un usuario mantiene el origen de datos y debe compartirse entre varios usuarios. Si la persona que mantiene el origen del vínculo cambia los datos, el cambio se actualiza automáticamente en todos los documentos que contienen un vínculo a esos datos. Además de crear vínculos simples, los usuarios pueden anidar vínculos y combinar objetos vinculados e incrustados para crear documentos complejos.
