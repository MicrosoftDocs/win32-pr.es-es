# <a name="scripting-com-objects-in-custom-applications"></a>Scripting de objetos COM en aplicaciones personalizadas

Microsoft proporciona varios entornos para crear scripts de objetos COM: Active Server Pages, Windows host de script, y así sucesivamente. Los proveedores de software independientes (ISV) también pueden licenciar los motores de scripting de Microsoft para su uso en sus aplicaciones. Por ejemplo, un ISV que crea un procesador de palabras puede querer agregar un lenguaje de macros a la aplicación para que los usuarios puedan automatizar tareas sencillas. Al otorgar licencias a un motor de scripting, el ISV puede usar un lenguaje como VBScript o JScript como lenguaje de macros de la aplicación.

Además de las licencias de VBScript y JScript de scripting, los ISV pueden escribir sus propios ActiveX de scripting. Estos motores de scripting se pueden conectar a cualquier entorno de host que admita el estándar ActiveX motor de scripting. Por ejemplo, un ISV podría escribir un motor de scripting perlScript e instalarlo en un servidor ASP, lo que permite que el servidor ASP procese scripts perlScript.
