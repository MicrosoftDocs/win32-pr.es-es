<span data-ttu-id="dfac0-107">Un PrintTicket específico del dispositivo se dirige a un dispositivo determinado y, por lo tanto, no suele ser portable en varios dispositivos.</span><span class="sxs-lookup"><span data-stu-id="dfac0-107">A device-specific PrintTicket targets a particular device and is therefore not generally portable across multiple devices.</span></span>

<span data-ttu-id="dfac0-108">Los pasos que se muestran en la lista siguiente deben usarse cuando se crea un PrintTicket específico del dispositivo.</span><span class="sxs-lookup"><span data-stu-id="dfac0-108">The steps shown in the following list should be used when you create a device-specific PrintTicket.</span></span>

1.  <span data-ttu-id="dfac0-109">Obtenga un documento de PrintCapabilities que contenga solo las instancias de características relevantes para el dispositivo.</span><span class="sxs-lookup"><span data-stu-id="dfac0-109">Obtain a PrintCapabilities document containing only the relevant Feature instances for the device.</span></span> <span data-ttu-id="dfac0-110">Solicitar un documento de PrintCapabilities predeterminado.</span><span class="sxs-lookup"><span data-stu-id="dfac0-110">Request a default PrintCapabilities document.</span></span> <span data-ttu-id="dfac0-111">Esto no requiere que especifique un PrintTicket de entrada porque los atributos del dispositivo (instancias de características y opciones y parámetros) que se usan para construir el PrintTicket son independientes de la configuración.</span><span class="sxs-lookup"><span data-stu-id="dfac0-111">This does not require you to specify an input PrintTicket because the device attributes (Feature and Option instances and parameters) that are used to construct the PrintTicket are independent of the configuration.</span></span>

2.  <span data-ttu-id="dfac0-112">Cree un documento XML vacío que contenga la raíz de PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-112">Create an empty XML document containing the PrintTicket root.</span></span> <span data-ttu-id="dfac0-113">Asigne un prefijo de espacio de nombres al espacio de nombres del esquema de impresión y asigne prefijos para el resto de espacios de nombres que usará el PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-113">Assign a namespace prefix to the Print Schema namespace, and assign prefixes for all other namespaces that will be used by the PrintTicket.</span></span> <span data-ttu-id="dfac0-114">Especifique la versión del esquema PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-114">Specify the PrintTicket Schema version.</span></span>

3.  <span data-ttu-id="dfac0-115">Puede definir la configuración en el PrintTicket para cada característica y instancia de ParameterDef que se notifique en el documento de PrintCapabilities, o solo las que le interesen.</span><span class="sxs-lookup"><span data-stu-id="dfac0-115">You may define settings in the PrintTicket for every Feature and ParameterDef instance reported in the PrintCapabilities document, or only those of interest to you.</span></span> <span data-ttu-id="dfac0-116">Si se presenta un PrintTicket parcial a la interfaz PrintTicket, se proporcionan automáticamente los valores predeterminados para las instancias de características y ParameterDef omitidas durante la validación de PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-116">If a partial PrintTicket is presented to the PrintTicket interface, defaults are automatically provided for omitted Feature and ParameterDef instances during PrintTicket validation.</span></span>

    <span data-ttu-id="dfac0-120">Para cada característica que transfiera, determine qué instancias de opción se aplican a su PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-120">For each Feature that you transfer, determine which Option instances are applicable to your PrintTicket.</span></span> <span data-ttu-id="dfac0-121">Transfiera toda la opción tal y como se define en el documento de PrintCapabilities y, a continuación, quite todas las instancias de propiedad que estén presentes.</span><span class="sxs-lookup"><span data-stu-id="dfac0-121">Transfer the entire Option as defined in the PrintCapabilities document, and then remove any Property instances that are present.</span></span> <span data-ttu-id="dfac0-122">Las instancias de propiedad se utilizan en los documentos de PrintCapabilities, pero no sirven para ningún propósito en el PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-122">The Property instances are used within PrintCapabilities documents, but serve no purpose in the PrintTicket.</span></span> <span data-ttu-id="dfac0-123">Mantenga todas las instancias de ScoredProperty, asegurándose de que conserva su ubicación; son una parte intrínseca de la definición de la opción.</span><span class="sxs-lookup"><span data-stu-id="dfac0-123">Keep all of the ScoredProperty instances, making sure that you preserve their location; they are an intrinsic part of the Option definition.</span></span>

5.  <span data-ttu-id="dfac0-124">También puede Agregar instancias de propiedad a cualquier instancia de ScoredProperty.</span><span class="sxs-lookup"><span data-stu-id="dfac0-124">You may also add Property instances to any ScoredProperty instance.</span></span> <span data-ttu-id="dfac0-125">Las instancias de propiedad solo se usan si el proveedor de PrintTicket admite explícitamente su uso.</span><span class="sxs-lookup"><span data-stu-id="dfac0-125">Property instances are used only if the PrintTicket provider explicitly supports their use.</span></span> <span data-ttu-id="dfac0-126">Cada proveedor debe documentar el propósito y el uso de todas las instancias de propiedad admitidas.</span><span class="sxs-lookup"><span data-stu-id="dfac0-126">Each provider should document the purpose and use of all supported Property instances.</span></span> <span data-ttu-id="dfac0-127">Tenga en cuenta que durante la validación se quitan todas las instancias de propiedad incluidas en una instancia de opción, a menos que el espacio de nombres de la propiedad sea reconocido por el proveedor de PrintCapabilities o PrintTicket de destino, y se encuentre una opción candidata en el documento de PrintCapabilities cuyas instancias de ScoredProperty coincidan exactamente con las definidas en la opción de referencia.</span><span class="sxs-lookup"><span data-stu-id="dfac0-127">Note that during validation, any Property instances contained within an Option instance are stripped off, unless the namespace of the Property is recognized by the target PrintCapabilities or PrintTicket provider, and a candidate Option is found in the PrintCapabilities document whose ScoredProperty instances perfectly match those defined in the reference Option.</span></span>

6.  <span data-ttu-id="dfac0-128">Si las palabras clave de esquema de impresión establecen la propiedad SelectionType en PickMany para una característica determinada, puede seleccionar más de una opción para esa característica, siempre que la opción designada como IdentityOption no sea una de las seleccionadas.</span><span class="sxs-lookup"><span data-stu-id="dfac0-128">If the Print Schema Keywords set the SelectionType Property to PickMany for a particular Feature, you may select more than one Option for that Feature, provided that the Option designated as IdentityOption is not one of those selected.</span></span> <span data-ttu-id="dfac0-129">IdentityOption en el esquema de impresión tiene el mismo efecto que la opción None en archivos PPD PostScript y archivos GPD de unidrv; sirve como una operación no operativa.</span><span class="sxs-lookup"><span data-stu-id="dfac0-129">IdentityOption in the Print Schema has the same effect as the None option in Postscript PPD files and Unidrv GPD files; it serves as a no-op.</span></span>

8.  <span data-ttu-id="dfac0-134">Examine el contenido de cada instancia de opción que aparece en el PrintTicket para las apariciones de instancias de ParameterRef.</span><span class="sxs-lookup"><span data-stu-id="dfac0-134">Examine the contents of each Option instance that appears in the PrintTicket for occurrences of ParameterRef instances.</span></span> <span data-ttu-id="dfac0-135">Si una instancia de ParameterInit correspondiente todavía no aparece en el PrintTicket, siga el paso anterior para crear una instancia de ParameterInit en el PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-135">If a corresponding ParameterInit instance does not already appear in the PrintTicket, follow the previous step to create a ParameterInit instance in the PrintTicket.</span></span> <span data-ttu-id="dfac0-136">El propósito de esta instancia de ParameterInit es inicializar el parámetro al que hace referencia la instancia de ParameterRef.</span><span class="sxs-lookup"><span data-stu-id="dfac0-136">The purpose of this ParameterInit instance is to initialize the parameter referenced by the ParameterRef instance.</span></span>

11. <span data-ttu-id="dfac0-142">Examine las instancias de propiedad opcionales en el esquema PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-142">Examine the optional Property instances in the PrintTicket Schema.</span></span> <span data-ttu-id="dfac0-143">Si hay alguna instancia de esta propiedad que debe estar en el PrintTicket, créela en el PrintTicket.</span><span class="sxs-lookup"><span data-stu-id="dfac0-143">If there are any such Property instances that should be in your PrintTicket, create them in your PrintTicket.</span></span>
