# <a name="s-rpc"></a>S (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) [F](f-glos.md) G H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [o](o-glos.md) [p](p-glos.md) [Q](q.md) [R](r-glos.md) s [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**escalabilidad**
</dt> <dd>

Capacidad de aprovechar al máximo la potencia de procesamiento adicional en un sistema multiprocesador (2, 4, 8, 32 o más procesadores). La escalabilidad es también la capacidad de atender un gran número de clientes.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Protocolo de paquetes secuenciados (SPP)**
</dt> <dd>

Protocolo de comunicación de Banyan Vines [*orientado a la conexión*](c-glos.md) para enrutar paquetes de información a través de redes de área local.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**Intercambio de paquetes secuenciados (SPX)**
</dt> <dd>

Protocolo de comunicación [*orientado a la conexión*](c-glos.md) Novell NetWare para enrutar paquetes de información en redes de área local y redes de área extensa.

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**serialización**
</dt> <dd>

Proceso de serialización de datos en (codificación) y de cálculo de referencias de los búferes (descodificar) que se controlan. Esto contrasta con el uso de RPC tradicional, en el que el código auxiliar y el tiempo de ejecución de RPC controlan los búferes de serialización. También se denomina selección. Vea también [*procedimientos de serialización*](p-glos.md), [*serialización de tipos*](t-glos.md).

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**código auxiliar de servidor**
</dt> <dd>

Código fuente del lenguaje C generado por MIDL que contiene todas las funciones necesarias para que la aplicación de servidor controle las solicitudes remotas mediante llamadas a procedimientos locales. Vea también [*código auxiliar de cliente*](c-glos.md).

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**sesión**
</dt> <dd>

Relación establecida entre una aplicación cliente y una aplicación de servidor. Vea también [*enlazar*](b-glos.md), [*identificador de enlace*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**función de devolución de llamada estática**
</dt> <dd>

Procedimiento remoto que forma parte del lado cliente de una aplicación distribuida, al que un servidor puede llamar para obtener información del cliente. El \[ atributo [callback](/windows/desktop/Midl/callback) \] designa una función de devolución de llamada estática.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**seguimiento de identidad estática**
</dt> <dd>

El seguimiento de identidades estáticas especifica que el tiempo de ejecución de RPC usa las credenciales de seguridad en el identificador de enlace del cliente para todas las llamadas RPC. Vea también [*seguimiento de identidad dinámica*](d-glos.md).

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**enlace de cadenas**
</dt> <dd>

Cadena de caracteres formada por el [*UUID*](u-glos.md)del objeto, la [*secuencia de protocolo*](p-glos.md), la [*dirección de red*](n-glos.md), el [*extremo*](e-glos.md)y las opciones de extremo, que se pueden utilizar para crear un [*identificador de enlace*](b-glos.md) para el servidor especificado.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**establecimiento fuerte de tipos**
</dt> <dd>

Aplicación del compilador de control estricto sobre los tipos de datos. En MIDL y RPC, el establecimiento de tipos seguros se usa para garantizar que los datos se interpretan de forma coherente en los distintos equipos de un entorno distribuido.

</dd> </dl>
