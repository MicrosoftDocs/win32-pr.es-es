## <a name="volume-types"></a><span data-ttu-id="56e07-107">Tipos de volúmenes</span><span class="sxs-lookup"><span data-stu-id="56e07-107">Volume Types</span></span>

<span data-ttu-id="56e07-108">VDS admite cinco tipos de volúmenes: simples, distribuidos, seccionados, reflejados y seccionados con paridad.</span><span class="sxs-lookup"><span data-stu-id="56e07-108">VDS supports five volume types: simple, spanned, striped, mirrored, and striped with parity.</span></span> <span data-ttu-id="56e07-109">Los volúmenes simples, distribuidos y seccionados no son tolerantes a errores. los volúmenes reflejados y de paridad son tolerantes a errores.</span><span class="sxs-lookup"><span data-stu-id="56e07-109">Simple, spanned, and striped volumes are non-fault tolerant; mirrored and parity volumes are fault tolerant.</span></span> <span data-ttu-id="56e07-110">En el resto de esta sección se describe cada uno de los tipos de volúmenes de VDS.</span><span class="sxs-lookup"><span data-stu-id="56e07-110">The remainder of this section describes each of the VDS volume types.</span></span>

-   <span data-ttu-id="56e07-113">Un volumen distribuido combina áreas de espacio sin asignar de varios discos en un volumen lógico, lo que le permite usar de forma más eficaz todo el espacio y todas las letras de unidad en un sistema de varios discos.</span><span class="sxs-lookup"><span data-stu-id="56e07-113">A spanned volume combines areas of unallocated space from multiple disks into one logical volume, allowing you to more efficiently use all of the space and all the drive letters on a multiple-disk system.</span></span>

-   <span data-ttu-id="56e07-119">Un volumen reflejado es un volumen tolerante a errores que proporciona redundancia de datos mediante el uso de dos copias, o complejos, del volumen para duplicar los datos almacenados en el volumen.</span><span class="sxs-lookup"><span data-stu-id="56e07-119">A mirrored volume is a fault-tolerant volume that provides data redundancy by using two copies, or plexes, of the volume to duplicate the data that is stored on the volume.</span></span> <span data-ttu-id="56e07-120">Todos los datos que se escriben en el volumen reflejado se escriben en ambos Plex, que se encuentran en discos físicos independientes.</span><span class="sxs-lookup"><span data-stu-id="56e07-120">All data that is written to the mirrored volume is written to both plexes, which are located on separate physical disks.</span></span> <span data-ttu-id="56e07-121">Si se produce un error en uno de los discos físicos, los datos del disco con errores dejarán de estar disponibles, pero el sistema continuará funcionando con el disco no afectado.</span><span class="sxs-lookup"><span data-stu-id="56e07-121">If one of the physical disks fails, the data on the failed disk becomes unavailable, but the system continues to operate using the unaffected disk.</span></span>
-   <span data-ttu-id="56e07-122">Un volumen seccionado con paridad es un volumen tolerante a errores con datos y paridad seccionados intermitentemente en tres o más discos físicos.</span><span class="sxs-lookup"><span data-stu-id="56e07-122">A striped with parity volume is a fault-tolerant volume with data and parity striped intermittently across three or more physical disks.</span></span> <span data-ttu-id="56e07-123">Si se produce un error en una parte de un disco físico, puede volver a crear los datos que se encontraban en la parte con errores a partir de los datos y la paridad restantes.</span><span class="sxs-lookup"><span data-stu-id="56e07-123">If a portion of a physical disk fails, you can recreate the data that was on the failed portion from the remaining data and parity.</span></span> <span data-ttu-id="56e07-124">Este tipo de volumen (también denominado volumen RAID-5) es una buena solución para la redundancia de datos en un entorno de equipo en el que la mayor parte de la actividad consiste en leer datos.</span><span class="sxs-lookup"><span data-stu-id="56e07-124">This volume type (also called a RAID-5 volume) is a good solution for data redundancy in a computer environment in which most activity consists of reading data.</span></span>
