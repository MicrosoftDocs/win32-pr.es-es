# <a name="how-to-design-a-domain-shader"></a><span data-ttu-id="d00da-103">Cómo: diseñar un sombreador de dominios</span><span class="sxs-lookup"><span data-stu-id="d00da-103">How To: Design a Domain Shader</span></span>

<span data-ttu-id="d00da-104">Un sombreador de dominios es el tercero de tres fases que funcionan conjuntamente para implementar la [teselación](direct3d-11-advanced-stages-tessellation.md).</span><span class="sxs-lookup"><span data-stu-id="d00da-104">A domain shader is the third of three stages that work together to implement [tessellation](direct3d-11-advanced-stages-tessellation.md).</span></span> <span data-ttu-id="d00da-105">El sombreador de dominios genera la geometría de la superficie a partir de los puntos de control transformados de un sombreador de casco y las coordenadas UV.</span><span class="sxs-lookup"><span data-stu-id="d00da-105">The domain shader generates the surface geometry from the transformed control points from a hull shader and the UV coordinates.</span></span> <span data-ttu-id="d00da-106">En este tema se muestra cómo diseñar un sombreador de dominios.</span><span class="sxs-lookup"><span data-stu-id="d00da-106">This topics shows how to design a domain shader.</span></span>

<span data-ttu-id="d00da-107">Un sombreador de dominio se invoca una vez para cada punto generado por la función Fixed del teselador.</span><span class="sxs-lookup"><span data-stu-id="d00da-107">A domain shader is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="d00da-108">Las entradas son las \[ coordenadas UV \] del punto de la revisión, así como todos los datos de salida del sombreador de casco, incluidos los puntos de control y las constantes de revisión.</span><span class="sxs-lookup"><span data-stu-id="d00da-108">The inputs are the UV\[W\] coordinates of the point on the patch, as well as all of the output data from the hull shader including control points and patch constants.</span></span> <span data-ttu-id="d00da-109">El resultado es un vértice definido de la forma que se desee.</span><span class="sxs-lookup"><span data-stu-id="d00da-109">The output is a vertex defined in whatever way is desired.</span></span> <span data-ttu-id="d00da-110">Si el resultado se envía al sombreador de píxeles, la salida debe incluir una posición (indicada con una semántica de posición de la VP \_ ).</span><span class="sxs-lookup"><span data-stu-id="d00da-110">If the output is being sent to the pixel shader, the output must include a position (denoted with a SV\_Position semantic).</span></span>

<span data-ttu-id="d00da-111">**Para diseñar un sombreador de dominios**</span><span class="sxs-lookup"><span data-stu-id="d00da-111">**To design a domain shader**</span></span>

1.  <span data-ttu-id="d00da-112">Defina el atributo de dominio.</span><span class="sxs-lookup"><span data-stu-id="d00da-112">Define the domain attribute.</span></span>

    ```
    [domain("quad")]
    ```

    

    <span data-ttu-id="d00da-113">El dominio se define para una revisión cuádruple.</span><span class="sxs-lookup"><span data-stu-id="d00da-113">The domain is defined for a quad patch.</span></span>

2.  <span data-ttu-id="d00da-114">Declare la ubicación en el casco con el valor del sistema de [Ubicación del dominio](/windows/desktop/direct3dhlsl/sv-domainlocation) .</span><span class="sxs-lookup"><span data-stu-id="d00da-114">Declare the location on the hull with the [domain location](/windows/desktop/direct3dhlsl/sv-domainlocation) system value.</span></span>

    -   <span data-ttu-id="d00da-115">Para una revisión cuádruple, use un float2.</span><span class="sxs-lookup"><span data-stu-id="d00da-115">For a quad patch, use a float2.</span></span>
    -   <span data-ttu-id="d00da-116">En el caso de una triple revisión, use float3 (para las coordenadas Barycentric).</span><span class="sxs-lookup"><span data-stu-id="d00da-116">For a tri patch, use a float3 (for barycentric coordinates)</span></span>
    -   <span data-ttu-id="d00da-117">Para una isolínea, use un float2.</span><span class="sxs-lookup"><span data-stu-id="d00da-117">For an isoline, use a float2.</span></span>

    <span data-ttu-id="d00da-118">Por lo tanto, la ubicación de dominio para una revisión cuádruple tiene el siguiente aspecto:</span><span class="sxs-lookup"><span data-stu-id="d00da-118">Therefore, the domain location for a quad patch looks like this:</span></span>

    ```
    float2 UV : SV_DomainLocation
    ```

    

3.  <span data-ttu-id="d00da-119">Defina las demás entradas.</span><span class="sxs-lookup"><span data-stu-id="d00da-119">Define the other inputs.</span></span>

    <span data-ttu-id="d00da-120">Las demás entradas proceden del sombreador de casco y están definidas por el usuario.</span><span class="sxs-lookup"><span data-stu-id="d00da-120">The other inputs come from the hull shader and are user defined.</span></span> <span data-ttu-id="d00da-121">Esto incluye los puntos de control de entrada para la revisión, de los cuales puede estar entre 1 y 32 puntos, y datos constantes de la revisión de entrada.</span><span class="sxs-lookup"><span data-stu-id="d00da-121">This includes the input control points for patch, of which there can be between 1 and 32 points, and input patch constant data.</span></span>

    <span data-ttu-id="d00da-122">Los puntos de control están definidos por el usuario, normalmente con una estructura como esta (definida en [Cómo: diseñar un sombreador de casco](direct3d-11-advanced-stages-hull-shader-design.md)):</span><span class="sxs-lookup"><span data-stu-id="d00da-122">The control points are user defined, usually with a structure such as this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch
    ```

    

    <span data-ttu-id="d00da-123">Los datos constantes de revisión también están definidos por el usuario y podrían parecerse a este (definido en [Cómo: diseñar un sombreador de casco](direct3d-11-advanced-stages-hull-shader-design.md)):</span><span class="sxs-lookup"><span data-stu-id="d00da-123">The patch constant data is also user defined, and might look like this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    HS_CONSTANT_DATA_OUTPUT input
    ```

    

4.  <span data-ttu-id="d00da-124">Agregue código definido por el usuario para calcular las salidas; crea el cuerpo del sombreador de dominios.</span><span class="sxs-lookup"><span data-stu-id="d00da-124">Add user-defined code to compute the outputs; this makes up the body of the domain shader.</span></span>

    <span data-ttu-id="d00da-125">Esta estructura contiene las salidas del sombreador de dominios definidas por el usuario.</span><span class="sxs-lookup"><span data-stu-id="d00da-125">This structure contains user-defined domain shader outputs.</span></span>

    ```
    struct DS_OUTPUT
    {
        float3 vNormal    : NORMAL;
        float2 vUV        : TEXCOORD;
        float3 vTangent   : TANGENT;
        float3 vBiTangent : BITANGENT;
        
        float4 vPosition  : SV_POSITION;
    };
    ```

    

    <span data-ttu-id="d00da-126">La función toma cada UV de entrada (desde el del teselador) y evalúa el parche de Bézier en esta posición.</span><span class="sxs-lookup"><span data-stu-id="d00da-126">The function takes each input UV (from the tessellator) and evaluates the Bezier patch at this position.</span></span>

    ```
    [domain("quad")]
    DS_OUTPUT BezierEvalDS( HS_CONSTANT_DATA_OUTPUT input, 
                            float2 UV : SV_DomainLocation,
                            const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch )
    {
        DS_OUTPUT Output;

        // Insert code to compute the output here.

        return Output;    
    }
    ```

    

    <span data-ttu-id="d00da-127">La función se invoca una vez para cada punto generado por la función Fixed del teselador.</span><span class="sxs-lookup"><span data-stu-id="d00da-127">The function is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="d00da-128">Dado que en este ejemplo se usa un cuádruple parche, la ubicación del dominio de entrada ([VP \_ DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) es un float2 (UV); un tri patch tendría una ubicación de entrada FLOAT3 (coordenadas UVW Barycentric) y una isolínea tendría una ubicación de dominio de entrada float2.</span><span class="sxs-lookup"><span data-stu-id="d00da-128">Since this example uses a quad patch, the input domain location ([SV\_DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) is a float2 (UV); a tri patch would have a float3 input location (UVW barycentric coordinates), and an isoline would have a float2 input domain location.</span></span>

    <span data-ttu-id="d00da-129">Las demás entradas de la función proceden directamente del sombreador de casco.</span><span class="sxs-lookup"><span data-stu-id="d00da-129">The other inputs for the function come from the hull shader directly.</span></span> <span data-ttu-id="d00da-130">En este ejemplo, es 16 puntos de control cada uno de los cuales es un **\_ \_ punto de control Bézier**, así como datos constantes de revisión (**\_ \_ \_ salida de datos constante HS**).</span><span class="sxs-lookup"><span data-stu-id="d00da-130">In this example it is 16 control points each one being a **BEZIER\_CONTROL\_POINT**, as well as patch constant data (**HS\_CONSTANT\_DATA\_OUTPUT**).</span></span> <span data-ttu-id="d00da-131">El resultado es un vértice que contiene los datos de **\_ salida de DS** deseado en este ejemplo.</span><span class="sxs-lookup"><span data-stu-id="d00da-131">The output is a vertex containing any data desired - **DS\_OUTPUT** in this example.</span></span>
