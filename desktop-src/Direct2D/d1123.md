# <a name="d1123-dxgi-format-unknown-not-allowed"></a>D1123: no se permite el formato de DXGI desconocido

\_ \_ No se permite el formato de DXGI desconocido como parámetro para esta API.






 

## <a name="possible-causes"></a>Causas posibles

Para la funcionalidad de Direct2D que acepta un formato de superficie como parámetro, el formato de superficie debe especificarse por completo. En algunos casos, se puede dejar desconocido; por ejemplo, al llamar a ID2D1RenderTarget:: CreateHwndRenderTarget o ID2D1RenderTarget:: CreateDxgiSurfaceRenderTarget, Direct2D puede inferir un formato de superficie a partir del formato de la cadena de intercambio o la copia de seguridad de la superficie de Direct3D, como podría ser.

## <a name="possible-fixes"></a>Posibles correcciones

En los casos en los que no es posible deducir el formato de la superficie, Direct2D no creará el recurso y generará este mensaje. Para solucionar este error, modifique la aplicación para especificar un formato de superficie distinto de desconocido para la operación que produce este mensaje. Un formato de superficie predeterminado razonable para usar, en caso de duda, es el formato de DXGI \_ \_ B8G8R8A8 \_ UNORM. Si la aplicación debe crear un recurso de Direct2D para interoperar con un recurso de Direct3D o WIC, elija un formato que coincida con el recurso interoperable. La semántica de los recursos de WIC difiere ligeramente de los recursos de DXGI; consulte [formatos de píxeles compatibles y modos alfa](supported-pixel-formats-and-alpha-modes.md) para obtener instrucciones adicionales.
