## <a name="remarks"></a>Observaciones

Estos tipos numéricos tienen expansiones de palabras clave Scalar, Vector y Matrix:

-   Float, int, uint, bool
-   min10float, min16float
-   min12int, min16int
-   min16uint

Las expansiones de estos tipos numéricos siguen este patrón, que usa Float como ejemplo:

-   Escalar

    <dl> FLOAT  
    </dl>

-   Vector

    <dl> float1, float2, float3, FLOAT4  
    </dl>

-   Matrix

    <dl> float1x1, float1x2, float1x3, float1x4  
    float2x1, float2x2, float2x3, float2x4  
    float3x1, float3x2, float3x3, float3x4  
    float4x1, float4x2, float4x3, float4x4  
    </dl>

HLSL es compatible con la [textura](dx-graphics-hlsl-texture.md) y la [muestra](dx-graphics-hlsl-sampler.md) en minúsculas por motivos de herencia. En su lugar, para las nuevas aplicaciones, se recomienda usar los nuevos objetos de textura de HLSL ([Texture2D](sm5-object-texture2d.md), [Texture3D](sm5-object-texture3d.md), etc.) y los objetos de muestra ([SamplerState y SamplerComparisonState](dx-graphics-hlsl-sampler.md)).

### <a name="export"></a>exportar

Use **exportar** para marcar las funciones que se empaquetan en una biblioteca.

Este es un ejemplo:

``` syntax
export float identity(float x)
{
    return x;
}
```

Al marcar la función **Identity** con la palabra clave **Export** , se hace que la función **Identity** esté disponible en una biblioteca para vincularla más adelante. Sin el marcado de **exportación** , la función de **identidad** no está disponible para la vinculación posterior.

El compilador omite la palabra clave **Export** para la compilación que no es de biblioteca.

> [!Note]  
> La palabra clave **Export** requiere el \_47.dll D3dcompiler o una versión posterior del archivo dll.
