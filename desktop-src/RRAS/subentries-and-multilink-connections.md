# <a name="subentries-and-multilink-connections"></a>Conexiones de varios vínculos y entradas

Windows NT Server 4.0 proporciona compatibilidad con las entradas de la libreta de teléfonos, que habilitan las conexiones de varios vínculos. Una conexión de varios vínculos combina el ancho de banda de varias conexiones para proporcionar una sola conexión con mayor ancho de banda.

Una entrada de libreta de teléfono RAS puede tener cero o más entradas. La [**función RasGetEntryProperties**](/windows/desktop/api/Ras/nf-ras-rasgetentrypropertiesa) recupera una estructura [**RASENTRY**](/previous-versions/windows/desktop/legacy/aa377274(v=vs.85)) que incluye información sobre las entradas de una libreta de teléfonos. El **miembro dwSubEntries** de la **estructura RASENTRY** indica el número de subentries. Teléfono entradas de libro no tienen entradas de entrada. Para agregar entradas de subbentries a una entrada de la libreta de teléfonos, use la [**función RasSetSubEntryProperties.**](/windows/desktop/api/Ras/nf-ras-rassetsubentrypropertiesa)

Las propiedades de cada subentrada incluyen un número de teléfono y el nombre y el tipo del dispositivo TAPI que se usará al marcar el subentrada. Además, un subentrada puede incluir una lista de números de teléfono alternativos para marcar si RAS no puede establecer una conexión con el número principal. Las [**funciones RasSetSubEntryProperties**](/windows/desktop/api/Ras/nf-ras-rassetsubentrypropertiesa) y [**RasGetSubEntryProperties**](/windows/desktop/api/Ras/nf-ras-rasgetsubentrypropertiesa) usan la estructura [**RASSUBENTRY**](/previous-versions/windows/desktop/legacy/aa377839(v=vs.85)) para establecer y recuperar las propiedades de una subentrada de libreta telefónica especificada. Los subentries se identifican mediante un índice basado en uno.

Puede llamar a la [**función RasSetEntryProperties para**](/windows/desktop/api/Ras/nf-ras-rassetentrypropertiesa) configurar una entrada RAS de varios vínculos para conectar todos los subentries cuando se marca por primera vez. Como alternativa, puede configurar una entrada para proporcionar ancho de banda variable. En este caso, RAS conecta inicialmente una única suentrada y, a continuación, conecta o desconecta entradas adicionales según sea necesario. Para una conexión de varios vínculos de ancho de banda variable, puede usar la estructura [**RASDIALPARAMS**](/previous-versions/windows/desktop/legacy/aa377238(v=vs.85)) para especificar la subentrada inicial para conectarse al llamar a la [**función RasDial.**](/windows/desktop/api/Ras/nf-ras-rasdiala) Al usar la [**función RasDialDlg**](/windows/desktop/api/Rasdlg/nf-rasdlg-rasdialdlga) para conectar una entrada de varios vínculos, puede usar la estructura [**RASDIALDLG**](/previous-versions/windows/desktop/legacy/aa377023(v=vs.85)) para especificar la subentrada inicial que se va a conectar.

Para una conexión de varios vínculos de ancho de banda variable, use la estructura [**RASENTRY**](/previous-versions/windows/desktop/legacy/aa377274(v=vs.85)) con la función [**RasSetEntryProperties**](/windows/desktop/api/Ras/nf-ras-rassetentrypropertiesa) para especificar los parámetros para conectar y desconectar los subentries individuales. RAS conecta una subentrada adicional cuando el ancho de banda que se usa supera un porcentaje especificado del ancho de banda disponible para un intervalo especificado.

Si llama a la [**función RasDial**](/windows/desktop/api/Ras/nf-ras-rasdiala) para establecer una conexión de varios vínculos, puede especificar una función de devolución de llamada [**RasDialFunc2**](/windows/desktop/api/Ras/nc-ras-rasdialfunc2) para recibir notificaciones sobre la conexión. **RasDialFunc2** es similar a la función de devolución de llamada [**RasDialFunc1,**](/windows/desktop/api/Ras/nc-ras-rasdialfunc1) salvo que proporciona información adicional para una conexión de varios vínculos, como el índice de la subentrada que provocó la notificación. RAS llama a **la función RasDialFunc2** cuando se conecta o desconecta una subentrada.

Puede usar un identificador **de conexión HRASCONN** para que no se pueda conectar o recuperar información sobre una conexión de varios vínculos. Puede obtener un identificador de conexión para cada una de las conexiones de subentrada que forma el vínculo múltiple, así como para la conexión de varios vínculos combinada. Cuando se llama a la [**función RasDial**](/windows/desktop/api/Ras/nf-ras-rasdiala) para establecer una conexión de varios vínculos, **RasDial** devuelve un identificador a la conexión de varios vínculos combinada. De forma similar, [**RasEnumConnections**](/windows/desktop/api/Ras/nf-ras-rasenumconnectionsa) devuelve el identificador de varios vínculos combinado al enumerar las conexiones. Para obtener un identificador para una de las conexiones de subentrada en una conexión de varios vínculos, llame a la [**función RasGetSubEntryHandle.**](/windows/desktop/api/Ras/nf-ras-rasgetsubentryhandlea)

Puede usar el identificador de conexión de varios vínculos combinado y los identificadores de conexión de subentrada en las funciones [**RasHangUp**](/windows/desktop/api/Ras/nf-ras-rashangupa), [**RasGetConnectStatus**](/windows/desktop/api/Ras/nf-ras-rasgetconnectstatusa)y [**RasGetProjectionInfo.**](/previous-versions/windows/embedded/ms897107(v=msdn.10)) Al **llamar a RasHangUp** con un identificador de varios vínculos combinado, se finaliza toda la conexión. llamarlo con un identificador de subentrada solo se queda para arriba esa conexión de subentrada. De forma similar, **RasGetConnectStatus devuelve** información para la conexión combinada o individual, en función del identificador especificado. La información de proyección devuelta por **RasGetProjectionInfo** para una entrada de varios vínculos es la misma para cada uno de los identificadores de conexión de subentrada que para el identificador de conexión principal.
