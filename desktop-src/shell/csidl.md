# <a name="csidl"></a>CSIDL

> [!Note]  
> A partir de Windows Vista, estos valores se han sustituido por valores [**KNOWNFOLDERID**](knownfolderid.md) . Consulte ese tema para obtener una lista de las nuevas constantes y sus valores de CSIDL correspondientes. Para mayor comodidad, los valores de **KNOWNFOLDERID** correspondientes también se indican aquí para cada valor de CSIDL.
>
> El sistema CSIDL es compatible con Windows Vista por motivos de compatibilidad. Sin embargo, el nuevo desarrollo debe usar valores [**KNOWNFOLDERID**](knownfolderid.md) en lugar de los valores CSIDL.

 

Los valores de CSIDL (lista de ID. de elemento especial constante) proporcionan una manera única independiente del sistema para identificar carpetas especiales usadas con frecuencia por aplicaciones, pero que pueden no tener el mismo nombre o ubicación en ningún sistema determinado. Por ejemplo, la carpeta del sistema puede ser "C: \\ Windows" en un sistema y "c: \\ WinNT" en otro. Estas constantes se definen en ShlObj. h.

## <a name="remarks"></a>Observaciones

Estos valores sustituyen al uso de variables de entorno para este fin. A su vez, se sustituyen por los valores [**KNOWNFOLDERID**](knownfolderid.md) en Windows Vista y versiones posteriores.

Se usa CSIDL junto con una de las cuatro funciones de Shell, [**SHGetFolderLocation**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderlocation), [**SHGetFolderPath**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderpatha), [**SHGetSpecialFolderLocation**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderlocation)y [**SHGetSpecialFolderPath**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderpatha), para recuperar la ruta de acceso de una carpeta especial o el puntero a una lista de identificadores de elementos (PIDL).

Combine * * * * la \_ marca \_ de CSIDL Create * * * * con cualquiera de los otros CSIDLs, con la excepción de * * * * la marca de CSIDL no \_ \_ Verify * * * * * \_ , para forzar la creación de la carpeta asociada. El resto de CSIDLs corresponden a carpetas del sistema de archivos o a carpetas virtuales. Donde CSIDL identifica una carpeta del sistema de archivos, se proporciona una ruta de acceso de uso común como ejemplo. Se pueden usar otras rutas de acceso. Algunos CSIDLs se pueden asignar a una variable de entorno *% VariableName%* equivalente. Sin embargo, los CSIDLs son más confiables y deben usarse si es posible.
