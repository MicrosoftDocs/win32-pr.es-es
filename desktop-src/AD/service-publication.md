# <a name="service-publication"></a>Publicación de servicios

Los servicios se anuncian mediante objetos almacenados en Active Directory Domain Services. Esto se conoce como *publicación de servicio*. Los clientes consultan el directorio para encontrar los servicios. Esto se denomina *cliente-servicio Rendezvous*. En esta sección se describen los tipos de objetos de directorio que se usan para la publicación de servicio y se explica cómo se usan para el encuentro de servicio de cliente.

En esta sección se tratan los temas siguientes:

-   [Información general de la publicación de servicio](about-service-publication.md)
-   [Problemas de seguridad para la publicación de servicio](security-issues-for-service-publication.md)
-   [Objetos de punto de conexión](connection-points.md)
-   [Publicación con puntos de conexión de servicio (SCP)](publishing-with-service-connection-points.md)
-   [Información que se va a almacenar en un punto de conexión de servicio](service-connection-point-properties.md)
-   [Dónde crear un punto de conexión de servicio](where-to-create-a-service-connection-point.md)
-   [Publicación de servicios replicables, basados en host y de base de datos mediante puntos de conexión de servicio](service-connection-points-for-replicated-host-based-and-database-services.md)
-   [Crear y mantener un punto de conexión de servicio](creating-and-maintaining-a-service-connection-point.md)
-   [Cómo un cliente consulta un SCP y lo usa para enlazar a una instancia de servicio](how-clients-find-and-use-a-service-connection-point.md)
-   [Uso de las API del servicio de nombres de RPC (RpcNs) para publicar un servicio RPC](publishing-with-the-rpc-name-service-rpcns.md)
-   [Registro y resolución de Windows Sockets (RnR) para publicar un servicio de Windows Sockets](publishing-with-windows-sockets-registration-and-resolution.md)
-   [Publicación de servicios basados en COM en el almacén de clases COM+](publishing-com-services.md)

Para obtener más información sobre cómo se autentican los servicios y los clientes entre sí, vea [autenticación mutua mediante Kerberos](mutual-authentication-using-kerberos.md). Para obtener más información sobre los contextos de seguridad del servicio y las cuentas de inicio de sesión, consulte [cuentas de inicio de sesión del servicio](service-logon-accounts.md).
