# <a name="responsibilities-of-winlogon"></a><span data-ttu-id="01cac-103">Responsabilidades de Winlogon</span><span class="sxs-lookup"><span data-stu-id="01cac-103">Responsibilities of Winlogon</span></span>

<span data-ttu-id="01cac-104">[*Winlogon*](../secgloss/w-gly.md) tiene las siguientes responsabilidades:</span><span class="sxs-lookup"><span data-stu-id="01cac-104">[*Winlogon*](../secgloss/w-gly.md) has the following responsibilities:</span></span>

-   <span data-ttu-id="01cac-105">Estación de ventana y protección de escritorio</span><span class="sxs-lookup"><span data-stu-id="01cac-105">Window station and desktop protection</span></span>

    <span data-ttu-id="01cac-106">Winlogon establece la protección de la estación de ventana y los escritorios correspondientes para asegurarse de que cada una sea accesible correctamente.</span><span class="sxs-lookup"><span data-stu-id="01cac-106">Winlogon sets the protection of the window station and corresponding desktops to ensure that each is properly accessible.</span></span> <span data-ttu-id="01cac-107">En general, esto significa que el sistema local tendrá acceso completo a estos objetos y que un usuario que haya iniciado sesión de forma interactiva tendrá acceso de lectura al objeto de la estación de ventana y acceso completo al objeto de escritorio de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="01cac-107">In general, this means that the local system will have full access to these objects and that an interactively logged-on user will have read access to the window station object and full access to the application desktop object.</span></span>

-   <span data-ttu-id="01cac-108">Reconocimiento de SAS estándar</span><span class="sxs-lookup"><span data-stu-id="01cac-108">Standard SAS recognition</span></span>

    <span data-ttu-id="01cac-109">Winlogon tiene enlaces especiales en el servidor user32 que le permiten supervisar los eventos de la secuencia de [*atención segura*](../secgloss/s-gly.md) (SAS) de Ctrl + Alt + Supr.</span><span class="sxs-lookup"><span data-stu-id="01cac-109">Winlogon has special hooks into the User32 server that allow it to monitor CTRL+ALT+DEL [*secure attention sequence*](../secgloss/s-gly.md) (SAS) events.</span></span> <span data-ttu-id="01cac-110">Winlogon hace que esta información de eventos de SAS esté disponible para que GINA la use como SAS o como parte de su SAS.</span><span class="sxs-lookup"><span data-stu-id="01cac-110">Winlogon makes this SAS event information available to GINAs to use as their SAS, or as part of their SAS.</span></span> <span data-ttu-id="01cac-111">En general, GINAs debe supervisar SASs por sí mismo; sin embargo, cualquier [*Gina*](../secgloss/g-gly.md) que tenga la SAS estándar Ctrl + Alt + Supr como uno de los Sass que reconoce debe usar la compatibilidad con Winlogon proporcionada para este fin.</span><span class="sxs-lookup"><span data-stu-id="01cac-111">In general, GINAs should monitor SASs on their own; however, any [*GINA*](../secgloss/g-gly.md) that has the standard CTRL+ALT+DEL SAS as one of the SASs it recognizes should use the Winlogon support provided for this purpose.</span></span>

-   <span data-ttu-id="01cac-112">Envío de rutinas SAS</span><span class="sxs-lookup"><span data-stu-id="01cac-112">SAS routine dispatching</span></span>

    <span data-ttu-id="01cac-113">Cuando Winlogon encuentra un evento de SAS o cuando se entrega una SAS a Winlogon mediante GINA, Winlogon establece el estado en consecuencia, cambia en el escritorio de Winlogon y llama a una de las funciones de procesamiento de SAS de GINA.</span><span class="sxs-lookup"><span data-stu-id="01cac-113">When Winlogon encounters a SAS event or when a SAS is delivered to Winlogon by the GINA, Winlogon sets the state accordingly, changes to the Winlogon desktop, and calls one of the SAS processing functions of the GINA.</span></span>

-   <span data-ttu-id="01cac-114">Carga de perfiles de usuario</span><span class="sxs-lookup"><span data-stu-id="01cac-114">User profile loading</span></span>

    <span data-ttu-id="01cac-115">Cuando los usuarios inician sesión, sus perfiles de usuario se cargan en el registro.</span><span class="sxs-lookup"><span data-stu-id="01cac-115">When users log on, their user profiles are loaded into the registry.</span></span> <span data-ttu-id="01cac-116">De esta manera, los procesos del usuario pueden usar la clave especial del registro HKEY \_ Current \_ User.</span><span class="sxs-lookup"><span data-stu-id="01cac-116">In this way, the processes of the user can use the special registry key HKEY\_CURRENT\_USER.</span></span> <span data-ttu-id="01cac-117">Winlogon realiza esta operación automáticamente después de un inicio de sesión correcto, pero antes de la activación del shell para el usuario que acaba de iniciar sesión.</span><span class="sxs-lookup"><span data-stu-id="01cac-117">Winlogon does this automatically after a successful logon but before activation of the shell for the newly logged-on user.</span></span>

-   <span data-ttu-id="01cac-118">Asignación de seguridad al shell de usuario</span><span class="sxs-lookup"><span data-stu-id="01cac-118">Assignment of security to user shell</span></span>

    <span data-ttu-id="01cac-119">Cuando un usuario inicia sesión, GINA es responsable de crear uno o más procesos iniciales para ese usuario.</span><span class="sxs-lookup"><span data-stu-id="01cac-119">When a user logs on, the GINA is responsible for creating one or more initial processes for that user.</span></span> <span data-ttu-id="01cac-120">Winlogon proporciona una función de soporte técnico para que GINA aplique la seguridad del usuario que ha iniciado sesión recientemente a estos procesos.</span><span class="sxs-lookup"><span data-stu-id="01cac-120">Winlogon provides a support function for the GINA to apply the security of the newly logged-on user to these processes.</span></span> <span data-ttu-id="01cac-121">Sin embargo, la mejor manera de hacerlo es que GINA llame a la función de Windows [**CreateProcessAsUser**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-createprocessasusera)y permita que el sistema proporcione el servicio.</span><span class="sxs-lookup"><span data-stu-id="01cac-121">However, the preferred way to do this is for the GINA to call the Windows function [**CreateProcessAsUser**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-createprocessasusera), and let the system provide the service.</span></span>

-   <span data-ttu-id="01cac-122">Control de protector de pantalla</span><span class="sxs-lookup"><span data-stu-id="01cac-122">Screen saver control</span></span>

    <span data-ttu-id="01cac-123">Winlogon supervisa la actividad del teclado y del mouse para determinar cuándo se deben activar los protectores de pantalla.</span><span class="sxs-lookup"><span data-stu-id="01cac-123">Winlogon monitors keyboard and mouse activity to determine when to activate screen savers.</span></span> <span data-ttu-id="01cac-124">Una vez activado el protector de pantalla, Winlogon continúa supervisando la actividad del teclado y del mouse para determinar cuándo terminar el protector de pantalla.</span><span class="sxs-lookup"><span data-stu-id="01cac-124">After the screen saver is activated, Winlogon continues to monitor keyboard and mouse activity to determine when to terminate the screen saver.</span></span> <span data-ttu-id="01cac-125">Si el protector de pantalla está marcado como seguro, Winlogon trata la estación de trabajo como bloqueada.</span><span class="sxs-lookup"><span data-stu-id="01cac-125">If the screen saver is marked as secure, Winlogon treats the workstation as locked.</span></span> <span data-ttu-id="01cac-126">Cuando hay actividad del mouse o del teclado, Winlogon invoca la función [**WlxDisplayLockedNotice**](/windows/desktop/api/Winwlx/nf-winwlx-wlxdisplaylockednotice) de los currículos de Gina y el comportamiento de la estación de trabajo bloqueado.</span><span class="sxs-lookup"><span data-stu-id="01cac-126">When there is mouse or keyboard activity, Winlogon invokes the [**WlxDisplayLockedNotice**](/windows/desktop/api/Winwlx/nf-winwlx-wlxdisplaylockednotice) function of the GINA and locked workstation behavior resumes.</span></span> <span data-ttu-id="01cac-127">Si el protector de pantalla no es seguro, cualquier actividad del teclado o del mouse finaliza el protector de pantalla sin notificación a GINA.</span><span class="sxs-lookup"><span data-stu-id="01cac-127">If the screen saver is not secure, any keyboard or mouse activity terminates the screen saver without notification to the GINA.</span></span>

-   <span data-ttu-id="01cac-128">Compatibilidad con varios proveedores de redes</span><span class="sxs-lookup"><span data-stu-id="01cac-128">Multiple network provider support</span></span>
