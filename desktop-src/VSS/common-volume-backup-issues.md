# <a name="common-volume-backup-issues"></a><span data-ttu-id="b4c85-103">Problemas comunes de copia de seguridad de volumen</span><span class="sxs-lookup"><span data-stu-id="b4c85-103">Common Volume Backup Issues</span></span>

<span data-ttu-id="b4c85-104">Cualquier operación de copia de seguridad que intente copiar una imagen completa y estable de un sistema debe tratar los siguientes aspectos:</span><span class="sxs-lookup"><span data-stu-id="b4c85-104">Any backup operation that attempts to copy a full and stable image of a system must deal with the following concerns:</span></span>

-   <span data-ttu-id="b4c85-105">Archivos inaccesibles durante una copia de seguridad.</span><span class="sxs-lookup"><span data-stu-id="b4c85-105">Inaccessible files during a backup.</span></span> <span data-ttu-id="b4c85-106">Con frecuencia, las aplicaciones en ejecución necesitan mantener los archivos abiertos en modo exclusivo durante una copia de seguridad, evitando que los programas de copia de seguridad los copien.</span><span class="sxs-lookup"><span data-stu-id="b4c85-106">Running applications frequently need to keep files open in exclusive mode during a backup, preventing backup programs from copying them.</span></span>
-   <span data-ttu-id="b4c85-107">Estado de archivo incoherente.</span><span class="sxs-lookup"><span data-stu-id="b4c85-107">Inconsistent file state.</span></span> <span data-ttu-id="b4c85-108">Incluso si una aplicación no tiene sus archivos abiertos en modo exclusivo, es posible que, debido al tiempo finito necesario para abrir, realizar una copia de seguridad y cerrar un archivo, los archivos copiados en el medio de almacenamiento no reflejen el mismo estado de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="b4c85-108">Even if an application does not have its files open in exclusive mode, it is possible—because of the finite time needed to open, back up, and close a file—that files copied to storage media may not all reflect the same application state.</span></span>
-   <span data-ttu-id="b4c85-109">Necesidad de minimizar las interrupciones del servicio.</span><span class="sxs-lookup"><span data-stu-id="b4c85-109">Need to minimize service interruptions.</span></span> <span data-ttu-id="b4c85-110">Para garantizar la accesibilidad de los archivos y la integridad de los datos de los que se realiza una copia de seguridad, es posible que se requiera la suspensión o finalización de todos los programas en ejecución durante una copia de seguridad del volumen.</span><span class="sxs-lookup"><span data-stu-id="b4c85-110">To ensure file accessibility and the integrity of the data being backed up can require the suspension and/or termination of all running programs during a volume backup.</span></span> <span data-ttu-id="b4c85-111">En el caso de los sistemas de discos de gran tamaño, esto podría ser horas de duración.</span><span class="sxs-lookup"><span data-stu-id="b4c85-111">For large disk systems, this could be hours in duration.</span></span>

    <span data-ttu-id="b4c85-112">Recientemente, algunos proveedores de almacenamiento han intentado resolver estos problemas proporcionando un mecanismo de captura de volumen, un medio para capturar una imagen de los archivos en disco en un momento dado en el tiempo, mediante un mecanismo de copia en escritura o de "reflejo dividido".</span><span class="sxs-lookup"><span data-stu-id="b4c85-112">Recently, some storage vendors have attempted to address these problems by providing a volume capture mechanism—a means to capture an image of the files on disk at a given instant in time—using either a copy-on-write or "split mirror" mechanism.</span></span> <span data-ttu-id="b4c85-113">Sin embargo, estas soluciones suponen dificultades propias:</span><span class="sxs-lookup"><span data-stu-id="b4c85-113">However, these solutions entail difficulties of their own:</span></span>

    -   <span data-ttu-id="b4c85-114">Implementaciones de los proveedores incompatibles de la captura de volumen.</span><span class="sxs-lookup"><span data-stu-id="b4c85-114">Incompatible vendor implementations of volume capture.</span></span> <span data-ttu-id="b4c85-115">Muchos proveedores de dispositivos RAID proporcionan mecanismos de captura de volumen.</span><span class="sxs-lookup"><span data-stu-id="b4c85-115">Many providers of RAID devices provide volume capture mechanisms.</span></span> <span data-ttu-id="b4c85-116">Sin embargo, cada proveedor tiene su propia interfaz y cada uno debe obtener soporte técnico de los proveedores de copia de seguridad para sus interfaces de captura de volumen.</span><span class="sxs-lookup"><span data-stu-id="b4c85-116">However, each vendor has its own interface and each must get support from the backup vendors for their volume capture interfaces.</span></span> <span data-ttu-id="b4c85-117">Esto significa que los proveedores de aplicaciones de copia de seguridad deben admitir varias implementaciones de captura de volumen, lo cual no es deseable.</span><span class="sxs-lookup"><span data-stu-id="b4c85-117">This means that backup application vendors must support multiple volume capture implementations, which is undesirable.</span></span>
    -   <span data-ttu-id="b4c85-118">Falta de coordinación de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="b4c85-118">Lack of application coordination.</span></span> <span data-ttu-id="b4c85-119">Muchos dispositivos que admiten una captura de volumen no admiten la coordinación de la ejecución de aplicaciones con la inmovilización de datos en el disco.</span><span class="sxs-lookup"><span data-stu-id="b4c85-119">Many devices that support a volume capture do not support the coordination of running applications with the freeze of data on disk.</span></span> <span data-ttu-id="b4c85-120">En el caso de los dispositivos que hacen, al igual que las aplicaciones de copia de seguridad, cada proveedor tiene una interfaz diferente.</span><span class="sxs-lookup"><span data-stu-id="b4c85-120">For those devices that do, as with the backup applications, each vendor has a different interface.</span></span>
    -   <span data-ttu-id="b4c85-121">Compatibilidad limitada para dispositivos que no son RAID.</span><span class="sxs-lookup"><span data-stu-id="b4c85-121">Limited support for non-RAID devices.</span></span> <span data-ttu-id="b4c85-122">Pocos si los proveedores de discos convencionales proporcionan compatibilidad con cualquier tipo de captura de volumen en sus controladores de dispositivos.</span><span class="sxs-lookup"><span data-stu-id="b4c85-122">Few if any conventional disk vendors provide support for any sort of volume capture in their device drivers.</span></span> <span data-ttu-id="b4c85-123">Esto significa que los mecanismos de captura se limitan a determinados sistemas de disco y, por lo general, no admiten la copia de seguridad de áreas del sistema.</span><span class="sxs-lookup"><span data-stu-id="b4c85-123">This means that capture mechanisms are limited to certain disk systems, and cannot typically support the backup of system areas.</span></span>
    -   <span data-ttu-id="b4c85-124">Es necesario controlar las actualizaciones del disco durante la captura de volumen.</span><span class="sxs-lookup"><span data-stu-id="b4c85-124">Need to handle updates to disk during volume capture.</span></span> <span data-ttu-id="b4c85-125">Aunque los mecanismos de captura de volumen proporcionados por el proveedor de almacenamiento pueden inmovilizar el estado de los datos en el disco, no siempre interoperan con las aplicaciones en ejecución.</span><span class="sxs-lookup"><span data-stu-id="b4c85-125">Although storage-vendor-provided volume capture mechanisms can freeze the state of data on disk, they do not always interoperate with running applications.</span></span> <span data-ttu-id="b4c85-126">Esto suele significar que se pueden perder los datos que se envían al volumen mientras un dispositivo de almacenamiento está llevando a cabo una captura de volumen.</span><span class="sxs-lookup"><span data-stu-id="b4c85-126">This frequently means that data sent to the volume while a storage device is undergoing volume capture may be lost.</span></span>
    -   <span data-ttu-id="b4c85-127">Copia de seguridad de multivolumen coherente.</span><span class="sxs-lookup"><span data-stu-id="b4c85-127">Consistent multivolume backup.</span></span> <span data-ttu-id="b4c85-128">El dispositivo de almacenamiento ejecuta estas capturas de volumen, por lo que normalmente no hay ningún mecanismo para coordinar la temporización de la inmovilización de datos.</span><span class="sxs-lookup"><span data-stu-id="b4c85-128">The storage device executes these volume captures, so there is generally no mechanism for coordinating the timing of the data freeze.</span></span> <span data-ttu-id="b4c85-129">Esto es especialmente cierto si los dispositivos proceden de proveedores independientes.</span><span class="sxs-lookup"><span data-stu-id="b4c85-129">This is particularly true if the devices come from separate vendors.</span></span> <span data-ttu-id="b4c85-130">Por lo tanto, si hay varios volúmenes de almacenamiento implicados en una copia de seguridad con una captura de volumen, es posible que la imagen de tiempo conservada para cada volumen no sea coherente.</span><span class="sxs-lookup"><span data-stu-id="b4c85-130">Therefore, if several storage volumes are involved in a backup with a volume capture, the time image preserved for each volume may not be consistent.</span></span>
