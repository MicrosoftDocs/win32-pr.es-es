# <a name="inputtype-complex-type"></a>Tipo complejo de InputType

Define un tipo de datos de entrada.

``` syntax
<xs:complexType name="InputType">
    <xs:sequence>
        <xs:element name="description"
            type="string"
         />
        <xs:element name="outType"
            type="OutputType"
            minOccurs="0"
            maxOccurs="unbounded"
         />
        <xs:any
            processContents="lax"
            namespace="##other"
         />
    </xs:sequence>
    <xs:attribute name="name"
        type="QName"
        use="required"
     />
    <xs:attribute name="value"
        type="string"
        use="required"
     />
    <xs:attribute name="symbol"
        type="CSymbolType"
        use="required"
     />
</xs:complexType>
```

## <a name="child-elements"></a>Elementos secundarios



| Elemento                                                                  | Tipo                                                             | Descripción                                                                                                                                                                                                                                                                                                                                                                                                   |
|--------------------------------------------------------------------------|------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**denominación**](eventmanifestschema-description-inputtype-element.md) | string                                                           | Descripción del tipo de entrada.<br/>                                                                                                                                                                                                                                                                                                                                                                   |
| [**outtype**](eventmanifestschema-outtype-inputtype-element.md)         | [**OutputType**](eventmanifestschema-outputtype-complextype.md) | Una lista de tipos de datos de salida que determinan cómo se representan los datos. Debe especificar al menos un tipo de salida. Especifique una lista si el tipo de entrada se puede representar como tipos de salida diferentes. Por ejemplo, el tipo de entrada Win: UInt16 se puede representar como un número corto, número de puerto o número hexadecimal sin signo. Uno de los tipos de salida que especifique debe tener el atributo predeterminado establecido en true.<br/> |
