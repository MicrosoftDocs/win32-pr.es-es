# <a name="persistent-object-state"></a>Estado del objeto persistente

Algunos objetos COM pueden guardar su estado interno cuando un cliente le pide que lo haga.

COM define los estándares a través de los cuales los clientes pueden solicitar que los objetos se inicialicen, carguen y guarden en un almacén de datos (por ejemplo, archivo plano, almacenamiento estructurado o memoria). Es responsabilidad del cliente administrar la ubicación donde se almacenan los datos persistentes del objeto, pero no el formato de los datos. Los objetos COM que se adhieren a estos estándares se denominan *objetos persistentes*.

Para obtener más información, vea los temas siguientes:

-   [Interfaces de objeto persistentes](persistent-object-interfaces.md)
-   [Inicializar objetos persistentes](initializing-persistent-objects.md)
