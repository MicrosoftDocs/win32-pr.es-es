# <a name="d1186-dc-render-target-requires-binddc"></a>D1186: el destino de representación de DC requiere BindDC

No es válido utilizar un destino de representación de DC antes de llamar a su método BindDC.






 

## <a name="possible-causes"></a>Causas posibles

La aplicación creó un ID2D1DCRenderTarget y lo usaba para llamar a BeginDraw, pero aún no llama a su método BindDC. O bien, BindDC produjo un error.

## <a name="possible-fixes"></a>Posibles correcciones

Antes de utilizar un destino de representación DC-interopped, las aplicaciones deben llamar a BindDC para asociar el destino a un contexto de dispositivo GDI. No se considera que el destino sea válido hasta que se haya asignado un contexto de dispositivo GDI. Vea ID2D1DCRenderTarget:: BindDC para obtener un ejemplo. Si la aplicación ya llama a BindDC, asegúrese de que BindDC no devolvió un error.
