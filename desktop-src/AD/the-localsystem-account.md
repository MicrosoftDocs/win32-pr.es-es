<span data-ttu-id="6c940-115">Cuando un servicio se ejecuta con la cuenta LocalSystem en un equipo que es miembro del dominio, el servicio tiene el acceso a la red que se concede a la cuenta de equipo o a los grupos de los que es miembro la cuenta de equipo.</span><span class="sxs-lookup"><span data-stu-id="6c940-115">When a service runs under the LocalSystem account on a computer that is a domain member, the service has whatever network access is granted to the computer account, or to any groups of which the computer account is a member.</span></span> <span data-ttu-id="6c940-116">Tenga en cuenta que en Windows 2000, una cuenta de equipo de dominio es una entidad de servicio, similar a una cuenta de usuario.</span><span class="sxs-lookup"><span data-stu-id="6c940-116">Be aware that in Windows 2000, a domain computer account is a service principal, similar to a user account.</span></span> <span data-ttu-id="6c940-117">Esto significa que una cuenta de equipo puede estar en un grupo de seguridad y una ACE en un descriptor de seguridad puede conceder acceso a una cuenta de equipo.</span><span class="sxs-lookup"><span data-stu-id="6c940-117">This means that a computer account can be in a security group, and an ACE in a security descriptor can grant access to a computer account.</span></span> <span data-ttu-id="6c940-118">Tenga en cuenta que no se recomienda agregar cuentas de equipo a grupos por dos motivos:</span><span class="sxs-lookup"><span data-stu-id="6c940-118">Be aware that adding computer accounts to groups is not recommended for two reasons:</span></span>

-   <span data-ttu-id="6c940-119">Las cuentas de equipo están sujetas a eliminación y recreación si el equipo sale del dominio y, a continuación, se vuelve a unir.</span><span class="sxs-lookup"><span data-stu-id="6c940-119">Computer accounts are subject to deletion and re-creation if the computer leaves and then rejoins the domain.</span></span>

<span data-ttu-id="6c940-123">Normalmente, las cuentas de equipo tienen pocos privilegios y no pertenecen a los grupos.</span><span class="sxs-lookup"><span data-stu-id="6c940-123">Computer accounts typically have few privileges and do not belong to groups.</span></span> <span data-ttu-id="6c940-124">La protección de ACL predeterminada en Active Directory Domain Services permite un acceso mínimo para las cuentas de equipo.</span><span class="sxs-lookup"><span data-stu-id="6c940-124">The default ACL protection in Active Directory Domain Services permits minimal access for computer accounts.</span></span> <span data-ttu-id="6c940-125">Por lo tanto, los servicios que se ejecutan como LocalSystem, en equipos que no sean controladores de sesión, solo tienen acceso mínimo a Active Directory Domain Services.</span><span class="sxs-lookup"><span data-stu-id="6c940-125">Consequently, services running as LocalSystem, on computers other than DCs, have only minimal access to Active Directory Domain Services.</span></span>

<span data-ttu-id="6c940-126">Si el servicio se ejecuta bajo LocalSystem, debe probar el servicio en un servidor miembro para asegurarse de que el servicio tiene derechos suficientes para leer y escribir en los controladores de Dominio de Active Directory.</span><span class="sxs-lookup"><span data-stu-id="6c940-126">If your service runs under LocalSystem, you must test your service on a member server to ensure that your service has sufficient rights to read/write to Active Directory Domain Controllers.</span></span> <span data-ttu-id="6c940-127">Un controlador de dominio no debe ser el único equipo de Windows en el que se prueba el servicio.</span><span class="sxs-lookup"><span data-stu-id="6c940-127">A domain controller should not be the only Windows computer on which you test your service.</span></span> <span data-ttu-id="6c940-128">Tenga en cuenta que un servicio que se ejecuta bajo LocalSystem en un controlador de dominio de Windows tiene acceso completo a Active Directory Domain Services y que un servidor miembro se ejecuta en el contexto de la cuenta de equipo que tiene muchos menos derechos.</span><span class="sxs-lookup"><span data-stu-id="6c940-128">Be aware that a service running under LocalSystem on a Windows domain controller has complete access to Active Directory Domain Services and that a member server runs in the context of the computer account which has substantially fewer rights.</span></span>
