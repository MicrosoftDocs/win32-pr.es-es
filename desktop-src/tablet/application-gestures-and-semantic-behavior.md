# <a name="application-gestures-and-semantic-behavior"></a>Gestos de aplicación y comportamiento semántico

Los gestos de la aplicación se entregan como parte del SDK Windows Vista. El reconocedor de gestos de Microsoft se ha creado para reconocer estos gestos. De forma predeterminada, no se habilita ningún gesto. Las aplicaciones deben elegir los gestos que se habilitarán. Además de reconocer gestos, el reconocedor de gestos de Microsoft también proporciona alternativas junto con valores de confianza como parte de los resultados del reconocimiento. Para usar estos gestos en la aplicación, debe suscribirse a ellos desde dentro de la aplicación mediante la referencia de [api de Tablet PC](tablet-pc-api-reference.md). Las API permiten elegir un subconjunto de estos gestos y consultar propiedades específicas, como el número de trazos, el punto de acceso rápido y otras propiedades, así como alternativas y sus valores de confianza.

En el caso de las aplicaciones con reconocimiento de lápiz que usan estos gestos, no se envían mensajes del mouse. En su lugar, los eventos de gesto se envían a estas aplicaciones para informarles de estos gestos. A continuación, las aplicaciones llevan a cabo acciones similares a las acciones basadas en el mouse en respuesta a estos gestos.

En la tabla siguiente se enumeran los gestos de aplicación admitidos por el reconocedor de gestos de Microsoft. Para garantizar la coherencia de los gestos usados para acciones comunes entre aplicaciones, debe cumplir las siguientes sugerencias:

-   La acción es el comportamiento semántico sugerido asociado al gesto.
-   Para los gestos etiquetados como Corregidos en la tabla siguiente, Microsoft recomienda no cambiar el comportamiento semántico sugerido. Si una aplicación no necesita el comportamiento semántico especificado, Microsoft recomienda no volver a usar el gesto para otra acción o comportamiento semántico.
-   Para los gestos etiquetados como específicos de la aplicación y que tienen un comportamiento semántico sugerido, Microsoft recomienda admitir el comportamiento semántico sugerido si esa funcionalidad existe en la aplicación. Para mantener la coherencia entre aplicaciones, no elija un comportamiento semántico diferente para este gesto si la funcionalidad correspondiente a la semántica sugerida existe en la aplicación. Sin embargo, si la aplicación no tiene funcionalidad que se corresponda con la semántica sugerida, no dude en asociar los comportamientos semánticos pertinentes al gesto. Esto también se aplica a todos los gestos que son específicos de la aplicación y no tienen una semántica sugerida.
-   El punto de acceso de un gesto es un punto distintivo en la geometría del gesto. El punto de acceso se puede usar para determinar dónde se realizó el gesto. Las API de gestos, específicamente la propiedad [HotPoint](/previous-versions/ms582125(v=vs.100)) del objeto [Gesture,](/previous-versions/ms583661(v=vs.100)) hacen posible determinar el punto de acceso para un gesto determinado. Sin embargo, no todos los gestos tienen un punto de referencia distintivo específico. Para aquellos que no tienen un punto de acceso rápido distintivo específico, el punto inicial se notifica como punto de acceso rápido.

    > Algunos de los gestos tienen un punto de acceso destacado que resulta ser el punto inicial. Se distinguen en la tabla.

> [!Note]  
> Las pulsaciones y las pulsaciones dobles se admiten como gestos de la aplicación, además de como gestos del sistema. Esto es para abordar escenarios en los que puede que desee habilitar un gesto de aplicación que tenga una pulsación o una doble pulsación como parte de él. Un ejemplo de este tipo de gesto es el gesto exclamación. En tal caso, habilite el gesto de pulsar la aplicación y no escuche el gesto del sistema de pulsación. Esto le permite escuchar un único componente que puede identificar y distinguir una pulsación de una pulsación dentro de un gesto. El caso de una pulsación dentro de un gesto da como resultado que solo se reconozca el gesto general.
