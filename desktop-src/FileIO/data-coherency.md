<span data-ttu-id="e7f48-114">Proporcionar almacenamiento en caché local de datos coherentes es bastante sencillo cuando tiene un subproceso en un cliente que tiene acceso a un archivo a través de la red a la vez.</span><span class="sxs-lookup"><span data-stu-id="e7f48-114">Providing local caching of coherent data is fairly simple when you have one thread on one client accessing a file across the network at a time.</span></span> <span data-ttu-id="e7f48-115">Sin embargo, en la mayoría de los casos, muchos subprocesos diferentes en uno o varios equipos pueden estar leyendo el mismo archivo.</span><span class="sxs-lookup"><span data-stu-id="e7f48-115">However, in most situations many different threads on one or more computers may be reading the same file.</span></span> <span data-ttu-id="e7f48-116">Esta situación sigue siendo bastante sencilla.</span><span class="sxs-lookup"><span data-stu-id="e7f48-116">This situation is still fairly straightforward.</span></span> <span data-ttu-id="e7f48-117">Dado que los datos del archivo son estáticos, cada equipo cliente puede tener su propia copia local sin implicaciones en la coherencia de los datos.</span><span class="sxs-lookup"><span data-stu-id="e7f48-117">Because the data in the file is static, each client computer can have its own local copy with no implications for data coherency.</span></span>
