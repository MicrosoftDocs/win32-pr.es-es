# <a name="ui-automation-control-patterns-overview"></a><span data-ttu-id="1ffc4-114">Información general acerca de los patrones de control de UI Automation</span><span class="sxs-lookup"><span data-stu-id="1ffc4-114">UI Automation Control Patterns Overview</span></span>

<span data-ttu-id="1ffc4-115">Un *patrón de control* es una implementación de interfaz que expone un aspecto concreto de la funcionalidad de un control a las aplicaciones cliente de automatización de la interfaz de usuario de Microsoft.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-115">A *control pattern* is an interface implementation that exposes a particular aspect of a control's functionality to Microsoft UI Automation client applications.</span></span> <span data-ttu-id="1ffc4-116">Los clientes usan las propiedades y los métodos expuestos a través de un patrón de control para recuperar información sobre una funcionalidad determinada del control o para manipular un aspecto determinado del comportamiento del control.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-116">Clients use the properties and methods exposed through a control pattern to retrieve information about a particular capability of the control, or to manipulate a particular aspect of the control's behavior.</span></span> <span data-ttu-id="1ffc4-117">Por ejemplo, un control que presenta una interfaz tabular utiliza el patrón de control [Grid](uiauto-implementinggrid.md) para exponer el número de filas y columnas de la tabla, y para permitir que un cliente recupere los elementos de la tabla.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-117">For example, a control that presents a tabular interface uses the [Grid](uiauto-implementinggrid.md) control pattern to expose the number of rows and columns in the table, and to enable a client to retrieve items from the table.</span></span>

<span data-ttu-id="1ffc4-118">La automatización de la interfaz de usuario utiliza patrones de control para representar comportamientos de control comunes.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-118">UI Automation uses control patterns to represent common control behaviors.</span></span> <span data-ttu-id="1ffc4-119">Por ejemplo, se usa el patrón de control [Invoke](uiauto-implementinginvoke.md) para los controles que se pueden invocar, como los botones, y el patrón de control [Scroll](uiauto-implementingscroll.md) para los controles que tienen barras de desplazamiento, como cuadros de lista, vistas de lista o cuadros combinados.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-119">For example, you use the [Invoke](uiauto-implementinginvoke.md) control pattern for controls that can be invoked, such as buttons, and the [Scroll](uiauto-implementingscroll.md) control pattern for controls that have scroll bars, such as list boxes, list views, or combo boxes.</span></span> <span data-ttu-id="1ffc4-120">Dado que cada patrón de control representa una funcionalidad independiente, los patrones de control se pueden combinar para describir el conjunto completo de funcionalidad que admite un control determinado.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-120">Because each control pattern represents a separate functionality, control patterns can be combined to describe the full set of functionality supported by a particular control.</span></span>

> [!Note]  
> <span data-ttu-id="1ffc4-121">Un control agregado se crea con controles secundarios que proporcionan la interfaz de usuario para la funcionalidad expuesta por el elemento primario y el elemento primario debe implementar todos los patrones de control que normalmente están asociados a sus controles secundarios.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-121">An aggregate control is built with child controls that provide the user interface for functionality that is exposed by the parent, and the parent should implement all control patterns that are typically associated with its child controls.</span></span> <span data-ttu-id="1ffc4-122">En cambio, no es necesario que los controles secundarios implementen esos mismos patrones de control.</span><span class="sxs-lookup"><span data-stu-id="1ffc4-122">In turn, those same control patterns are not required to be implemented by the child controls.</span></span>
