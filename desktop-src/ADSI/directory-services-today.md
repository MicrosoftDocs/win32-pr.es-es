# <a name="directory-services-today"></a>Servicios de directorio hoy en día

Es habitual encontrar varios directorios administrativos implementados dentro de una sola organización. Estos directorios incluyen directorios de recursos de red, como directorios basados en LDAP, como el servicio de directorio de Microsoft Active Directory, el servicio de directorio del sistema operativo Windows, así como directorios específicos de la aplicación, como Microsoft Exchange.

![implementación de varios directorios](images/ds2chal.png)

## <a name="the-directory-challenge"></a>El desafío de directorio

Varios directorios de la organización suponen desafíos complejos para los usuarios, administradores y desarrolladores. Estos problemas tienen una implementación limitada de directorios anchos. Los usuarios se enfrentan a varios inicios de sesión y una variedad de interfaces para obtener información en varios directorios. Los administradores se enfrentan a la complejidad de administrar varios directorios. Los usuarios finales y administradores quieren que los desarrolladores de aplicaciones usen un directorio administrativo existente, pero los desarrolladores se enfrentan a un dilema sobre cuál usar. Cada directorio ofrece interfaces de aplicación únicas. Un desarrollador debe elegir una implementación de directorio específica o admitir varias versiones de su aplicación. Como resultado, los desarrolladores rara vez usan los servicios de directorio existentes.
