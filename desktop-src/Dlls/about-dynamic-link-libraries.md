# <a name="about-dynamic-link-libraries"></a><span data-ttu-id="48855-103">Acerca de las bibliotecas de Dynamic-Link</span><span class="sxs-lookup"><span data-stu-id="48855-103">About Dynamic-Link Libraries</span></span>

<span data-ttu-id="48855-104">La vinculación dinámica permite a un módulo incluir solo la información necesaria para buscar una función de DLL exportada en tiempo de carga o en tiempo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="48855-104">Dynamic linking allows a module to include only the information needed to locate an exported DLL function at load time or run time.</span></span> <span data-ttu-id="48855-105">La vinculación dinámica difiere de la vinculación estática más familiar, en la que el vinculador copia el código de una función de biblioteca en cada módulo que lo llama.</span><span class="sxs-lookup"><span data-stu-id="48855-105">Dynamic linking differs from the more familiar static linking, in which the linker copies a library function's code into each module that calls it.</span></span>

## <a name="types-of-dynamic-linking"></a><span data-ttu-id="48855-106">Tipos de vinculación dinámica</span><span class="sxs-lookup"><span data-stu-id="48855-106">Types of Dynamic Linking</span></span>

<span data-ttu-id="48855-107">Hay dos métodos para llamar a una función en un archivo DLL:</span><span class="sxs-lookup"><span data-stu-id="48855-107">There are two methods for calling a function in a DLL:</span></span>

-   <span data-ttu-id="48855-108">En la *vinculación dinámica en tiempo de carga*, un módulo realiza llamadas explícitas a las funciones de dll exportadas como si fueran funciones locales.</span><span class="sxs-lookup"><span data-stu-id="48855-108">In *load-time dynamic linking*, a module makes explicit calls to exported DLL functions as if they were local functions.</span></span> <span data-ttu-id="48855-109">Esto requiere vincular el módulo con la biblioteca de importación del archivo DLL que contiene las funciones.</span><span class="sxs-lookup"><span data-stu-id="48855-109">This requires you to link the module with the import library for the DLL that contains the functions.</span></span> <span data-ttu-id="48855-110">Una biblioteca de importación proporciona al sistema la información necesaria para cargar el archivo DLL y ubicar las funciones DLL exportadas cuando se carga la aplicación.</span><span class="sxs-lookup"><span data-stu-id="48855-110">An import library supplies the system with the information needed to load the DLL and locate the exported DLL functions when the application is loaded.</span></span>
-   <span data-ttu-id="48855-111">En la *vinculación dinámica en tiempo de ejecución*, un módulo utiliza la función [**LoadLibrary**](/windows/win32/api/libloaderapi/nf-libloaderapi-loadlibrarya) o [**LoadLibraryEx**](/windows/desktop/api/LibLoaderAPI/nf-libloaderapi-loadlibraryexa) para cargar el archivo dll en tiempo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="48855-111">In *run-time dynamic linking*, a module uses the [**LoadLibrary**](/windows/win32/api/libloaderapi/nf-libloaderapi-loadlibrarya) or [**LoadLibraryEx**](/windows/desktop/api/LibLoaderAPI/nf-libloaderapi-loadlibraryexa) function to load the DLL at run time.</span></span> <span data-ttu-id="48855-112">Una vez cargado el archivo DLL, el módulo llama a la función [**GetProcAddress**](/windows/win32/api/libloaderapi/nf-libloaderapi-getprocaddress) para obtener las direcciones de las funciones DLL exportadas.</span><span class="sxs-lookup"><span data-stu-id="48855-112">After the DLL is loaded, the module calls the [**GetProcAddress**](/windows/win32/api/libloaderapi/nf-libloaderapi-getprocaddress) function to get the addresses of the exported DLL functions.</span></span> <span data-ttu-id="48855-113">El módulo llama a las funciones DLL exportadas con los punteros de función devueltos por **GetProcAddress**.</span><span class="sxs-lookup"><span data-stu-id="48855-113">The module calls the exported DLL functions using the function pointers returned by **GetProcAddress**.</span></span> <span data-ttu-id="48855-114">Esto elimina la necesidad de una biblioteca de importación.</span><span class="sxs-lookup"><span data-stu-id="48855-114">This eliminates the need for an import library.</span></span>

## <a name="dlls-and-memory-management"></a><span data-ttu-id="48855-115">Archivos dll y administración de memoria</span><span class="sxs-lookup"><span data-stu-id="48855-115">DLLs and Memory Management</span></span>

<span data-ttu-id="48855-116">Cada proceso que carga el archivo DLL lo asigna a su espacio de direcciones virtuales.</span><span class="sxs-lookup"><span data-stu-id="48855-116">Every process that loads the DLL maps it into its virtual address space.</span></span> <span data-ttu-id="48855-117">Una vez que el proceso carga el archivo DLL en su dirección virtual, puede llamar a las funciones de DLL exportadas.</span><span class="sxs-lookup"><span data-stu-id="48855-117">After the process loads the DLL into its virtual address, it can call the exported DLL functions.</span></span>

<span data-ttu-id="48855-118">El sistema mantiene un recuento de referencias por proceso para cada archivo DLL.</span><span class="sxs-lookup"><span data-stu-id="48855-118">The system maintains a per-process reference count for each DLL.</span></span> <span data-ttu-id="48855-119">Cuando un subproceso carga el archivo DLL, el recuento de referencias se incrementa en uno.</span><span class="sxs-lookup"><span data-stu-id="48855-119">When a thread loads the DLL, the reference count is incremented by one.</span></span> <span data-ttu-id="48855-120">Cuando finaliza el proceso, o cuando el recuento de referencias se convierte en cero (solo vinculación dinámica en tiempo de ejecución), el archivo DLL se descarga del espacio de direcciones virtuales del proceso.</span><span class="sxs-lookup"><span data-stu-id="48855-120">When the process terminates, or when the reference count becomes zero (run-time dynamic linking only), the DLL is unloaded from the virtual address space of the process.</span></span>

<span data-ttu-id="48855-121">Al igual que cualquier otra función, una función de DLL exportada se ejecuta en el contexto del subproceso que la llama.</span><span class="sxs-lookup"><span data-stu-id="48855-121">Like any other function, an exported DLL function runs in the context of the thread that calls it.</span></span> <span data-ttu-id="48855-122">Por lo tanto, se aplican las condiciones siguientes:</span><span class="sxs-lookup"><span data-stu-id="48855-122">Therefore, the following conditions apply:</span></span>

-   <span data-ttu-id="48855-123">Los subprocesos del proceso que llamó a la DLL pueden usar los identificadores abiertos por una función DLL.</span><span class="sxs-lookup"><span data-stu-id="48855-123">The threads of the process that called the DLL can use handles opened by a DLL function.</span></span> <span data-ttu-id="48855-124">De forma similar, los identificadores abiertos por cualquier subproceso del proceso de llamada se pueden usar en la función DLL.</span><span class="sxs-lookup"><span data-stu-id="48855-124">Similarly, handles opened by any thread of the calling process can be used in the DLL function.</span></span>
-   <span data-ttu-id="48855-125">El archivo DLL utiliza la pila del subproceso que realiza la llamada y el espacio de direcciones virtuales del proceso que realiza la llamada.</span><span class="sxs-lookup"><span data-stu-id="48855-125">The DLL uses the stack of the calling thread and the virtual address space of the calling process.</span></span>
-   <span data-ttu-id="48855-126">El archivo DLL asigna memoria del espacio de direcciones virtuales del proceso que realiza la llamada.</span><span class="sxs-lookup"><span data-stu-id="48855-126">The DLL allocates memory from the virtual address space of the calling process.</span></span>
