## <a name="listeners-and-emitters"></a><span data-ttu-id="2eaa0-106">Agentes de escucha y emisores</span><span class="sxs-lookup"><span data-stu-id="2eaa0-106">Listeners and Emitters</span></span>

-   <span data-ttu-id="2eaa0-119">X3DAudio usa coordenadas de la izquierda.</span><span class="sxs-lookup"><span data-stu-id="2eaa0-119">X3DAudio uses left-handed coordinates.</span></span> <span data-ttu-id="2eaa0-120">Para usar con coordenadas a la derecha, los desarrolladores deben negar el elemento. z de los miembros OrientTop, OrientFront, Position y Velocity de X3DAUDIO \_ Listener y X3DAUDIO \_ emisor.</span><span class="sxs-lookup"><span data-stu-id="2eaa0-120">To use with right-handed coordinates, developers need to negate the .z element of the OrientTop, OrientFront, Position, and Velocity members of X3DAUDIO\_LISTENER and X3DAUDIO\_EMITTER.</span></span>

<span data-ttu-id="2eaa0-121">Además de la posición, los agentes de escucha y los emisores pueden incluir la velocidad.</span><span class="sxs-lookup"><span data-stu-id="2eaa0-121">In addition to position, listeners and emitters can include velocity.</span></span> <span data-ttu-id="2eaa0-122">A diferencia de un motor de representación en 3D, X3DAudio solo usa Velocity para calcular los efectos de Doppler (no se usa para calcular la posición).</span><span class="sxs-lookup"><span data-stu-id="2eaa0-122">Unlike a 3D rendering engine, X3DAudio only uses velocity to calculate Doppler effects (it is not used to calculate position).</span></span>
