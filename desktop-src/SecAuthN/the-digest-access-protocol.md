# <a name="the-digest-access-protocol"></a><span data-ttu-id="2a1cf-103">El protocolo de acceso implícita</span><span class="sxs-lookup"><span data-stu-id="2a1cf-103">The Digest Access Protocol</span></span>

<span data-ttu-id="2a1cf-104">El [*proveedor de compatibilidad para seguridad*](../secgloss/s-gly.md) (SSP) de Microsoft Digest implementa el protocolo de acceso implícita especificado por [RFC 2617](https://www.ietf.org/rfc/rfc2617.txt) .</span><span class="sxs-lookup"><span data-stu-id="2a1cf-104">The Digest Access protocol specified by [RFC 2617](https://www.ietf.org/rfc/rfc2617.txt) is implemented by the Microsoft Digest [*security support provider*](../secgloss/s-gly.md) (SSP).</span></span> <span data-ttu-id="2a1cf-105">La implementación de consta de un conjunto de funciones de contexto de seguridad de la [interfaz del proveedor de compatibilidad para seguridad](sspi.md) (SSPI) de Microsoft a las que llaman las aplicaciones cliente/servidor:</span><span class="sxs-lookup"><span data-stu-id="2a1cf-105">The implementation consists of a set of [Microsoft Security Support Provider Interface](sspi.md) (SSPI) security context functions that client/server applications call to:</span></span>

-   <span data-ttu-id="2a1cf-106">Establecer un [*contexto de seguridad*](../secgloss/s-gly.md) para el intercambio de mensajes.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-106">Establish a [*security context*](../secgloss/s-gly.md) for message exchange.</span></span>
-   <span data-ttu-id="2a1cf-107">Obtener objetos de datos requeridos por el SSP de síntesis, como [*credenciales*](../secgloss/c-gly.md) e identificadores de contexto.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-107">Obtain data objects required by the Digest SSP, such as [*credentials*](../secgloss/c-gly.md) and context handles.</span></span>
-   <span data-ttu-id="2a1cf-108">Mecanismos de confidencialidad y [*integridad*](../secgloss/i-gly.md) del mensaje de acceso.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-108">Access message [*integrity*](../secgloss/i-gly.md) and confidentiality mechanisms.</span></span>

<span data-ttu-id="2a1cf-109">La autenticación de acceso implícita tiene lugar dentro de las transacciones de solicitud/respuesta emparejadas, con las solicitudes que se originan en el cliente y las respuestas que se originan en el servidor.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-109">Digest Access authentication takes place within paired request/response transactions, with requests originating on the client and responses originating on the server.</span></span> <span data-ttu-id="2a1cf-110">Una autenticación de acceso implícita correcta requiere dos pares de solicitud/respuesta.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-110">A successful Digest Access authentication requires two request/response pairs.</span></span>

<span data-ttu-id="2a1cf-111">Cuando el SSP de síntesis se utiliza para la autenticación HTTP, no se mantiene ninguna conexión entre el primer y el segundo par de solicitud-respuesta.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-111">When the Digest SSP is used for HTTP authentication, there is no connection maintained between the first and second request/response pair.</span></span> <span data-ttu-id="2a1cf-112">En otras palabras, el servidor no espera la segunda solicitud después de enviar la primera respuesta.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-112">In other words, the server does not wait for the second request after it sends the first response.</span></span>

<span data-ttu-id="2a1cf-113">En la siguiente ilustración se muestran los pasos realizados en la ruta de acceso HTTP por un cliente y un servidor para completar una autenticación mediante el SSP de síntesis.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-113">The following illustration shows the steps taken on the HTTP path by a client and server to complete an authentication using the Digest SSP.</span></span> <span data-ttu-id="2a1cf-114">El mecanismo SASL hace uso de la autenticación mutua, por lo que los datos de autenticación se devuelven en la llamada final de ASC Server al cliente que comprueba que el cliente se comunica con el servidor correcto.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-114">The SASL mechanism makes use of mutual authentication, so authentication data is sent back on the final ASC server call to the client that verifies that the client is communicating with the correct server.</span></span>

<span data-ttu-id="2a1cf-116">El proceso se inicia con el cliente que solicita un recurso protegido de acceso desde el servidor mediante el envío de la solicitud HTTP 1.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-116">The process starts with the client requesting an access-protected resource from the server by sending HTTP Request 1.</span></span>

<span data-ttu-id="2a1cf-117">El servidor recibe la solicitud HTTP 1 y determina que el recurso requiere información de autenticación que no se incluyó en la solicitud.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-117">The server receives HTTP Request 1 and determines that the resource requires authentication information that was not included in the request.</span></span> <span data-ttu-id="2a1cf-118">El servidor genera un desafío para el cliente de la manera siguiente:</span><span class="sxs-lookup"><span data-stu-id="2a1cf-118">The server generates a challenge for the client as follows:</span></span>

1.  <span data-ttu-id="2a1cf-119">El servidor obtiene sus [*credenciales*](../secgloss/c-gly.md) mediante una llamada a la función [**AcquireCredentialsHandle**](/windows/win32/api/sspi/nf-sspi-acquirecredentialshandlea) .</span><span class="sxs-lookup"><span data-stu-id="2a1cf-119">The server obtains its [*credentials*](../secgloss/c-gly.md) by calling the [**AcquireCredentialsHandle**](/windows/win32/api/sspi/nf-sspi-acquirecredentialshandlea) function.</span></span>
2.  <span data-ttu-id="2a1cf-120">El servidor genera el desafío de síntesis mediante una llamada a la función [**AcceptSecurityContext (general)**](/windows/win32/api/sspi/nf-sspi-acceptsecuritycontext) .</span><span class="sxs-lookup"><span data-stu-id="2a1cf-120">The server generates the Digest challenge by calling the [**AcceptSecurityContext (General)**](/windows/win32/api/sspi/nf-sspi-acceptsecuritycontext) function.</span></span>
3.  <span data-ttu-id="2a1cf-121">El servidor envía un encabezado WWW-Authenticate como respuesta a la solicitud del cliente (se muestra como respuesta HTTP 1).</span><span class="sxs-lookup"><span data-stu-id="2a1cf-121">The server sends a WWW-Authenticate header as its response to the client's request (shown as HTTP Response 1).</span></span> <span data-ttu-id="2a1cf-122">El encabezado contiene el desafío de síntesis y una directiva opaca que contiene una referencia a un [*contexto de seguridad*](../secgloss/s-gly.md) parcial establecido para el cliente.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-122">The header contains the Digest challenge and an opaque directive that contains a reference to a partial [*security context*](../secgloss/s-gly.md) established for the client.</span></span> <span data-ttu-id="2a1cf-123">El encabezado se envía con un código de estado 401 que indica que la solicitud de cliente ha generado un error de acceso no autorizado.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-123">The header is sent with a 401 status code that indicates that the client request generated an unauthorized access error.</span></span> <span data-ttu-id="2a1cf-124">Para obtener más información sobre el desafío de síntesis, consulte [contenido de un desafío de síntesis](contents-of-a-digest-challenge.md) y [generación del desafío de síntesis](generating-the-digest-challenge.md).</span><span class="sxs-lookup"><span data-stu-id="2a1cf-124">For more information about the Digest challenge, see [Contents of a Digest Challenge](contents-of-a-digest-challenge.md) and [Generating the Digest Challenge](generating-the-digest-challenge.md).</span></span>
4.  <span data-ttu-id="2a1cf-125">El cliente recibe la respuesta HTTP 1, extrae el desafío de síntesis enviado por el servidor y genera una respuesta de desafío de síntesis como se indica a continuación:</span><span class="sxs-lookup"><span data-stu-id="2a1cf-125">The client receives HTTP Response 1, extracts the Digest challenge sent by the server, and generates a Digest challenge response as follows:</span></span>
    1.  <span data-ttu-id="2a1cf-126">Las credenciales del usuario se obtienen mediante una llamada a la función [**AcquireCredentialsHandle**](/windows/win32/api/sspi/nf-sspi-acquirecredentialshandlea) o mediante la solicitud interactiva del usuario de las credenciales.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-126">The user's credentials are obtained either by calling the [**AcquireCredentialsHandle**](/windows/win32/api/sspi/nf-sspi-acquirecredentialshandlea) function or by interactively prompting the user for credentials.</span></span>
    2.  <span data-ttu-id="2a1cf-127">La información sobre el desafío y las credenciales se pasa a la función [**InitializeSecurityContext (general)**](/windows/win32/api/sspi/nf-sspi-initializesecuritycontexta) , que genera la respuesta del desafío de síntesis.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-127">The challenge and credentials information are passed to the [**InitializeSecurityContext (General)**](/windows/win32/api/sspi/nf-sspi-initializesecuritycontexta) function, which generates the Digest challenge response.</span></span>
5.  <span data-ttu-id="2a1cf-128">El cliente envía un encabezado de autorización que contiene la respuesta de desafío al servidor (que se muestra como solicitud HTTP 2).</span><span class="sxs-lookup"><span data-stu-id="2a1cf-128">The client sends an Authorization header that contains the challenge response to the server (shown as HTTP Request 2).</span></span> <span data-ttu-id="2a1cf-129">Para obtener más información acerca de la respuesta de desafío de síntesis, consulte [contenido de una respuesta de desafío de síntesis](contents-of-a-digest-challenge-response.md) y [generación de la respuesta de desafío de síntesis](generating-the-digest-challenge-response.md).</span><span class="sxs-lookup"><span data-stu-id="2a1cf-129">For more information about the Digest challenge response, see [Contents of a Digest Challenge Response](contents-of-a-digest-challenge-response.md) and [Generating the Digest Challenge Response](generating-the-digest-challenge-response.md).</span></span>
6.  <span data-ttu-id="2a1cf-130">El servidor recibe la solicitud HTTP 2, extrae la respuesta de desafío enviada por el cliente y autentica la información mediante una llamada a la función [**AcceptSecurityContext (general)**](/windows/win32/api/sspi/nf-sspi-acceptsecuritycontext) .</span><span class="sxs-lookup"><span data-stu-id="2a1cf-130">The server receives HTTP Request 2, extracts the challenge response sent by the client, and authenticates the information by calling the [**AcceptSecurityContext (General)**](/windows/win32/api/sspi/nf-sspi-acceptsecuritycontext) function.</span></span> <span data-ttu-id="2a1cf-131">Para obtener más información sobre el proceso de autenticación, vea [autenticación inicial mediante Microsoft Digest](initial-authentication-using-microsoft-digest.md).</span><span class="sxs-lookup"><span data-stu-id="2a1cf-131">For details about the authentication process, see [Initial Authentication using Microsoft Digest](initial-authentication-using-microsoft-digest.md).</span></span>
7.  <span data-ttu-id="2a1cf-132">El servidor envía la respuesta HTTP 2 al cliente como la segunda y última respuesta requerida por el protocolo de acceso implícita.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-132">The server sends HTTP Response 2 back to the client as the second and final response required by the Digest Access protocol.</span></span> <span data-ttu-id="2a1cf-133">Si la autenticación es correcta, esta respuesta contiene el recurso solicitado.</span><span class="sxs-lookup"><span data-stu-id="2a1cf-133">If the authentication is successful, this response contains the requested resource.</span></span>
