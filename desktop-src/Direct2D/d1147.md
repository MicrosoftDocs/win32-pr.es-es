# <a name="d1147-print-control-closed-without-addpage"></a>D1147: control de impresión cerrado sin AddPage

Print de Direct2D: se llama a Close sin ninguna llamada a AddPage anterior






 

## <a name="possible-causes"></a>Causas posibles

Este mensaje puede producirse si un [**ID2D1PrintControl**](/windows/win32/api/d2d1_1/nn-d2d1_1-id2d1printcontrol) se cierra sin ninguna invocación anterior a [**ID2D1PrintControl:: AddPage**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-addpage). La impresión en Direct2D considera que un control de impresión sin páginas es erróneo y el comando [**ID2D1PrintControl:: Close**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-close) correspondiente devolverá un error.

## <a name="possible-fixes"></a>Posibles correcciones

Para direccionar el mensaje, agregue una página al control de impresión antes de cerrarlo. Si la aplicación no necesita imprimir ningún contenido, abstenerse de cerrar el control de impresión o crearlo.
