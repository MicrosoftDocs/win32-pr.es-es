# <a name="peer-to-peer"></a>Punto a punto

## <a name="purpose"></a>Propósito

Las tecnologías punto a punto se usan para facilitar la comunicación en tiempo real y la colaboración en redes distribuidas.

En el modelo punto a punto, sin usar servidores de Internet, cada usuario de equipo puede hacer lo siguiente:

-   Datos de Exchange
-   Compartir recursos
-   Buscar otros usuarios
-   Comunicarse
-   Colaborar directamente en tiempo real

Mediante el uso de tecnologías punto a punto, las aplicaciones que coordinan el uso de ciclos y almacenamiento de la CPU del equipo pueden compartir recursos entre grupos pequeños o grandes de equipos conectados a Internet.

## <a name="where-applicable"></a>Donde sea aplicable

Los desarrolladores pueden usar la infraestructura del mismo nivel para crear una amplia gama de aplicaciones distribuidas, ad hoc y de punto a punto.

## <a name="developer-audience"></a>Audiencia de desarrolladores

Los desarrolladores que usan la infraestructura del mismo nivel deben estar familiarizados con los conceptos de programación de C. Los desarrolladores que usan el proveedor de espacios de nombres de Winsock de PNRP deben estar familiarizados con la API de Winsock.

## <a name="run-time-requirements"></a>Requisitos de tiempo de ejecución

La infraestructura del mismo nivel es compatible con Windows Vista, Windows XP con Service Pack 2 (SP2) y versiones posteriores, así como el paquete de redes avanzadas para Windows XP disponible para Windows XP con Service Pack 1 (SP1). La infraestructura punto a punto requiere que IPv6 se instale y se inicie para permitir que funcionen las aplicaciones de redes del mismo nivel. El uso de la colaboración punto a punto solo se admite en Windows Vista.
