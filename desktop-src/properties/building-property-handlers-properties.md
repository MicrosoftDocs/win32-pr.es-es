## <a name="legacy-technology"></a><span data-ttu-id="dfcbb-148">Tecnología heredada</span><span class="sxs-lookup"><span data-stu-id="dfcbb-148">Legacy Technology</span></span>

<span data-ttu-id="dfcbb-149">Microsoft Windows tecnología de flujo secundario del sistema de archivos NT (NTFS) se desarrolló para admitir la persistencia de información adicional con el archivo a través de un flujo alternativo establecido en la capa del sistema de archivos.</span><span class="sxs-lookup"><span data-stu-id="dfcbb-149">Microsoft Windows NT File System (NTFS) secondary stream technology was developed to support the persistence of extra information with the file through an alternate stream set at the file system layer.</span></span> <span data-ttu-id="dfcbb-150">Es posible que se pregunte por qué esas secuencias secundarias no se usan como método principal para almacenar propiedades, especialmente con compatibilidad con metadatos abiertos.</span><span class="sxs-lookup"><span data-stu-id="dfcbb-150">One might wonder why those secondary streams are not used as the main method to store properties, especially with open metadata support.</span></span> <span data-ttu-id="dfcbb-151">La razón principal es la transportabilidad de esta información adicional.</span><span class="sxs-lookup"><span data-stu-id="dfcbb-151">The primary reason is the transportability of this extra information.</span></span> <span data-ttu-id="dfcbb-152">Desafortunadamente, estas secuencias alternativas se quitan en muchos escenarios, incluida la compatibilidad con el almacenamiento en caché del lado cliente (CSC), el envío de archivos como datos adjuntos y la copia de archivos a un almacén que no es NTFS.</span><span class="sxs-lookup"><span data-stu-id="dfcbb-152">Unfortunately, these alternate streams are removed in many scenarios including client-side caching support (CSC), sending files as attachments, and copying files to a non-NTFS store.</span></span>
