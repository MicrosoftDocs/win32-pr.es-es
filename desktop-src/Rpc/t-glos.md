<dl> <dt>

<span id="_rpc_top_level_pointer_glos"></span><span id="_RPC_TOP_LEVEL_POINTER_GLOS"></span>**puntero de nivel superior**
</dt> <dd>

Puntero que se especifica como el nombre de un parámetro en un prototipo de función. Vea también [*puntero incrustado*](e-glos.md).

</dd> <dt>

<span id="_rpc_tcp_glos"></span><span id="_RPC_TCP_GLOS"></span>**Protocolo de control de transmisión (TCP)**
</dt> <dd>

Transporte de red [*orientado a la conexión*](c-glos.md) en la parte superior del Protocolo de Internet ( [*IP)*](i-glos.md).

</dd> <dt>

<span id="_rpc_tunneling_glos"></span><span id="_RPC_TUNNELING_GLOS"></span>**tunelización**
</dt> <dd>

Protocolo TCP/IP para la transmisión de datos desde una aplicación de servidor de contenido a un enrutador de difusión.

</dd> <dt>

<span id="_rpc_type_serialization_glos"></span><span id="_RPC_TYPE_SERIALIZATION_GLOS"></span>**serialización de tipo**
</dt> <dd>

[*Serialización*](s-glos.md) de datos que utiliza rutinas generadas por MIDL para ajustar el tamaño, codificar y descodificar objetos de un tipo especificado. La aplicación cliente llama a estas rutinas para serializar los datos. La serialización de tipos se logra aplicando los atributos de \[ [codificación](/windows/desktop/Midl/encode) \] y \[ [descodificación](/windows/desktop/Midl/decode) \] a un solo tipo de datos, o a una interfaz, en el archivo ACF. Vea también [*procedimientos de serialización*](p-glos.md).

</dd> </dl>
