# <a name="d1155-fillopacitymask-requires-aliased-rendering"></a>D1155: FillOpacityMask requiere la representación con alias

FillOpacityMask requiere que el modo antialias se establezca en el \_ modo ANTIALIAS D2D1 con \_ \_ alias.



|             |             |
|-------------|-------------|
| Nivel de error | Información |



 

## <a name="possible-causes"></a>Causas posibles

Cuando una aplicación llama a ID2D1RenderTarget:: FillOpacityMask, el modo antialias de rendertarget implicado debe ser D2D1 con \_ alias del modo antialias \_ \_ . El comportamiento predeterminado de un destino de representación es usar el suavizado de contorno, por lo que es necesario llamar a ID2D1RenderTarget:: SetAntialiasMode antes de rellenar una máscara de opacidad.

## <a name="possible-fixes"></a>Posibles correcciones

Use ID2D1RenderTarget:: SetAntialiasMode para asegurarse de que rendertarget usa la representación con alias al rellenar una máscara de opacidad. Si las operaciones subsiguientes requieren suavizado de contorno, vuelva a establecer el modo antialias del destino en el \_ modo D2D1 antialias \_ \_ por \_ primitiva. Asegúrese de que las llamadas a ID2D1RenderTarget:: SetAntialiasMode y ID2D1RenderTarget:: FillOpacityMask se emitan en el mismo destino.
