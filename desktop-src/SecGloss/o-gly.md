<span id="_security_object_identifier_gly"></span><span id="_SECURITY_OBJECT_IDENTIFIER_GLY"></span>**identificador de objeto**
</dt> <dd>

OID Número que identifica de forma única una clase de objeto o un atributo. Un identificador de objeto se representa como una cadena decimal con puntos, como 1.2.3.4. Los identificadores de objetos se organizan en una jerarquía global. Las entidades de registro nacionales expiden identificadores de objeto raíz a personas u organizaciones que administran la jerarquía por debajo de su identificador de objeto raíz.

Vea también [*notación de sintaxis abstracta 1*](a-gly.md), [*atributo*](a-gly.md).

</dd> <dt>

<span id="_security_oid_gly"></span><span id="_SECURITY_OID_GLY"></span>**OID**

<span id="_security_online_certificate_status_protocol_gly"></span><span id="_SECURITY_ONLINE_CERTIFICATE_STATUS_PROTOCOL_GLY"></span>**Protocolo de estado de certificados en línea**
</dt> <dd>

SIGAN Protocolo de Internet que se usa para obtener el estado de revocación de un certificado digital X. 509.

</dd> <dt>

<span id="_security_opaque_blob_gly"></span><span id="_SECURITY_OPAQUE_BLOB_GLY"></span>**BLOB opaco**
</dt> <dd>

Secuencia de bytes que se usa para almacenar las claves de sesión. Los blobs opacos contienen el material de clave base y toda la información de estado actual. Esto incluye información como el valor Salt, el vector de inicialización y la tabla de claves. El formato de los blobs opacos no se publica. Cada proveedor de CSP determina su propio formato de BLOB, que debe incluir el cifrado de los blobs opacos con algún tipo de clave simétrica.

</dd> <dt>

<span id="_security_outer_content_gly"></span><span id="_SECURITY_OUTER_CONTENT_GLY"></span>**contenido externo**
</dt> <dd>

Mejoras en algunos datos encapsulados. Este término se usa principalmente cuando se analizan datos mejorados (el contenido interno) en \# mensajes PKCS 7.

</dd> <dt>

<span id="_security_output_feedback_gly"></span><span id="_SECURITY_OUTPUT_FEEDBACK_GLY"></span>**Comentarios de salida**
</dt> <dd>

(OFB) Modo de cifrado de bloques que usa comentarios similares al modo de comentarios de cifrado (CFB). La única diferencia entre los dos modos es cómo se rellena el registro de desplazamiento.

Vea también [*comentarios de cifrado*](c-gly.md).

</dd> </dl>
