# <a name="characteristics-of-attributes"></a>Características de los atributos

Cada atributo de un objeto de Active Directory Domain Services se define mediante un objeto [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) en el contenedor de esquemas. Las propiedades de un objeto **attributeSchema** especifican las características del atributo, como:

-   Identificadores de atributo. Los atributos tienen varios identificadores, de los cuales el más interesante de la perspectiva de programación es el [**lDAPDisplayName**](/windows/desktop/ADSchema/a-ldapdisplayname), que usan los clientes LDAP para leer y escribir el atributo, y el [**schemaIDGUID**](/windows/desktop/ADSchema/a-schemaidguid), que se usa en los descriptores de seguridad para controlar el acceso al atributo.
-   El tipo de datos que contienen las instancias del atributo. Las propiedades de sintaxis de un atributo determinan el tipo de datos, como un entero, una cadena o un archivo binario. Las propiedades adicionales pueden especificar el intervalo de valores que se permiten para el atributo y si una instancia del atributo puede tener varios valores.
-   Incluido el atributo en grupos. Otras propiedades etiquetan un atributo que se va a incluir en un conjunto de propiedades, que es un conjunto de propiedades relacionadas, o que se va a incluir en el conjunto de atributos que se replican en el catálogo global o se indizan para optimizar el rendimiento de la búsqueda.

En la tabla siguiente se enumeran los [**lDAPDisplayName**](/windows/desktop/ADSchema/a-ldapdisplayname) y la descripción de las propiedades de clave de un objeto [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) . Para obtener más información y una lista completa de las propiedades obligatorias y opcionales de un objeto **attributeSchema** , vea [**clase de esquema de atributo**](/windows/desktop/ADSchema/c-attributeschema).
