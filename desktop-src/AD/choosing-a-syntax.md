# <a name="choosing-a-syntax"></a>Elección de una sintaxis

Hay 23 sintaxis definidas en Active Directory Domain Services. Este tema contiene una lista de las sintaxis recomendadas que se deben usar al definir un nuevo atributo. Para obtener más información, vea [Sintaxis de atributos en Active Directory Domain Services](syntaxes-for-attributes-in-active-directory-domain-services.md).

En la tabla siguiente se proporciona una lista de recomendaciones.



| Datos para almacenar en el atributo      | Sintaxis que se va a usar                                                      | Comentario                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|---------------------------------|--------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Datos binarios                     | [**String(Octet)**](/windows/desktop/ADSchema/s-string-octet)                       | Se utiliza para almacenar datos binarios. Se trata de una matriz de bytes.                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Datos binarios con una referencia DN | [**Object(DN-Binary)**](/windows/desktop/ADSchema/s-object-dn-binary)               | Contiene un valor binario y un nombre distintivo (DN). El servidor de Active Directory mantiene el DN actualizado.                                                                                                                                                                                                                                                                                                                                                                                                     |
| Boolean                         | [**Booleano**](/windows/desktop/ADSchema/s-boolean)                                  | Se usa para los valores booleanos.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Referencia de DN                    | [**Object(DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn)                       | Use para almacenar nombres distintivos que desee mantener actualizados por el servidor de Active Directory. Cuando se crea un atributo de sintaxis DN con un DN válido, el servidor trata el atributo como una referencia al objeto representado por el DN que se estableció. Si se cambia el nombre del objeto al que se hace referencia o se mueve, el servidor se asegura de que el atributo refleje el cambio. Si el atributo se restablece con un nuevo DN, el atributo es una referencia al objeto representado por el nuevo DN.                                |
| Entero                         | [**Entero**](/windows/desktop/ADSchema/s-integer)                                  | Se utiliza para los enteros.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Entero grande (valores de 64 bits)   | [**LargeInteger**](/windows/desktop/ADSchema/s-largeinteger)                        | Se usa para los valores de 64 bits.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| DN vinculado                       | [**Object(DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn)                       | Esta sintaxis de cadena se puede usar para DNs vinculado. Los vínculos hacia atrás deben ser de DN de sintaxis. Los vínculos hacia delante pueden tener el DN de la sintaxis, el objeto ( [**DN-cadena)**](/windows/desktop/ADSchema/s-object-dn-string), el [**objeto (DN-binario)**](/windows/desktop/ADSchema/s-object-dn-binary), el [**objeto (punto de acceso)**](/windows/desktop/ADSchema/s-object-access-point)o [**el objeto (o-Name)**](/windows/desktop/ADSchema/s-object-or-name). Los atributos vinculados deben tener un **LinkId** definido. Vea la descripción de **LinkId** en propiedades [**de esquema de atributo**](/windows/desktop/ADSchema/c-attributeschema) . |
| Descriptor de seguridad             | [**String(NT-Sec-Desc)**](/windows/desktop/ADSchema/s-string-nt-sec-desc)           | Cadena de octetos que contiene un descriptor de seguridad.                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Identificador de seguridad (SID)       | [**Cadena (SID)**](/windows/desktop/ADSchema/s-string-sid)                           | Cadena de octetos que contiene un identificador de seguridad (SID). Use esta sintaxis para almacenar solo valores de SID.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| String                          | [**String(Unicode)**](/windows/desktop/ADSchema/s-string-unicode)                   | Se usa para la mayoría de los atributos de cadena. Admite el juego de caracteres Unicode. Cuando el servidor de Active Directory realiza comparaciones con los atributos de esta sintaxis (como la evaluación de una consulta), realiza comparaciones que no distinguen mayúsculas de minúsculas. Use las demás sintaxis de cadena ([**cadena (IA5)**](/windows/desktop/ADSchema/s-string-ia5), [**cadena (numérica)**](/windows/desktop/ADSchema/s-string-numeric), etc.) para almacenar cadenas que solo deben contener los juegos de caracteres específicos admitidos por la sintaxis.<br/>                          |
| Datos de cadena con una referencia DN | [**Object(DN-String)**](/windows/desktop/ADSchema/s-object-dn-string)               | Cadena que contiene un valor de cadena y un nombre distintivo (DN). El servidor de Active Directory mantiene el DN actualizado.                                                                                                                                                                                                                                                                                                                                                                                            |
| Hora                            | [**String(Generalized-Time)**](/windows/desktop/ADSchema/s-string-generalized-time) | Use la sintaxis de [**cadena (en tiempo generalizado)**](/windows/desktop/ADSchema/s-string-generalized-time) para almacenar valores de hora en lugar de la sintaxis de [**cadena (hora UTC)**](/windows/desktop/ADSchema/s-string-utc-time) , ya que la **cadena (en tiempo generalizado)** utiliza cuatro caracteres para el año y la **cadena (hora UTC)** solo usa dos.                                                                                                                                                                                                                 |
