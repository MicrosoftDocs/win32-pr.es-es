# <a name="the-vss-model"></a><span data-ttu-id="6dc39-103">El modelo VSS</span><span class="sxs-lookup"><span data-stu-id="6dc39-103">The VSS Model</span></span>

<span data-ttu-id="6dc39-104">VSS está diseñado para solucionar los problemas descritos en los [problemas comunes de copia de seguridad de volumen](common-volume-backup-issues.md).</span><span class="sxs-lookup"><span data-stu-id="6dc39-104">VSS is designed to address the problems described in [Common Volume Backup Issues](common-volume-backup-issues.md).</span></span>

<span data-ttu-id="6dc39-105">El modelo de VSS incluye lo siguiente:</span><span class="sxs-lookup"><span data-stu-id="6dc39-105">The VSS model includes the following:</span></span>

-   <span data-ttu-id="6dc39-106">Mecanismo de instantáneas.</span><span class="sxs-lookup"><span data-stu-id="6dc39-106">The shadow copy mechanism.</span></span> <span data-ttu-id="6dc39-107">VSS proporciona una captura de volumen rápida del estado de un disco en un instante en el tiempo, una [*instantánea*](vssgloss-s.md) del volumen.</span><span class="sxs-lookup"><span data-stu-id="6dc39-107">VSS provides fast volume capture of the state of a disk at one instant in time—a [*shadow copy*](vssgloss-s.md) of the volume.</span></span> <span data-ttu-id="6dc39-108">Esta copia de volumen existe en paralelo con el volumen activo y contiene copias de todos los archivos del disco que se guardan de forma eficaz y están disponibles como un dispositivo independiente.</span><span class="sxs-lookup"><span data-stu-id="6dc39-108">This volume copy exists side by side with the live volume, and contains copies of all files on disk effectively saved and available as a separate device.</span></span>
-   <span data-ttu-id="6dc39-109">Estado de archivo coherente mediante la coordinación de aplicaciones.</span><span class="sxs-lookup"><span data-stu-id="6dc39-109">Consistent file state via application coordination.</span></span> <span data-ttu-id="6dc39-110">VSS proporciona un mecanismo de comunicación entre procesos basado en COM y orientado a eventos que los procesos participantes pueden utilizar para determinar el estado del sistema con respecto a las operaciones de copia de seguridad, restauración y instantáneas (captura de volumen).</span><span class="sxs-lookup"><span data-stu-id="6dc39-110">VSS provides a COM-based, event-driven interprocess communication mechanism that participating processes can use to determine system state with respect to backup, restore, and shadow copy (volume capture) operations.</span></span> <span data-ttu-id="6dc39-111">Estos eventos definen las fases por las que las aplicaciones que modifican los datos en disco ([*escritores*](vssgloss-w.md)) pueden poner todos sus archivos en un estado coherente antes de la creación de la instantánea.</span><span class="sxs-lookup"><span data-stu-id="6dc39-111">These events define stages by which applications modifying data on disk ([*writers*](vssgloss-w.md)) can bring all their files into a consistent state prior to the creation of the shadow copy.</span></span>
-   <span data-ttu-id="6dc39-112">Minimizar el tiempo de inactividad de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="6dc39-112">Minimizing application downtime.</span></span> <span data-ttu-id="6dc39-113">La instantánea de VSS existe en paralelo con una copia en directo del volumen del que se va a hacer una copia de seguridad, por lo que, salvo el breve período de preparación y creación de la instantánea, una aplicación puede continuar su trabajo.</span><span class="sxs-lookup"><span data-stu-id="6dc39-113">The VSS shadow copy exists in parallel with a live copy of the volume to be backed up, so except for the brief period of the shadow copy's preparation and creation, an application can continue its work.</span></span> <span data-ttu-id="6dc39-114">El tiempo necesario para crear realmente una instantánea, que se produce entre los eventos [*Freeze*](vssgloss-f.md) y [*reanudar*](vssgloss-t.md) , normalmente tarda aproximadamente un minuto.</span><span class="sxs-lookup"><span data-stu-id="6dc39-114">The time needed to actually create a shadow copy, which occurs between [*Freeze*](vssgloss-f.md) and [*Thaw*](vssgloss-t.md) events, typically takes about one minute.</span></span>

    <span data-ttu-id="6dc39-115">Aunque la preparación de un escritor para una instantánea, incluido el estado de almacenamiento e/s y guardado (consulte [información general de las tareas anteriores a la copia de seguridad](overview-of-pre-backup-tasks.md)), puede no ser trivial, es mucho menor que el tiempo necesario para realizar realmente una copia de seguridad de un volumen, lo que puede tardar horas.</span><span class="sxs-lookup"><span data-stu-id="6dc39-115">While a writer's preparation for a shadow copy, including flushing I/O and saving state (see [Overview of Pre-Backup Tasks](overview-of-pre-backup-tasks.md)), may be nontrivial, it is significantly shorter than the time required to actually back up a volume—which for large volumes may take hours.</span></span>

-   <span data-ttu-id="6dc39-116">Interfaz unificada para VSS.</span><span class="sxs-lookup"><span data-stu-id="6dc39-116">Unified interface to VSS.</span></span> <span data-ttu-id="6dc39-117">VSS abstrae los mecanismos de instantáneas dentro de una interfaz común y permite a un proveedor de hardware agregar y administrar las características únicas de sus propios proveedores.</span><span class="sxs-lookup"><span data-stu-id="6dc39-117">VSS abstracts the shadow copy mechanisms within a common interface while enabling a hardware vendor to add and manage the unique features of its own providers.</span></span> <span data-ttu-id="6dc39-118">Cualquier aplicación de copia de seguridad ([*solicitante*](vssgloss-r.md)) y cualquier escritor debe poder ejecutarse en cualquier sistema de almacenamiento en disco que admita la interfaz de VSS.</span><span class="sxs-lookup"><span data-stu-id="6dc39-118">Any backup application ([*requester*](vssgloss-r.md)) and any writer should be able to run on any disk storage system that supports the VSS interface.</span></span>
-   <span data-ttu-id="6dc39-119">Copia de seguridad multivolumen.</span><span class="sxs-lookup"><span data-stu-id="6dc39-119">Multivolume backup.</span></span> <span data-ttu-id="6dc39-120">VSS admite [*conjuntos de instantáneas*](vssgloss-s.md), que son colecciones de instantáneas, en varios tipos de volúmenes de disco de varios proveedores.</span><span class="sxs-lookup"><span data-stu-id="6dc39-120">VSS supports [*shadow copy sets*](vssgloss-s.md), which are collections of shadow copies, across multiple types of disk volumes from multiple vendors.</span></span> <span data-ttu-id="6dc39-121">Todas las instantáneas de un conjunto de instantáneas se crearán con la misma marca de tiempo y presentarán el mismo estado de disco para un estado de disco multivolumen.</span><span class="sxs-lookup"><span data-stu-id="6dc39-121">All shadow copies in a shadow copy set will be created with the same time stamp and will present the same disk state for a multivolume disk state.</span></span>
-   <span data-ttu-id="6dc39-122">Compatibilidad con instantáneas nativas.</span><span class="sxs-lookup"><span data-stu-id="6dc39-122">Native shadow copy support.</span></span> <span data-ttu-id="6dc39-123">A partir de Windows XP, la compatibilidad con la instantánea está disponible a través de VSS como parte nativa del sistema operativo Windows.</span><span class="sxs-lookup"><span data-stu-id="6dc39-123">Beginning with Windows XP, shadow copy support is available through VSS as a native part of the Windows operating system.</span></span> <span data-ttu-id="6dc39-124">Siempre que haya al menos un disco NTFS presente en un sistema, estos sistemas se pueden configurar para admitir instantáneas de todos los sistemas de disco montados en ellos.</span><span class="sxs-lookup"><span data-stu-id="6dc39-124">As long as at least one NTFS disk is present on a system, these systems can be configured to support shadow copies of all disk systems mounted on them.</span></span>
