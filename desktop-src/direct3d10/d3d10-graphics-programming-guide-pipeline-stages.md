-   <span data-ttu-id="6d36b-113">[Fase del sombreador de vértices](https://www.bing.com/search?q=Vertex-Shader+Stage). -   <span data-ttu-id="6d36b-115">[Fase de sombreador de geometría](https://www.bing.com/search?q=Geometry-Shader+Stage) : la fase de sombreador de geometría procesa los primitivos completos.</span><span class="sxs-lookup"><span data-stu-id="6d36b-115">[Geometry-Shader Stage](https://www.bing.com/search?q=Geometry-Shader+Stage) - The geometry-shader stage processes entire primitives.</span></span> <span data-ttu-id="6d36b-116">Su entrada es un primitivo completo (que es tres vértices para un triángulo, dos vértices para una línea o un solo vértice para un punto).</span><span class="sxs-lookup"><span data-stu-id="6d36b-116">Its input is a full primitive (which is three vertices for a triangle, two vertices for a line, or a single vertex for a point).</span></span> <span data-ttu-id="6d36b-117">Además, cada primitiva también puede incluir los datos de vértices para cualquier primitivo adyacente perimetral.</span><span class="sxs-lookup"><span data-stu-id="6d36b-117">In addition, each primitive can also include the vertex data for any edge-adjacent primitives.</span></span> <span data-ttu-id="6d36b-118">Esto podría incluir como máximo tres vértices adicionales para un triángulo o dos vértices adicionales para una línea.</span><span class="sxs-lookup"><span data-stu-id="6d36b-118">This could include at most an additional three vertices for a triangle or an additional two vertices for a line.</span></span> <span data-ttu-id="6d36b-119">El sombreador de geometría también admite amplificación y desamplificación de geometría limitada.</span><span class="sxs-lookup"><span data-stu-id="6d36b-119">The Geometry Shader also supports limited geometry amplification and de-amplification.</span></span> <span data-ttu-id="6d36b-120">Dado un primitivo de entrada, el sombreador de geometría puede descartar la primitiva o emitir una o más primitivas nuevas.</span><span class="sxs-lookup"><span data-stu-id="6d36b-120">Given an input primitive, the Geometry Shader can discard the primitive, or emit one or more new primitives.</span></span>
-   <span data-ttu-id="6d36b-121">[Fase de flujo de salida](../direct3d11/d3d10-graphics-programming-guide-output-stream-stage.md) : la fase de salida de flujo está diseñada para el streaming de datos primitivos de la canalización a la memoria en su camino al rasterizador.</span><span class="sxs-lookup"><span data-stu-id="6d36b-121">[Stream-Output Stage](../direct3d11/d3d10-graphics-programming-guide-output-stream-stage.md) - The stream-output stage is designed for streaming primitive data from the pipeline to memory on its way to the rasterizer.</span></span> <span data-ttu-id="6d36b-122">Los datos pueden transmitirse o pasarse al rasterizador.</span><span class="sxs-lookup"><span data-stu-id="6d36b-122">Data can be streamed out and/or passed into the rasterizer.</span></span> <span data-ttu-id="6d36b-123">Los datos trasmitidos a la memoria pueden regresar a la canalización como datos de entrada o volver a leerse en la CPU.</span><span class="sxs-lookup"><span data-stu-id="6d36b-123">Data streamed out to memory can be recirculated back into the pipeline as input data or read-back from the CPU.</span></span>
-   <span data-ttu-id="6d36b-124">[Fase de rasterización](../direct3d11/d3d10-graphics-programming-guide-rasterizer-stage.md) : el rasterizador es responsable de recortar los primitivos, preparar los primitivos para el sombreador de píxeles y determinar cómo invocar los sombreadores de píxeles.</span><span class="sxs-lookup"><span data-stu-id="6d36b-124">[Rasterizer Stage](../direct3d11/d3d10-graphics-programming-guide-rasterizer-stage.md) - The rasterizer is responsible for clipping primitives, preparing primitives for the pixel shader and determining how to invoke pixel shaders.</span></span>
-   <span data-ttu-id="6d36b-125">[Fase del sombreador de píxeles](https://www.bing.com/search?q=Pixel-Shader+Stage): La fase del sombreador de píxeles recibe datos interpolados para un primitivo y genera datos por píxel, como el color.</span><span class="sxs-lookup"><span data-stu-id="6d36b-125">[Pixel-Shader Stage](https://www.bing.com/search?q=Pixel-Shader+Stage) - The pixel-shader stage receives interpolated data for a primitive and generates per-pixel data such as color.</span></span>
-   <span data-ttu-id="6d36b-126">[Fase de combinación de salida](../direct3d11/d3d10-graphics-programming-guide-output-merger-stage.md) : la fase de combinación de salida es responsable de combinar varios tipos de datos de salida (valores del sombreador de píxeles, información de profundidad y de estarcido) con el contenido de los búferes de destino de representación y de profundidad y estarcido para generar el resultado final de la canalización.</span><span class="sxs-lookup"><span data-stu-id="6d36b-126">[Output-Merger Stage](../direct3d11/d3d10-graphics-programming-guide-output-merger-stage.md) - The output-merger stage is responsible for combining various types of output data (pixel shader values, depth and stencil information) with the contents of the render target and depth/stencil buffers to generate the final pipeline result.</span></span>
