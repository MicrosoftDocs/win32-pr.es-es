# <a name="s-upnp-apis"></a>S (API de UPnP)

Contiene términos relacionados con UPnP que comienzan por la letra S.

[A](a-gly.md) B [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F G H [I](h-gly.md) J K L M [N O](n-gly.md) P [P](p-gly.md) [Q R](r-gly.md) S T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.s_1_gly"></span><span id="UPNP.S_1_GLY"></span>**Servicio**
</dt> <dd>

Un servicio es una unidad funcional lógica. Los servicios exponen acciones y modela algunos o todos los estados de un dispositivo físico mediante variables de estado.

</dd> <dt>

<span id="upnp.s_2_gly"></span><span id="UPNP.S_2_GLY"></span>**descripción del servicio**
</dt> <dd>

Una descripción del servicio es una lista de las acciones que proporciona un servicio y las variables de estado asociadas a las acciones.

</dd> <dt>

<span id="upnp.s_3_gly"></span><span id="UPNP.S_3_GLY"></span>**objeto de servicio**
</dt> <dd>

Un objeto de servicio es la representación de la API de punto de control de un servicio. Un objeto de servicio implementa la [**interfaz IUPnPService.**](/windows/desktop/api/Upnp/nn-upnp-iupnpservice)

</dd> <dt>

<span id="upnp.s_4_gly"></span><span id="UPNP.S_4_GLY"></span>**tipo de servicio**
</dt> <dd>

Un tipo de servicio es un URN que identifica un servicio. Hay una relación uno a uno entre una plantilla de servicio UPnP y un tipo de servicio. Los comités de trabajo del foro de UPnP definen varios tipos de servicio estándar. Los tipos de servicio tienen el formato urn:schemas-upnp-org:service:serviceType:version. Por ejemplo, un tipo de servicio de escáner podría ser urn:schemas-upnp-org:service:scanner:1.

Los proveedores de UPnP pueden especificar servicios adicionales. Estos servicios se indican mediante urn:domain-name:service:serviceType:version, donde domain-name es un nombre de dominio registrado en el proveedor.

</dd> <dt>

<span id="upnp.s_5_gly"></span><span id="UPNP.S_5_GLY"></span>**state variable**
</dt> <dd>

Una variable de estado es un fragmento de datos que describe alguna parte del estado de un servicio.

</dd> </dl>
