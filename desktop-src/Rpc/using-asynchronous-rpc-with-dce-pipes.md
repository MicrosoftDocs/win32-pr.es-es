# <a name="using-asynchronous-rpc-with-dce-pipes"></a>Usar RPC asincrónico con canalizaciones de DCE

Microsoft RPC admite el uso de canalizaciones de DCE. Aunque comparten un nombre similar, las canalizaciones de DCE no están relacionadas con las [canalizaciones con nombre](asynchronous-rpc-over-the-named-pipe-protocol.md). Las canalizaciones con nombre son un protocolo de transporte. Las canalizaciones de DCE son un método independiente del Protocolo de comunicación entre cliente y servidor.

En esta sección se presenta una descripción de RPC mediante canalizaciones asincrónicas de DCE. Se divide en los siguientes temas:

-   [Canalizaciones asincrónicas](asynchronous-pipes.md)
-   [Declarar canalizaciones asincrónicas](declaring-asynchronous-pipes.md)
-   [Control de canalizaciones asincrónicas en el cliente](client-side-asynchronous-pipe-handling.md)
-   [Control de canalizaciones asincrónicas en el servidor](server-side-asynchronous-pipe-handling.md)

Para obtener una aplicación RPC de canalización asincrónica de ejemplo, vea el ejemplo FileRep en el kit de desarrollo de software (SDK) de la plataforma.
