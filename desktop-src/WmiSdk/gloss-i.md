# <a name="i-wmi"></a>I (WMI)

<dl> <dt>

<span id="wmi.gloss_impersonation"></span><span id="WMI.GLOSS_IMPERSONATION"></span>**suplantación**
</dt> <dd>

La capacidad de un subproceso en un proceso de asumir la identidad de seguridad y los privilegios de otro proceso.

</dd> <dt>

<span id="wmi.gloss_instance_provider"></span><span id="WMI.GLOSS_INSTANCE_PROVIDER"></span>**proveedor de instancias**
</dt> <dd>

Un servidor COM que proporciona instancias de clases e implementa métodos de la interfaz [**IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) para admitir la recuperación de datos, la modificación, la eliminación, la enumeración o el procesamiento de consultas.

</dd> <dt>

<span id="wmi.gloss_intrinsic_event"></span><span id="WMI.GLOSS_INTRINSIC_EVENT"></span>**evento intrínseco**
</dt> <dd>

Evento que proporciona WMI, que es una notificación de un cambio en el modelo de datos WMI estándar. Por ejemplo, la creación de una nueva instancia del [**\_ proceso de Win32**](/windows/desktop/CIMWin32Prov/win32-process) es un tipo [**\_ \_ InstanceCreationEvent**](--instancecreationevent.md) de evento intrínseco. Vea también [*eventos extrínsecos*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_instrumentation"></span><span id="WMI.GLOSS_INSTRUMENTATION"></span>**instrumentación**
</dt> <dd>

Mecanismo utilizado para notificar el estado del hardware y el software del sistema. WMI implementa la instrumentación mediante el modelado de un sistema mediante clases basadas en el [*modelo de información común (CIM)*](gloss-c.md). WMI en el .NET Framework crea clases instrumentadas agregando atributos.

</dd> </dl>
