<dl> <dt>

<span id="_rras_aged_out_gly"></span><span id="_RRAS_AGED_OUT_GLY"></span>**antigüedad**
</dt> <dd>

Además, se ha *quedado anticuado en la memoria caché*. En RRAS, una ruta caduca cuando se quita de la tabla de enrutamiento después de un período de tiempo especificado.

</dd> <dt>

<span id="_rras_authentication_gly"></span><span id="_RRAS_AUTHENTICATION_GLY"></span>**autenticación**
</dt> <dd>

La autenticación es el proceso de comprobar que un usuario es quien dice ser. Normalmente, la autenticación se realiza mediante un nombre de usuario y una contraseña.

</dd> <dt>

<span id="_rras_autonomous_system_gly"></span><span id="_RRAS_AUTONOMOUS_SYSTEM_GLY"></span>**sistema autónomo (AS)**
</dt> <dd>

Un sistema autónomo es un grupo de enrutadores que comparten una única directiva administrativa. Todos estos enrutadores usan el mismo protocolo de enrutamiento para comunicarse (denominado [*Protocolo de puerta de enlace interior*](i-gly.md)). Vea también [*Protocolo de puerta de enlace exterior*](e-gly.md).

</dd> <dt>

<span id="_rras_autostatic_route_gly"></span><span id="_RRAS_AUTOSTATIC_ROUTE_GLY"></span>**Ruta autoestática**
</dt> <dd>

Una ruta autoestática es una ruta que se obtiene dinámicamente a partir de un protocolo de enrutamiento. Una vez que se obtiene una ruta de esta manera, se comporta como [*STATICROUTE*](s-gly.md).

</dd> </dl>
