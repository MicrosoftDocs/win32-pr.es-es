# <a name="filtering-layer-identifiers"></a>Filtrar identificadores de capa

Los Windows de capa de plataforma de filtrado de filtros (WFP) se representan mediante un GUID. Estos identificadores se definen de la manera siguiente.

Los sufijos V4 y V6 al final de los identificadores de capa indican si la capa se encuentra en la pila de red IPv4 o en la pila de red IPv6.

<dl> <dt>

<span id="FWPM_LAYER_INBOUND_IPPACKET_V4___FWPM_LAYER_INBOUND_IPPACKET_V6"></span><span id="fwpm_layer_inbound_ippacket_v4___fwpm_layer_inbound_ippacket_v6"></span>**CAPA FWPM \_ \_ INBOUND \_ IPPACKET \_ V4 /FWPM \_ LAYER INBOUND \_ \_ IPPACKET \_ V6**
</dt> <dd> <dl> <dt>



Esta capa de filtrado se encuentra en la ruta de acceso de recepción justo después de analizar el encabezado IP de un paquete recibido, pero antes de que se lleve a cabo cualquier procesamiento de encabezados IP. No se ha producido ningún descifrado o reensamblado de IPsec.


</dt> </dl> </dd> <dt>

<span id="FWPM_LAYER_INBOUND_IPPACKET_V4_DISCARD___FWPM_LAYER_INBOUND_IPPACKET_V6_DISCARD"></span><span id="fwpm_layer_inbound_ippacket_v4_discard___fwpm_layer_inbound_ippacket_v6_discard"></span>**FWPM \_ LAYER \_ INBOUND \_ IPPACKET \_ V4 DISCARD \_ /FWPM \_ LAYER INBOUND \_ \_ IPPACKET \_ V6 \_ DISCARD**
</dt> <dd> <dl> <dt>



Esta capa de filtrado se encuentra en la ruta de acceso de recepción para inspeccionar los paquetes recibidos que se han descartado en la capa de red.


</dt> </dl> </dd> <dt>

<span id="FWPM_LAYER_OUTBOUND_IPPACKET_V4___FWPM_LAYER_OUTBOUND_IPPACKET_V6"></span><span id="fwpm_layer_outbound_ippacket_v4___fwpm_layer_outbound_ippacket_v6"></span>**FWPM \_ LAYER \_ OUTBOUND \_ IPPACKET \_ V4 /FWPM \_ LAYER OUTBOUND \_ \_ IPPACKET \_ V6**
</dt> <dd> <dl> <dt>



Esta capa de filtrado se encuentra en la ruta de acceso de envío justo antes de evaluar la fragmentación del paquete enviado. Todo el procesamiento de encabezados IP se ha completado y todos los encabezados de extensión están en su lugar. Ya se ha producido cualquier autenticación y cifrado de IPsec.


</dt> </dl> </dd> <dt>

<span id="FWPM_LAYER_OUTBOUND_IPPACKET_V4_DISCARD___FWPM_LAYER_OUTBOUND_IPPACKET_V6_DISCARD"></span><span id="fwpm_layer_outbound_ippacket_v4_discard___fwpm_layer_outbound_ippacket_v6_discard"></span>**FWPM \_ LAYER \_ OUTBOUND \_ IPPACKET \_ V4 DISCARD \_ /FWPM \_ LAYER OUTBOUND \_ \_ IPPACKET \_ V6 \_ DISCARD**
</dt> <dd> <dl> <dt>



Esta capa de filtrado se encuentra en la ruta de acceso de envío para inspeccionar los paquetes enviados que se han descartado en la capa de red.


</dt> </dl> </dd> <dt>

<span id="FWPM_LAYER_IPFORWARD_V4___FWPM_LAYER_IPFORWARD_V6"></span><span id="fwpm_layer_ipforward_v4___fwpm_layer_ipforward_v6"></span>**FWPM \_ LAYER \_ IPFORWARD \_ V4 /FWPM \_ LAYER \_ IPFORWARD \_ V6**
</dt> <dd> <dl> <dt>



Esta capa de filtrado se encuentra en la ruta de acceso de reenvío en el punto donde se reenvía un paquete recibido.

Esta capa de filtrado se encuentra en la parte superior de la capa de red en lugar de en la parte inferior de la capa de transporte para que los paquetes enviados por transportes de terceros o como paquetes sin procesar se filtren en esta capa.

## <a name="remarks"></a>Comentarios

Estos identificadores de capa de filtrado también se conocen como identificadores de capa de filtrado de administración. La API de WFP también contiene un conjunto de identificadores de capa de filtrado en tiempo de ejecución, [documentados](/windows-hardware/drivers/network/management-filtering-layer-identifiers)en Windows Driver Kit (WDK). Los identificadores de capa de filtrado en tiempo de ejecución son LUID y, por tanto, tienen un tamaño de solo 64 bits, en comparación con los identificadores de capa de filtrado de administración, que tienen un tamaño de 128 bits.

Los identificadores de capa de filtrado de administración y los identificadores de capa de filtrado en tiempo de ejecución apuntan a las mismas capas.

Las funciones que interactúan con el motor de filtrado base (BFE) usan los identificadores de capa de filtrado de administración desde el modo de usuario o el modo kernel (por ejemplo, [**FwpmFilterAdd0).**](/windows/desktop/api/Fwpmu/nf-fwpmu-fwpmfilteradd0)

Las funciones que interactúan con el motor de filtro solo usan identificadores de capa de filtrado en tiempo de ejecución desde el modo kernel (por ejemplo, [FwpsFlowAssociateContext0](/windows-hardware/drivers/ddi/fwpsk/nf-fwpsk-fwpsflowassociatecontext0), [FwpsStreamInjectAsync0)](/windows-hardware/drivers/ddi/fwpsk/nf-fwpsk-fwpsstreaminjectasync0)y en estructuras de datos que proceden directamente del kernel (por ejemplo, [**FWPM \_ NET EVENT \_ CLASSIFY \_ \_ DROP0).**](/windows/desktop/api/Fwpmtypes/ns-fwpmtypes-fwpm_net_event_classify_drop0)
