<dl> <dt>

<span id="_msi_cabinet_file_using_windows_installer_gly"></span><span id="_MSI_CABINET_FILE_USING_WINDOWS_INSTALLER_GLY"></span>**archivo. cab**
</dt> <dd>

Archivo único, normalmente con una extensión. cab, que almacena archivos comprimidos en una biblioteca de archivos. El formato de archivo. cab es una manera eficaz de empaquetar varios archivos, ya que la compresión se realiza a través de los límites del archivo, lo que mejora significativamente la relación de compresión. Para obtener información sobre la creación de archivadores, consulte [archivos. cab](cabinet-files.md).

</dd> <dt>

<span id="_msi_checksum_gly"></span><span id="_MSI_CHECKSUM_GLY"></span>**MD5**
</dt> <dd>

Valor calculado que depende del contenido de un archivo. Se almacena con datos para detectar daños en los archivos. El sistema comprueba este valor para asegurarse de que los datos no estén dañados.

</dd> <dt>

<span id="_msi_component_using_windows_installer_gly"></span><span id="_MSI_COMPONENT_USING_WINDOWS_INSTALLER_GLY"></span>**pone**
</dt> <dd>

Parte más pequeña de una instalación controlada por el instalador, también un bloque de creación de una aplicación o característica desde la perspectiva del programador. Algunos ejemplos de componentes son un grupo de archivos relacionados, un objeto COM o una biblioteca. Para obtener más información, vea [componentes y características](components-and-features.md).

</dd> <dt>

<span id="_msi_committing_databases_using_windows_installer_gly"></span><span id="_MSI_COMMITTING_DATABASES_USING_WINDOWS_INSTALLER_GLY"></span>**confirmar bases de datos**
</dt> <dd>

Cambios acumulados realizados en una base de datos. Los cambios no se reflejan en la base de datos real hasta que se confirma la base de datos. Para obtener más información, consulte [confirmación de bases](committing-databases.md)de datos.

</dd> <dt>

<span id="_msi_controlevent_gly"></span><span id="_MSI_CONTROLEVENT_GLY"></span>**ControlEvent,**
</dt> <dd>

Aspecto de la funcionalidad de la interfaz de usuario del instalador. Un ControlEvent, típico desencadena alguna acción por parte del instalador al activar un control de cuadro de diálogo u otro evento. Para obtener más información, consulte [información general de ControlEvent,](controlevent-overview.md).

</dd> <dt>

<span id="_msi_costing_gly"></span><span id="_MSI_COSTING_GLY"></span>**costos**
</dt> <dd>

Método utilizado por el instalador para determinar los requisitos de espacio en disco para la instalación. El costo tiene en cuenta factores como, por ejemplo, si es necesario sobrescribir los archivos existentes. Para obtener más información, consulte [costo de archivos](file-costing.md).

</dd> <dt>

<span id="_msi_.cub_file_gly"></span><span id="_MSI_.CUB_FILE_GLY"></span>**archivo. Cub**
</dt> <dd>

Módulo de validación que almacena y proporciona acceso a las acciones personalizadas de ICE. Para obtener más información, consulte [sample. Cub File](sample--cub-file.md). Para obtener más información, consulte también [Windows Installer extensiones de archivo](windows-installer-file-extensions.md).

</dd> <dt>

<span id="_msi_custom_action_using_windows_installer_gly"></span><span id="_MSI_CUSTOM_ACTION_USING_WINDOWS_INSTALLER_GLY"></span>**acción personalizada**
