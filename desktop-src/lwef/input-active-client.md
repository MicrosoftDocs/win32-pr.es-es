<span data-ttu-id="06b77-105">Dado que varias aplicaciones cliente pueden compartir el mismo carácter y que varios clientes pueden usar caracteres diferentes al mismo tiempo, el servidor designa a un cliente como cliente *de entrada-activo* y envía la entrada de mouse y de voz solo a esa aplicación cliente.</span><span class="sxs-lookup"><span data-stu-id="06b77-105">Because multiple client applications can share the same character and because multiple clients can use different characters at the same time, the server designates one client as the *input-active* client and sends mouse and voice input only to that client application.</span></span> <span data-ttu-id="06b77-106">Esto mantiene la administración ordenada de los datos proporcionados por el usuario para que un cliente adecuado responda a la entrada.</span><span class="sxs-lookup"><span data-stu-id="06b77-106">This maintains the orderly management of user input, so that an appropriate client responds to the input.</span></span>

<span data-ttu-id="06b77-111">Cuando se oculta un carácter, el cliente de ese carácter dejará de ser de entrada-activo para ese carácter.</span><span class="sxs-lookup"><span data-stu-id="06b77-111">When a character is hidden, the client of that character will no longer be input-active for that character.</span></span> <span data-ttu-id="06b77-112">El servidor convierte automáticamente en activo el cliente activo de cualquier carácter restante (s).</span><span class="sxs-lookup"><span data-stu-id="06b77-112">The server automatically makes the active client of any remaining character(s) input-active.</span></span> <span data-ttu-id="06b77-113">Cuando todos los caracteres están ocultos, no hay ningún cliente de entrada activo.</span><span class="sxs-lookup"><span data-stu-id="06b77-113">When all characters are hidden, no client is input-active.</span></span> <span data-ttu-id="06b77-114">Sin embargo, en esta situación, si el usuario presiona la tecla de micrófono de escucha, el agente continuará escuchando sus comandos (mediante el motor de reconocimiento de voz que coincide con el carácter superior del último cliente de entrada-activo).</span><span class="sxs-lookup"><span data-stu-id="06b77-114">However, in this situation, if the user presses the Listening hotkey, Agent will continue to listen for its commands (using the speech recognition engine matching the topmost character of the last input-active client).</span></span>
