# <a name="t-telephony-api"></a>T (API de telefonía)

<dl> <dt>

<span id="tapi2.t1_e1_tapgloss"></span><span id="TAPI2.T1_E1_TAPGLOSS"></span>**T1/E1**
</dt> <dd>

Un vínculo de transmisión digital con una capacidad de 1.544 Mbps. En Estados Unidos, este vínculo se denomina *T1*. En la Unión Europea, se denomina *E1*. Consulte [*megabits por segundo (Mbps)*](m-tapgloss.md).

</dd> <dt>

<span id="tapi2.tapi_tapgloss"></span><span id="TAPI2.TAPI_TAPGLOSS"></span>**TAPI**
</dt> <dd>

Vea *interfaz de programación de aplicaciones de telefonía (TAPI)*.

</dd> <dt>

<span id="tapi2.tapisrv_tapgloss"></span><span id="TAPI2.TAPISRV_TAPGLOSS"></span>**tapisrv**
</dt> <dd>

Una aplicación de servicio de soporte técnico que proporciona un contexto de ejecución para que la biblioteca de vínculos dinámicos TAPI lleve a cabo tareas como la asignación de memoria o el acceso a estructuras de datos que se han intercambiado en el disco.

</dd> <dt>

<span id="tapi2.tdd_tapgloss"></span><span id="TAPI2.TDD_TAPGLOSS"></span>**TDD**
</dt> <dd>

Vea *dispositivos de telecomunicaciones para sordos (TDD)*.

</dd> <dt>

<span id="tapi2.telecommucations_devices_for_the_deaf_tdd__tapgloss"></span><span id="TAPI2.TELECOMMUCATIONS_DEVICES_FOR_THE_DEAF_TDD__TAPGLOSS"></span>**Dispositivos Telecommucations para sordos (TDD)**
</dt> <dd>

Un dispositivo de datos para la transmisión de señales codificadas. Normalmente, el dispositivo es un terminal de equipo simple que consta de un teclado, un par acústico y una pantalla, y se ejecuta a 300 baudios.

</dd> <dt>

<span id="tapi2.telephony_tapgloss"></span><span id="TAPI2.TELEPHONY_TAPGLOSS"></span>**telefonía**
</dt> <dd>

La ciencia de transmitir señales de voz, datos, vídeo o imagen a través de una distancia mediante la tecnología que integra los equipos con la red telefónica.

</dd> <dt>

<span id="tapi2.telephony_application_programming_interface_tapi__tapgloss"></span><span id="TAPI2.TELEPHONY_APPLICATION_PROGRAMMING_INTERFACE_TAPI__TAPGLOSS"></span>**Interfaz de programación de aplicaciones de telefonía (TAPI)**
</dt> <dd>

Conjunto de funciones que permite programar dispositivos basados en líneas telefónicas de forma independiente del dispositivo, lo que proporciona a los usuarios la telefonía personal. TAPI admite la transmisión de datos y voz, permite una variedad de dispositivos de terminal y admite tipos de conexión complejos y técnicas de administración de llamadas, como llamadas de conferencia, llamada en espera y correo de voz. TAPI permite que todos los elementos de uso telefónico (desde la llamada simple de acceso telefónico y de lectura a correo electrónico internacional) se controlen dentro de las aplicaciones desarrolladas para Microsoft Windows. Consulte la [Introducción a la telefonía de Microsoft](./microsoft-telephony-overview.md)

</dd> <dt>

<span id="tapi2.telephony_service_provider_interface_tspi__tapgloss"></span><span id="TAPI2.TELEPHONY_SERVICE_PROVIDER_INTERFACE_TSPI__TAPGLOSS"></span>**Interfaz del proveedor de servicios de telefonía (TSPI)**
</dt> <dd>

Herramienta para crear proveedores de servicios para los sistemas operativos Microsoft Windows. TSPI define el modo en que la red comparte información con telefonía de Windows, que a su vez se comunica con la API, que se comunica con las aplicaciones de telefonía de Windows. Para obtener más información, vea [interfaz del proveedor de servicios de telefonía (TSPI)](./telephony-service-provider-interface-tspi-.md).

</dd> <dt>

<span id="tapi2.third_party_call_control_tapgloss"></span><span id="TAPI2.THIRD_PARTY_CALL_CONTROL_TAPGLOSS"></span>**control de llamadas de terceros**
</dt> <dd>

Modelo de control de llamada de entidad que permite que una aplicación establezca o responda a una llamada entre dos partes cualesquiera.

</dd> <dt>

<span id="tapi2.time_to_live_ttl__tapgloss"></span><span id="TAPI2.TIME_TO_LIVE_TTL__TAPGLOSS"></span>**período de vida (TTL)**
</dt> <dd>

Un valor en el intervalo comprendido entre 0 y 255 que define el ámbito en el que se deben enviar los paquetes de multidifusión a través de una red mediante el protocolo de Internet (IP). El ámbito se define en lo que respecta al modo en que el destino de un paquete es local o remoto. Cada enrutador reduce el TTL en uno. Cuando el valor alcanza un límite inferior predefinido, el enrutador produce el paquete de inmediato. Los requisitos actuales de la red troncal de multidifusión (MBONE), disponibles en el sitio de ftp://ftp.isi.edu/mbone/faq.txt, definen los siguientes ámbitos estándar: red local, 1; sitio local, 15; región, 63; mundo, 127. Otras opciones pueden tener un significado local. por ejemplo, 31 podría indicar todos los sitios de una organización determinada.

</dd> <dt>

<span id="tapi2.tspi_tapgloss"></span><span id="TAPI2.TSPI_TAPGLOSS"></span>**TSPI**
</dt> <dd>

Vea [interfaz del proveedor de servicios de telefonía (TSPI)](./telephony-service-provider-interface-tspi-.md).

</dd> <dt>

<span id="tapi2.ttl_tapgloss"></span><span id="TAPI2.TTL_TAPGLOSS"></span>**PERÍODO**
</dt> <dd>

Vea *período de vida (TTL)*.

</dd> <dt>

<span id="tapi2.twisted_pair_tapgloss"></span><span id="TAPI2.TWISTED_PAIR_TAPGLOSS"></span>**par trenzado**
</dt> <dd>

Cableado telefónico normal. En realidad, cada conexión telefónica es un par de cables.

</dd> </dl>
