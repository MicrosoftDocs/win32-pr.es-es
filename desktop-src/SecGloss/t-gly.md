<dl> <dt>

<span id="_security_t_0_protocol_gly"></span><span id="_SECURITY_T_0_PROTOCOL_GLY"></span>**T = protocolo 0**
</dt> <dd>

Un Protocolo asincrónico de transmisión dúplex orientado a caracteres.

</dd> <dt>

<span id="_security_t_1_protocol_gly"></span><span id="_SECURITY_T_1_PROTOCOL_GLY"></span>**T = 1 Protocolo**
</dt> <dd>

Un Protocolo asincrónico de transmisión dúplex orientado a bloque.

</dd> <dt>

<span id="_security_terminal_gly"></span><span id="_SECURITY_TERMINAL_GLY"></span>**terminal**
</dt> <dd>

Una combinación de monitor, teclado, mouse y periféricos colocalizados, como lectores de tarjetas inteligentes. Se pueden asociar varios procesos a un solo terminal, pero solo un proceso controla el terminal en un momento dado.

</dd> <dt>

<span id="_security_three_phase_key_exchange_protocol_gly"></span><span id="_SECURITY_THREE_PHASE_KEY_EXCHANGE_PROTOCOL_GLY"></span>**Protocolo de intercambio de claves de tres fases**
</dt> <dd>

Protocolo que se usa para generar una conexión autenticada y cifrada entre dos usuarios de una red no segura. Los usuarios intercambian un conjunto de mensajes para negociar un par de claves de cifrado. El remitente usa una clave para cifrar los mensajes enviados al receptor y el otro lo usa el receptor para cifrar los mensajes enviados al remitente. Este protocolo garantiza que ambos usuarios estén activos y envíen mensajes directamente entre sí.

</dd> <dt>

<span id="_security_thumbprint_gly"></span><span id="_SECURITY_THUMBPRINT_GLY"></span>**huella**
</dt> <dd>

El hash [*SHA-1*](s-gly.md) de un certificado.

</dd> <dt>

<span id="_security_tls_gly"></span><span id="_SECURITY_TLS_GLY"></span>**EMPLEO**
</dt> <dd>

Consulte *Protocolo de seguridad* de la capa de transporte.

</dd> <dt>

<span id="_security_token_gly"></span><span id="_SECURITY_TOKEN_GLY"></span>**muestras**
</dt> <dd>

Consulte [*token de acceso*](a-gly.md).

</dd> <dt>

<span id="_security_transaction_gly"></span><span id="_SECURITY_TRANSACTION_GLY"></span>**transacción**
</dt> <dd>

Operación que permite realizar varias interacciones como una sola operación. Los pasos intermedios no se realizan realmente hasta que todas las interacciones se completen correctamente. Si se produce un error en alguna interacción, todos los pasos se devuelven a su configuración original.

</dd> <dt>

<span id="_security_transport_layer_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_GLY"></span>**nivel de transporte**
</dt> <dd>

El nivel de red que es responsable de la calidad de servicio y de la entrega precisa de la información. Entre las tareas realizadas en esta capa se encuentran la corrección y detección de errores.

</dd> <dt>

<span id="_security_transport_layer_security_protocol_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_SECURITY_PROTOCOL_GLY"></span>**Protocolo de seguridad de la capa de transporte**
</dt> <dd>

EMPLEO Protocolo que proporciona privacidad y seguridad de las comunicaciones entre dos aplicaciones que se comunican a través de una red.

</dd> <dt>

<span id="_security_triple_des_gly"></span><span id="_SECURITY_TRIPLE_DES_GLY"></span>**triple DES**
</dt> <dd>

Una variación del algoritmo de cifrado de bloques DES que cifra texto sin formato con una clave, cifra el texto cifrado resultante con una segunda clave y, por último, cifra el resultado del segundo cifrado con una tercera clave. Triple DES es un algoritmo simétrico que usa el mismo algoritmo y las mismas claves para el cifrado y el descifrado.

</dd> <dt>

<span id="_security_trustee_gly"></span><span id="_SECURITY_TRUSTEE_GLY"></span>**confianza**
</dt> <dd>

La cuenta de usuario, la cuenta de grupo o la sesión de inicio de sesión a la que se aplica una entrada de control de acceso (ACE). Cada ACE de una lista de control de acceso (ACL) se aplica a un administrador de confianza.

Consulte también [*entrada de control de acceso*](a-gly.md) y [*lista de control de acceso*](a-gly.md).

</dd> <dt>

<span id="_security_trust_list_gly"></span><span id="_SECURITY_TRUST_LIST_GLY"></span>**lista de confianza**
</dt> <dd>

Consulte la [*lista de certificados de confianza*](c-gly.md).

</dd> <dt>

<span id="_security_trust_provider_gly"></span><span id="_SECURITY_TRUST_PROVIDER_GLY"></span>**proveedor de confianza**
</dt> <dd>

El software que decide si un archivo determinado es o no de confianza. Esta decisión se basa en el certificado asociado al archivo.

</dd> </dl>
