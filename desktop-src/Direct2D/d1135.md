# <a name="d1135-bitmap-option-must-be-subset"></a>D1135: la opción Bitmap debe ser de subconjunto

El valor de las opciones de mapa \[  \] de bits debe ser un subconjunto de las marcas asociadas a la superficie de DXGI.

</dd> </dl>

## <a name="possible-causes"></a>Causas posibles

Mientras interoperan las texturas de Direct3D con Direct2D como ID2D1Bitmaps, las opciones especificadas en el \_ campo de opciones de mapa de bits D2D1 \_ deben coincidir con los atributos de Direct3D de la textura. En concreto:

-   La opción de lectura de la [**\_ CPU de opciones de mapa de bits \_ \_ \_ D2D1**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) requiere la opción de lectura de acceso de CPU de Direct3D \_ \_ .
-   La opción de [**destino de opciones de mapa de bits de D2D1 \_ \_ \_**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) requiere la opción de destino de representación de BIND \_ \_ .
-   La [**opción \_ \_ \_ None de D2D1 Bitmap**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) no es compatible con la opción de \_ almacenamiento provisional del uso de Direct3D con \_ solo escritura de acceso de CPU \_ .

## <a name="possible-fixes"></a>Posibles correcciones

Para solucionar este problema, asegúrese de que la creación de [**ID2D1Bitmap**](/windows/win32/api/d2d1/nn-d2d1-id2d1bitmap) usa especificadores de [**\_ \_ Opciones de mapa de bits D2D1**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) que están de acuerdo con las propiedades de la textura de Direct3D subyacente. Para usar **\_ \_ las opciones de mapa de bits de D2D1**, esto puede implicar el uso de [**ID2D1DeviceContext:: CreateBitmap**](id2d1devicecontext-createbitmap-overload.md) en lugar de [**ID2D1RenderTarget:: CreateBitmap**](id2d1rendertarget-createbitmap.md), ya que es miembro de [**D2D1 \_ Bitmap \_ PROPERTIES1**](/windows/desktop/api/D2D1_1/ns-d2d1_1-d2d1_bitmap_properties1).
