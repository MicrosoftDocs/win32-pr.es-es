<span data-ttu-id="743fa-104">Para mejorar la eficacia de la CPU de las aplicaciones de Direct3D, a partir de la versión 12, Direct3D ya no admite un contexto inmediato asociado a un dispositivo.</span><span class="sxs-lookup"><span data-stu-id="743fa-104">To improve the CPU efficiency of Direct3D apps, as of version 12, Direct3D no longer supports an immediate context associated with a device.</span></span> <span data-ttu-id="743fa-105">En su lugar, la aplicación registra y envía *listas de comandos*, que contienen llamadas de administración de dibujos y recursos.</span><span class="sxs-lookup"><span data-stu-id="743fa-105">Instead, your application records and then submits *command lists*, which contain drawing and resource management calls.</span></span> <span data-ttu-id="743fa-106">Puede enviar estas listas de comandos desde varios subprocesos a una o varias colas de comandos, que administran la ejecución de los comandos.</span><span class="sxs-lookup"><span data-stu-id="743fa-106">You can submit these command lists from multiple threads to one or more command queues, which manage the execution of the commands.</span></span> <span data-ttu-id="743fa-107">Este cambio fundamental aumenta la eficacia de un solo subproceso, ya que permite que la aplicación procese previamente el trabajo de representación para su reutilización posterior y aprovecha los sistemas de varios núcleos mediante la propagación del trabajo de representación entre varios subprocesos.</span><span class="sxs-lookup"><span data-stu-id="743fa-107">This fundamental change increases single-threaded efficiency by allowing your application to pre-compute rendering work for later re-use, and it takes advantage of multi-core systems by spreading rendering work across multiple threads.</span></span>
