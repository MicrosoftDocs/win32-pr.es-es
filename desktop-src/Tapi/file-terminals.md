# <a name="file-terminals"></a>Terminales de archivo

En TAPI 3.1, el objeto de terminal se expande para incluir un terminal de archivo. Los terminales de archivo proporcionan un medio sencillo para realizar operaciones comunes, por ejemplo, grabar una secuencia de audio o secuencias en un archivo o la reproducción posterior de audio grabado en una secuencia.

Se usan cinco objetos básicos para manipular terminales de archivo:

-   Terminal de reproducción de archivos
-   Terminal de grabación de archivos
-   Seguimiento de reproducción de archivos
-   Seguimiento de grabación de archivos
-   Evento de archivo

Los objetos terminales de archivo se pueden pensar como una abstracción para el almacenamiento, que normalmente es un archivo. Las pistas corresponden a las secuencias de datos de audio individuales de ese almacenamiento, como cuando se graba la voz.

Para obtener más información sobre los terminales de archivo, vea:

-   [Terminal de reproducción de archivos y terminal de grabación de archivos](file-playback-terminal-and-file-recording-terminal.md)
-   [Seguimiento de terminales](track-terminals.md)

Para obtener más información y ejemplos de código y listas de tareas relacionadas con terminales de archivo, vea [Using File Terminals](using-file-terminals.md).
