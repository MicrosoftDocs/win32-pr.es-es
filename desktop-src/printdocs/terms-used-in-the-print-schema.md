# <a name="terms-used-in-the-print-schema"></a>Términos usados en el esquema de impresión

Este tema no está actualizado. Para obtener la información más reciente, consulte la [especificación del esquema de impresión](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

La siguiente lista contiene términos que son específicos del esquema de impresión y de las tecnologías relacionadas.



| Término                                                                                                                                                                 | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>nombre del atributo<br/>                                                                          | Un nombre de atributo es el token en el lado izquierdo del signo igual en un atributo XML estándar. La representación estándar de un atributo XML es *Attribute Name*= "*Attribute Value*". Aunque es similar, el término "nombre de atributo" no se debe confundir con el atributo de nombre definido en el marco de trabajo del esquema de impresión. El atributo de nombre definido en el marco de esquema de impresión es una instancia específica de un nombre de atributo. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>buen<br/>                                                                                         | Desea. Cualquier opción que pertenezca a la lista que se está evaluando durante el proceso de puntuación. Vea las opciones términos, puntuación o asignación y referencia.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Configuraciones<br/>                                             | Describe uno de los Estados que un dispositivo puede asumir o colocar en. El dispositivo realiza su tarea de forma distinta en cada Estado. Existe una diferencia distinta e identificable entre el comportamiento del dispositivo en cada uno de estos Estados. Por ejemplo, en un estado, el medio se extrae de una bandeja y termina en otra bandeja. Un cambio en una o ambas bandejas determina un estado diferente.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Restricción<br/>                                                         | Información necesaria para determinar la validez de una configuración. Una restricción tiene a menudo el formato: Featurea establecido en Opción1 no es compatible con FeatureB establecido en opción2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>elementos correspondientes<br/>                                                  | Dos elementos que tienen la misma ubicación en relación con un par de elementos de origen especificados. Los elementos de origen son con frecuencia dos elementos Option que se están comparando.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>predeterminada<br/>                                                                                               | Desea. La opción designada por el esquema de impresión o el proveedor de PrintCapabilities como la opción que se asigna a una característica cuando PrintTicket no contiene una configuración explícita para esa característica. Cada característica tiene una opción predeterminada designada.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>atributo de configuración de dispositivo<br/>                          | Un atributo de dispositivo que contribuye a la definición de la configuración del dispositivo. Un atributo de configuración de dispositivo se representa en el marco de trabajo de esquema de impresión mediante las construcciones característica/opción o parámetro. Tenga en cuenta que el atributo utilizado en este contexto no tiene ninguna relación con los atributos XML ni con los elementos de atributo definidos por el marco de esquema.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>tipo de elemento<br/>                                                                                | El tipo de cada elemento XML definido en el marco de esquema de impresión se identifica mediante su etiqueta de elemento XML.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>elementos de Common<br/>                                                              | Vea los elementos correspondientes.<br/> Ofrecen. Una construcción definida por el marco de esquema de impresión que representa un atributo de configuración de dispositivo específico. Cada instancia de característica representa un atributo de dispositivo. Una instancia de característica se define definiendo explícitamente cada uno de sus Estados.<br/> Opción de identidad. Opción que no tiene ningún efecto observable cuando se selecciona; en efecto, un "no operativo". El resultado siempre es el mismo, independientemente del número de veces que se seleccione esta opción.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>nombre de instancia<br/>                                                                             | Atributo XML que identifica la instancia específica de un tipo de elemento. Vea el atributo name.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>cliente de interfaz<br/>                                                                    | Entidad que consume documentos PrintCapabilities y que construye y consume elementos PrintTicket. Entre los ejemplos comunes de clientes de interfaz se incluyen módulos de interfaz de usuario (IU) y módulos de representación.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>proveedor de interfaz<br/>                                                              | Entidad que implementa y expone la funcionalidad por medio de una interfaz. En el esquema de impresión, esto equivale a crear documentos de PrintCapabilities y validar elementos PrintTicket. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>Ubicación de un elemento<br/>                                                  | Posición de una instancia determinada de un elemento dentro de la jerarquía de elementos. La ubicación se define mediante el elemento determinado y todos los antecesores de dicho elemento. En otras palabras, Location es el equivalente de una ruta de acceso completa en un sistema de archivos. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>Name (atributo)<br/>                                                                          | Designación asignada al atributo XML con un nombre de atributo "Name". Este atributo aparece en la mayoría de los tipos de elemento definidos en el marco de trabajo del esquema de impresión. Identifica y define instancias de cada tipo de elemento. Los dos términos siguientes hacen referencia a los atributos name: la regla de unicidad del mismo nivel y la Convención de espacio de nombres.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Convención de espacio de nombres<br/>                 | Indica que los atributos de nombre definidos por el esquema, así como cualquier objeto definido por el esquema, existen en un espacio de nombres dedicado, el espacio de nombres público y que los atributos de nombre definidos de forma privada deben existir en un espacio de nombres diferente, uno que se puede identificar como perteneciente a la entidad de definición.<br/> Desea. Una construcción de marco de esquema de impresión que se usa para representar los Estados que cada característica puede asumir o colocar en. Por ejemplo, si la característica representa el tamaño del medio, los valores de opción típicos pueden ser letras, legales, A4, entre otros.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Parámetro<br/>                                                             | El esquema de impresión admite la capacidad de describir datos con parámetros para la entrada del usuario. Hay tres tipos de elemento de marco de esquemas de impresión que juegan roles en la entrada paramétrica: ParameterDef, ParameterInit y ParameterRef. Un elemento ParameterDef caracteriza un parámetro (su intervalo de valores, el tipo de sus datos y otras características). Se usa un elemento ParameterInit para inicializar un parámetro. Un elemento ParameterRef se usa para hacer referencia a una instancia determinada de un elemento ParameterInit.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>con parámetros<br/>                                                                             | Desea. Instancia de la opción que contiene al menos un elemento ScoredProperty que, a su vez, contiene un elemento ParameterRef en lugar de un elemento de valor.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>coincidencia perfecta<br/>                                                              | Opciones. Se considera que dos instancias de opción coinciden perfectamente si para cada elemento ScoredProperty de una instancia de opción hay una instancia de ScoredProperty correspondiente en la otra instancia de la opción y ambos elementos de valor de las instancias de ScoredProperty correspondientes son idénticos.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>Esquema de PrintCapabilities<br/> | El esquema que define la sintaxis y la semántica que usa un documento de PrintCapabilities. El esquema de PrintCapabilities y el esquema de PrintTicket son subconjuntos del esquema de impresión. Vea también Print Schema Framework.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>Esquema PrintTicket<br/>                         | El esquema que define la sintaxis y la semántica que usa el PrintTicket. El esquema PrintTicket y el esquema PrintCapabilities son subconjuntos del esquema de impresión. Vea también Print Schema Framework.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Marco de esquema de impresión<br/>         | El esquema que define la sintaxis y la semántica que utilizan las tecnologías de esquema de impresión. Por ejemplo, el marco de trabajo del esquema de impresión define una colección de tipos de elementos y sus usos, pero no enumera todas las instancias de elemento.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Imprimir palabras clave de esquema<br/>             | Esquema que define las instancias individuales de los elementos de propiedades, características y opciones que se usan normalmente para describir los dispositivos y el formato del trabajo. Este esquema define un lenguaje común que convierte los documentos de PrintCapabilities y elementos PrintTicket en portable. También se conoce como esquema público.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>definido de forma privada<br/>                                                                 | Cualquier instancia de los elementos que aparecen en un documento de PrintCapabilities o PrintTicket que no se definen explícitamente en las palabras clave del esquema de impresión. Estas instancias de elemento se deben calificar explícitamente con un espacio de nombres privado por proveedor.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>referencia<br/>                                                                                         | Desea. La opción en la que se compara cada opción candidata. Normalmente, una opción de referencia se origina desde PrintTicket. Vea los términos, la puntuación o la opción de asignación y la opción candidato.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>nivel raíz<br/>                                                                                      | El nivel raíz es la base de la estructura jerárquica del marco de trabajo del esquema de impresión. Un elemento de nivel de raíz se define como elemento secundario de un elemento PrintCapabilities o un elemento PrintTicket.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>Prefijo de ámbito<br/>                                                                          | Una etiqueta de texto previamente anexada a una palabra clave de esquema para proporcionar el ámbito contextual. Los prefijos de ámbito de los esquemas de impresión permitidos son trabajo, documento y página. Por ejemplo, el prefijo de ámbito para JobInputBin es ' Job '.<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | Una construcción de marco de esquema de impresión utilizada para representar las cualidades que distinguen o diferencian una instancia de opción de otra. Un ejemplo es la diferencia entre la carta y el tamaño de papel oficio.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>puntuación o asignación<br/>                                                              | Desea. El proceso de selección en una lista de la instancia de la opción (candidata) que mejor coincida con una instancia de opción (referencia) determinada. Los términos relacionados son la opción de candidato y la opción de referencia. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Regla de unicidad del mismo nivel<br/>     | Indica que todos los elementos del mismo nivel del mismo tipo de elemento deben tener atributos de nombre único. Algunos tipos de elemento están exentos de esta regla. Estas excepciones se definen en el marco de trabajo del esquema de impresión.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>valor único frente a varios valores<br/>                    | Propiedad. Una propiedad cuyo valor es invariable (no depende de la configuración) se considera de un solo valor. Este tipo de propiedad solo tiene un valor en cualquier configuración posible. Una propiedad que depende de la configuración se considera multivalor.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>sufijo/sufijos<br/>                                                                       | Palabra clave de esquema que representa la característica o el nombre del parámetro. Por ejemplo, el sufijo de JobInputBin es ' InputBin' '.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Valida<br/>                                                         | El proceso de comprobar que un PrintTicket se ajusta al esquema y representa una configuración que puede ser ejecutada por un dispositivo específico.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Indefinido\_<br/>                                                   | Los datos específicos que se representan aquí son un marcador de posición para el controlador, el dispositivo o la aplicación específicos que se va a producir. No transmite que los datos deben ser una cadena.<br/>                                                                                                                                                                                                                                                                                                                                                            |
