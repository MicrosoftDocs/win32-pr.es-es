## <a name="remarks"></a><span data-ttu-id="04cd5-205">Observaciones</span><span class="sxs-lookup"><span data-stu-id="04cd5-205">Remarks</span></span>

<span data-ttu-id="04cd5-206">La función **AcceptSecurityContext (NTLM)** es el homólogo del servidor para la función [**InitializeSecurityContext (NTLM)**](initializesecuritycontext--ntlm.md) .</span><span class="sxs-lookup"><span data-stu-id="04cd5-206">The **AcceptSecurityContext (NTLM)** function is the server counterpart to the [**InitializeSecurityContext (NTLM)**](initializesecuritycontext--ntlm.md) function.</span></span>

<span data-ttu-id="04cd5-207">Cuando el servidor recibe una solicitud de un cliente, el servidor usa el parámetro *fContextReq* para especificar lo que requiere de la sesión.</span><span class="sxs-lookup"><span data-stu-id="04cd5-207">When the server receives a request from a client, the server uses the *fContextReq* parameter to specify what it requires of the session.</span></span> <span data-ttu-id="04cd5-208">De este modo, un servidor puede especificar que los clientes deben ser capaces de usar una sesión confidencial o comprobada por [*integridad*](../secgloss/i-gly.md), y puede rechazar clientes que no puedan satisfacer esa demanda.</span><span class="sxs-lookup"><span data-stu-id="04cd5-208">In this fashion, a server can specify that clients must be capable of using a confidential or [*integrity*](../secgloss/i-gly.md)-checked session, and it can reject clients that cannot meet that demand.</span></span> <span data-ttu-id="04cd5-209">Como alternativa, un servidor no puede necesitar nada y lo que el cliente puede proporcionar o requiera se devuelve en el parámetro *pfContextAttr* .</span><span class="sxs-lookup"><span data-stu-id="04cd5-209">Alternatively, a server can require nothing, and whatever the client can provide or requires is returned in the *pfContextAttr* parameter.</span></span>
