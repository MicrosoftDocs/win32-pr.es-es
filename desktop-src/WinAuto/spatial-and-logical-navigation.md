## <a name="establishing-the-order-in-logical-navigation"></a><span data-ttu-id="28003-125">Establecer el orden en la navegación lógica</span><span class="sxs-lookup"><span data-stu-id="28003-125">Establishing the Order in Logical Navigation</span></span>

<span data-ttu-id="28003-126">En la navegación lógica, los desarrolladores que diseñan los objetos establecen las relaciones entre ellos.</span><span class="sxs-lookup"><span data-stu-id="28003-126">In logical navigation, the developers who design the objects establish the relationships between them.</span></span> <span data-ttu-id="28003-127">La navegación lógica es más subjetiva que la navegación espacial.</span><span class="sxs-lookup"><span data-stu-id="28003-127">Logical navigation is more subjective than spatial navigation.</span></span> <span data-ttu-id="28003-128">Además, el orden en la navegación lógica no es el mismo que el orden que se usa con los identificadores secundarios.</span><span class="sxs-lookup"><span data-stu-id="28003-128">Also, the order in logical navigation is not the same as the order used with child IDs.</span></span>

<span data-ttu-id="28003-129">En el caso de los objetos que tienen ubicaciones de pantalla, los desarrolladores de servidor deben establecer el orden de navegación de la forma en que la mayoría de los usuarios considerarían lógicos.</span><span class="sxs-lookup"><span data-stu-id="28003-129">For objects that have screen locations, server developers should establish the navigation order in the way that most users would consider logical.</span></span> <span data-ttu-id="28003-130">Por ejemplo, en países o regiones de habla inglesa, esto significa una ordenación de izquierda a derecha y de arriba a abajo.</span><span class="sxs-lookup"><span data-stu-id="28003-130">In English-speaking countries/regions, for instance, this means a left-to-right, top-to-bottom ordering.</span></span>

<span data-ttu-id="28003-131">El orden de navegación lógico debe estar en paralelo en el orden de navegación del teclado.</span><span class="sxs-lookup"><span data-stu-id="28003-131">Logical navigation order must parallel keyboard navigation order.</span></span> <span data-ttu-id="28003-132">Por ejemplo, un cuadro de diálogo contiene los botones **Aceptar** y **Cancelar** y algunos controles de edición.</span><span class="sxs-lookup"><span data-stu-id="28003-132">For example, a dialog box contains **OK** and **Cancel** push buttons and a few edit controls.</span></span> <span data-ttu-id="28003-133">Un cliente que llama a [**IAccessible:: accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accnavigate) para desplazarse al objeto siguiente o anterior de ese cuadro de diálogo se mueve en el mismo orden que un usuario que presiona tab o Mayús + Tab para mover el foco entre elementos.</span><span class="sxs-lookup"><span data-stu-id="28003-133">A client that calls [**IAccessible::accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accnavigate) to navigate to the next or previous object in that dialog box moves in the same order as a user pressing TAB or SHIFT+TAB to move the focus between items.</span></span>

<span data-ttu-id="28003-134">En el caso de los objetos que no tienen ubicaciones de pantalla definidas, el orden lógico lo deciden los desarrolladores de servidores y los desarrolladores de cliente no deben realizar suposiciones sobre él.</span><span class="sxs-lookup"><span data-stu-id="28003-134">For objects that do not have defined screen locations, the logical order is decided by server developers, and client developers should make no assumptions about it.</span></span> <span data-ttu-id="28003-135">Por ejemplo, es aceptable que los objetos no visibles, como los objetos que solo están ocultos temporalmente, se intercalan con objetos visibles.</span><span class="sxs-lookup"><span data-stu-id="28003-135">For instance, it is acceptable for non-visible objects, such as objects that are only temporarily hidden, to be interspersed with visible objects.</span></span>
